> **版本**：v1.0 · 2026-05
> **目的**：定义"AI 生成内容 → 落盘到 `content/` → 一键导入数据库"的全流程契约。
> **适用范围**：
> - 发现中国（china）：12 类目下的中文短文与 5 语言对照

## 1. 总体设计原则（生成数据的 7 条铁律）

1. **以库表为唯一真源**：所有字段名、类型、长度、枚举、正则必须 1:1 对应 `system/supabase/migrations/*` + `function/0X-xxx/ai/F1-AI-数据模型规范/*`。出现冲突以 SQL migration 为准。
2. **数据文件而非 SQL**：AI 输出业务约定的数据文件，不要写 `INSERT` 语句。china 域使用单篇 `*.article.json`；course 域继续使用 `*.jsonl`。导入由脚本完成，方便校验、去重、增量、回滚。
3. **业务定位明确**：
   - **每条 KP/题目从生成那一刻就必须知道自己归属"哪个 stage / chapter / lesson"**；通过外部业务 key（`lesson_code` 如 `ec-2-3-1`）声明，导入脚本反查 uuid 写关联。
   - 不允许"先生成一堆，再人工分配"。
4. **内容静态化**：所有学员看到的字段（讲解、例句、翻译、音频文案）都在生成阶段一次写齐 5 语言（zh / en / vi / th / id），运行时不再调 LLM。
5. **来源可追溯**：每个数据文件都必须能追溯到来源。china 单篇 JSON 用 `article_slug` + manifest 追踪；course JSONL 用 `source_batch`、文件名、sha256 与批次号追踪。
6. **幂等可重入**：用业务键（`code` / `kp_code` / `q_code` / `lesson_code` / 句子的 `(article_code, seq_no)`）作为 `ON CONFLICT` 目标。重复跑同一文件不会产生脏数据。
7. **失败不阻塞**：单行校验失败 → 写 `*.errors.jsonl` 并跳过；不要因为 1 行报错就回滚整个文件。
8. **内容合规先行**：面向越南、泰国、印尼等东南亚市场的首期内容，默认避开政治、宗教、民族、领土争议、社会冲突、成人性内容、色情暗示、暴力猎奇和可能触发当地红线的敏感表达。

### 文件命名规则

- **china 文件**：`<2位序号>-<英文slug>.article.json`，一篇文章一个文件；同目录 `manifest.json` 只做清单。
- **course 批次 ID** = `<batch_code>` = `batch_<YYYYMMDD>_<3位序号>`，例：`batch_20260515_001`。
- course 一个"批次"= 同一次 AI 生成会话的所有文件，共享同一个 manifest。
- **后缀严格**：china 使用 `.article.json` / `manifest.json`；course 使用 `.kp.jsonl` / `.questions.jsonl` / `.lesson_kp.jsonl` / `.exam.json` / `.assets.jsonl` / `.manifest.json`。

## 3. 数据文件通用规约

### 3.0 china 单篇 JSON

- 每个 `*.article.json` 只含一篇文章。
- 顶层必须包含 `schema/article/content_policy/seo/geo/sentences`。
- `sentences` 是数组，`seq_in_article` 从 1 连续递增，最多 120。
- 不写 DB `id/code/article_id/created_at/audio_url_zh`。

### 3.1 course JSONL 文件级

- **UTF-8 无 BOM**，LF 换行
- 每行一个 JSON 对象，**不允许换行符**出现在 value 中（用 `\n`）
- 第 1 行可以是注释行：`// schema=course.kp.v1`（导入脚本忽略 `//` 开头的行，便于人读）
- 文件第一行之外的注释一律不允许

### 3.2 字段级铁律

| 类型 | 要求 |
|------|------|
| `_i18n` 多语言 jsonb | **必须包含全部 5 个 key**：`zh / en / vi / th / id`。漏一个，CHECK 约束 `?& array['zh','en','vi','th','id']` 会抛错 |
| 拼音 | 一律带声调（`jià gé`，不要 `jia ge`），多音节空格分隔；标调用组合字符不要数字（`jia4 ge2` 是错的） |
| 汉字 | 简体；`accept_traditional=false` 默认；不允许出现繁体 |
| URL | 全部以 `https://cdn.zhiyu.app/` 或者 `/cdn/` 开头；生成阶段允许占位 `cdn://placeholder/...`，导入时由媒资脚本替换 |
| 时间戳 | `null`（生成阶段不写 `created_at` / `published_at`，由 DB 默认值或发布动作填充） |
| uuid | **不要** AI 自己生成 uuid！uuid 由 DB 默认值产生。所有跨表关联通过业务 key（如 `lesson_code` / `kp_code` / `category_code`）传递 |
| 枚举值 | 全部小写英文，严格按 [§4 枚举速查](#4-枚举速查表必背) |
| 文本长度 | 严格按表字段 CHECK 约束（`title_zh` 1–60、`pinyin` 1–200、`content_xx` 1–400 等）。生成时主动控制，超长直接截断或重写 |

### 3.3 内容安全与东南亚合规

所有 `content/` 数据生成都必须默认适配越南、泰国、印尼等东南亚首发市场：

- 不生成政治立场表达、现实政治评价、民族矛盾叙事、宗教争辩、领土争议、国家间冲突动员等敏感内容。
- 不生成成人性内容、色情暗示、性交易、露骨身体描写、未成年人相关不当内容。
- 历史、文化、课程示例都使用中立事实、生活场景和学习语境；必要历史事件只做低刺激、事实型说明，避免煽动性措辞。
- 课程例句优先使用购物、工作、安全、交通、餐饮、旅游、家庭等低风险场景。
- 发现中国文章优先写饮食、节日、艺术、景点、文学、汉字、神话等文化科普；涉及历史时避免现实政治类比。
- 多语言翻译不得把中性中文改写成更刺激、更带立场或更成人化的表达。

### 3.4 行级容错

- 一行报错只标红该行，写入 `*.errors.jsonl`，不影响同文件其它行
- 字段缺失：必填字段缺失 → 拒绝；可选字段缺失 → 默认 null
- 类型错误：自动转换 string→int 失败 → 拒绝
- 多语言不全 5 key：拒绝并提示缺失的 key

## 5. 业务定位（最重要！）

> 每条内容都必须在生成时**显式指定归属**，否则就是孤儿数据。

### 5.1 china 域

每篇 article JSON 必须指定 `category_code`（`01..12`）。每条 sentence 写在同一个 JSON 的 `sentences` 数组内，通过 `seq_in_article` 表示文章内顺序。AI 不写 DB `article_id`。

### 5.2 course 域

**KP 必须指定挂在哪些 lesson 下**：通过同批次的 `lesson_kp.jsonl` 写入 `(lesson_code, kp_code, position, is_new_in_lesson)`。

**question 必须指定考的是哪个 KP**：通过 `kp_code` 关联（不是 uuid）。

**question.exam_scope 必须显式列出**：默认所有 KP 关联的题至少包含 `practice`，节末/章末/阶段考的题在生成时就划好。

**lesson 与上层 chapter / stage 必须先存在**：先跑 `_skeleton/` 下的 tracks/stages/chapters/lessons 4 个文件建好骨架，再跑各节内容。

详见各业务 dataspec。

## 6. 生成顺序与导入流程

### 6.1 生成顺序（严格遵守，否则外键报错）

#### china 域
```
1. 类目（已在 migration 0005 内置 12 条种子，无需生成）
2. 同一类目下：一篇文章一个 `*.article.json`，每篇文件内同时包含 article 标题、SEO/GEO 元数据与 sentences 数组
3. 首期每类目 12 篇；后续按搜索热点持续追加，不设总量上限
```

#### course 域
```
1. _skeleton/tracks.jsonl       (5 行，已有种子可跳过)
2. _skeleton/stages.jsonl       (25 行)
3. _skeleton/chapters.jsonl     (~148 行)
4. _skeleton/lessons.jsonl      (~888 行)
   ↑ 以上 4 步是"骨架"，由 PM 一次性生成，AI 只填业务字段
5. 每节循环：
   a. 该节的 8–14 个 KP（kp.jsonl）
   b. 该节 KP 的 4–6 题/KP（questions.jsonl，含 exam_scope 划分）
   c. 节-KP 关联（lesson_kp.jsonl）
6. _exams/ 下的试卷模板（lesson_quiz / chapter_test / stage_exam / hsk_mock）
7. _media/ 媒资清单（在 KP/题目都生成完后再补，hash 去重）
```

### 6.2 导入工具

- china 脚本：`system/scripts/db/import-china-article-json.mjs`
- 用法：
  ```bash
  node system/scripts/db/import-china-article-json.mjs \
    --data-root content/01-china/data/articles/01-history/phase1 \
    --env-file system/docker/env/.env.dev \
    --replace \
    --publish
  ```
- course 统一脚本（待开发）：`system/scripts/db/import_content.ts`
- course 用法（计划）：
  ```bash
  pnpm tsx system/scripts/db/import_content.ts \
    --domain course \
    --batch content/02-course/data/ec/stage2/chapter03/lesson01/batch_20260515_001.manifest.json
  ```
- 行为：
  1. 解析 manifest，按 `files[].path` 读 jsonl
  2. 计算 sha256 与 manifest 比对（不匹配 → 拒绝）
  3. 写 `course_import_batches`（status 流转：`draft` → `validating` → `ready` → `imported`/`failed`）
  4. 按 `_skeleton/` → `kp` → `lesson_kp` → `questions` → `exams` 顺序逐表 upsert
  5. 失败行写 `*.errors.jsonl`，成功行 `success_count++`
  6. 全部完成 → 更新批次 `status='imported'`

> 用户操作"导入即用"的关键：导入脚本默认不强制发布。china 可显式传 `--publish` 自动调用 `fn_publish_article`；course 后续可在 manifest 里加 `"auto_publish": true`。

### 6.3 回滚

按 `source_batch_id` 一键删除：
```sql
delete from zhiyu.course_questions where source_batch_id = '<batch_uuid>';
delete from zhiyu.course_lesson_kp where (lesson_id, kp_id) in (...);  -- 通过反查
delete from zhiyu.course_knowledge_points where source_batch_id = '<batch_uuid>';
update zhiyu.course_import_batches set status='cancelled' where id='<batch_uuid>';
```

china 域同理。

## 7. SEO / GEO 融合要点（生成时主动嵌入）

> 内容生成阶段就把 SEO/GEO 友好结构嵌入，避免后期改翻译。
> SEO/GEO 是内容字段的一部分，不是独立的外置内容包：china 写入每篇 `*.article.json` 的 `seo` / `geo`；course 写入 lesson / KP / question 的标题、标签、解释、entities、sources 等字段。

### 7.1 SEO（搜索引擎）友好

- **标题**（`title_zh` / `title_i18n`）：含核心关键词，每条不超过 40 字。china 文章标题可参考"`<具体主体> + <类目长尾词>`"，例：「四川火锅的辣是怎么来的？」（含"四川火锅"主词 + "为什么"问句长尾）。
- **首句 BLUF**（china `sentences.seq_no=1` / course KP `explanation_md` 第一句）：把答案和定义放在第一句，便于摘要抓取。
- **多语言对齐**：5 语言版本不仅是翻译，还要把目标语言常用搜索词嵌入（如 vi 用越南人搜中文学习的高频词 "học tiếng Trung"）。
- **结构化数据 schema.org**：china 在单篇 JSON 的 `seo.schema_type` 写 `Article`；course KP 写 `LearningResource` / `HowTo` 等元字段，运行时由 web 端注入 JSON-LD。

### 7.2 GEO（AI 答案引擎）友好

- **问句式标题**：china 文章 30% 以问句式命名（"为什么…?" "…是什么?"），AI 引擎更易识别 QA 对。
- **小颗粒答案**：每条 sentence ≤ 80 字最佳；KP `explanation_md` ≤ 80 字（与 PRD 对齐）；让 AI 引擎能直接整段引用。
- **可引用单位**：每条数据自带稳定的 `code`（`china_articles.code` 12 位 / `course_knowledge_points.kp_code`）作为永久 URL slug，例：`https://zhiyu.app/china/A1B2C3D4E5F6` 永远稳定，AI 引擎索引后不会失效。
- **来源透明**：JSON-LD 中加 `author: { "@type":"Organization", "name":"Zhiyu" }` + `dateModified`，AI 答案引擎更愿意引用有出处的内容。
- **llms.txt**：把每个类目的种子描述和持续更新文章索引输出为 `https://zhiyu.app/llms.txt` 的章节。生成 china 类目内容时，category 的 `description_i18n.en` 必须是完整可独立成章的一句话（已 seed）。

### 7.3 GEO 与合规共同约束

- GEO 需要高事实密度，但不得为了“可引用”而制造争议性、成人性或敏感政治内容。
- AI 摘要句必须可直接面向东南亚学习者展示，避免让越南、泰国、印尼用户误解为政治评论或价值输出。
- 当搜索热点本身涉及敏感议题时，首期内容跳过该选题，改选低风险文化、语言或生活类热点。

> 详细策略见 [marking/seo/](../marking/seo/) 与 [marking/geo/](../marking/geo/)。生成 AI 不需要全读，但每个 dataspec 文件末尾会给出该业务的"必带"SEO/GEO 字段清单。
