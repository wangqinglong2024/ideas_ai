# 发现中国 · 01 · 生成流程与目录约定

> 本文件给生成 AI 看的"操作手册"。读完即可开工。

---

## 1. 单批次工作流（每个类目独立循环）

```
准备
  ├─ 选定类目（01..12）
  ├─ 选定本批次主题（如"八大菜系"）
  └─ 决定文章数量（推荐 5 篇/批）

生成
  ├─ 给每篇文章拟标题（zh + 拼音 + 4 语言翻译）
  ├─ 给每篇文章写 8–15 句正文，5 语言齐全 + 拼音
  └─ 输出：articles.jsonl / sentences.jsonl / manifest.json

自检
  ├─ 字数：title ≤ 40，pinyin ≤ 200，sentence content ≤ 400
  ├─ 5 语言全齐（zh/en/vi/th/id）
  ├─ pinyin 带声调，按音节空格分
  └─ 句子顺序 seq_in_batch 从 1 开始无跳号

提交
  └─ 把整个批次目录交给 PM / 导入脚本
```

---

## 2. 目录与文件命名

```
content/01-china/data/articles/<类目目录>/
  batch_<YYYYMMDD>_<3位序号>.articles.jsonl
  batch_<YYYYMMDD>_<3位序号>.sentences.jsonl
  batch_<YYYYMMDD>_<3位序号>.manifest.json
```

类目目录见 [00-index.md §4](./00-index.md)。

**批次序号**：当天若有多批，从 `001` 递增。同一类目同一天可以多批，互不冲突。

**单文件上限**：单个 jsonl 不超过 2000 行，超出请拆批次。articles 一般不会超，sentences 取决于句数。

## 3. manifest.json（每批次必带，导入脚本据此工作）

```jsonc
{
  "batch_code": "batch_20260515_001",
  "domain": "china",
  "import_type": "article+sentence",
  "category_code": "02",
  "files": [
    { "path": "batch_20260515_001.articles.jsonl",  "rows": 5,  "sha256": "<填>" },
    { "path": "batch_20260515_001.sentences.jsonl", "rows": 52, "sha256": "<填>" }
  ],
  "generator": {
    "model": "claude-opus-4.7",
    "operator": "ai-runner-01",
    "generated_at": "2026-05-15T10:11:23+08:00",
    "prompt_template": "china_article_v1"
  },
  "auto_publish": false,
  "notes": "中国美食 · 第 1 批 5 篇 · 八大菜系（鲁/川/粤/苏/闽）"
}
```

字段说明：
- `batch_code`：与文件名前缀一致
- `category_code`：本批次所有文章共享同一个类目编码（不同类目不要混批）
- `files[].sha256`：可由生成 AI 留空，导入脚本会重新计算并填回
- `auto_publish`：默认 `false`（推荐人工点检后再发布）；若内容已 100% 校对过，可置 `true` 让导入脚本调 `fn_publish_article`
- `prompt_template`：用于追溯哪个生成模板，方便后期改进

## 4. 同批次内的"本地引用"机制

> 句子要关联文章，但 article 的 uuid / code 都由 DB 在导入时生成。AI 怎么把句子挂到正确的 article？

**约定**：在同一批次内使用 `__article_local_id`（任意字符串，批次内唯一）作为占位。

`articles.jsonl` 一行：
```json
{"__article_local_id":"a1","category_code":"02","title_pinyin":"...","title_i18n":{...}}
```

`sentences.jsonl` 多行：
```json
{"__article_local_id":"a1","seq_in_batch":1,"pinyin":"...","content_zh":"...","content_en":"...","content_vi":"...","content_th":"...","content_id":"..."}
{"__article_local_id":"a1","seq_in_batch":2,"pinyin":"...","content_zh":"...",...}
{"__article_local_id":"a2","seq_in_batch":1,"pinyin":"...","content_zh":"...",...}
```

**导入脚本行为**：
1. 先逐条 insert article，调 `fn_gen_article_code()` 拿到真实 `code` 与 `id`，建立 `__article_local_id → uuid` 映射
2. 再逐条 insert sentence，按 `(__article_local_id, seq_in_batch)` 排序，对每个 article 调 `fn_next_sentence_seq()` 顺序分配 `seq_no`

> AI **不要**在 jsonl 里写 `seq_no` 或 `code`/`id`/`article_id`。只写本地占位字段。

## 5. 幂等键

| 表 | 业务唯一键 | 重复策略 |
|----|-----------|---------|
| `china_articles` | `(category_id, title_i18n->>'zh')` 推荐 + 同批次 `__article_local_id` 防止同批重复 | 同标题已存在 → 跳过该行（写 errors.jsonl 警告） |
| `china_sentences` | `(article_id, content_zh)` 推荐 | 同文章 + 同中文内容已存在 → 跳过 |

> 真正的物理唯一是 `articles.code` 和 `sentences.(article_id, seq_no)`，由 DB 保证。
> 业务唯一只是为了"重复跑同一文件不会插重复内容"。

如果你确实需要更新已有文章（比如修正翻译），不要重跑导入；走 web-admin 的编辑接口。

## 6. 生成阶段不要做的事

❌ 不要写 `code`（文章 code）
❌ 不要写 `seq_no`（句子序号）
❌ 不要写 `id` / `article_id`（uuid）
❌ 不要写 `audio_url_zh` / `audio_status` / `audio_*`（TTS 是用户触发）
❌ 不要写 `created_at` / `updated_at` / `published_at`
❌ 不要写 `created_by` / `updated_by`
❌ 不要把 `status` 设为 `published`（一律 `draft`，运营在管理端发布）
❌ 不要在 jsonl 里加注释（除文件第 1 行 `// schema=...`）
❌ 不要省略 5 语言中的任何一个
❌ 不要在 sentence 里使用换行符（用 `\n` 转义，但通常根本不需要换行）
❌ 不要使用繁体字
❌ 不要混入英文标点到中文里（中文用全角 ，。？！；：""''《》）
