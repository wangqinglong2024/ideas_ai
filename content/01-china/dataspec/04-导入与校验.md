## 1. 导入脚本（计划路径）

`system/scripts/db/import_china_content.ts`（待实现，骨架见下文）。

### 1.1 用法

```bash
# dry-run（仅校验，不写库）
pnpm tsx system/scripts/db/import_china_content.ts \
  --manifest content/01-china/data/articles/02-cuisine/batch_20260515_001.manifest.json \
  --dry-run

# 正式导入
pnpm tsx system/scripts/db/import_china_content.ts \
  --manifest content/01-china/data/articles/02-cuisine/batch_20260515_001.manifest.json \
  --operator 00000000-0000-0000-0000-000000000001
```

### 1.2 内部行为（伪代码）

```ts
// 1. 解析 manifest
const m = readJSON(args.manifest);
assert(m.domain === 'china');

// 2. 读两个 jsonl 并校验 sha256
const articles = readJsonl(dirname(args.manifest) + '/' + m.files[0].path);
const sentences = readJsonl(dirname(args.manifest) + '/' + m.files[1].path);
verifyHash(articles, m.files[0].sha256);
verifyHash(sentences, m.files[1].sha256);

// 3. 行级校验（用 zod schema 与 DB CHECK 一致）
const errors = validate(articles, ARTICLE_ZOD);
errors.push(...validate(sentences, SENTENCE_ZOD));
writeErrors(args.manifest + '.errors.jsonl', errors);
if (args.dryRun) return;

// 4. 反查 category_id
const catMap = await sql`select code, id from zhiyu.china_categories`;

// 5. insert articles, 拿到 uuid 与 code
const localIdToArticleId: Record<string, {id: string, code: string}> = {};
for (const a of articles) {
  if (errors.has(a.__article_local_id)) continue;
  const code = await sql`select zhiyu.fn_gen_article_code() as code`;
  const row = await sql`
    insert into zhiyu.china_articles (code, category_id, title_pinyin, title_i18n, status, created_by)
    values (${code}, ${catMap[a.category_code]}, ${a.title_pinyin}, ${rawClient.json(a.title_i18n)}, 'draft', ${args.operator})
    on conflict (code) do nothing
    returning id, code
  `;
  localIdToArticleId[a.__article_local_id] = row;
}

// 6. insert sentences（按 __article_local_id + seq_in_batch 排序）
const sorted = sentences.sort((x, y) => {
  if (x.__article_local_id !== y.__article_local_id)
    return x.__article_local_id.localeCompare(y.__article_local_id);
  return x.seq_in_batch - y.seq_in_batch;
});
for (const s of sorted) {
  const article = localIdToArticleId[s.__article_local_id];
  if (!article) continue;
  const seqNo = await sql`select zhiyu.fn_next_sentence_seq(${article.id})`;
  await sql`
    insert into zhiyu.china_sentences (article_id, seq_no, pinyin, content_zh, content_en, content_vi, content_th, content_id)
    values (${article.id}, ${seqNo}, ${s.pinyin}, ${s.content_zh}, ${s.content_en}, ${s.content_vi}, ${s.content_th}, ${s.content_id})
  `;
}

// 7. 可选自动发布
if (m.auto_publish) {
  for (const article of Object.values(localIdToArticleId)) {
    await sql`select zhiyu.fn_publish_article(${article.id})`;
  }
}

// 8. 汇总
console.log({success: ..., failed: ..., article_codes: [...]});
```

> jsonb 写入注意：drizzle-orm 0.30 + postgres-js 双重编码 jsonb，必须用 `rawClient.json(obj)`，详见 user memory `drizzle-jsonb.md`。本脚本直接用 postgres-js raw client。

## 2. 校验规则（与 DB CHECK 完全一致）

### 2.1 article 行

```ts
const ARTICLE_ZOD = z.object({
  __article_local_id: z.string().min(1).max(32),
  category_code: z.enum(['01','02','03','04','05','06','07','08','09','10','11','12']),
  title_pinyin: z.string().min(1).max(200),
  title_i18n: z.object({
    zh: z.string().min(1).max(40),
    en: z.string().min(1).max(40),
    vi: z.string().min(1).max(40),
    th: z.string().min(1).max(40),
    id: z.string().min(1).max(40),
  }).strict(),
}).strict();
```

### 2.2 sentence 行

```ts
const SENTENCE_ZOD = z.object({
  __article_local_id: z.string().min(1).max(32),
  seq_in_batch: z.number().int().min(1).max(9999),
  pinyin: z.string().min(1).max(600),
  content_zh: z.string().min(1).max(400),
  content_en: z.string().min(1).max(400),
  content_vi: z.string().min(1).max(400),
  content_th: z.string().min(1).max(400),
  content_id: z.string().min(1).max(400),
}).strict();
```

### 2.3 跨行校验

- 同 `__article_local_id` 的 sentence 行，`seq_in_batch` 必须从 1 开始连续（1,2,3,...N），不允许跳号或重复
- 每个 sentence 行的 `__article_local_id` 必须能在 articles.jsonl 中找到
- articles.jsonl 中 `__article_local_id` 不允许重复
- manifest 中 `category_code` 与所有 article 行的 `category_code` 必须一致

## 3. errors.jsonl 格式

校验失败时生成 `<batch_code>.errors.jsonl`：

```jsonl
{"file":"batch_20260515_001.articles.jsonl","line":3,"local_id":"a3","field":"title_i18n.th","error":"required"}
{"file":"batch_20260515_001.sentences.jsonl","line":12,"local_id":"a2","seq_in_batch":5,"field":"pinyin","error":"contains digit tone (jia4 ge2)"}
{"file":"batch_20260515_001.sentences.jsonl","line":20,"local_id":"a3","seq_in_batch":3,"field":"seq_in_batch","error":"gap: expected 2"}
```

修正方式：
1. 编辑生成的 jsonl，修正报错行
2. 重新计算 sha256，更新 manifest
3. 再次跑导入（幂等：已成功的行不会重复插）

## 4. 幂等与回滚

### 4.1 幂等

- 同一 batch_code 重复跑：导入脚本检测 `china_import_runs` 表（如有）的状态，已 `imported` 直接跳过
- article 业务唯一键：`(category_id, lower(title_i18n->>'zh'))`；已存在则跳过该行
- sentence 业务唯一键：`(article_id, content_zh)`；已存在则跳过

### 4.2 回滚（按文章 code）

```sql
-- 找出本批次所有 article code（从 errors / 日志拿）
with target as (
  select id from zhiyu.china_articles
  where code in ('A1B2C3D4E5F6','...')
)
delete from zhiyu.china_sentences where article_id in (select id from target);
delete from zhiyu.china_articles where id in (select id from target);
```

> 不推荐生产环境物理删除；优先用 `fn_delete_article` 软删，30 天后 cron 清理。

## 5. 发布流程（导入完成后）

1. 默认 `status='draft'`，`china_articles_select_published` RLS 策略对 anon/auth 不可见
2. 运营在 web-admin 文章列表页点击"发布"按钮
3. 后端调 `select zhiyu.fn_publish_article('<uuid>')`
4. RPC 校验：至少 1 条 sentence + status 必须是 draft → 改为 `published`，写 `published_at`
5. 应用端 RLS 立即放行

如果 manifest 设了 `auto_publish: true`，导入脚本在第 7 步自动调用 `fn_publish_article`，省略人工点检。**生产环境推荐 `false`**。

## 6. 故障速查

| 现象 | 排查 |
|------|------|
| `CHINA_ARTICLE_CODE_GEN_FAILED` | RPC 5 次随机生成 code 都撞了，几乎不可能；如出现说明 `china_articles` 极端密度，重跑即可 |
| `CHINA_ARTICLE_PUBLISH_NO_SENTENCES` | sentences.jsonl 没生成或 sentence 行全部校验失败；检查 errors.jsonl |
| `CHINA_SENTENCE_SEQ_OVERFLOW` | 单文章超过 9999 句；不可能出现，除非误用 |
| jsonb CHECK 报错 `?& array['zh','en','vi','th','id']` | `title_i18n` 缺少某 key；查 errors.jsonl 找 `field=title_i18n.<lang>` |
| `length must be ...` | 字段超长；按 [02 §1.1 / §2.1](./02-字段与JSONL格式.md) 表格截断或重写 |
