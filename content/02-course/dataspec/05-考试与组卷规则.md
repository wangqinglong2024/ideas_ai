# 课程 · 05 · 考试与组卷规则

> 本节定义 `course_exams` 表的 jsonl/json 输出格式（试卷模板，**不是**学员答卷）。
> 试卷模板存的是**抽题蓝图（blueprint）**，运行时从已发布题库实时抽题。

---

## 1. 试卷四种范围

| scope_type | 中文 | 关联 | 时长 | 通过分 | 一节几套 |
|------------|------|------|------|-------|--------|
| `lesson_quiz` | 节末小测 | lesson | 不限时（建议 5min） | 60 | 一节 1 套 |
| `chapter_test` | 章测 | chapter | 15min | 70 | 一章 1 套 |
| `stage_exam` | 阶段考 | stage | 45min | 70 | 一阶段 1 套 |
| `hsk_mock` | HSK 模考 | stage（仅 hsk 主题） | 90min | 60（按 HSK 标准） | 一 stage 多套 |

前 3 种 (`lesson_quiz/chapter_test/stage_exam`) **每范围最多 1 套有效**（DB 唯一索引 `(scope_type, scope_ref_id)`）；`hsk_mock` 不唯一。

---

## 2. 试卷模板文件命名

```
content/02-course/data/_exams/
  ├── lesson_quiz/
  │     ec-2-3-1.exam.json      ← 文件名 = lesson_code
  │     ec-2-3-2.exam.json
  │     share-0-1-1.exam.json
  ├── chapter_test/
  │     ec-2-3.exam.json        ← 文件名 = <track>-<stage>-<chapter>
  │     ec-2-4.exam.json
  ├── stage_exam/
  │     ec-2.exam.json          ← 文件名 = <track>-<stage>
  │     ec-3.exam.json
  └── hsk_mock/
        hsk-3-mock-01.exam.json ← <track>-<stage>-mock-<两位序号>
        hsk-3-mock-02.exam.json
```

每文件一个 JSON 对象（不是 jsonl，因为单文件一份卷）。

## 3. 试卷 JSON 字段

```jsonc
{
  "scope_type":   "lesson_quiz",                   // 必填
  "scope_ref": {                                   // 用业务码 + 类型，导入脚本反查 uuid
    "lesson_code":   "ec-2-3-1",                   // scope_type=lesson_quiz 时
    "chapter_code":  null,                         // scope_type=chapter_test 时填 "ec-2-3"
    "stage_code":    null                          // scope_type in ('stage_exam','hsk_mock') 时填 "ec-2" 或 "hsk-3"
  },
  "title_zh":     "电商优惠与支付节末小测",
  "title_i18n": {
    "zh":"电商优惠与支付节末小测","en":"...","vi":"...","th":"...","id":"..."
  },
  "pass_score":   60.00,
  "total_score":  100.00,
  "time_limit_minutes": null,                      // null=不限时；章测 15；阶段考 45；HSK 90
  "blueprint": {
    "items": [ /* 见 §4 */ ],
    "kp_filter": {                                 // 必填，按 scope_type 不同
      "lesson_code":  "ec-2-3-1",                  // lesson_quiz 必填
      "chapter_code": null,                        // chapter_test 必填
      "stage_code":   null,                        // stage_exam / hsk_mock 必填
      "include_review_pct": 20                     // 0–50；见 §5
    },
    "shuffle":          true,                      // 选项打乱
    "show_explanation": false,                     // 考试期间不立刻反馈
    "allow_skip":       true                       // lesson_quiz / chapter_test = true；stage_exam / hsk_mock = false
  },
  "is_published": false                            // AI 一律 false
}
```

> 导入脚本会按 scope_type 反查 scope_ref_id（lesson_code/chapter_code/stage_code → uuid），并把 `kp_filter.lesson_code` 等替换为 lesson_id 等 uuid 后写入 jsonb。

## 4. blueprint.items（出卷蓝图）

每条 item 描述一类题的抽取规则：

```jsonc
{
  "q_type":           "mcq_meaning",   // 12 类之一
  "count":            3,                // 抽几道
  "difficulty_range": [1, 2],           // 可选：[min,max]，1–6
  "exam_scope_filter": "lesson_quiz",   // 必填：与该试卷 scope_type 对齐
  "kp_filter_extra":  null              // 可选 jsonb：在 blueprint.kp_filter 之上叠加约束
}
```

`exam_scope_filter`：抽题时只取 `course_questions.exam_scope @> array['<filter>']` 的题，确保题与卷的层级匹配。

## 5. 4 种范围的 blueprint 标准模板

### 5.1 `lesson_quiz`（节末小测）

- 题量：约 10 题，总分 100，每题 10 分（动态均分）
- 配比：**80% 本节 KP + 20% 上节复习 KP**（`include_review_pct: 20`）
- 题型：从该节 12 KP 中按 KP 配比抽题，全题型混合

```jsonc

  "scope_type": "lesson_quiz",
  "scope_ref": { "lesson_code": "ec-2-3-1", "chapter_code": null, "stage_code": null },
  "title_zh": "ec-2-3-1 优惠券与折扣 节末小测",
  "title_i18n": { /* ... */ },
  "pass_score": 60.00,
  "total_score": 100.00,
  "time_limit_minutes": null,
  "blueprint": {
    "items": [
      { "q_type": "mcq_meaning",       "count": 3, "exam_scope_filter": "lesson_quiz" },
      { "q_type": "listen_pick",       "count": 2, "exam_scope_filter": "lesson_quiz" },
      { "q_type": "fill_blank_choice", "count": 2, "exam_scope_filter": "lesson_quiz" },
      { "q_type": "sort_words",        "count": 2, "exam_scope_filter": "lesson_quiz" },
      { "q_type": "type_zh_ime",       "count": 1, "exam_scope_filter": "lesson_quiz" }
    ],
    "kp_filter": {

      "include_review_pct": 20
    },
    "shuffle": true,
    "show_explanation": false,
    "allow_skip": true
  },
  "is_published": false
}
```

- 例：HSK3 模考 80 题（听力 40 + 阅读 30 + 综合 10）

## 6. 出卷顺序（重要）

```
1. 先生成 KP + Question + lesson_kp（保证题库充足）
2. 校验：每节 exam_scope=lesson_quiz 题 ≥ 12（实际抽 10）
3. 校验：每章 exam_scope=chapter_test 题 ≥ 20（实际抽 15）
4. 校验：每阶段 exam_scope=stage_exam 题 ≥ 40（实际抽 30）
5. HSK 主题：每 stage exam_scope=hsk_mock 题 ≥ 100（实际每套抽 80）
6. 校验通过后才生成对应的 .exam.json
```

如果题库不足，**先回到内容批次补题**，不要降低试卷题量。

---

## 7. 动态均分

引入数据后，前端发卷流程：
1. 按 blueprint 抽题，得到当前生效题数 `N`（可能因题被下架而 < 计划数）
2. 每题分 = `total_score / N`
3. 已发出的卷不重算

所以试卷 `total_score` 是**目标满分**而非实际分配，AI 只填 100 即可。

---

## 8. 自检清单

- [ ] 文件命名与 scope_type 一致（lesson_quiz → lesson_code）
- [ ] `scope_ref` 三个码恰好填一个，其它为 null
- [ ] `time_limit_minutes` 与范围匹配（lq=null/5；ct=15；se=45；hm 按 HSK 等级）
- [ ] `pass_score ≤ total_score`，HSK 模考 `pass_score=60`
- [ ] `blueprint.items` 每条 `q_type` 在 12 类内，`count > 0`
- [ ] 每条 item `exam_scope_filter` 与试卷 scope_type 一致
- [ ] sum(items.count) ≈ 节测 10 / 章测 15 / 阶段考 30 / HSK 模考 80（按 HSK 等级调整）
- [ ] `include_review_pct`：lq=20 / ct=30 / se=0 / hm=0
- [ ] `allow_skip`：lq/ct=true / se/hm=false
- [ ] HSK 模考的 `scope_ref.stage_code` 必须以 `hsk-` 开头
- [ ] `is_published=false`
