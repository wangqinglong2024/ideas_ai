# 课程 · 01 · 生成流程与目录约定

> 给生成 AI 看的"操作手册"。
> 课程域比 china 域复杂得多，请务必区分**骨架层**与**内容层**。

---

## 1. 三层文件组织

```
content/02-course/data/
  ├── _skeleton/                ← 骨架层（先于内容生成，一次性建好）
  │     tracks.jsonl            5 行
  │     stages.jsonl            25 行
  │     chapters.jsonl          ~148 行
  │     lessons.jsonl           ~888 行
  │     skeleton.manifest.json
  │
  ├── share/                    ← Stage 0 共享层
  │   stage0/
  │     chapter01-pinyin/
  │       lesson01/
  │         batch_xxx.kp.jsonl
  │         batch_xxx.lesson_kp.jsonl
  │         batch_xxx.questions.jsonl
  │         batch_xxx.manifest.json
  │       lesson02/
  │       ...
  │     chapter02-listening/
  │     chapter03-strokes/
  │     chapter04-radicals/
  │
  ├── ec/   fc/   hsk/   dl/    ← 4 主题
  │   stage1/chapter01/lesson01/{kp,lesson_kp,questions,manifest}
  │   ...
  │   stage6/chapter06/lesson06/{kp,lesson_kp,questions,manifest}
  │
  ├── _exams/                   ← 试卷模板（文件名 = scope）
  │   lesson_quiz/<lesson_code>.exam.json
  │   chapter_test/<chapter_code>.exam.json   chapter_code = <track>-<stage>-<chapter>
  │   stage_exam/<track>-<stage>.exam.json
  │   hsk_mock/hsk-<stage>-mock-<nn>.exam.json
  │
  └── _media/                   ← 媒资清单（音频/图片/笔顺）
      media_<batch>.assets.jsonl
```

### 1.1 命名规范

- `lesson_code` = `<track>-<stage_no>-<chapter_no>-<lesson_no>`，例：`ec-2-3-1`
- `chapter_code` = `<track>-<stage_no>-<chapter_no>`，例：`ec-2-3`（仅文件名用，不入库）
- 目录中 `chapter01` / `lesson01` 用 2 位零填充，便于排序
- Stage 0 章目录加描述后缀：`chapter01-pinyin` / `chapter02-listening` / `chapter03-strokes` / `chapter04-radicals`，便于人读
- 主题章目录可以加描述也可以纯数字（`chapter03` 或 `chapter03-coupon`），推荐加描述

### 1.2 批次（batch）

- 一个批次对应**一个节**的全部内容（KP + lesson_kp + Question）
- 一个 manifest 对应一个批次
- 批次粒度小是为了：失败影响范围小、并行生成友好、回滚精准

## 2. 生成顺序（严格遵守）

### 2.1 全局首次

```
1. _skeleton/tracks.jsonl              ← 5 主题种子（migration 已含，可跳过）
2. _skeleton/stages.jsonl              ← 25 阶段（PM 一次填齐）
3. _skeleton/chapters.jsonl            ← ~148 章
4. _skeleton/lessons.jsonl             ← ~888 节
5. 跑 import_skeleton.ts，把 4 张目录表灌入
```

### 2.2 内容生成（每节循环）

```
对每个 lesson_code：
  1. 读 lesson 学习目标 + 词汇清单（PM 提供）
  2. 生成 batch_xxx.kp.jsonl（8–14 行）
  3. 生成 batch_xxx.lesson_kp.jsonl（与 kp 同行数，写 position 1..N）
  4. 生成 batch_xxx.questions.jsonl（每 KP 4–6 题，含 exam_scope）
  5. 生成 batch_xxx.manifest.json
  6. 自检后交导入
```

### 2.3 试卷模板（节内容齐全后）

- 节末小测：节 KP 全发布后写 `_exams/lesson_quiz/<lesson_code>.exam.json`
- 章测：本章所有节 KP 全发布后写
- 阶段考：本阶段所有章 KP 全发布后写
- HSK 模考：HSK 主题某 stage 全发布后写

## 4. 同批次内的本地引用

> 与 china 类似但更复杂。

| 跨表引用 | 通过 |
|---------|-----|
| Question → KP | `kp_code` 字符串。AI 不知道真实 `kp_code`（DB 生成），所以用 `__kp_local_id` 占位 |
| lesson_kp → KP | 同上 `__kp_local_id` |
| lesson_kp → Lesson | `lesson_code`（已知，AI 直接写） |

### 4.1 `kp.jsonl` 中

```json
{"__kp_local_id":"k1","kp_type":"word","primary_track":"ec","title_zh":"优惠券",...}
{"__kp_local_id":"k2","kp_type":"word","primary_track":"ec","title_zh":"满减",...}
```

### 4.2 `lesson_kp.jsonl` 中

```json
{"lesson_code":"ec-2-3-1","__kp_local_id":"k1","position":1,"is_new_in_lesson":true}
{"lesson_code":"ec-2-3-1","__kp_local_id":"k2","position":2,"is_new_in_lesson":true}
```

### 4.3 `questions.jsonl` 中

```json
{"__kp_local_id":"k1","q_type":"mcq_meaning","exam_scope":["practice","lesson_quiz"],"payload":{...}}
{"__kp_local_id":"k1","q_type":"listen_pick","exam_scope":["practice"],"payload":{...}}
```

### 4.4 导入脚本流程

```
1. 解析 kp.jsonl，逐条 insert，DB 触发器生成 kp_code，建立 __kp_local_id → {id, kp_code} 映射
2. 解析 lesson_kp.jsonl，按 (lesson_code, kp) 反查 lesson_id 与 kp_id，insert
3. 解析 questions.jsonl，按 __kp_local_id 反查 kp_id，DB 触发器生成 q_code，insert
   注意：触发器会校验 (kp_id, q_type, payload_hash) 唯一性，去重
```

> 跨主题复用 KP（如共享层 KP 被 ec 第 1 节复用）：复用时**不要再生成新 KP**，只在 `lesson_kp.jsonl` 写：`{"lesson_code":"ec-1-1-1","kp_code":"kp_share_pinyin_000001","position":1,"is_new_in_lesson":false}`。
> 注意此时直接用 `kp_code`（已存在的 DB code）而非 `__kp_local_id`。导入脚本两种都识别。

## 5. 幂等键

| 表 | 业务唯一键（导入幂等） | 物理唯一键（DB） |
|----|--------------------|----------------|
| `course_tracks` | `code` | `code` |
| `course_stages` | `(track_code, stage_no)` | `(track_id, stage_no)` |
| `course_chapters` | `(stage_id, chapter_no)` | `(stage_id, chapter_no)` |
| `course_lessons` | `code` | `code` |
| `course_knowledge_points` | 由 `(primary_track, kp_type, payload_hash_of_content)` 推导生成 `kp_code`；但实际 dedup 通过 `(primary_track, title_zh, kp_type)` 软去重 | `kp_code` |
| `course_lesson_kp` | `(lesson_id, kp_id)` | `(lesson_id, kp_id)` |
| `course_questions` | `(kp_id, q_type, payload_hash)` | 同上 |

> 题目去重 hash：`encode(sha256(payload::text::bytea), 'hex')`，由 DB 计算，AI 不需要写。

重复跑同 batch：
- KP：title 已存在 → 跳过该行（写 errors 警告"duplicate KP title"）
- Lesson_kp：(lesson, kp) 已绑定 → 跳过
- Question：payload_hash 重复 → 跳过

## 6. 生成阶段不要做的事（与 china 类似 + 课程域专项）

❌ 不要写 `id` / `kp_code` / `q_code`（DB 生成）
❌ 不要写 `created_at` / `updated_at` / `published_at`
❌ 不要把 `is_published` 设为 `true`（一律 `false`，发布动作在管理端）
❌ 不要写 `source_batch_id`（导入脚本根据 manifest 自动填充）
❌ 不要写 `payload_hash`（DB 触发器计算）
❌ 不要写 `report_count` / `version`（DB 默认）
❌ 不要在 KP `content` 或 Question `payload` 里写音频 URL 真实地址；用占位 `cdn://placeholder/<...>`，后期由媒资脚本替换
❌ 不要混用 5 主题的内容（一个批次只生成一个 lesson_code 的内容）
❌ 不要在题目 `exam_scope` 同时写 `chapter_test` + `stage_exam` + `hsk_mock`（互斥规则，详见 [05](./05-考试与组卷规则.md)）
❌ 不要为 Stage 0（共享层）使用 word/grammar/sentence/dialog 类型的 KP
❌ 不要为 Stage 1-6 使用 Stage 0 类型为 pinyin 的 KP（要复用就用 lesson_kp 关联，不要新建）
❌ 不要使用繁体字
❌ 不要在 jsonl 里加注释（除文件第 1 行 `// schema=...`）

## 7. 自检清单（提交批次前必跑）

- [ ] manifest 的 `lesson_code` 与目录路径一致（`ec/stage2/chapter03/lesson01` ↔ `ec-2-3-1`）
- [ ] kp.jsonl 与 lesson_kp.jsonl 行数相等（除非有跨主题复用）
- [ ] lesson_kp.jsonl 的 position 从 1 开始连续
- [ ] questions.jsonl 行数 = sum(每 KP 的题数)，落在 `4*kp_count` ~ `6*kp_count`
- [ ] 每条 question 的 `__kp_local_id` 在 kp.jsonl 中存在（或是已存在的 kp_code）
- [ ] 每条 question 的 `exam_scope` 至少含 `practice`，且 chapter/stage/hsk_mock 三者互斥
- [ ] KP `kp_type` 在 `[pinyin,hanzi,word,phrase,grammar,sentence,dialog]` 中
- [ ] Question `q_type` 在 12 类中
- [ ] KP `primary_track` 与 manifest.track 一致（除非是引用共享层）
- [ ] KP `content` jsonb 按 7 类差异结构齐全（详见 [03](./03-知识点KP格式与7类差异.md)）
- [ ] Question `payload` 按 12 类差异结构齐全（详见 [04](./04-题目Question格式与12类差异.md)）
- [ ] 5 语言全齐（KP 的 translations / explanation_md，Question 的 explanation 与 i18n options）
- [ ] 拼音带声调，无数字调
