# 三 · 并发与冲突处理（模板）

> 根据 F2-用户中的边界情况，定义技术上的并发处理方案。

---

## 场景：[例：两人同时编辑同一文章]

**方案**：`[最后保存为准 / 乐观锁（version 字段） / 悲观锁（select for update） / 幂等键 + 先到先得]`

**实现方式**：

```
- 数据库：[在表上加 version int / 加 updated_at 比较]
- 接口：[入参带 if-match: <version> 头 / 写接口必须带 idempotency-key]
- 失败提示：[409 + 最新版本数据回吐 / 重试指引]
```

---

## 场景：[例：高并发下的库存扣减]

**方案**：`[原子 SQL update where stock > 0 / Redis 计数器 / 队列串行化]`

**实现方式**：略

## 限流策略

| 接口范围 | 限流维度 | 阈值 | 触发后行为 |
|---------|---------|------|-----------|
| 默认 | IP | 60/分 | 429 + Retry-After |
| 默认 | 用户 | 120/分 | 同上 |
| 高成本（如 TTS、AI 生成） | IP | 20/分 | 429 |
| 全局搜索 | 用户 | 30/分 | 429 |
