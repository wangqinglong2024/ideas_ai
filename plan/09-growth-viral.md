## 一、增长飞轮模型

### 1.1 核心飞轮

知语的增长飞轮围绕三大模块构建：**发现中国**（免费内容引流） → **系统课程**（12 级体系驱动付费） → **游戏模式**（12 款游戏驱动留存与传播）。

```
发现中国 12 类目免费内容 → SEO/社交 吸引注册
       ↓
L1-L3 免费课程 + 12 款免费游戏 → 体验产品价值
       ↓
内容墙 / 课程墙（L4+ 需会员） → 付费转化
       ↓
深度学习 + 游戏段位提升 → 成就感 + 社交炫耀
       ↓
分享成绩卡 / 推荐链接 → 好友注册
       ↓
好友付费 → 推荐人获 20% 佣金 → 持续推荐
       ↓
新用户进入飞轮 → 循环加速
```

### 1.2 飞轮加速器

| 加速器 | 来源 | 作用 |
|--------|------|------|
| **发现中国（12 类目免费内容）** | [plan/02](02-discover-china.md) | SEO 自然流量入口，降低获客成本 |

## 二、推荐返佣系统

### 2.1 核心机制

| 规则 | 说明 |
|------|------|
| **推荐方式** | 每个用户拥有**唯一的推荐链接**，链接直接绑定上下级关系 |
| **无需邀请码** | 被推荐人点击链接注册即自动绑定，无需手动输入任何邀请码 |
| **单层推荐** | 仅推荐人获得被推荐人付费的佣金（不做多级分销） |
| **佣金比例** | 被推荐人**每次付费**的 **20%** 作为推荐人佣金 |
| **佣金形式** | **知语币**（不可提现、不可转让），仅可在平台内消费（兑换会员、皮肤、道具） |
| **所有用户可参与** | 免费用户和会员均可生成推荐链接 |

### 2.2 推荐链接

**链接格式**：

```
https://zhiyu.app/ref/{用户唯一ID}
```

- 每个用户注册时自动生成唯一推荐链接
- 链接中不含邀请码，ID 即身份标识
- 被推荐人通过链接访问时，`ref` 参数存入浏览器（Cookie + localStorage，7 天有效）
- 注册时自动绑定推荐关系，**无需用户手动输入任何内容**

**归属规则**：

| 规则 | 说明 |
|------|------|
| 绑定时机 | 点击推荐链接 → ref 参数自动存储 → 注册时绑定 |
| 有效期 | 点击链接后 **7 天内** 注册有效 |
| 唯一性 | 每个用户只能被一个推荐人绑定（首次归属原则） |
| 不可改 | 绑定后推荐关系永久不变 |
| 自推禁止 | 用户不能推荐自己（后端校验 IP + 设备指纹） |

### 2.3 佣金规则

| 规则 | 说明 |
|------|------|
| 佣金比例 | 固定 **20%**（不设阶梯，简单透明） |
| 佣金来源 | 被推荐人的**每次续费**都产生佣金（非仅首次） |
| 佣金形式 | **知语币**（1 币 = $0.01），30 天冷却期后到账 |
| 退款扣回 | 被推荐人退款 → 对应佣金自动扣回 |

**佣金计算示例**：

```
张三分享自己的推荐链接 → 李四点击注册
李四购买月度会员 $2

佣金 = $2 × 20% = $0.40 = 40 知语币

下一个月李四续费 $2
佣金 = $2 × 20% = 40 知语币（持续获得）
```

### 2.4 知语币说明

推荐佣金以**知语币**形式发放，不可提现、不可转让。知语币详细规则见 [06-business-model.md](06-business-model.md) 第五章。

| 规则 | 说明 |
|------|------|
| 比价 | 1 币 = $0.01（固定） |
| 用途 | 兑换会员 / 皮肤 / 特效 / 赛季通行证 |
| 不可做 | 不可提现、不可转让、不可交易 |
| 有效期 | 获得后 365 天 |

> **为什么不做现金提现**：Paddle 是 Merchant of Record，仅处理向用户收款，不支持向终端用户打款。越南用户提现至银行卡会触发 FCT（外国承包商税）、SBV AML（越南国家银行反洗钱）、FATF 合规风险。Apple App Store 明确禁止 "cash referral rewards"。对 $2/月的低价产品，$0.4 佣金的跨境汇款手续费在经济上不可行。虚拟币闭环消费是唯一合理的推广激励形式。

### 2.5 反刷单机制

推荐返佣的最大风险是刷单。核心防线：

| 风险行为 | 检测方式 | 处理 |
|---------|---------|------|
| 自推自买 | 推荐人和被推荐人同 IP / 同设备指纹 | 不发放佣金 |
| 批量假注册 | 同一推荐链接单日新增 > 10 人 | 冻结推荐功能，人工审核 |
| 刷退款 | 被推荐人频繁购买后退款 | 扣回佣金，标记风险账号 |
| 多账号互刷 | 同一设备/IP 多个账号互推 | 合并计算，超额部分不发放 |
| 支付工具关联 | 同一支付卡关联 > 3 个账号 | 超出部分不计佣金 |

**量化阈值**：
- 单个推荐人每日新增上限：**50 人**（超出自动冻结）
- 单日新增 > 10 人：触发人工审核
- 被推荐用户 7 日留存 < 10% 的批次：该批次推荐不计佣金

**处罚机制**：
- 轻度违规：佣金冻结 + 警告
- 重度违规：永久取消推荐资格，扣回所有未提现佣金

### 2.6 被推荐人视角

- 通过推荐链接注册的流程与普通用户**完全一致**
- 不显示任何"您被 XXX 邀请"类文案
- 被推荐人自动获得**首单 9 折优惠**
- ref 参数静默处理，用户无感

### 2.7 推广中心

每个用户的个人主页内设「推广中心」入口：

- 显示个人唯一推荐链接（一键复制）
- 显示推荐数据：推荐人数 / 总佣金 / 可提现金额
- 佣金明细记录（时间、来源、金额、状态）
- 分享功能：复制链接 / 生成海报（带二维码）/ 分享到社交媒体

## 四、游戏段位系统

> 参考王者荣耀段位设计，12 款游戏共享统一段位。

### 4.1 段位层级

| 段位 | 图标 | 小段位数 | 每段星数 | 该段总星数 | 累计星数 |
|------|------|---------|---------|----------|---------|
| 青铜 | 🥉 | III → I（3 段） | 3 🌟 | 9 🌟 | 9 |
| 白银 | 🥈 | III → I（3 段） | 3 🌟 | 9 🌟 | 18 |
| 黄金 | 🥇 | IV → I（4 段） | 3 🌟 | 12 🌟 | 30 |
| 铂金 | 💎 | IV → I（4 段） | 4 🌟 | 16 🌟 | 46 |
| 钻石 | 💠 | V → I（5 段） | 4 🌟 | 20 🌟 | 66 |
| 星耀 | ⭐ | V → I（5 段） | 5 🌟 | 25 🌟 | 91 |
| 王者 | 👑 | 排名制 | — | — | 91+ |

### 4.2 核心规则

| 规则 | 说明 |
|------|------|
| **赢了** | +1 🌟 |
| **输了** | -1 🌟 |
| **最低** | 1 🌟（青铜 III 最低 1 星，不会再掉） |
| **升段** | 星满自动升入下一段位 |
| **掉段** | 当前段位星数归零时掉回上一段位 |
| **跨游戏联动** | 12 款游戏共享同一段位，任何游戏的胜负都影响段位 |
| **赛季重置** | 每赛季（3 个月）段位软重置，保留基础段位 |

### 4.3 段位设计逻辑

```
新手：从青铜 III 1🌟 开始
       ↓
保持 50% 胜率：约 180 局可达王者（每日 5 局 ≈ 36 天）
保持 60% 胜率：约 120 局可达王者（每日 5 局 ≈ 24 天）
       ↓
王者段位：按排名竞争，持续活跃才能保持排名
```

### 4.4 段位排行榜

| 维度 | 刷新周期 | 可见范围 |
|------|---------|---------|
| 全服段位排行 | 实时 | 所有用户 |
| 好友段位排行 | 实时 | 好友间可见 |
| 赛季排行 | 赛季结束时结算 | 全服公开 |

赛季结束奖励：

| 排名 | 奖励 |
|------|------|
| 王者 Top 10 | 赛季限定头像框 + 专属称号 |
| 王者 | 限定段位边框 |
| 星耀 | 赛季纪念徽章 |
| 钻石 | 赛季参与徽章 |

### 4.5 段位社交传播

- 段位升级时自动弹出分享引导（含段位卡片）
- 个人主页展示当前段位 + 赛季最高段位
- 分享卡片展示段位图标（社交炫耀）
- 好友可以互相发起 PK 挑战

## 五、用户生命周期管理

### 5.1 生命周期阶段

```
新用户 → 活跃用户 → 付费用户 → 忠诚用户 → 传播用户
  ↓          ↓          ↓          ↓          ↓
沉默用户 ← 流失用户（各阶段都可能流失）
```

### 5.2 各阶段策略

#### 新用户（注册后 0-7 天）

| 策略 | 说明 | 依据 |
|------|------|------|
| 新手引导 | 选目标 → 选模块 → 创角色 → 进入学习 | [plan/01](01-executive-summary.md) §5.3 |
| 试学引导 | 引导体验 L1-L3 免费课程 | [plan/03](03-course-system.md) |
| 游戏入门 | 引导体验全部 12 款免费游戏 | [plan/04](04-game-design.md) |
| 文化入门 | 引导浏览发现中国 12 类目免费内容 | [plan/02](02-discover-china.md) |
| 首周签到 | 连续 7 天学习获得递增奖励 | — |
| 首日推送 | 欢迎 + "你的第一课已准备好" | — |

#### 活跃用户（D7+，未付费）

| 策略 | 说明 |
|------|------|
| 学习提醒 | 每日固定时间推送（用户可设置） |
| 进度展示 | 强调"你已经学了 X 个汉字 / 达到 HSK X 水平" |
| 付费墙 | 到达 L4 课程边界时弹出会员引导 |
| 限时优惠 | 注册 7 天内专享首月优惠 |
| 游戏段位 | 段位提升带来的成就感驱动继续活跃 |

#### 付费用户

| 策略 | 说明 |
|------|------|
| 专属内容 | 解锁全部 L4-L12 课程 |
| 学习报告 | 周报 / 月报推送学习成果 |
| 续费提醒 | 到期前 7 天、3 天、1 天温馨提醒 |
| 证书体系 | 每通过一个 Level 获得数字证书（可分享） |

#### 忠诚 / 传播用户

| 策略 | 说明 |
|------|------|
| 推荐返佣 | 突出 20% 佣金激励，推广中心入口 |
| KOC 培养 | 邀请创作学习分享内容 |
| 内测资格 | 新功能优先体验 |
| 专属称号 | 社区内的特殊标识 |

### 5.3 流失挽回

| 流失阶段 | 策略 |
|---------|------|
| 3 天未登录 | Push：你的学习进度在等你 |
| 7 天未登录 | Push + 邮件：你的连续学习天数即将中断 |
| 14 天未登录 | Push + 邮件：限时回归优惠 |
| 30 天未登录 | 邮件：专属回归礼包（免费 7 天会员体验） |
| 60 天未登录 | 最后一封（不再频繁打扰） |

## 六、病毒传播系数

### 6.1 K-Factor 模型

```
K = 平均每用户邀请数 × 邀请转化率

目标：K > 0.5（每 2 个用户带来 1 个新用户）
理想：K > 1（自然增长，无需付费获客）
```

### 6.2 各阶段预期

| 阶段 | K 值估计 | 核心驱动 |
|------|---------|---------|
| MVP | 0.2-0.3 | 推荐链接 + 分享卡片 |
| 增长期 | 0.4-0.6 | 返佣体系成熟 + KOL 推荐 |
| 成熟期 | 0.6-0.8 | 品牌口碑 + 自然传播 |
| 目标 | > 0.8 | 接近自增长临界点 |

### 6.3 K-Factor 优化杠杆

| 杠杆 | 预期影响 | 来源 |
|------|---------|------|
| 推荐返佣 20% | 提升邀请意愿 | 本文档 §2 |
| 游戏 PK 社交 | 提升自然分享频次 | [plan/04](04-game-design.md) §4.5 |
| 段位炫耀 | 提升分享动力 | 本文档 §4 |
| 数字证书分享 | 高价值分享内容 | [plan/03](03-course-system.md) §6.3 |
| 文化内容分享 | 低门槛传播载体 | [plan/02](02-discover-china.md) |

## 七、增长实验计划

### 7.1 A/B 测试优先级

| 实验 | 假设 | 指标 |
|------|------|------|
| 付费墙位置 | L3 完成时 vs L3 期间预告 | 付费转化率 |
| 推荐返佣比例 | 15% vs 20% vs 25% | 推荐数量 / 成本 |
| 新手引导步数 | 3 步 vs 5 步 | D1 留存 |
| 分享卡片样式 | 成果展示型 vs 优惠引导型 | 分享率 |
| 推送时间 | 早 8 点 vs 晚 8 点 vs 自定义 | 打开率 |
| 首单折扣 | 9 折 vs 8 折 vs 首月 $0.99 | 被推荐人转化 |

### 7.2 数据驱动

- 所有核心页面埋点（注册、首课、付费、分享、推荐）
- 周度增长会议：复盘数据 → 制定实验 → 执行 → 评估
- 月度增长报告：渠道 ROI、LTV、CAC、K-Factor
