# 商业模式设计

> 知语 Zhiyu 变现体系与定价策略
> 版本：v6.0（终版） | 日期：2026-04-16

---

## 一、收入结构总览

知语 Zhiyu 采用 **"免费游戏 + 会员订阅"** 双引擎模式。12 款游戏全部免费开放吸引用户，通过课程和发现中国内容的分层访问驱动会员转化。

```
收入结构
  ├── 会员订阅（核心收入，占比 70-80%）
  ├── 游戏皮肤 / 特效 / 赛季通行证（占比 10-20%）
  └── 推荐返佣（驱动增长，占比 5-10%）
```

### 核心策略

| 策略 | 说明 |
|------|------|
| **游戏全免费** | 12 款游戏对所有用户完全免费，零门槛上手 |
| **内容分层** | 非会员可体验前 3 个模块 / 前 3 级课程，会员解锁全部 |
| **皮肤变现** | 游戏皮肤和特效是唯一的游戏内付费项，纯装饰不影响平衡 |
| **未来拓展** | 游戏皮肤和特效收费空间大，随游戏生态成熟持续增长 |

## 五、推荐返佣体系

### 5.1 机制

```
用户 A 分享邀请链接 → 用户 B 注册 → 用户 B 首次付费
     ↓
用户 A 获得返佣（B 首次支付金额的 20%）
用户 B 获得优惠（首单 9 折）
```

### 5.2 返佣规则

| 规则 | 说明 |
|------|------|
| 返佣比例 | 被推荐人首次付费的 **20%** |
| 生效条件 | 被推荐人完成首次付费且未退款（7天确认期） |
| 提现门槛 | 累计 ¥50 起提现 |
| 提现方式 | 银行转账 / PayPal / 平台余额（可购买会员/皮肤） |
| 推荐上限 | 无上限 |
| 有效期 | 推荐关系绑定永久有效 |

### 5.3 推荐人激励

| 累计推荐人数 | 额外奖励 |
|------------|---------|
| 5 人 | 赠送 1 个月会员 |
| 10 人 | 赠送稀有皮肤礼包 |
| 20 人 | 赠送 3 个月会员 |
| 50 人 | 赠送年度会员 |
| 100 人 | 赠送终身会员 |

## 六、付费转化路径

### 6.1 免费用户转化漏斗

```
下载/访问 → 浏览发现中国前 3 模块（免费）
           → 学习 L1-L3 课程（免费）
           → 玩全部 12 款游戏（免费）
                         ↓
              遇到内容墙 / AI 功能限制
                         ↓
              弹出优惠引导 → 会员订阅 → 付费
```

### 6.2 付费墙触发点

| 触发场景 | 展示内容 |
|---------|----------|
| L3 课程学完，进入 L4 | “解锁全部 12 级课程，会员仅 ¥22/月” |
| 发现中国基础内容看完，点击深度内容 | “成为会员，解锁全部 12 个类目的深度内容” |
| AI 功能达到免费上限 | "升级会员，无限使用 AI 助手" |
| 升级考试（免费用户不可用） | "通过考试获得证书，成为会员解锁" |
| 看到高级课程对应的游戏题目 | "学习 L4+ 课程，解锁更多游戏题库" |

### 6.3 促销策略

| 策略 | 时机 | 内容 |
|------|------|------|
| 新用户 7 天优惠 | 注册后 7 天内 | 年度会员 7 折 |
| 节日促销 | 春节/中秋/双11 | 终身会员限时优惠 |
| 学习里程碑 | 完成 L3 / 连续学习 7 天 | 弹出限时折扣 |
| 流失挽回 | 30天未登录 | Push 通知 + 专属优惠 |
| 游戏成瘾转化 | 游戏段位达到白银 | 引导"学更多课程，游戏题目更多更有趣" |

## 七、支付实现

### 7.1 Stripe

| 维度 | 说明 |
|------|------|
| 角色 | 支付处理商（公司为商户记录方） |
| 优势 | 费率低、支持中国公司直接签约、全球覆盖、API 完善 |
| 费率 | ~2.9% + $0.30/笔 |
| 支持方式 | 信用卡、Apple Pay、Google Pay、本地支付（Stripe 支持的地区） |
| 订阅管理 | Stripe Billing 内置订阅管理（续费/取消/退款） |
| 税务 | 可启用 Stripe Tax 自动计算（后期） |

### 7.2 支付流程

```
用户选择套餐 → 前端调用 Stripe Checkout → Stripe 处理支付
                                              ↓
                                    Stripe Webhook 通知后端
                                              ↓
                                    后端更新用户会员状态
                                              ↓
                                    前端刷新权限
```

### 7.3 iOS 特殊处理

- iOS App 内的数字内容购买**必须走 Apple IAP**（苹果 30% 抽成）
- 策略：引导用户通过 **Web 端** 完成订阅（Stripe），绕开苹果税
- App 内不展示购买按钮，只展示"已有会员？登录恢复"
- Web 端定价 = 标准价；iOS IAP 定价 = 标准价 + 30%（如果必须上架 IAP）

### 7.4 中国公司签约说明

- **Stripe 支持中国大陆企业直接注册**（需营业执照 + 法人身份证）
- 初期以中国公司身份签约 Stripe，接收国际支付
- 结算货币：美元/人民币（根据 Stripe 中国的结算规则）
- 后续如注册新加坡公司，可迁移 Stripe 账户至新主体
