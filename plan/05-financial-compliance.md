# PlayLingo — 财务合规与公司架构方案

> 中国有限公司 + 东南亚运营 + 新加坡服务器
> 版本：v2.0 | 日期：2026-04-15

---

## 一、公司架构

### 1.1 推荐架构

```
中国有限公司（主体）
├── 产品开发与技术中心
├── 内容生产与审核
├── 财务与行政
├── 知识产权持有
│
├── [Phase 3+] 新加坡公司（月收入 >$5,000 后）
│   ├── 海外收入结算
│   ├── 东南亚客户合同
│   └── 应用商店开发者账号
│
└── [可选] 越南/印尼当地代理
    └── 本地化运营支持
```

### 1.2 阶段选择

**Phase 0-2（前 6 个月）**：仅中国主体，不增加复杂度
- 零额外注册成本，运营简单
- 海外收入通过 Paddle/LemonSqueezy 代收（Merchant of Record 模式）

**Phase 3+（年收入 >$50K）**：注册新加坡公司
- 新加坡企业税 17%（前 $200K 减免至 ~8.5%）
- 海外收入直接在新加坡结算
- 东南亚用户/企业信任度高
- 触发条件：月收入稳定 >$5,000 或准备接 B2B 合同

## 二、收入结算

### 2.1 初创期方案

```
用户付费（美元/越南盾/印尼盾）
    │
    ▼
Paddle / LemonSqueezy（Merchant of Record）
    │  ← 处理全球税务、退款、发票
    │  ← 抽成约 5-10%
    ▼
中国银行外汇账户
    │
    ▼
人民币结汇入对公账户
```

**为什么用 MoR 模式而非直接 Stripe**：
- Stripe 不直接支持中国大陆企业
- MoR 平台（Paddle/LemonSqueezy）代为处理全球税务合规
- 省去每个国家单独处理增值税的复杂度
- 对接本地支付（MoMo/GoPay）通过 MoR 平台集成

### 2.2 规模化后方案

```
新加坡公司 Stripe 账户
    ├── 直接对接 Stripe + MoMo/Midtrans
    ├── 留存运营资金
    └── 技术服务费 → 中国母公司
```

## 三、税务规划

### 3.1 中国税务

| 税种 | 税率 | 优化建议 |
|------|------|---------|
| 企业所得税 | 25%（小微 5%） | 保持小微资质（≤300 万） |
| 增值税 | 6% | 出口软件服务申请免征 |
| 研发加计扣除 | 100% | AI 和游戏开发支出可加计 |

### 3.2 东南亚税务

```
越南（无本地实体）：
  · 2022 年起要求外国数字服务提供商注册纳税
  · 简化税率：增值税 5% + 企业所得税 5%
  · 如通过 MoR 平台（Paddle），由平台代扣代缴

印尼：
  · 增值税 12%（2025 年起）
  · 如通过 Midtrans/Google Play/App Store，由平台代扣
  · >1000 笔交易可能触发本地代表机构要求
```

## 四、数据合规

### 4.1 各国要求

| 国家 | 法规 | 应对 |
|------|------|------|
| 越南 | PDPD 2023 | 数据存新加坡，隐私政策越南语版，跨境传输评估 |
| 印尼 | PDP 2022 | 同上，隐私政策印尼语版 |
| 新加坡 | PDPA | 服务器在新加坡，天然合规 |
| 中国 | PIPL | 用户数据不回传中国，开发团队访问脱敏数据 |

### 4.2 数据架构原则

```
1. 用户数据存储在新加坡（腾讯云新加坡节点）
2. 隐私政策多语言版本（越南语/印尼语/英语/中文）
3. 首次注册获取明确同意
4. AI 对话数据不用于训练，仅提供即时服务
5. 支付信息由第三方处理，不自行存储卡号
6. 用户可删除账号和所有数据
```

## 五、佣金系统合规

### 5.1 法律风险分析

| 国家 | 相关法规 | 风险评估 | 合规方案 |
|------|---------|---------|---------|
| 越南 | 竞争法（禁止非法多层分销） | 低风险 | 严格一级，佣金基于实际交易 |
| 印尼 | 电商法规（GR 80/2019） | 低风险 | 监控交易量阈值 |
| 泰国 | 反金字塔法 | 低风险 | 单级推荐不构成金字塔 |
| 菲律宾 | 消费者法（禁止连锁分销） | 低风险 | 同上 |

### 5.2 合规要点

```
1. 结构合规
   · 仅一级分销，无下线收益
   · 佣金仅基于被推荐人的实际付费交易
   · 不奖励"纯拉人头"

2. 支付合规
   · 佣金打款通过正规支付渠道（MoMo/银行转账）
   · 单次提现需 KYC 验证（身份证/银行账户）
   · 大额提现（>$100）需额外验证

3. 税务合规
   · 佣金属于推荐人的应税收入
   · 平台保留所有交易记录供税务审计
   · 年度超过一定金额时提醒用户报税

4. AML 合规
   · 越南 2022 年反洗钱法要求第三方支付报告可疑交易
   · 我们通过正规支付平台（MoMo）打款，平台已负责 AML 合规
   · 设置月度提现上限（$500），超出需人工审核

5. 退出预案
   · 系统支持佣金 ↔ 会员奖励热切换
   · 如遇法律政策变化可 24 小时内切换为非现金奖励模式
```

```

```

特殊注意：
  · 不允许在 App 内引导到外部订阅页面
