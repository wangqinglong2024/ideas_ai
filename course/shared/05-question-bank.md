# shared · 固定题型库与批量出题引擎

> 4 轨道 × 12 阶段 × 全部考核 **只用以下 10 类固定题型**。题型由系统根据知识点的"内容单位"自动匹配，并在节小测 / 章单元测 / 阶段综合考核中**随机抽取**。
>
> 所有题型 **不依赖 ASR 语音识别**；用户只需"听 / 读 / 选 / 拖 / 写"。
> 所有题型可由"批量导入一行文本"自动生成，无需逐题人工命题。

---

## 1. 10 类固定题型

| 代号 | 名称 | 题面 | 选项形式 | 适用单位 |
|:---:|---|---|---|---|
| **Q1** | 听辨选项 | 播放中文音频（点喇叭） | 4 个文本选项（汉字/词/句）| 字 / 词 / 句 |
| **Q2** | 辨音选项 | 显示 1 个汉字/词 | 4 个可点播放的音频，选读对的 | 字 / 词 |
| **Q3** | 拼音→汉字 | 显示拼音（带声调） | 4 个汉字/词文本 | 字 / 词 |
| **Q4** | 汉字→拼音 | 显示汉字/词 | 4 个拼音（含声调干扰）| 字 / 词 |
| **Q5** | 汉字→意思 | 显示中文 | 4 个母语翻译选项 | 字 / 词 / 句 |
| **Q6** | 意思→汉字 | 显示母语翻译 | 4 个中文选项 | 字 / 词 / 句 |
| **Q7** | 选词填空 | 句子挖一空 | 4 个候选词 | 短句 / 中句 / 长句 |
| **Q8** | 词序排序 | 句子词序打乱 | 拖拽排序 | 短句 / 中句 / 长句 |
| **Q9** | 声调辨识 | 显示去声调拼音 + 播放音频 | 4 组声调组合 | 字 / 词（仅阶段 1-4 必含）|
| **Q10** | 偏旁部首 | 显示汉字 | 4 个部首选项 | 字（仅阶段 1-3 必含）|

> 10 类已穷举：所有"听/读/写"维度的静默考核都能用这 10 类拼接出来。
> 不再增加新题型；新内容只往这 10 类里灌数据。

## 2. 内容单位 → 可用题型映射表

系统按知识点的 `unit_type` 字段自动决定可抽题型集合：

| `unit_type` | 字数 | 主用阶段 | 可抽题型集 |
|---|:---:|:---:|---|
| `char` | 1 | S1-S4 | Q1, Q2, Q3, Q4, Q5, Q6, Q9, Q10 |
| `word` | 2 | S2-S6 | Q1, Q2, Q3, Q4, Q5, Q6, Q9 |
| `phrase` | 3-4 | S4-S7 | Q1, Q3, Q4, Q5, Q6 |
| `short_sentence` | 5-8 | S5-S8 | Q1, Q5, Q6, Q7, Q8 |
| `mid_sentence` | 9-15 | S7-S10 | Q1, Q5, Q6, Q7, Q8 |
| `long_sentence` | 16-25 | S9-S11 | Q1, Q5, Q6, Q7, Q8 |
| `complex_sentence` | 26-40 | S11-S12 | Q5, Q6, Q7, Q8 |

> **不收文章 / 段落**。整段阅读由「发现中国」「小说专区」承载，与课程系统解耦。
> 阶段 12 最大单位 = 复合长句（≤ 40 字），仍是单行可导入数据。

## 3. 节小测 12 题随机抽题规则

```
输入：本节 12 个知识点 + 每个知识点的 unit_type
对每个知识点 i：
  available = 该 unit_type 的可抽题型集
  forced_q = 该知识点首次出现 → 必含 Q1（听辨）保证音感
  题型[i] = forced_q if 首次出现 else random.choice(available)
输出：12 道题（覆盖 4-6 种题型，每节随机不同）
```

干扰项生成：
- 4 选 1 类（Q1, Q3, Q5, Q6）：干扰项从**同章其他 11 节**同 `unit_type` 的知识点池随机抽 3 条
- Q2 / Q9：干扰项 = 该字/词的常见误读（系统预存"易错音 / 易错调"表）
- Q4：干扰项 = 同声母不同声调 / 同韵母不同声母 / 整体替换 1 个声调
- Q7：干扰项 = 同词性近义词
- Q8：直接对原句分词后打乱
- Q10：干扰项 = 字形相近的部首（如 氵/冫/丬）

## 6. 批量导入数据格式（CSV / JSONL）

每行 = 1 个知识点。系统读到此行后自动生成 TTS 音频 + 配套题。

### CSV 列定义
```csv
id,track,stage,chapter,lesson,kpoint,unit_type,chinese,pinyin,vi,th,key_point
ec-s01-c01-l01-k01,ec,1,1,1,1,word,老板,lǎobǎn,ông chủ,เจ้านาย,电商对话开场万能称呼
```

### JSONL 等价
```json
{"id":"ec-s01-c01-l01-k01","track":"ec","stage":1,"chapter":1,"lesson":1,"kpoint":1,
 "unit_type":"word","chinese":"老板","pinyin":"lǎobǎn","vi":"ông chủ","th":"เจ้านาย",
 "key_point":"电商对话开场万能称呼"}
```

### 字段约束（导入时校验）

| 字段 | 必填 | 校验 |
|---|:---:|---|
| `id` | ✓ | 正则 `^(ec\|fc\|hsk\|dl)-s\d{2}-c\d{2}-l\d{2}-k\d{2}$` |
| `track` | ✓ | 枚举 ec/fc/hsk/dl |
| `stage`/`chapter`/`lesson`/`kpoint` | ✓ | 1-12 |
| `unit_type` | ✓ | 7 类枚举（见第 2 节）|
| `chinese` | ✓ | ≤ 40 字（按 unit_type 长度限制再校验）|
| `pinyin` | ✓ | GB/T 16159 带声调；多音节空格分词 |
| `vi` | ✓ | 越南语母语审校通过 |
| `th` | ✓ | 泰语母语审校通过 |
| `key_point` | ✓ | ≤ 30 字，1 行讲解 |

> 导入失败行写入错误日志；通过行立即可在 App 内出现。
> **整套系统无须按知识点逐题命题**——所有题面 / 选项 / 干扰项由本规范第 3 节算法自动生成。

## 7. App 端通用展示组件（4 轨共用）

每个知识点页面固定布局：

```
┌─────────────────────────────────────────┐
│  [🔊 播放]   你好                         │  ← chinese 大字
│              nǐ hǎo                      │  ← pinyin 拼音
│              Xin chào / สวัสดี            │  ← vi/th 母语对照
├─────────────────────────────────────────┤
│  💡 key_point：见面问候万能词              │
├─────────────────────────────────────────┤
│  [示范视频 ▶]   [我会了]   [收藏]         │
└─────────────────────────────────────────┘
```

- 🔊 按钮：播放 TTS（首次播放后缓存音频文件）
- 母语对照默认显示用户注册时选择的语言（vi 或 th），可一键切换
- 视频按钮：跳转该知识点的 30s 课程视频（如已录制；未录制时灰显）
- 无任何"录音""跟读评分"按钮

## 8. 拼音 / 笔画类知识点的特殊处理

仅阶段 1 第 1-2 章使用；不参与上述 10 类题型，使用专用 3 题型：

| 代号 | 名称 | 题面 | 选项 |
|:---:|---|---|---|
| **P1** | 声韵母拼读 | 播放音节 | 4 组声+韵+调组合 |
| **P2** | 笔顺选择 | 显示汉字静态 | 4 段笔顺动画，选正确 |
| **P3** | 笔画数 | 显示汉字 | 4 个数字选项 |

> 阶段 1 完成后（≈ 30h），P1-P3 不再出现，全面切换到 Q1-Q10。
