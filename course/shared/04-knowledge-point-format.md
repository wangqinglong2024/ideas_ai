# shared · 知识点统一规范（批量导入）

> 4 轨道 × 12 阶段 × 12 章 × 12 节 × 12 知识点 共 8.3 万行，必须遵循本规范。
> 这是课程内容、课程视频、考核生成、私域日推的**单一数据源**。

---

## 1. 知识点字段（CSV / JSONL，每行 1 条）

| 字段 | 必填 | 类型 | 校验规则 |
|---|:---:|---|---|
| `id` | ✓ | string | 正则 `^(ec\|fc\|hsk\|dl)-s\d{2}-c\d{2}-l\d{2}-k\d{2}$` |
| `track` | ✓ | enum | `ec / fc / hsk / dl` |
| `stage` | ✓ | int | 1-12 |
| `chapter` | ✓ | int | 1-12 |
| `lesson` | ✓ | int | 1-12 |
| `kpoint` | ✓ | int | 1-12 |
| `unit_type` | ✓ | enum | 7 类，见第 2 节 |
| `chinese` | ✓ | string | ≤ 40 字（按 unit_type 上限再校验）|
| `pinyin` | ✓ | string | GB/T 16159 带声调；多音节空格分词 |
| `vi` | ✓ | string | 越南语母语审校通过 |
| `th` | ✓ | string | 泰语母语审校通过 |
| `key_point` | ✓ | string | ≤ 30 字，1 行核心讲解 |

### 示例（电商轨道）

```csv
id,track,stage,chapter,lesson,kpoint,unit_type,chinese,pinyin,vi,th,key_point
ec-s01-c01-l01-k01,ec,1,1,1,1,word,老板,lǎobǎn,ông chủ,เจ้านาย,电商对话开场万能称呼
ec-s05-c03-l02-k04,ec,5,3,2,4,short_sentence,能便宜点吗？,néng piányi diǎn ma?,Có thể rẻ hơn được không?,ลดได้อีกไหมคะ?,能+V+吗 砍价万能句型
```

JSONL 等价格式略（同字段名）。

## 2. `unit_type` 7 类枚举（核心约束）

| 值 | 字数 | 主用阶段 | 示例 |
|---|:---:|:---:|---|
| `char` | 1 | S1-S4 | 你 / 好 / 买 / 卖 / 贵 |
| `word` | 2 | S2-S6 | 老板 / 包邮 / 电话 / 朋友 |
| `phrase` | 3-4 | S4-S7 | 起订量 / 多少钱 / 包清关 |
| `short_sentence` | 5-8 | S5-S8 | 能便宜点吗？/ 我要这个。|
| `mid_sentence` | 9-15 | S7-S10 | 这款产品的 MOQ 是 500 件起。|
| `long_sentence` | 16-25 | S9-S11 | 如果数量超过 1000 件，我们可以谈进一步的折扣。|
| `complex_sentence` | 26-40 | S11-S12 | 在跨境贸易中，30% 定金加 70% 见提单副本付款的方式被广泛接受。|

> **不允许**整段 / 多句组合 / 文章。最大单位 = 单行复合长句 ≤ 40 字。
> 整段阅读 → 由 [`/china/`](../../china/) 与小说专区承载，与本课程系统解耦。

## 3. 一节 12 知识点的内容编排建议

每节 12 知识点采用"由易到难 + 同主题闭环"原则；**主单位由阶段决定**，同节内可下浮 1 级（如阶段 5 主用 `short_sentence`，节内仍可有 1-2 个 `word` 作为新词卡）。

| 顺序 | 类型建议（以阶段 5 为例）|
|:---:|---|
| 1-2 | 新词卡（`word` / `phrase`，本节关键词）|
| 3-9 | 核心句（`short_sentence`，主单位）|
| 10-11 | 句型变体 / 易错点 |
| 12 | 综合应用句 |

> 这是建议，非强制。具体节内编排由轨道负责人决定。

## 4. 与课程视频的对应

每个知识点 = 1 节 App 课 = 1 条 30s 课程视频。视频由平台自录。

| 视频段 | 时长 | 内容 | 字段映射 |
|---|:---:|---|---|
| 钩子 | 0-3s | 母语提问 / 痛点 | （视频独有）|
| 呈现 | 3-8s | 中文 + 拼音 + 母语 | `chinese` + `pinyin` + `vi`/`th` |
| 拆解 | 8-18s | 1 个关键点 | `key_point` |
| 示范跟读 | 18-26s | 讲师慢速 + 正常（**用户静默跟读，无 AI 评分**）| `chinese` |
| CTA | 26-30s | 关注 / 评论 / 入群 | （视频独有）|

---

## 5. 与考核题的对应

每个知识点 1 题考核，题型从 10 类固定题型中按 `unit_type` 自动抽取。完整映射表与抽题算法见 [`05-question-bank.md`](./05-question-bank.md)。

> 内容团队**只填 12 字段**，不写题目；题目由系统按 `unit_type` 自动生成。

---

## 6. 跨轨重复字词的处理

同一字 / 词在多轨同时出现时（如"老板"在电商和日常都用）：
- 同 `chinese` + 同 `unit_type` → 合并为同一"基础知识点 ID"（如 `base-laoban`）
- 各轨保留各自 `id`，但共享 SRS 记忆栈
- 用户在轨 A 学过 → 轨 B 节内仅作复习提示
- `key_point` 在不同轨道可不同（电商讲砍价，日常讲称呼）

---

## 7. 内容版权与原创

- ✅ 全部句子由内容团队原创或来自公版（古诗 / 成语 / 谚语）
- ❌ 不直接复制 HSK 官方教材原句
- ❌ 不复制其他 App（HelloChinese / SuperChinese）的句子
- ✅ 改写 / 同义重述允许，需标注"改写自 X 文化常识"

---

## 8. 审校与版本

- 每轮新知识点录入：母语审校 → 编辑校对 → 上线
- 上线后 7 天内学员反馈错误 → 48h 修订（同步课程 + 视频 + 考核）
- 知识点版本号 `v1.0` / `v1.1` ... 在数据库字段单独记录

---

## 9. 一行批量导入命令（后台 CLI）

```bash
# CSV 导入（推荐，便于 Excel 编辑）
$ zhiyu-cli kp:import --file ec-s01.csv --validate-only   # 仅校验
$ zhiyu-cli kp:import --file ec-s01.csv --commit          # 提交入库

# JSONL 导入（推荐 CI/CD 流水线）
$ zhiyu-cli kp:import --file ec-s01.jsonl --commit

# 校验失败行写入 ec-s01.errors.csv，不阻断成功行
```

入库后 5 分钟内：
1. TTS 自动生成中文音频（缓存到 OSS）
2. 自动按 `unit_type` 生成可用题型集
3. 节小测 / 章单元测 / 阶段综合考核可立即抽题
4. App 端无需发版，热加载新内容
