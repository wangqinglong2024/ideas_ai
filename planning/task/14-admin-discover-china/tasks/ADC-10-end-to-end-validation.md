# ADC-10 · 实现前后台闭环验收

## PRD 原文引用

- `planning/rules.md`：“列表 → 详情 → 关键交互（学习/游戏/解锁/购买/阅读）一条主链路。”
- `AD-AC-007`：“4 模块内容 CRUD + 发布。”
- `DC-AC-001~014` 覆盖 DC 前台验收。

## 需求落实

- 页面：后台 DC 编辑页、前台 `/discover` 链路。
- 组件：无新增。
- API：后台 CRUD + 前台查询。
- 数据表：DC 内容表、admin_audit_logs、events。
- 状态逻辑：后台发布后前台可见；撤回后前台不可见；权限门禁始终生效。

## 不明确 / 风险

- 风险：只测后台不测前台会漏内容状态。
- 处理：验收必须同时跑后台和前台链路。

## 技术假设

- E2E 使用 MCP Puppeteer 直连 3100/4100。

## 最终验收清单

- [ ] 后台创建文章并发布。
- [ ] 前台登录用户可读新文章。
- [ ] 未登录访问受限类目新文章被拦截。
- [ ] 撤回后前台不可读。
