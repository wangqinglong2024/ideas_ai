> ⚠️ **2025-11 变更**：系统内批量生成页面整体废弃；内容由运营线下生成/整理后直接导入；不设置复杂流转流程，学员举报在 P-A-6 处理。
> 详见 [function/02-course/ai/CHANGELOG-2025-11.md](../function/02-course/ai/CHANGELOG-2025-11.md)。本文档历史段落若与变更冲突，以 CHANGELOG 为准。

## 6.3 流程 B：批量导入 KP（最高频流程）

```
content_admin              web-admin                DB
  │                          │                      │
  │ 1.上传 KP CSV/JSON       │                      │
  ├─────────────────────────▶│ 2.本地解析 + 字段校验
  │                          │ 3.展示预览 / 错误行  │
  │ 4.确认导入               │                      │
  ├─────────────────────────▶│ 5.写入导入批次       │
  │                          ├─────────────────────▶ knowledge_points is_published=false
  │                          │                      │ source_batch_id / payload_hash
```

要点：
- 导入必须**幂等**：同文件 hash + 同 source_batch 不重复入库。
- 失败行可导出修正后再次导入。
- 每条 KP 保留来源批次、payload hash、操作者和导入时间。

---

## 6.4 流程 C：批量导入 Question（依赖 KP 已存在）

类似流程 B，输入是题目 CSV/JSON；每题必须绑定已存在 KP，并声明 `q_type`、可抽层级、payload、答案与解析。
- 关键约束：干扰项**只能引用已存在的真实词**；系统导入校验会拦截不存在的 KP / 选项引用。

## 6.5 流程 D：人工点检（每天日常工作）

```
content_admin 进入 P-A-4 / P-A-5
  → 按主题 / 题型 / 状态筛选
  → 查看详情或题目预览
  → 直接编辑字段并保存
  → 保存、发布、归档或删除
  → 节内全部 KP/Q 达到发布前置条件后，节卡片右上角 [发布] 按钮亮
```

---

## 6.6 流程 E：发布（节级开关）

```
content_admin 点节卡片[发布]
  ↓
后端校验：
  ✓ 节内 KP 全部已发布
  ✓ 每个 KP 至少有 N(=4) 道已发布题
  ✓ 5 语言内容齐全（zh/en/vi/th/id）
  ✓ 音频/图片 URL 可访问
  ↓
更新 lessons.is_published=true，触发 CDN 缓存预热（KP/Q/媒资 URL）
  ↓
学员端学习地图实时刷新（轮询 / WS 推送）
```

## 6.7 流程 F：学员学习一节（用户端最重要流程）

```
用户点[继续学习]
  ↓ GET /app/lesson/ec-2-3-1
  ↓ 服务端聚合：lessons + lesson_kp + knowledge_points + questions(每KP 抽 2~4) + 翻译(按 ui_lang)
  ↓
本地 SDK 预加载所有音频
  ↓
逐 KP 卡片流：[讲解卡] → [练习题×N] → 下一个 KP
  每答一题：
    POST /app/answer  { question_id, user_answer, duration_ms }
    服务端：
      ① 校验答案，写 user_answers
      ② 错→ user_srs 入 box1，due=明天
        对→ 若该 KP 此前已在 SRS，升一盒、改 due
      ③ 返回 { correct, correct_answer, explanation }
  ↓
节末小测 6 题
  ↓
POST /app/lesson/ec-2-3-1/finish
  ↓
服务端：写 user_progress.status=passed/in_progress, best_score, attempts++
  ↓
解锁判断：若节通过 → 看是否需要解锁下一节（同章顺次） / 章测 / 阶段考
```

## 6.8 流程 G：每日复习（SRS）

```
用户点[今日复习]
  ↓ GET /app/srs/today
  ↓ 服务端：
    SELECT kp_id FROM user_srs
    WHERE user_id=? AND due_at <= now()
    ORDER BY box ASC, due_at ASC LIMIT 30
  ↓ 对每个 KP 抽 1 道题（题型轮换：今天选择，明天听音，避免背答案）
  ↓ 返回题包
  ↓
用户做题，每题：
  POST /app/answer (context_type=srs_review)
  → 对：box +1，due += 间隔表
  → 错：box=1，due=明天
```

---

## 6.9 流程 H：考试

```
用户点[阶段考]
  ↓ POST /app/exam/{exam_id}/start
  ↓ 服务端：
    ① 按 exam.blueprint 抽题 → snapshot=[q_id1, q_id2, ...]
    ② 写 user_exam_attempts (started_at, snapshot)
    ③ 返回快照题包（不返回答案）
  ↓
倒计时 + 全程不反馈 + 不能退出
  ↓
POST /app/exam/{attempt_id}/submit { answers:[{q_id, user_answer}, ...] }
  ↓
服务端：
  ① 逐题判分（同 user_answers 写流水，context_type=stage_exam）
  ② 计算总分，写 user_exam_attempts.score / passed / finished_at
  ③ 若 passed=true 且是阶段考 → 解锁下一阶段所有节
  ④ 错题入 SRS box1
  ↓
返回结果页 + 错题清单
```

## 6.10 流程 I：学员举报 → 修正 → 影响范围

```
学员答题反馈页 🚩 举报
  ↓ POST /app/feedback { question_id, type, comment }
  ↓ 写 content_action_log target_type=question action=report
  ↓
管理端 P-A-6「学员举报处理」按 question_id 聚合：3+ 学员举报自动置顶
  ↓
content_admin 处理：
  · 确实错 → 编辑 questions.payload，version+1，按需要保存或发布
  · 不算错 → 标记 dismissed，给学员 push 解释
  ↓
修正后：
  · 历史 user_answers 保持不变（不追溯改分）
  · 新作答取最新版本
  · 若涉及音频 → 替换媒资或等待学员端再次命中缓存，必要时 CDN 失效该 URL
```

## 6.11 流程 J：内容修正与版本治理

```
content_admin 在 KP/Q 编辑抽屉手动修正
  ↓ 后端：
    ① 写 version+1 与修改 diff
    ② 保留旧版本归档记录
    ③ 如需批量修正，可通过新的导入批次覆盖同 code 内容
  ↓ 保存后按发布规则对学员端生效
```

## 6.12 流程 K：订阅与主题访问控制

```
用户访问 ec 主题学习地图
  ↓ 中间件：检查 users.subscription
     · 含 ec 或 ALL → 通过
     · 仅 fc → 提示升级，仅显示前 1 章试学
  ↓
试学完后强制订阅页
  ↓ 支付成功（Stripe / 本地支付网关）→ webhook 更新 subscription
```

---

## 6.13 流程 L：上线后的"内容版本治理"

| 场景 | 做法 |
|---|---|
| 一道题答案错了被举报 | 见流程 I，version+1，旧成绩不动 |
| 一个 KP 翻译要全平台改 | 编辑 → version+1，CDN 单文件失效 |
| 一节要改章节归属 | 改 `lessons.chapter_id`，`code` 字段保持，避免学员历史进度断 |
| 一个 stage 要废弃 | `is_published=false`，已通过该 stage 的学员保持，未达学员看不到 |

## 6.14 关键非功能流程

### 6.14.1 数据备份
- PG 每日全量 + WAL 流式（PITR），保留 30 天
- `user_*` 表是命脉，独立异地副本

### 6.14.2 监控
- 应用：错误率 / P95 延迟 / 在线人数
- 内容：每日导入失败行、待点检数量、举报数 Top
- 业务：DAU / 完课率 / 续费率

### 6.14.3 灰度
- 任何内容模板升级先在「沙箱主题」（不展示给学员）导入样本，运营人评分 ≥ 4 / 5 才推到正式主题。
