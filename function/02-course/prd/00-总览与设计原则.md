## 0.2 业务定位（一页讲清楚）

| 维度 | 设定 |
|---|---|
| 目标用户 | 东南亚（首发越南 / 泰国 / 印尼）成人零基础 ~ HSK4 |
| 四大主题（学习主题） | **电商（E-commerce）** / **工厂（Factory）** / **HSK（应试）** / **日常（Daily）** |
| 学习产物 | 听得懂、看得懂、能打字回复（**不练口语发音**） |
| 商业模式 | 免费试学 + 订阅制（按主题 / 全主题） |
| 核心成本压低点 | 内容生产 = 线下批量生成/整理 + 发布前检查；运营期对每个用户**零额外算力** |
| **明确不做** | 实时 AI 对话、AI 发音评测、AI 写作批改、千人千面学习路径 |

## 0.3 全局设计原则（之后所有决定的判据）

1. **内容静态化原则**：所有学员看到的题、音、图、讲解，都是**预生成的不可变资源**，CDN 分发。运行时只做"取下一道题 / 判分 / 写进度"。
2. **题型纯静默原则**：所有题目都能在**没有麦克风、没有手写板**的设备上完成（选择、连线、排序、点选、键盘输入即可）。
3. **主题同骨架原则**：5 个主题（share / EC / FC / HSK / DL）用**同一套表结构、同一套题型、同一套学习引擎**，只是内容不同——降低开发成本和管理成本。
4. **三层共享原则**：
   - **共享层**：拼音、声调、基础笔画、最基础 300 高频词 / 80 高频部首（5 个主题都用）
   - **主题层**：每个主题独有的词汇、场景、对话
   - **混合层**：句型 / 语法是共享的，但例句使用各自主题的词汇
5. **来源可追溯原则**：任何 KP、题目、媒资都必须保存来源批次、导入文件、版本号和修改记录，方便追溯、回滚和人工修正。
6. **应用端五语言与单一中文目标**：应用端 UI / 讲解 / 翻译适配 `zh / en / vi / th / id` 5 语言；中文教学目标本体保留一套原文（汉字 + 拼音 + 音频）。管理后台 UI 保持中文，后台多语言只指内容编辑字段支持五语言录入。
7. **进度可恢复**：所有用户进度只存"做到哪、答得对不对、需不需要复习"，**不存 AI 推理结果**。

## 0.4 术语表（之后文档所有出现都按这里）

| 术语 | 含义 |
|---|---|
| **Track（主题）** | 电商 / 工厂 / HSK / 日常，4 个业务主题；另有共享基础主题 share |
| **Stage（阶段）** | 一个主题内的难度阶梯，建议 6 个（入门→进阶→精通） |
| **Chapter（章）** | 一个阶段下的话题包，建议 6 章 |
| **Lesson（节）** | 一个章下的最小教学单元，默认 6 节，每节 10–20 分钟可完成 |
| **KnowledgePoint（知识点 / KP）** | 课程内容的最小颗粒，**7 类**（见 §0.5） |
| **Question（题）** | 围绕 1 个 KP 出的题目，**12 类题型**（见 02 文件） |
| **Lesson Bundle** | 一节 = 若干 KP 的有序列表 + 该节的节测配置 |
| **Stage Exam** | 阶段末考核，覆盖该阶段所有章 |
| **SRS** | 间隔重复，本系统采用 **Leitner 5 盒（简化）+ 错题加权** |

## 0.5 七类知识点（KP Type）

颗粒度从小到大：

| 编号 | 类型 | 例子 | 典型时长 |
|---|---|---|---|
| KP1 | **音节 / 拼音**（pinyin） | `mā` / `zhōng` | 10 秒 |
| KP2 | **汉字 / 部首**（hanzi） | 「好」/部首「氵」 | 20 秒 |
| KP3 | **词汇**（word） | 「价格」「质检」 | 30 秒 |
| KP4 | **短语 / 搭配**（phrase） | 「下订单」「请假条」 | 30 秒 |
| KP5 | **句型 / 语法**（grammar） | 「把字句」「比……还……」 | 60–90 秒 |
| KP6 | **短句**（sentence） | 「这个多少钱？」 | 30 秒 |
| KP7 | **对话 / 长句**（dialog） | 客服 3–5 轮对话 | 60–120 秒 |

> 数量级估算（全平台）：
> - 共享层 ≈ 1 200 KP（拼音 410 + 部首 80 + 高频词 600 + 通用句型 110）
> - 每主题独有 ≈ 2 500 KP
> - **全平台 KP 总量 ≈ 1 200 + 4 × 2 500 = 11 200 个**
> - 每个 KP 配 4–6 题 → 题库 ≈ **5 万–7 万道题**，线下批量生成与人工整理后导入。

## 0.6 学员一日学习闭环（产品 MVP 闭环）

```
登录 → 选主题 → 看到学习地图（阶段/章/节）
      ↓
    进入今日推荐节（=断点续学 + 1 节新内容 + 1 节复习）
      ↓
   节内循环：[KP讲解卡] → [2~4 题随堂练] → 下一个 KP
      ↓
   节末小测（6 题，错的进 SRS 队列）
      ↓
   章末测验 / 阶段末考试（解锁下一阶段）
```

## 0.7 内容生产流水线（管理端的灵魂）

```
[线下生成 / 人工整理] → [批量导入] → [人工点检] → [配音 / 配图入库] → [冻结发布] → [CDN 同步]
     ↑                                                                        ↓
     └────────────── 学员举报 / 人工修正 / 重新导入版本 ←──────────────────────┘
```

每个导入批次必须存：来源文件、批次号、操作者、导入时间、校验结果、版本号和原始 payload hash，保证**可追溯、可回滚、可比对**。
