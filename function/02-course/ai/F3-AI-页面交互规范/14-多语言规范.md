# F3-14 · 多语言（i18n）全局规范

> 适用范围：课程学习引擎**应用端**页面、组件、弹窗、Toast、占位符、按钮、列、表头、错误提示。**管理端 UI 不做界面语言切换**，管理端的“多语言”仅指内容编辑字段支持 `zh/en/vi/th/id` 五语言录入与发布前检查。

---

## 一、5 语言枚举

| code | 名称（在切换器中显示）| 使用场景 |
|------|----------------------|---------|
| `zh` | 简体中文 | 默认；中文学习者母语兜底；示意翻译列在 `zh` 时隐藏 `.zy-trans-only` |
| `en` | English | |
| `vi` | Tiếng Việt | |
| `th` | ภาษาไทย | |
| `id` | Bahasa Indonesia | |

应用端 UI 语言变量挂在 `<html data-ui-lang="xx">`；`localStorage.zy.uiLang` 持久化。管理端不读取 / 不展示该切换器，后台操作界面保持中文。

---

## 二、三类翻译来源（必须分清）

### 1. 应用端 UI 文本（按钮/导航/弹窗标题/提示等）
- 来源：前端字典 `window.zyI18nDict[lang][key]`（生产由 `@zhiyu/shared-i18n` 包统一维护）。
- 标记：`<button data-i18n="btn.save">保存</button>`。
- 属性翻译：`<input data-i18n-attr="placeholder:search.placeholder; title:search.tip" />`。
- 兜底规则：`xx → zh → 字面值`。
- **接入点**：每一个新增按钮/列名/弹窗标题/Toast 文案都必须先登记 key 到字典，再使用。
- **接口侧**：后端无需返回 UI 文本；接口仅返回数据 ID/枚举码（如 `q_type='listen_pick'`），前端用 `qt.listen_pick` 渲染中文友好名。
- **管理端边界**：P-A-* 后台页面不放 UI 语言切换器，不要求按钮 / 菜单 / 表头做五语言展示；后台操作文案保持中文，避免把内容多语言编辑误实现成后台界面多语言。

### 2. 学习内容（KP 主词翻译、例句翻译、解析 feedback、考试 / 章 / 节 / 主题名称）
- 来源：数据库 jsonb 字段 `i18n: { zh, en, vi, th, id }`。
- 表/列示意：
  ```sql
  kps.i18n      jsonb  -- {zh: '早上', en: 'morning', ...}
  examples.i18n jsonb  -- {zh: '...', en: '...', ...}
  questions.feedback_i18n jsonb
  questions.stem_i18n     jsonb
  themes.name_i18n / stages.name_i18n / chapters.name_i18n / lessons.name_i18n / exam_templates.name_i18n
  ```
- 前端模板：
  ```html
  <span data-zh="早上" data-trans-en="morning"
        data-trans-vi="buổi sáng" data-trans-th="ตอนเช้า"
        data-trans-id="pagi"
        class="zy-trans-only">morning</span>
  ```
- **隐藏规则**：`html[data-ui-lang="zh"] .zy-trans-only { display: none }` —— 中文 UI 下，所有学习内容的翻译版本一律隐藏，只显示中文原文。
- 兜底规则：`xx → zh`。
- **管理端**：所有学习内容编辑表单**强制 5 语言并排输入**（`.zy-i18n-grid` 网格）；待发布内容可暂存缺译，发布 / 应用端展示前必须补齐 `zh/en/vi/th/id`。

### 3. 题型 / 难度 / 状态 / 角色 等枚举
- 数据库字段 = 英文 code（`listen_pick` / `pending` / `editor` 等）。
- 前端永远不直接显示 code；通过 `qt.<code>` / `status.<code>` / `role.<code>` 字典翻译。
- 应用端不显示 code，只显示翻译；管理端列表默认显示中文友好名，必要时 tooltip 显示 code 便于排查。

## 三、Key 命名规范

| 前缀 | 含义 | 例 |
|------|------|----|
| `brand.*` | 品牌字 | `brand.learn` = "学中文" |
| `nav.*` | 顶栏 / 底栏导航 | `nav.learn`、`nav.me`、`nav.exams` |
| `btn.*` | 按钮文字 | `btn.save`、`btn.adopt`、`btn.viewAnalysis` |
| `common.*` | 列标题、状态副本 | `common.score`、`common.duration` |
| `qt.*` | 题型枚举 | `qt.listen_pick` |
| `exam.*` | 考试相关 | `exam.chapter`、`exam.stage`（不再保留 `exam.hsk` 单独抽题） |
| `page.*` | 页面标题 | `page.learnMap` |
| `modal.*` | 弹窗标题 / 主体 | `modal.langTitle` |
| `settings.*` | 设置项 | `settings.dailySrsCap` |
| `theme.*` | 主题名 | `theme.ec` = "🛒 电商" |
| `me.*` | 个人统计 | `me.kpMastered` |
| `srs.*` / `wrong.*` | 子领域 | `srs.title` |

**禁止**：在按钮文本中拼接动态数字时硬编码翻译；改用 `{n}` 占位符 + JS `String.replace('{n}', x)`，或使用模板化 `<span data-i18n="...">前缀</span> 7/30 <span data-i18n="...">后缀</span>`。

## 四、切换交互规范

1. 应用端页面顶栏右侧必须有 **🌐 lang-pill**：
   ```html
   <button class="zy-lang-pill" data-open-modal="mLang" title="语言">
     <span data-icon="globe"></span><span data-lang-label>中文</span>
   </button>
   ```
2. 对应的 `mLang` 弹窗必须包含 5 个 radio：
   ```html
   <input type="radio" name="lang" data-set-lang="zh" /> 简体中文
   <input type="radio" name="lang" data-set-lang="en" /> English
   <input type="radio" name="lang" data-set-lang="vi" /> Tiếng Việt
   <input type="radio" name="lang" data-set-lang="th" /> ภาษาไทย
   <input type="radio" name="lang" data-set-lang="id" /> Bahasa Indonesia
   ```
3. JS 行为（`zyI18n.set(lang)`）：
   - `localStorage.zy.uiLang = lang`
   - `<html>` 加 `data-ui-lang`
   - 自动勾选所有匹配的 `[data-set-lang]` radio（**所有页面，所有位置**）
   - 自动更新所有 `[data-lang-label]` 文本
   - 自动调用 `apply(lang)` 翻译所有 `[data-i18n]` / `[data-i18n-attr]` / `[data-trans-xx]`
   - 关闭语言弹窗 + Toast 提示

> 管理端 P-A-* 不创建 `mLang`、不放 `data-set-lang` radio、不放 `zy-lang-pill`。管理端需要多语言时，只在内容表单中使用 `.zy-i18n-grid` 并排编辑五语言内容。

## 五、验收清单（每个新页面提交前自检）

### 应用端 P-C-*

- [ ] 顶栏含 lang-pill 与 `data-lang-label`
- [ ] 所有可见按钮 / 列名 / 弹窗标题 / 弹窗按钮 / 占位符 / Toast 都有 `data-i18n` 或 `data-i18n-attr`
- [ ] 学习内容文本块用 `data-zh` + `data-trans-*` + `.zy-trans-only`
- [ ] 题型 tag 文本走 `qt.*` key（不写裸 `listen_pick`）
- [ ] 5 个 radio 切换语言后能立刻看到对应语言的按钮 / 标题 / 占位符
- [ ] 重新进入页面（刷新）后，UI 仍保持上次选定的语言；语言弹窗中对应 radio 已勾选

### 管理端 P-A-*

- [ ] 后台操作界面保持中文，不出现 UI 语言切换入口
- [ ] 主题 / 阶段 / 章 / 节 / KP / 题面 / 解析 / 考试名称等学习内容字段使用 `.zy-i18n-grid` 五语言编辑
- [ ] 待发布内容可暂存缺译，发布 / 应用端展示前必须补齐 `zh/en/vi/th/id`
- [ ] 列表页优先显示中文内容，需要排查时展示 code / 版本 / 状态，不展示后台 UI 多语言切换

## 六、接口契约约束

- 所有 `*_i18n` jsonb 列：必须含 `zh`，其他语言可缺失。后端不做 fallback，由前端 `data-trans-*` + `data-zh` 兜底。
- API 响应不允许返回"已本地化"的 UI 字符串（如 `"已通过"`），改为：
  ```json
  { "status": "passed", "score": 85 }
  ```
  由前端 `common.passed` 映射。
- 题目接口返回的题面 / 解析 / 选项标签等学习内容字段**必须为完整 i18n 对象**：
  ```json
  {
    "stem": {"zh":"请选择...", "en":"Pick the word...", ...},
    "feedback": {"zh":"...", "en":"..."},
    "options": [{"label":{"zh":"早上","en":"morning"}, "is_correct":true}]
  }
  ```

---

## 七、迁移与维护

- 字典在 `system/packages/shared-i18n/src/dict.ts` 集中维护，按 namespace 拆分（`btn.ts` / `nav.ts` / `qt.ts` …）。
- 新 key：先在字典 5 语言全部补齐才允许 PR 合并（CI 校验缺失 key）。
- 删除 key：先全仓搜索 `data-i18n="<key>"` 引用为 0，再删除。
- 命名约束：仅 `^[a-z][a-zA-Z0-9_.]+$`，禁止包含中文。
