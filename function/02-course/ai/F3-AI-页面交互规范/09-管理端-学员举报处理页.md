# P-A-6 管理端 · 学员举报处理

> 管理端 UI 固定中文；多语言仅指内容字段可编辑 `zh / en / vi / th / id`。
> 学员举报处理只负责查看举报、跳转调整、采纳、忽略、通知举报人；不产生集中处理队列，也不自动改变内容发布状态。

---

## P-A-6 · 学员举报处理页

**路由**：`/admin/course/reports`
**权限**：content_admin+

### 关键约定

- 学员答错时若发现题目错误，可在 P-C-3 / P-C-7 题目内点举报（D-14）→ 写 `course_content_action_log(action='report')` 或举报聚合表。
- 处理动作：忽略（无效）/ 调整（进入对应编辑抽屉手动修正）/ 采纳（举报状态改为已采纳并通知举报人）。
- 采纳不自动修改 KP/Q 发布状态；内容调整必须由「调整」按钮进入 D-3 / D-4 完成，再按需要保存或发布。

### 数据来源

- [A18 GET /admin/v1/course/reports](../F2-AI-接口规范/07-管理端-发布与举报处理.md)：分页 + 状态（默认 open）。

### 交互

| # | 操作 | 实现 |
|---|------|------|
| A6-01 | 进入 | 调 A18；默认展示待处理举报 |
| A6-02 | 点击「详情」 | 弹 D-7：举报全文 + 题目 / KP 预览 + 学员答题历史 |
| A6-03 | 点击「调整」 | 根据对象类型打开 D-3 KP 编辑抽屉或 D-4 题目编辑抽屉；保存后回到 P-A-6 |
| A6-04 | 点击「忽略」 | D-8 输入说明 → [A19 POST /reports/:id:dismiss](../F2-AI-接口规范/07-管理端-发布与举报处理.md)；report status → dismissed |
| A6-05 | 点击「采纳」 | D-8 二次确认 → [A19 POST /reports/:id:adopt](../F2-AI-接口规范/07-管理端-发布与举报处理.md)；report status → adopted；通知举报人 |
| A6-06 | 隐藏应用端举报入口 | 写系统配置；学员端 D-14 入口立即隐藏，历史举报仍可处理 |

### 条件显示

| 条件 | 行为 |
|------|------|
| 同一题目报错次数 ≥ 3 | 顶部红条「该题被多人举报，建议优先处理」+ 列表行高亮 |
| 已处理（dismissed / adopted） | 灰色行 + 状态标 + 不可再操作（仅查看） |

## 操作留痕与通知

| 操作 | 实现 |
|------|------|
| 调整题目 / KP | 写对应内容表版本记录 + `course_content_action_log(action='edit')` |
| 采纳 / 忽略 | 写举报状态、处理人、处理时间、通知内容 |
| 查看历史 | 通过举报详情 Drawer 展示处理记录，不再单独提供集中处理日志页 |
