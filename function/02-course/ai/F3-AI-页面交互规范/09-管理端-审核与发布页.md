## P-A-6 · 学员举报处理页

**路由**：`/admin/course/reports`
**权限**：reviewer+

### 关键约定

- 学员答错时若发现题目错误，可在 P-C-3 / P-C-7 题目内点举报（D-14）→ 写 `course_content_reports`。
- 处理动作：忽略（无效）/ 调整（进入对应编辑抽屉手动修正）/ 采纳（举报状态改为已采纳并通知举报人）。
- 采纳不自动修改 KP/Q 状态；内容调整必须由「调整」按钮进入 D-3 / D-4 完成。

### 数据来源

- [A51 GET /admin/v1/course/reports](../F2-AI-接口规范/07-管理端-审核与发布.md)：分页 + 状态（默认 pending）。

### 交互

| # | 操作 | 实现 |
|---|------|------|
| A6-01 | 进入 | 调 A51；默认展示待处理举报 |
| A6-02 | 点击「详情」 | 弹 D-7：举报全文 + 题目 / KP 预览 + 学员答题历史 |
| A6-03 | 点击「调整」 | 根据对象类型打开 D-3 KP 编辑抽屉或 D-4 题目编辑抽屉；保存后回到 P-A-6 |
| A6-04 | 点击「忽略」 | D-8 输入理由 → [A52 POST /reports/:id/dismiss](../F2-AI-接口规范/07-管理端-审核与发布.md)；status → dismissed |
| A6-05 | 点击「采纳」 | D-8 二次确认 → [A53 POST /reports/:id/adopt](../F2-AI-接口规范/07-管理端-审核与发布.md)；report status → adopted；通知举报人 |
| A6-06 | 隐藏应用端举报入口 | 写系统配置；学员端 D-14 入口立即隐藏，历史举报仍可处理 |

### 条件显示

| 条件 | 行为 |
|------|------|
| 同一题目报错次数 ≥ 3 | 顶部红条「该题被多人举报，建议优先处理」+ 列表行高亮 |
| 已处理（dismissed / adopted） | 灰色行 + 状态标 + 不可再操作（仅查看） |

## 审计与通知

| 操作 | 实现 |
|------|------|
| 调整题目 / KP | 写对应内容表版本记录 + `admin_audit_log` |
| 采纳 / 忽略 | 写 `course_content_reports.status`、处理人、处理时间、通知内容 |
