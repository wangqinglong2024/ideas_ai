# 管理端：内容导入

> KP 与题目由运营在线下生成、整理后导入系统；管理端只负责文件上传、结构校验、预览、导入和来源追溯。

---

## A15 · 内容导入批次

### A15.1 上传并预览

**方法**：POST
**路径**：`/admin/v1/course/import-batches:preview`
**权限**：`content_admin+`

**Content-Type**：`multipart/form-data`

| 字段 | 必填 | 说明 |
|------|------|------|
| `file` | ✅ | CSV / JSON 文件 |
| `import_type` | ✅ | `kp` / `question` / `lesson` / `exam` |
| `track` | ❌ | 主题过滤，写入默认上下文 |
| `source_note` | ❌ | 来源说明 |

**业务校验**：
- 计算 `payload_hash`，同文件同类型重复上传时返回已有批次。
- 校验列名、必填字段、多语言 key、题型 payload、可抽层级、KP 引用和重复编码。
- 预览阶段不写 KP / Question 正表。

**响应 200**：

```json
{
  "code": 0,
  "data": {
    "batch_id": "<uuid>",
    "status": "ready",
    "summary": { "total": 120, "valid": 118, "error": 2, "duplicate": 3 },
    "errors": [
      { "row": 17, "field": "title_i18n.en", "message": "英文内容缺失" }
    ]
  }
}
```

### A15.2 确认导入

**方法**：POST
**路径**：`/admin/v1/course/import-batches/:id:import`
**权限**：`content_admin+`

**业务校验**：
- 仅 `status=ready` 的批次可导入。
- 正表写入后内容默认 `is_published=false`，管理端显示「待发布」。
- 同 `kp_code / q_code` 命中已有内容时按导入策略覆盖或跳过，保留版本记录。

**响应 200**：

```json
{ "code": 0, "data": { "imported": 118, "skipped": 3, "failed": 0 } }
```

### A15.3 批次列表 / 详情

**方法**：GET
**路径**：`/admin/v1/course/import-batches` / `/admin/v1/course/import-batches/:id`
**权限**：`content_admin+`

**请求参数**：`import_type / status / created_by / from / to / page`

**响应 200**：返回批次基础信息、校验摘要、错误行和导入结果。

## 共通错误

| HTTP | code | 触发 |
|------|------|------|
| 400 | `COURSE_IMPORT_FILE_REQUIRED` | 未上传文件 |
| 400 | `COURSE_IMPORT_SCHEMA_MISMATCH` | 文件结构不匹配 |
| 409 | `COURSE_IMPORT_DUPLICATE` | 同文件同类型已存在可用批次 |
| 422 | `COURSE_IMPORT_HAS_ERRORS` | 仍有阻断错误，不能确认导入 |
