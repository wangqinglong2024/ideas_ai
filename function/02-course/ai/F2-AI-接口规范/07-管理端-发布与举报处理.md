# 管理端：发布与举报处理

> 本文件不再定义集中处理队列、通过 / 退回或其它复杂流转动作。
> 课程内容只有发布状态：`is_published=true` 显示为「已发布」，`is_published=false` 显示为「待发布」（包含未发布与下架）。
> 学员举报只进入 P-A-6「学员举报处理」，由运营查看、调整、采纳或忽略。

---

## A16 · 发布 / 下架

通用：`POST /admin/v1/course/:target_type/:id:publish` / `:unpublish`

| Target | 权限 | 备注 |
|--------|------|------|
| kp / question / lesson / exam | `content_admin+` | 仅切换 `is_published` |

**业务校验**：
- publish：要求 `is_published=false and deleted_at is null`。
- unpublish：要求 `is_published=true and deleted_at is null`。
- 节发布前必须确保：节内 KP 均已发布、每个 KP 至少有足够已发布题、内容 5 语言字段满足前端展示要求。
- 题目发布前必须确保：关联 KP 已发布，payload 校验通过，所勾选的可抽层级合法。
- 考试配置发布前必须确保：最近一次试抽成功，题量和题型权重可满足。

**副作用**：
- publish：`is_published=true`，写 `published_by/published_at`，学员端立即可见或可抽取。
- unpublish：`is_published=false`，管理端显示「待发布」，学员端立即不可见。
- 事务内写 `course_content_action_log`，action=`publish` 或 `unpublish`。

**响应 200**：

```json
{ "code": 0, "data": { "id":"<uuid>", "is_published": true, "state_label": "已发布" } }
```

## A19 · 处理举报

| 操作 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 忽略 | POST | `/admin/v1/course/reports/:report_id:dismiss` | 判定举报无效，必须填写给举报人的说明 |
| 采纳 | POST | `/admin/v1/course/reports/:report_id:adopt` | 表示已按举报完成调整并通知举报人 |

### dismiss 请求体

```json
{ "note": "核对后题目无误，感谢反馈。" }
```

### adopt 请求体

```json
{
  "note": "已修正题目解析。",
  "fixed_target_id": "<uuid>"
}
```

**adopt 副作用**：
- 仅把举报状态改为 `adopted` 并通知举报人。
- 不自动改变 KP / Question 的发布状态；内容调整必须由 P-A-4 / P-A-5 编辑抽屉手动保存或发布。
- 写 `course_content_action_log`，action=`adopt`。

**响应 200**：

```json
{ "code": 0, "data": { "report_status": "adopted", "target_publish_state": "已发布" } }
```

**错误**：

| HTTP | code | 触发 |
|------|------|------|
| 400 | `COURSE_REPORT_NOTE_REQUIRED` | 忽略举报但未填写说明 |
| 409 | `COURSE_REPORT_ALREADY_HANDLED` | 已处理 |
| 404 | 40400 | report 不存在 |
