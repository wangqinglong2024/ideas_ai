# 管理端：发布与举报处理

> 本文件不再定义集中处理队列或其它复杂流转动作。
> 课程内容只有发布状态：`is_published=true` 显示为「已发布」，`is_published=false` 显示为「待发布」（包含未发布与下架）。
> 学员举报只进入 P-A-6「学员举报处理」，由运营查看、调整、采纳或忽略。

---

## A16 · 发布 / 下架

通用：`POST /admin/v1/course/:target_type/:id:publish` / `:unpublish`

| Target | 权限 | 备注 |
|--------|------|------|
| stage | `content_admin+` | 阶段级可见性开关；不自动改子级内容状态 |
| chapter | `content_admin+` | 发布章时默认发布章下全部节；下架章时默认下架章下全部节 |
| lesson | `content_admin+` | 发布节时默认发布节下 KP 与题目；下架节只下架节本身 |
| kp | `content_admin+` | KP 可单独发布 / 下架；下架后学员端不可见 |
| question | `content_admin+` | 题目可单独发布 / 下架；下架后不再被节测、章测、阶段测抽取 |
| exam | `content_admin+` | 阶段考 / 章测 / 节测配置必须发布后应用端可见 |

**业务校验**：
- publish：要求 `is_published=false and deleted_at is null`。
- unpublish：要求 `is_published=true and deleted_at is null`。
- 章发布前必须确保：章本身 5 语言标题满足展示要求；章下至少存在 1 个未软删节。发布成功后同步将章下未软删节设为已发布，并按“节发布”默认级联 KP 与题目。
- 章下架只同步下架章下未软删节；KP / Question 不被全局下架，以免影响其它已发布节或考试引用。
- 节发布前必须确保：节内 KP 与题目满足最小数量、内容 5 语言字段满足前端展示要求。发布成功后同步将本节绑定 KP 与这些 KP 下可抽层级合法的题目设为已发布。
- 节下架只切换本节 `is_published=false`；已发布 KP / Question 可继续被其它已发布节、章测或阶段考引用。
- KP 下架后，该 KP 不在学员端展示；若题目仍已发布，只有在其所属抽题范围仍有效时才可被考试抽取。
- 题目发布前必须确保：关联 KP 已发布，payload 校验通过，所勾选的可抽层级合法。
- 题目下架后，不再参与任何新生成的节测、章测、阶段测；已生成 attempt 的 snapshot 不回溯修改。
- 考试配置发布前必须确保：最近一次试抽成功，题量和题型权重可满足；保存配置不会自动发布。

**副作用**：
- publish：目标记录 `is_published=true`，写 `published_by/published_at`；章 / 节按上面的默认级联规则同步子级状态。
- unpublish：目标记录 `is_published=false`，管理端显示「待发布」；章按规则同步下架子节，其它目标只切换自身。
- 题目下架后的测试分值不落库重写；发卷 / 交卷时按当前有效已发布题数动态计算，`per_question_score = total_score / effective_question_count`。
- 事务内写 `course_content_action_log`，action=`publish` 或 `unpublish`。

**响应 200**：

```json
{
  "code": 0,
  "data": {
    "id":"<uuid>",
    "target_type":"lesson",
    "is_published": true,
    "state_label": "已发布",
    "cascade_summary": {
      "lessons_published": 0,
      "lessons_unpublished": 0,
      "kps_published": 12,
      "questions_published": 48,
      "questions_excluded_from_pool": 0
    }
  }
}
```

`cascade_summary` 按 target 返回相关字段即可；无影响项可省略或返回 0。

## A19 · 处理举报

| 操作 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 忽略 | POST | `/admin/v1/course/reports/:report_id:dismiss` | 判定举报无效，必须填写给举报人的说明 |
| 采纳 | POST | `/admin/v1/course/reports/:report_id:adopt` | 表示已按举报完成调整并通知举报人 |

### dismiss 请求体

```json
{ "note": "核对后题目无误，感谢反馈。" }
```

### adopt 请求体

```json
{
  "note": "已修正题目解析。",
  "fixed_target_id": "<uuid>"
}
```

**adopt 副作用**：
- 仅把举报状态改为 `adopted` 并通知举报人。
- 不自动改变 KP / Question 的发布状态；内容调整必须由 P-A-4 / P-A-5 编辑抽屉手动保存或发布。
- 写 `course_content_action_log`，action=`adopt`。

**响应 200**：

```json
{ "code": 0, "data": { "report_status": "adopted", "target_publish_state": "已发布" } }
```

**错误**：

| HTTP | code | 触发 |
|------|------|------|
| 400 | `COURSE_REPORT_NOTE_REQUIRED` | 忽略举报但未填写说明 |
| 409 | `COURSE_REPORT_ALREADY_HANDLED` | 已处理 |
| 404 | 40400 | report 不存在 |
