## 1. 内容发布状态机（适用：Stage / Chapter / Lesson / KP / Question / Exam）

来源：[prd/04 §4.4 / §4.7](../../prd/04-管理端模块设计.md) + [prd/06 §6.4 流程 C](../../prd/06-关键业务流程.md)

```
┌──────────┐      publish       ┌──────────┐
│ 待发布   │ ─────────────────► │ 已发布   │
│ false    │                    │ true     │
└────┬─────┘                    └────┬─────┘
     │ edit / import / save          │ unpublish / edit and save as pending
     └───────────────────────────────┘
```

| 当前状态 | 允许动作 | 角色 | 目标状态 | 副作用 |
|---------|---------|------|---------|-------|
| 待发布 | `save` | content_admin/super | 待发布 | 写操作日志；学员不可见 |
| 待发布 | `publish` | content_admin/super | 已发布 | `is_published=true`；写 `published_by/published_at`；目标及有效子级按规则可见 |
| 已发布 | `save` | content_admin/super | 已发布 | version+1；发布状态不变，学员端读取当前已发布内容 |
| 已发布 | `unpublish` | content_admin/super | 待发布 | `is_published=false`；目标立即不可见 |
| 任意 | `delete` | content_admin/super | 软删 | `deleted_at=now()`；学员不可见 |

### 1.1 默认级联

| 目标 | 发布默认行为 | 下架默认行为 | 说明 |
|---|---|---|---|
| Stage | 只切换阶段自身 | 只切换阶段自身 | 子级由章 / 节列表单独控制 |
| Chapter | 章设为已发布；章下全部未软删节设为已发布；每个被发布节继续触发节发布默认行为 | 章设为待发布；章下全部未软删节设为待发布 | 章发布后学员可见章；章下架后学员不可见章及其节 |
| Lesson | 节设为已发布；本节绑定 KP 设为已发布；这些 KP 下可抽层级合法的题目设为已发布 | 只切换节自身为待发布 | KP / 题目可被其它节或考试复用，节下架不全局下架它们 |
| KP | 只切换 KP 自身 | 只切换 KP 自身 | 下架后学员端不展示该知识点 |
| Question | 只切换题目自身 | 只切换题目自身 | 下架后从所有新抽题池排除 |
| Exam | 只切换考试配置自身 | 只切换考试配置自身 | 阶段考 / 章测 / 节测保存后仍待发布，发布后应用端可见 |

实现：RPC `fn_content_publish(p_target_type, p_target_id, p_action, p_diff)`，事务内统一写入 `course_content_action_log`，并返回 `cascade_summary`。不存在额外流转状态。

## 2. 导入批次状态机

来源：[prd/06 §6.3 流程 B](../../prd/06-关键业务流程.md)

```
   ┌────────┐
   │ draft  │
   └───┬────┘
        │ validate
        ▼
┌────────────┐
│ validating │
└────┬───────┘
      │ success
      ▼
 ┌────────┐      import       ┌──────────┐
 │ ready  │ ────────────────► │ imported │
 └───┬────┘                   └──────────┘
      │ fail / cancel
      ▼
┌──────────┐
│ failed   │ / cancelled
└──────────┘
```

约束：
- `draft → validating`：上传后异步校验列名、必填、枚举、重复编码与 JSON schema。
- `validating → ready`：写入 `row_count / success_count / validation_errors=[]`。
- `ready → imported`：事务内插入 KP / 题目 / 媒资，并把 `source_batch_id` 写入内容表。
- `validating / ready → failed`：保留 `validation_errors`，不写入内容主表。
- `draft / ready → cancelled`：仅管理员可取消；取消后不可导入。

## 3. 节级进度状态机（`course_user_progress.status`）

来源：[prd/05 §5.4](../../prd/05-用户端模块设计.md) + [prd/06 §6.7](../../prd/06-关键业务流程.md)

```
   ┌────────┐
   │ locked │ ◄── 默认（前置节未通过）
   └───┬────┘
       │ unlock_predecessor_passed
       ▼
   ┌──────────┐
   │ unlocked │
   └────┬─────┘
        │ first_enter
        ▼
   ┌─────────────┐
   │ in_progress │
   └─┬─────────┬─┘
     │         │
quiz_pass     quiz_fail (允许重试，状态保持)
     │         │
     ▼         └────► 仍 in_progress
   ┌────────┐
   │ passed │ ─── (终态，可复习不改状态)
   └────────┘
```

| 触发 | 当前 | 目标 | 写入字段 |
|------|------|------|---------|
| 学员首次进入节 | unlocked | in_progress | `started_at=now()`, `status='in_progress'` |
| 通过节末小测 | in_progress | passed | `passed_at=now()`, `best_score=max(...)`, `status='passed'` |
| 节末小测未过 | in_progress | in_progress | `attempts += 1`, `best_score=max(...)` |
| 上一节通过（触发） | locked | unlocked | `status='unlocked'` |
| 节内退出 | in_progress | in_progress | `last_kp_id`, `last_position`（断点续学） |

实现：RPC `fn_lesson_pass(p_user_id, p_lesson_id, p_score)` + 触发器 `tg_after_lesson_pass_unlock_next`。

> ⚠️ 节末小测是否强制（不强制 → unlocked → 直接进下一节），见 [12-待确认问题清单.md Q4](./12-待确认问题清单.md)。

## 4. 考试答卷状态机（`course_user_exam_attempts.state`）

来源：[prd/06 §6.9 流程 H](../../prd/06-关键业务流程.md)

```
   ┌─────────────┐
   │ in_progress │ ◄── start_exam (RPC 抽题)
   └─┬────┬────┬─┘
     │    │    │
submit│ exit│  │ time_up (cron)
     │    │    │
     ▼    ▼    ▼
  ┌──────────┐ ┌───────────┐ ┌─────────┐
  │submitted │ │ abandoned │ │ expired │
  └──────────┘ └───────────┘ └─────────┘
   学员主动提交  退出弹窗确认放弃   超时
   计算总分      score=0          按已答计分
```

| 当前 | 触发 | 目标 | 写入 |
|------|------|------|-----|
| in_progress | submit | submitted | score, passed, finished_at=now() |
| in_progress | exit_confirm | abandoned | score=0, passed=false, finished_at=now() |
| in_progress | cron(now > expires_at) | expired | score=按已答计算, passed, finished_at=now() |
| 任意终态 | — | — | 不可改 |

cron 任务：`scripts/cron/expire-exam-attempts.ts`，每分钟扫一次。
