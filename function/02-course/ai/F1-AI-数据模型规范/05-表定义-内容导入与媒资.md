# 一 · 表定义 · 内容导入与媒资（3 表）

> 2025-11 起，课程内容生产不在系统内编排生成任务。运营线下生成、整理 KP 与题目后，通过批量导入进入系统；系统内记录导入批次、点检日志与媒资资产。

---

## 1. 表：`course_import_batches`（内容导入批次）

**对应需求中的对象**：[temp/03 §3.4](../../../temp/03-数据库schema.md) + [temp/06 §6.3 / §6.4](../../../temp/06-关键业务流程.md)
**性质**：管理端导入 KP / Question / 媒资时写入的批次记录，用于追溯来源文件、校验结果与导入人。

### 一、字段定义

| 字段名 | 类型 | 必填 | 默认值 | 索引 | 对应需求中的信息 | 校验规则 |
|--------|------|------|--------|------|----------------|---------|
| `id` | `uuid` | ✅ | `gen_random_uuid()` | PK | — | — |
| `batch_code` | `text` | ✅ | 由 RPC `fn_gen_import_batch_code()` 生成 | `uq_course_import_batch_code` | 批次编号 | 正则 `^imp_(kp\|q\|media)_\d{8}$` |
| `import_type` | `text` | ✅ | — | `idx_course_import_type` | 导入类型 | 枚举 `kp\|question\|media` |
| `track` | `text` | ❌ | `null` | `idx_course_import_track` | 所属主题 | FK → `course_tracks.code` |
| `source_file_name` | `text` | ✅ | — | — | 原始文件名 | 1–255 |
| `source_file_url` | `text` | ❌ | `null` | — | 源文件归档地址 | URL |
| `payload_hash` | `text` | ✅ | — | `idx_course_import_hash` | 文件内容 hash | 64 字符 hex |
| `row_count` | `int` | ✅ | `0` | — | 总行数 | ≥ 0 |
| `success_count` | `int` | ✅ | `0` | — | 成功行数 | ≥ 0 |
| `failed_count` | `int` | ✅ | `0` | — | 失败行数 | ≥ 0 |
| `validation_errors` | `jsonb` | ❌ | `null` | — | 行级错误 | `[{row, field, message}]` |
| `status` | `text` | ✅ | `'draft'` | `idx_course_import_status` | 导入状态 | 枚举 `draft\|validating\|ready\|imported\|failed\|cancelled` |
| `created_by` | `uuid` | ❌ | `null` | `idx_course_import_creator` | 导入人 | FK → `auth.users.id`，`ON DELETE SET NULL` |
| `created_at` | `timestamptz` | ✅ | `now()` | `idx_course_import_created` | — | — |
| `updated_at` | `timestamptz` | ✅ | `now()` | — | — | — |

### 二、DDL 草案

```sql
create table zhiyu.course_import_batches (
  id                uuid primary key default gen_random_uuid(),
  batch_code        text not null,
  import_type       text not null,
  track             text,
  source_file_name  text not null,
  source_file_url   text,
  payload_hash      text not null,
  row_count         int not null default 0,
  success_count     int not null default 0,
  failed_count      int not null default 0,
  validation_errors jsonb,
  status            text not null default 'draft',
  created_by        uuid,
  created_at        timestamptz not null default now(),
  updated_at        timestamptz not null default now(),

  constraint fk_course_import_track
    foreign key (track) references zhiyu.course_tracks(code) on delete restrict,
  constraint fk_course_import_creator
    foreign key (created_by) references auth.users(id) on delete set null,
  constraint uq_course_import_batch_code unique (batch_code),
  constraint ck_course_import_type check (import_type in ('kp','question','media')),
  constraint ck_course_import_status check (status in ('draft','validating','ready','imported','failed','cancelled')),
  constraint ck_course_import_counts check (row_count >= 0 and success_count >= 0 and failed_count >= 0),
  constraint ck_course_import_hash check (payload_hash ~ '^[a-f0-9]{64}$')
);

create index idx_course_import_type    on zhiyu.course_import_batches (import_type);
create index idx_course_import_track   on zhiyu.course_import_batches (track);
create index idx_course_import_status  on zhiyu.course_import_batches (status);
create index idx_course_import_creator on zhiyu.course_import_batches (created_by);
create index idx_course_import_created on zhiyu.course_import_batches (created_at desc);
create index idx_course_import_hash    on zhiyu.course_import_batches (payload_hash);
```

### 三、RLS 策略

| 操作 | 角色 | 策略 |
|------|------|------|
| `select` | `content_admin` / `reviewer` / `super` | 可查批次与校验结果 |
| 写 | `service_role` | 管理端 API 代写 |
| 其它 | — | 拒绝 |

## 2. 表：`course_content_review_log`（点检 / 学员举报留痕）

**对应需求中的对象**：[temp/03 §3.4.4](../../../temp/03-数据库schema.md) + [temp/06 §6.5 / §6.10](../../../temp/06-关键业务流程.md)
**性质**：高写入审计表。承载运营点检 approve/reject/edit 与学员 report。

### 一、字段定义

| 字段名 | 类型 | 必填 | 默认值 | 索引 | 对应需求中的信息 | 校验规则 |
|--------|------|------|--------|------|----------------|---------|
| `id` | `uuid` | ✅ | `gen_random_uuid()` | PK | — | — |
| `target_type` | `text` | ✅ | — | `idx_course_crl_target` | 目标对象类型 | 枚举 `kp\|question\|lesson\|exam` |
| `target_id` | `uuid` | ✅ | — | `idx_course_crl_target` | 目标对象 id | — |
| `action` | `text` | ✅ | — | `idx_course_crl_action` | 操作类型 | 枚举 `approve\|reject\|edit\|report\|dismiss\|adopt` |
| `actor_id` | `uuid` | ❌ | `null` | `idx_course_crl_actor` | 操作人（学员或运营） | FK → `auth.users.id`，`ON DELETE SET NULL` |
| `actor_role` | `text` | ✅ | — | — | 角色（learner/reviewer/content_admin/super） | 枚举 |
| `reason` | `text` | ❌ | `null` | — | 备注 | ≤ 500 |
| `report_type` | `text` | ❌ | `null` | — | 学员举报类型（仅 action='report' 时） | 枚举 `wrong_answer\|wrong_translation\|wrong_audio\|wrong_image\|other` |
| `diff` | `jsonb` | ❌ | `null` | — | 修改前后差异（仅 action='edit'） | — |
| `created_at` | `timestamptz` | ✅ | `now()` | `idx_course_crl_created` | — | — |

### 二、DDL 草案

```sql
create table zhiyu.course_content_review_log (
  id          uuid primary key default gen_random_uuid(),
  target_type text not null,
  target_id   uuid not null,
  action      text not null,
  actor_id    uuid,
  actor_role  text not null,
  reason      text,
  report_type text,
  diff        jsonb,
  created_at  timestamptz not null default now(),

  constraint fk_course_crl_actor
    foreign key (actor_id) references auth.users(id) on delete set null,
  constraint ck_course_crl_target_type check (target_type in ('kp','question','lesson','exam')),
  constraint ck_course_crl_action check (action in ('approve','reject','edit','report','dismiss','adopt')),
  constraint ck_course_crl_actor_role check (actor_role in ('learner','reviewer','content_admin','super')),
  constraint ck_course_crl_report_type
    check (report_type is null or report_type in ('wrong_answer','wrong_translation','wrong_audio','wrong_image','other')),
  constraint ck_course_crl_reason_len check (reason is null or char_length(reason) <= 500),
  constraint ck_course_crl_report_type_only_with_report
    check ((action = 'report' and report_type is not null) or (action <> 'report' and report_type is null))
);

create index idx_course_crl_target  on zhiyu.course_content_review_log (target_type, target_id);
create index idx_course_crl_action  on zhiyu.course_content_review_log (action);
create index idx_course_crl_actor   on zhiyu.course_content_review_log (actor_id);
create index idx_course_crl_created on zhiyu.course_content_review_log (created_at desc);
```

### 三、RLS 策略

| 操作 | 角色 | 策略 |
|------|------|------|
| `insert` | `authenticated` | 仅允许 `action='report' and actor_id=auth.uid()`（通过 RPC `fn_report_question` 写入） |
| `select / update / delete` | `authenticated` | 拒绝 |
| 全部 | `service_role` | 全量 |

## 3. 表：`course_media_assets`（媒资库）

**对应需求中的对象**：[temp/03 §3.4.3](../../../temp/03-数据库schema.md) + [temp/04 §4.8](../../../temp/04-管理端模块设计.md)
**性质**：所有音频 / 图片 / 视频 / Lottie 笔顺统一存储；按 `hash` 去重。

### 一、字段定义

| 字段名 | 类型 | 必填 | 默认值 | 索引 | 对应需求中的信息 | 校验规则 |
|--------|------|------|--------|------|----------------|---------|
| `id` | `uuid` | ✅ | `gen_random_uuid()` | PK | — | — |
| `kind` | `text` | ✅ | — | `idx_course_ma_kind` | 媒资类型 | 枚举 `audio\|image\|video\|svg\|lottie` |
| `url` | `text` | ✅ | — | — | CDN 链接 | URL，1–500 |
| `hash` | `text` | ✅ | — | `uq_course_ma_hash` | sha256 内容 hash（去重用） | 64 字符 hex |
| `meta` | `jsonb` | ✅ | `'{}'` | — | 元信息 | 见 §meta schema |
| `source` | `text` | ✅ | — | `idx_course_ma_source` | 来源 | 枚举 `tts\|human_recorded\|uploaded\|stock_library` |
| `voice_profile` | `text` | ❌ | `null` | — | 声线（仅 kind='audio'） | 1–80 |
| `ref_kp_id` | `uuid` | ❌ | `null` | `idx_course_ma_kp` | 关联 KP（弱引用） | FK → `course_knowledge_points.id`，`ON DELETE SET NULL` |
| `ref_q_id` | `uuid` | ❌ | `null` | `idx_course_ma_q` | 关联 Question | FK → `course_questions.id`，`ON DELETE SET NULL` |
| `created_by` | `uuid` | ❌ | `null` | — | — | FK → `auth.users.id`，`ON DELETE SET NULL` |
| `created_at` | `timestamptz` | ✅ | `now()` | — | — | — |

### 二、DDL 草案

```sql
create table zhiyu.course_media_assets (
  id            uuid primary key default gen_random_uuid(),
  kind          text not null,
  url           text not null,
  hash          text not null,
  meta          jsonb not null default '{}'::jsonb,
  source        text not null,
  voice_profile text,
  ref_kp_id     uuid,
  ref_q_id      uuid,
  created_by    uuid,
  created_at    timestamptz not null default now(),

  constraint fk_course_ma_kp foreign key (ref_kp_id) references zhiyu.course_knowledge_points(id) on delete set null,
  constraint fk_course_ma_q foreign key (ref_q_id) references zhiyu.course_questions(id) on delete set null,
  constraint fk_course_ma_creator foreign key (created_by) references auth.users(id) on delete set null,
  constraint uq_course_ma_hash unique (hash),
  constraint ck_course_ma_kind check (kind in ('audio','image','video','svg','lottie')),
  constraint ck_course_ma_source check (source in ('tts','human_recorded','uploaded','stock_library')),
  constraint ck_course_ma_hash_format check (hash ~ '^[a-f0-9]{64}$'),
  constraint ck_course_ma_url_len check (char_length(url) between 1 and 500)
);

create index idx_course_ma_kind   on zhiyu.course_media_assets (kind);
create index idx_course_ma_source on zhiyu.course_media_assets (source);
create index idx_course_ma_kp     on zhiyu.course_media_assets (ref_kp_id);
create index idx_course_ma_q      on zhiyu.course_media_assets (ref_q_id);
```

### 三、RLS 策略

| 操作 | 角色 | 策略 |
|------|------|------|
| `select` | `anon`、`authenticated` | 全部放行（CDN 链接本身公开） |
| 写 | `service_role` | 全量 |

策略命名：`course_ma_select_public`、`course_ma_write_service`。
