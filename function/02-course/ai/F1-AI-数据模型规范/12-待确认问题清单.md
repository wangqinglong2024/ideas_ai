# 12 · F1 已封板决策清单

> 本文件保留原编号，用于承接历史引用；当前 F1 数据模型决策已封板。

---

| 编号 | 决策 | 数据模型影响 |
|---|---|---|
| Q1 | 全表主键统一使用 `uuid` | 业务编码独立列保留，如 `kp_code`、`q_code`、`lesson.code`。 |
| Q3 | 学员资料归属账号域 | 课程域只通过 `auth.users.id` 引用，不重复定义账号资料表。 |
| Q4 | 节测默认可跳过 | 强制与否由测试配置表达；课程目录页不维护抽题逻辑。 |
| Q5 | 管理员资料归属账号域 | 课程域只定义课程角色约束和日志引用。 |
| Q7 | 题型全集按 12 类建模 | MVP 可按产品节奏开关，但 schema 预留完整题型。 |
| Q8 | 高写入流水按月分区 | 用户答题、考试 attempt、内容操作日志按增长情况分区。 |
| Q9 | 题目去重使用 payload hash | 同 KP + 同题型 + 同 hash 不重复入库。 |
| Q10 | 多语言内容字段固定 5 key | `zh / en / vi / th / id`，发布前必须补齐。 |
| Q13 | 媒资供应商通过抽象 profile 映射 | 表内记录平台统一字段，不暴露具体供应商实现。 |
| Q14 | 统计物化视图按 5 分钟刷新 | 管理端统计允许分钟级延迟。 |

## 发布状态封板

- 内容表统一使用 `is_published` 表达学员端可见性。
- `is_published=true AND deleted_at IS NULL` 才能被应用端读取或被测试抽取。
- 未发布、下架内容统一显示为「待发布」。
