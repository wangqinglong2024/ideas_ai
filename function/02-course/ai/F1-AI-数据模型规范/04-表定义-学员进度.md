# 一 · 表定义 · 学员进度（4 表）

> 学员侧"用户写为主"的表，承载所有学习行为数据。
> 对应需求：[temp/03 §3.3](../../../temp/03-数据库schema.md) + [temp/06 §6.7–6.9 学习/复习/考试流程](../../../temp/06-关键业务流程.md)。
>
> **学员账户本身**复用平台公共表 `auth.users`（Supabase Auth）+ 扩展表 `auth_user_profiles`（含 `ui_lang / native_lang / current_track / shared_stage_done / subscription` 等字段）。
> 该扩展表归属"账号域"而非课程域，本规范不重复定义；具体 schema 详见 [12-Q3](./12-待确认问题清单.md)。

## 1. 表：`course_user_progress`（节级进度）

**对应需求中的对象**：[temp/03 §3.3.2](../../../temp/03-数据库schema.md) + [temp/05 §5.4 学习地图节状态](../../../temp/05-用户端模块设计.md)
**性质**：每个 `(user_id, lesson_id)` 一行，记录该学员对该节的学习状态与最佳成绩。

### 一、字段定义

| 字段名 | 类型 | 必填 | 默认值 | 索引 | 对应需求中的信息 | 校验规则 |
|--------|------|------|--------|------|----------------|---------|
| `user_id` | `uuid` | ✅ | — | PK 一部分 | 学员 | FK → `auth.users.id`，`ON DELETE CASCADE` |
| `lesson_id` | `uuid` | ✅ | — | PK 一部分 | 节 | FK → `course_lessons.id`，`ON DELETE CASCADE` |
| `status` | `text` | ✅ | `'unlocked'` | `idx_course_up_status` | 进度状态 | 枚举 `locked\|unlocked\|in_progress\|passed` |
| `last_kp_id` | `uuid` | ❌ | `null` | — | 断点续学：上次停在哪个 KP | FK → `course_knowledge_points.id`，`ON DELETE SET NULL` |
| `last_position` | `int` | ❌ | `null` | — | 节内位置（卡片序号） | 0–100 |
| `best_score` | `numeric(5,2)` | ❌ | `null` | — | 节末小测最高得分 | 0–100 |
| `attempts` | `int` | ✅ | `0` | — | 节末小测尝试次数 | ≥ 0 |
| `passed_at` | `timestamptz` | ❌ | `null` | — | 首次通过时间 | `status='passed'` 时必须非空 |
| `started_at` | `timestamptz` | ❌ | `null` | — | 首次进入时间 | — |
| `updated_at` | `timestamptz` | ✅ | `now()` | — | — | — |

### 二、AI 补充字段

| 字段名 | 类型 | 补充理由 |
|--------|------|---------|
| `last_position` | `int` | [temp/05 §5.5.3](../../../temp/05-用户端模块设计.md) 退出弹窗"已答题进度自动保存" |
| `started_at` | `timestamptz` | 学习数据分析：从首次进入到通过的时间分布 |
| 无 `created_at` | — | 用 `started_at` 替代，复合主键不需要单独 created_at |

### 三、DDL 草案

```sql
create table zhiyu.course_user_progress (
  user_id        uuid not null,
  lesson_id      uuid not null,
  status         text not null default 'unlocked',
  last_kp_id     uuid,
  last_position  int,
  best_score     numeric(5,2),
  attempts       int  not null default 0,
  passed_at      timestamptz,
  started_at     timestamptz,
  updated_at     timestamptz not null default now(),

  primary key (user_id, lesson_id),
  constraint fk_course_up_user
    foreign key (user_id) references auth.users(id) on delete cascade,
  constraint fk_course_up_lesson
    foreign key (lesson_id) references zhiyu.course_lessons(id) on delete cascade,
  constraint fk_course_up_last_kp
    foreign key (last_kp_id) references zhiyu.course_knowledge_points(id) on delete set null,

  constraint ck_course_up_status
    check (status in ('locked','unlocked','in_progress','passed')),
  constraint ck_course_up_best_score
    check (best_score is null or best_score between 0 and 100),
  constraint ck_course_up_attempts check (attempts >= 0),
  constraint ck_course_up_passed_consistency
    check ((status = 'passed' and passed_at is not null) or status <> 'passed')
);

create index idx_course_up_status   on zhiyu.course_user_progress (user_id, status);
create index idx_course_up_lesson   on zhiyu.course_user_progress (lesson_id);
create index idx_course_up_passed   on zhiyu.course_user_progress (lesson_id, passed_at) where status = 'passed';

create trigger tg_course_up_before_update_set_updated_at
  before update on zhiyu.course_user_progress
  for each row execute function zhiyu.set_updated_at();
```

### 四、RLS 策略

| 操作 | 角色 | 策略 |
|------|------|------|
| `select / insert / update / delete` | `authenticated` | 仅 `user_id = auth.uid()` |
| 全部 | `service_role` | 全量 |

策略命名：`course_up_self_rw`、`course_up_service_full`。

## 2. 表：`course_user_answers`（每道题作答流水）

**对应需求中的对象**：[temp/03 §3.3.3](../../../temp/03-数据库schema.md)
**性质**：高写入流水表。建议按 `answered_at` 月分区（pg_partman），避免单表过大。

### 一、字段定义

| 字段名 | 类型 | 必填 | 默认值 | 索引 | 对应需求中的信息 | 校验规则 |
|--------|------|------|--------|------|----------------|---------|
| `id` | `uuid` | ✅ | `gen_random_uuid()` | PK | — | — |
| `user_id` | `uuid` | ✅ | — | `idx_course_ua_user_kp` | 学员 | FK → `auth.users.id`，`ON DELETE CASCADE` |
| `question_id` | `uuid` | ✅ | — | `idx_course_ua_question` | 题目 | FK → `course_questions.id`，`ON DELETE RESTRICT` |
| `question_version` | `int` | ✅ | — | — | 答题时的题目版本 | ≥ 1 |
| `kp_id` | `uuid` | ✅ | — | `idx_course_ua_user_kp` | 关联 KP（冗余便于 SRS） | FK → `course_knowledge_points.id`，`ON DELETE RESTRICT` |
| `context_type` | `text` | ✅ | — | `idx_course_ua_context` | 答题上下文 | 枚举 `practice\|lesson_quiz\|chapter_test\|stage_exam\|hsk_mock\|srs_review` |
| `context_ref_id` | `uuid` | ❌ | `null` | — | 上下文实体 id（lesson_id / attempt_id 等） | — |
| `is_correct` | `bool` | ✅ | — | — | 是否答对 | — |
| `score` | `numeric(5,2)` | ✅ | — | — | 本题得分（连线题可能 0.25） | 0–10 |
| `user_answer` | `jsonb` | ✅ | — | — | 学员提交内容 | — |
| `duration_ms` | `int` | ❌ | `null` | — | 答题耗时 | 0–600000 |
| `answered_at` | `timestamptz` | ✅ | `now()` | `idx_course_ua_answered_at` | 答题时间 | 用作分区键 |

### 二、AI 补充字段

| 字段名 | 类型 | 补充理由 |
|--------|------|---------|
| `question_version` | `int` | 题目修订后历史回溯：可知"该次作答用的是哪个版本的题" |
| `score` | `numeric(5,2)` | 连线题部分分（0.25/对），非二值 |
| 月分区 | — | [temp/03 §3.7](../../../temp/03-数据库schema.md) 性能注意，按 `answered_at` 分区 |

### 三、DDL 草案

```sql
create table zhiyu.course_user_answers (
  id               uuid not null default gen_random_uuid(),
  user_id          uuid not null,
  question_id      uuid not null,
  question_version int  not null,
  kp_id            uuid not null,
  context_type     text not null,
  context_ref_id   uuid,
  is_correct       bool not null,
  score            numeric(5,2) not null,
  user_answer      jsonb not null,
  duration_ms      int,
  answered_at      timestamptz not null default now(),

  primary key (id, answered_at),  -- 分区表 PK 必须含分区键
  constraint fk_course_ua_user
    foreign key (user_id) references auth.users(id) on delete cascade,
  constraint fk_course_ua_question
    foreign key (question_id) references zhiyu.course_questions(id) on delete restrict,
  constraint fk_course_ua_kp
    foreign key (kp_id) references zhiyu.course_knowledge_points(id) on delete restrict,
  constraint ck_course_ua_context
    check (context_type in ('practice','lesson_quiz','chapter_test','stage_exam','hsk_mock','srs_review')),
  constraint ck_course_ua_score check (score between 0 and 10),
  constraint ck_course_ua_duration check (duration_ms is null or duration_ms between 0 and 600000),
  constraint ck_course_ua_q_version check (question_version >= 1)
) partition by range (answered_at);

-- 示例：建第一个月分区（实际由 pg_partman 自动维护）
create table zhiyu.course_user_answers_y2026m05
  partition of zhiyu.course_user_answers
  for values from ('2026-05-01') to ('2026-06-01');

create index idx_course_ua_user_kp     on zhiyu.course_user_answers (user_id, kp_id);
create index idx_course_ua_user_ans    on zhiyu.course_user_answers (user_id, answered_at desc);
create index idx_course_ua_question    on zhiyu.course_user_answers (question_id);
create index idx_course_ua_context     on zhiyu.course_user_answers (context_type, context_ref_id);
create index idx_course_ua_answered_at on zhiyu.course_user_answers (answered_at);
```

### 四、RLS 策略

| 操作 | 角色 | 策略 |
|------|------|------|
| `select` | `authenticated` | 仅 `user_id = auth.uid()` |
| `insert` | `authenticated` | 仅 `user_id = auth.uid()`（且经 RPC `fn_submit_answer` 写入，禁止前端直插） |
| `update / delete` | — | 全部禁止（流水不可篡改） |
| 全部 | `service_role` | 全量 |

策略命名：`course_ua_select_self`、`course_ua_insert_self_via_rpc`、`course_ua_service_full`。

## 3. 表：`course_user_srs`（SRS 队列 · Leitner 5 盒）

**对应需求中的对象**：[temp/02 §2.6 SRS 调度](../../../temp/02-知识点与题型内容模板.md) + [temp/03 §3.3.4](../../../temp/03-数据库schema.md)
**性质**：每个 `(user_id, kp_id)` 一行；运行时**零 AI 推理**，纯查询调度。

### 一、字段定义

| 字段名 | 类型 | 必填 | 默认值 | 索引 | 对应需求中的信息 | 校验规则 |
|--------|------|------|--------|------|----------------|---------|
| `user_id` | `uuid` | ✅ | — | PK 一部分 | 学员 | FK → `auth.users.id`，`ON DELETE CASCADE` |
| `kp_id` | `uuid` | ✅ | — | PK 一部分 | KP | FK → `course_knowledge_points.id`，`ON DELETE CASCADE` |
| `box` | `smallint` | ✅ | `1` | `idx_course_srs_box` | Leitner 盒位 | 1–5 |
| `due_at` | `timestamptz` | ✅ | `now()` | `idx_course_srs_due` | 下次复习到期时间 | — |
| `correct_streak` | `int` | ✅ | `0` | — | 连续答对次数 | ≥ 0 |
| `wrong_count` | `int` | ✅ | `0` | — | 累计答错次数 | ≥ 0 |
| `last_seen_at` | `timestamptz` | ❌ | `null` | — | 上次出现时间 | — |
| `last_q_type` | `text` | ❌ | `null` | — | 上次出题用的题型（轮换避免背答案） | 12 题型枚举 |

### `box → due_at` 调度公式（[temp/02 §2.6](../../../temp/02-知识点与题型内容模板.md)）

```
答对：box = min(box + 1, 5)
答错：box = 1，wrong_count += 1，correct_streak = 0
盒位 → 下次到期：
  box=1 → due_at = now() + 1 day
  box=2 → due_at = now() + 3 days
  box=3 → due_at = now() + 7 days
  box=4 → due_at = now() + 14 days
  box=5 → due_at = now() + 30 days
```

实现位置：RPC `fn_srs_update(p_user_id uuid, p_kp_id uuid, p_correct bool, p_q_type text)`，由 `fn_submit_answer` 在事务末尾调用。

### 二、AI 补充字段

| 字段名 | 类型 | 补充理由 |
|--------|------|---------|
| `last_q_type` | `text` | [temp/06 §6.8](../../../temp/06-关键业务流程.md) "今天选择，明天听音"——抽题时 `where q_type <> last_q_type` 优先 |
| `wrong_count` | `int` | 错题本展示与统计：`select * from course_user_srs where user_id=? and wrong_count>=1 order by wrong_count desc` |

### 三、DDL 草案

```sql
create table zhiyu.course_user_srs (
  user_id        uuid not null,
  kp_id          uuid not null,
  box            smallint not null default 1,
  due_at         timestamptz not null default now(),
  correct_streak int not null default 0,
  wrong_count    int not null default 0,
  last_seen_at   timestamptz,
  last_q_type    text,

  primary key (user_id, kp_id),
  constraint fk_course_srs_user
    foreign key (user_id) references auth.users(id) on delete cascade,
  constraint fk_course_srs_kp
    foreign key (kp_id) references zhiyu.course_knowledge_points(id) on delete cascade,
  constraint ck_course_srs_box check (box between 1 and 5),
  constraint ck_course_srs_streak check (correct_streak >= 0),
  constraint ck_course_srs_wrong_count check (wrong_count >= 0),
  constraint ck_course_srs_last_q_type
    check (last_q_type is null or last_q_type in (
      'mcq_meaning','mcq_zh','listen_pick','listen_pinyin','tone_pick',
      'match_pairs','sort_words','fill_blank_choice','type_pinyin',
      'type_zh_ime','image_pick','dialog_cloze'
    ))
);

create index idx_course_srs_due on zhiyu.course_user_srs (user_id, due_at);
create index idx_course_srs_box on zhiyu.course_user_srs (user_id, box, due_at);
```

### 四、RLS 策略

同 `course_user_progress`：自己读写自己。

## 4. 表：`course_user_exam_attempts`（考试答卷快照）

**对应需求中的对象**：[temp/03 §3.3.5](../../../temp/03-数据库schema.md) + [temp/06 §6.9 流程 H](../../../temp/06-关键业务流程.md)
**性质**：每次发起考试一行；存"抽到的题 id 列表 + 顺序"作为快照，便于争议判定。

### 一、字段定义

| 字段名 | 类型 | 必填 | 默认值 | 索引 | 对应需求中的信息 | 校验规则 |
|--------|------|------|--------|------|----------------|---------|
| `id` | `uuid` | ✅ | `gen_random_uuid()` | PK | — | — |
| `user_id` | `uuid` | ✅ | — | `idx_course_uea_user` | 学员 | FK → `auth.users.id`，`ON DELETE CASCADE` |
| `exam_id` | `uuid` | ✅ | — | `idx_course_uea_exam` | 试卷模板 | FK → `course_exams.id`，`ON DELETE RESTRICT` |
| `attempt_no` | `int` | ✅ | — | — | 该学员对该卷第几次尝试 | ≥ 1 |
| `score` | `numeric(5,2)` | ❌ | `null` | — | 总得分 | 0–100 |
| `passed` | `bool` | ❌ | `null` | `idx_course_uea_passed` | 是否通过 | — |
| `snapshot` | `jsonb` | ✅ | — | — | 快照 | 见 §snapshot schema |
| `state` | `text` | ✅ | `'in_progress'` | `idx_course_uea_state` | 答卷状态 | 枚举 `in_progress\|submitted\|expired\|abandoned` |
| `started_at` | `timestamptz` | ✅ | `now()` | — | 开始时间 | — |
| `expires_at` | `timestamptz` | ❌ | `null` | — | 到期时间（限时卷） | started_at + time_limit |
| `finished_at` | `timestamptz` | ❌ | `null` | — | 提交/过期时间 | `state ≠ in_progress` 时必须非空 |

### `snapshot` JSON Schema

```jsonc
{
  "question_ids": ["<uuid>", "<uuid>", ...],   // 抽到的题 id 顺序
  "question_versions": [3, 1, 2, ...],         // 与上面一一对应；用于历史回溯
  "blueprint_used": { /* 抽题时使用的 blueprint 副本 */ }
}
```

### 二、AI 补充字段

| 字段名 | 类型 | 补充理由 |
|--------|------|---------|
| `attempt_no` | `int` | [temp/07 §H5](../../../temp/07-待确认问题清单.md) "阶段考过了不能重考"，但其它卷可多次；`(user_id, exam_id, attempt_no)` 唯一 |
| `state` | `text` | 区分"答题中 / 已提交 / 超时未交 / 主动放弃"；[temp/05 §5.7](../../../temp/05-用户端模块设计.md) "退出 = 计零分"对应 `abandoned` |
| `expires_at` | `timestamptz` | 限时卷：服务端定时 cron 把 `now() > expires_at and state='in_progress'` 的标 `expired`，按当前已答题计分 |
| `question_versions` | `int[]` | 提交时对照版本，避免修题影响历史 |

### 三、DDL 草案

```sql
create table zhiyu.course_user_exam_attempts (
  id           uuid primary key default gen_random_uuid(),
  user_id      uuid not null,
  exam_id      uuid not null,
  attempt_no   int  not null,
  score        numeric(5,2),
  passed       bool,
  snapshot     jsonb not null,
  state        text not null default 'in_progress',
  started_at   timestamptz not null default now(),
  expires_at   timestamptz,
  finished_at  timestamptz,

  constraint fk_course_uea_user
    foreign key (user_id) references auth.users(id) on delete cascade,
  constraint fk_course_uea_exam
    foreign key (exam_id) references zhiyu.course_exams(id) on delete restrict,
  constraint uq_course_uea_attempt
    unique (user_id, exam_id, attempt_no),
  constraint ck_course_uea_attempt_no check (attempt_no >= 1),
  constraint ck_course_uea_score check (score is null or score between 0 and 100),
  constraint ck_course_uea_state
    check (state in ('in_progress','submitted','expired','abandoned')),
  constraint ck_course_uea_finished_consistency
    check ((state = 'in_progress' and finished_at is null) or
           (state <> 'in_progress' and finished_at is not null))
);

create index idx_course_uea_user   on zhiyu.course_user_exam_attempts (user_id, started_at desc);
create index idx_course_uea_exam   on zhiyu.course_user_exam_attempts (exam_id);
create index idx_course_uea_passed on zhiyu.course_user_exam_attempts (user_id, exam_id, passed);
create index idx_course_uea_state  on zhiyu.course_user_exam_attempts (state, expires_at) where state = 'in_progress';
```

### 四、RLS 策略

同上：自己读自己；写由 RPC `fn_start_exam` / `fn_submit_exam` 完成（service-role 内部调用）。
