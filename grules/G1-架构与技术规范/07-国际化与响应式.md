# 07 · 国际化与响应式

> 引用：[00-index.md](./00-index.md)
> 适用：所有 UI（应用端 + 管理后台）+ 所有面向用户的接口文案

## 一、支持语言矩阵

| 代码 | 名称 | 应用端 | 管理后台 | 默认语言 |
|------|------|-------|---------|---------|
| `zh` | 简体中文 | ✅ | ✅ | — |
| `en` | English | ✅ | ✅ | **回退兜底** |
| `vi` | Tiếng Việt（越南语）| ✅ | ✅ | — |
| `th` | ภาษาไทย（泰语）| ✅ | ✅ | — |
| `id` | Bahasa Indonesia（印尼语）| ✅ | ✅ | — |

- **缺省策略**：用户语言 → `Accept-Language` 头 → 浏览器 `navigator.language` → `en`。
- **回退链**：当前语言 key 缺失 → `en` → `zh`；都缺则显示 key 本身（dev）/ 空字符串（prod）。
- **文案产出**：**AI 自动翻译占位**（每条 key 标 `needs_review: true`），上线前 PM 安排人工校对（PM 决策）。

---

## 二、前端实现

- 框架：`i18next` + `react-i18next`。
- 文案源：`packages/shared-i18n/src/<locale>/<namespace>.json`，**5 种语言一一对应**，缺 key CI 报错。
- 命名空间按 PRD 模块切分：`common`、`auth`、`courses`、`games`、`novels`、`payment` ……
- 调用：`const { t } = useTranslation('courses'); t('list.empty')`。
- 切换：`/settings/language`，写入 Supabase user metadata + `localStorage`，全应用即时生效。
- 数字 / 日期 / 货币：使用 `Intl.NumberFormat` / `Intl.DateTimeFormat`，禁止手写格式串。
- 复数：使用 i18next 的 `_one/_other` 后缀。
- 富文本插值：使用 `<Trans>` 组件，禁止字符串拼接。

---

## 三、后端实现

- Hono 中间件：`middlewares/i18n.ts`，按 `Accept-Language` 解析得到 `c.var.locale`。
- 错误信息走 `packages/shared-i18n` 同一文案库：`t(c.var.locale, 'errors.40001')`。
- 多语言业务文案存 jsonb（结构 `{zh,en,vi,th,id}`，详见 [03-数据库规范.md §四 §八](./03-数据库规范.md)）。
- 邮件 / 短信模板按语言分文件：`templates/<channel>/<locale>/<name>.hbs`。
- Edge Functions 同样使用 `_shared/i18n.ts`，文案表通过 `scripts/sync-shared-to-edge.ts` 同步。

---

## 四、文案规范

1. 不允许在代码中硬编码任何用户可见文案；CI ESLint 规则 `i18n/no-literal-string` 强制。
2. key 命名：`<namespace>.<page>.<element>`，全小写下划线；嵌套深度 ≤ 4。
3. 长度差异：UI 必须为最长语言（一般为越南语 / 印尼语 + 30%）预留空间。
4. RTL：当前 5 语言均为 LTR；不实现 RTL，避免过度设计。
5. 中文相关教学内容（汉字本体、拼音、释义示例）属于"业务内容"，不走 i18n，按 [03-数据库规范.md §八](./03-数据库规范.md) 的多语言 jsonb 存储。

## 五、响应式断点

> Tailwind CSS 默认断点 + 项目语义命名，二者保持一致。

| 名称 | min-width | 设备 |
|------|-----------|------|
| `xs` | 0 | 小屏手机（iPhone SE 等）|
| `sm` | 640px | 普通手机横屏 / 平板竖屏 |
| `md` | 768px | 平板 |
| `lg` | 1024px | 小笔记本 |
| `xl` | 1280px | 桌面 |
| `2xl` | 1536px | 大屏 |

- **设计基线**：移动端 360×780（iPhone SE 2 物理像素）；桌面端 1440×900。
- **应用端（C 端）**：移动优先（mobile-first），所有页面必须在 `xs` 完整可用。
- **管理后台**：桌面优先（desktop-first），`< md` 仅保证只读查看可用，关键编辑提示切换大屏。

> v1 仅 Web 响应式，**不打包原生 App**；后期视需要再评估 React Native（PM 决策）。

## 六、移动端适配细则

1. 触控热区最小 44×44 CSS 像素。
2. 关键按钮固定底部安全区（iOS `env(safe-area-inset-bottom)`）。
3. 输入框聚焦时禁止页面缩放：`<meta name="viewport" content="width=device-width,initial-scale=1,maximum-scale=1,user-scalable=no">`。
4. 长列表必须虚拟滚动（TanStack Virtual）。
5. 游戏页（PixiJS）使用 `resize` 监听 + `devicePixelRatio` 适配；横竖屏切换需提示。

## 七、字体与排版

> 用户决策：字体要好看；Supabase 是本地化的，**禁止走 Google Fonts CDN**；其余由 AI 自定。

### 7.1 字体托管

- 全部字重以 `woff2` 形式存放在 `apps/web-*/public/fonts/`，由 Vite 构建时连同静态资源一起部署；**不**外链 CDN。
- 关键字重在 `index.html` 中 `<link rel="preload" as="font" crossorigin>` 预载。
- 仅引入"应用真用到的字重"（一般 `400 / 500 / 700`），减少首屏体积。

### 7.2 字体选型（按语言）

| 语言 | 首选字体 | 兜底栈 | 选择理由 |
|------|---------|--------|---------|
| zh | **Noto Sans SC**（开源、字重齐全、字形端正）| `"PingFang SC", "Microsoft YaHei", system-ui` | 教学场景对汉字字形清晰度要求高 |
| en | **Inter**（现代几何、字距精良）| `system-ui, -apple-system, sans-serif` | 与 shadcn/ui 默认审美一致 |
| vi | **Be Vietnam Pro**（越南本土字体厂出品，附符号字形优化）| `"Inter", "Noto Sans", system-ui` | 越南文 diacritics 不被截断 |
| th | **Noto Sans Thai Looped**（环形字形，泰文母语者更易读）| `"Sarabun", system-ui` | 泰文上下标层叠多，需要专门字形 |
| id | **Inter**（拉丁字符同英文）| `"Noto Sans", system-ui` | 印尼文为拉丁字母，无需独立字体 |
| 标题 / 中文教学卡片 | **Noto Serif SC**（搭配 Noto Sans SC）| `"Source Han Serif SC", serif` | 课程页面需要"庄重感"凸显汉字结构 |

### 7.3 排版

- 行高：中文 1.6、拉丁 1.5、泰文 1.7（避免上下标重叠）。
- 字号 token（rem）：`xs=12 / sm=14 / base=16 / lg=18 / xl=20 / 2xl=24 / 3xl=30 / 4xl=36`，移动端 base 不小于 14px。
- 中英混排时显式 `font-feature-settings: "palt"` 关闭，否则中文标点错位。

## 八、图片与图标

- 所有图标使用 `lucide-react`；禁止内联 SVG > 5 处复用。
- 图片必须提供 `alt` 文案（走 i18n）。
- 静态营销图按断点提供 `1x/2x` + `webp/avif`，全部存放在 `apps/web-*/public/`，**不**走外部 CDN。

---

## 九、可访问性（A11y）

- 颜色对比度 ≥ WCAG AA（4.5:1 正文 / 3:1 大字）。
- 所有交互元素键盘可达；模态框焦点陷阱由 Radix 提供。
- 表单控件必有 `<label>` 关联或 `aria-label`。
- 动画提供 `prefers-reduced-motion` 适配。
