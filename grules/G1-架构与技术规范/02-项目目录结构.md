## 一、顶层布局

```
zhiyu/
├── apps/                       # 可独立部署的应用
│   ├── web-app/                # 应用端前端（C 端用户，端口 3100）
│   ├── web-admin/              # 管理端前端（端口 4100）
│   ├── api-app/                # 应用端后端（端口 8100）
│   └── api-admin/              # 管理端后端（端口 9100）
│
├── packages/                   # 跨应用共享代码
│   ├── shared-schemas/         # Zod schema + JSDoc 类型，前后端共用
│   ├── shared-config/          # 常量、错误码、枚举
│   ├── shared-utils/           # 纯函数工具（日期、字符串、汉字处理）
│   ├── shared-i18n/            # 5 语言文案（zh/en/vi/th/id）
│   ├── ui-kit/                 # 基于 shadcn/ui 的项目级组件
│   ├── db/                     # Drizzle schema + 迁移脚本（唯一数据真相源）
│   ├── ai-adapters/            # LLM/Workflow/TTS/ASR 适配器 + mock fixture
│   └── game-engine/            # PixiJS 游戏共享内核
│
├── docker/                     # Docker 配置（唯一 dev 环境）
│   ├── compose.yaml            # 唯一 compose 文件
│   ├── Dockerfile.web          # 前端镜像（多阶段）
│   ├── Dockerfile.api          # 后端镜像
│   ├── caddy/                  # 反向代理配置
│   ├── postgres/               # 初始化脚本、扩展（pgvector）
│   ├── redis/                  # redis.conf
│   └── env/
│       ├── .env.example        # 全部变量模板（必交付）
│       └── README.md           # 各 API Key 来源、申请方式
│
├── grules/                     # 本规范目录（G 系列规则）
├── prds/                       # 产品需求文档
├── content/                    # 静态内容（中国文化、课程素材、小说大纲）
├── prompt/                     # 用户提供的提示词原文
├── template/                   # 文档模板
├── scripts/                    # 一次性脚本（数据迁移、数据导入）
│
├── pnpm-workspace.yaml
├── package.json                # workspace 根
├── tsconfig.base.json          # 前端共享 TS 配置
├── jsconfig.base.json          # 后端共享 JSDoc/JS 配置
├── .editorconfig
├── .gitignore
└── README.md
```

## 二、`apps/web-app`（应用端前端，C 端）

│   ├── features/               # 业务模块（按 PRD 编号对齐）
│   │   ├── 02-discover-china/
│   │   ├── 03-courses/
│   │   ├── 04-games/
│   │   ├── 05-novels/
│   │   ├── 06-user-account/
│   │   ├── 08-economy/
│   │   ├── 09-referral/
│   │   ├── 10-payment/
│   │   ├── 11-customer-service/
│   │   └── 15-i18n/
│   ├── components/             # 跨 feature 的展示组件
│   ├── hooks/
│   ├── lib/                    # supabase client、http、analytics

> **feature 内部统一三层**：`api/`（请求 hooks）+ `ui/`（组件）+ `model/`（store/derive）。

## 三、`apps/web-admin`（管理端前端）

结构同 `web-app`，但：

- `routes/` 全部位于 `_admin/` 守卫之下；
- `features/` 与 PRD `12-admin/` 子模块一一对齐；
- 引入 `TanStack Table` + `Tiptap`；
- 不引入 `game-engine`、`framer-motion`（保持轻量）。

---

## 四、`apps/api-app`（应用端后端，JS）

```
apps/api-app/
├── src/
│   ├── server.mjs              # 入口
│   ├── app.mjs                 # express 实例 + 中间件装配
│   ├── env.mjs                 # 启动时 Zod 校验环境变量
│   ├── routes/                 # 仅做路由声明，调用 controller
│   │   ├── index.mjs
│   │   ├── auth.routes.mjs
│   │   ├── courses.routes.mjs
│   │   └── ...
│   ├── controllers/            # 解析 req → 调用 service → 包装 res
│   ├── services/               # 业务逻辑，纯函数优先
│   ├── repositories/           # Drizzle 查询封装
│   ├── adapters/               # 外部服务调用入口（依赖 packages/ai-adapters）
│   ├── middlewares/            # auth、rateLimit、i18n、errorHandler
│   ├── jobs/                   # BullMQ 队列与 worker
│   ├── crons/                  # node-cron 定义
│   ├── ws/                     # Realtime 桥接（如有）
│   ├── errors/                 # 业务错误类
│   └── utils/
├── tests/                      # vitest + supertest
├── package.json
└── jsconfig.json
```

## 五、`apps/api-admin`（管理端后端，JS）

结构同 `api-app`；额外：

- `middlewares/rbac.mjs` 强制 `admin` 角色校验；
- 不暴露 C 端业务接口，仅审核/配置/统计。

## 六、`packages/db`（Drizzle 唯一数据源）

```
packages/db/
├── src/
│   ├── schema/                 # 按业务域拆分文件
│   │   ├── users.js
│   │   ├── courses.js
│   │   ├── games.js
│   │   ├── novels.js
│   │   ├── economy.js
│   │   ├── payments.js
│   │   └── index.js            # 汇出
│   ├── client.js               # postgres-js + drizzle 实例
│   └── helpers/                # jsonb 写入辅助等（见 user memory）
├── drizzle.config.js
├── migrations/
└── package.json
```

> **JSONB 写入**：必须使用 `helpers/jsonb-insert.js` 提供的 raw 客户端封装，避免 drizzle 0.30 + postgres-js 的双重编码 bug。

## 八、命名规则汇总

| 对象 | 规则 | 示例 |
|------|------|------|
| 目录 | kebab-case | `web-admin/`、`game-engine/` |
| TS/JS 普通文件 | kebab-case | `course-list.controller.mjs` |
| React 组件文件 | PascalCase | `CourseCard.tsx` |
| Hook 文件 | camelCase 加 `use` 前缀 | `useCourseProgress.ts` |
| Zustand store | camelCase + `.store.ts` | `gameSession.store.ts` |
| 测试文件 | 与被测同名 + `.test.{ts,mjs}` | `course-list.controller.test.mjs` |

---

## 九、禁止事项

1. **跨 app 直接 import**：必须通过 `packages/*`。
2. **在 app 内写 SQL/迁移**：所有 schema 变更只能在 `packages/db`。
3. **绕过 adapter 直接 fetch 第三方 LLM**：必须经 `ai-adapters`。
4. **新增顶层目录**：除非更新本规范，否则一律拒绝。
