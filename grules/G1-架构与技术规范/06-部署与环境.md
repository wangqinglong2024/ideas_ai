# 06 · 部署与环境

> 引用：[00-index.md](./00-index.md) · [env.md](../../env.md)

> **核心约束**：Docker-only，**唯一一个 dev 环境**用于自动化测试，**禁止多环境**。生产部署由用户自行处理，本仓库不输出生产 compose。

## 一、Docker 拓扑（唯一 dev 环境）

```
┌──────────────── docker compose (file: docker/compose.yaml) ────────────────┐
│                                                                            │
│  caddy           :80/:443  →  反向代理（dev 简化 HTTPS、统一入口）         │
│  web-app         :3100     →  应用端前端（Vite preview / dev server）      │
│  web-admin       :4100     →  管理端前端                                 │
│  api-app         :8100     →  应用端后端（Express）                        │
│  api-admin       :9100     →  管理端后端                                 │
│  postgres        :5432     →  Supabase Postgres（含 pgvector 扩展）         │
│  supabase-*      —          →  auth(GoTrue) / storage / realtime / kong    │
│  redis           :6379     →  缓存 + BullMQ                                │
│  worker          —          →  api-app 的 BullMQ worker（独立进程同镜像）   │
│                                                                            │
└────────────────────────────────────────────────────────────────────────────┘
```

### 1.1 端口策略

| 用途 | 端口 | 说明 |
|------|------|------|
| 应用端前端 | **3100** | 用户指定 |
| 应用端后端 | **8100** | 用户指定 |
| 管理端前端 | **4100** | 用户指定 |
| 管理端后端 | **9100** | 用户指定 |
| Postgres | 5432 | 仅 host 调试用，业务走 docker 网络 |
| Redis | 6379 | 同上 |

> **占用即释放**：`scripts/dev/free-ports.sh` 在 `docker compose up` 前自动 `kill` 占用 `3100/4100/8100/9100` 的进程；CI/dev 强制启用，不询问。

### 1.2 启动命令

```bash
# 一键启动唯一 dev 环境
pnpm dev               # 等价于：scripts/dev/free-ports.sh && docker compose up --build

# 跑测试
docker compose run --rm api-app   pnpm test
docker compose run --rm api-admin pnpm test
docker compose run --rm web-app   pnpm test
docker compose run --rm web-admin pnpm test
```

## 二、镜像与多阶段构建

| 镜像 | 基础镜像 | 阶段 |
|------|---------|------|
| `web-*` | `node:20-alpine` → `nginx:1.27-alpine` | install → build → static serve |
| `api-*` | `node:20-alpine` | install（含 dev deps）→ prune → run |
| `worker` | 同 `api-*`，仅 entrypoint 不同 | — |
| `postgres` | `supabase/postgres:15.x`（含 pgvector）| 一阶段 |

> 镜像内**严禁**复制：`.github/`、`.agents`、`.claude`、`_bmad`、`grules/`、`prompt/`、`backup/`、`template/`、`prds/`、`content/`、本地 `.env`。
> `.dockerignore` 必须显式列出上述目录。

## 三、环境变量

### 3.1 来源与优先级

`.env`（dev 默认）  ←  `docker/env/.env.dev`（仓库提交，**仅占位**）  ←  本机 `docker/env/.env.local`（不提交，覆盖密钥）

### 3.2 必备变量（应用端 + 管理端共用）

| 变量 | 示例 | 说明 |
|------|------|------|
| `APP_ENV` | `dev` | 唯一环境 |
| `APP_NAME` | `zhiyu` | 项目名 |
| `DATABASE_URL` | `postgres://postgres:postgres@postgres:5432/zhiyu_dev` | Drizzle / postgres-js 连接 |
| `REDIS_URL` | `redis://redis:6379/0` | 缓存 + BullMQ |
| `SUPABASE_URL` | `http://supabase-kong:8000` | 后端调用 |
| `SUPABASE_ANON_KEY` | `…` | 前端公开 |
| `SUPABASE_SERVICE_ROLE_KEY` | `…` | 后端绕过 RLS |
| `JWT_SECRET` | 32+ 字符 | 与 Supabase 一致 |
| `VITE_API_BASE_APP` | `http://localhost:8100/api/v1` | 应用端前端 |
| `VITE_API_BASE_ADMIN` | `http://localhost:9100/admin/v1` | 管理端前端 |
| `VITE_SUPABASE_URL` | `http://localhost:8000` | 前端浏览器直连 |
| `VITE_SUPABASE_ANON_KEY` | `…` | 同上 |
| `LOG_LEVEL` | `debug` | dev 默认 |
| `RATE_LIMIT_RPM` | `60` | 限流默认 |

### 3.3 第三方 / AI Key（缺则 mock，不阻塞）

> `docker/env/.env.example` 必须列出**全部**变量与说明，缺失时由对应 adapter fallback 到 mock。

| 变量 | 用途 | 缺失行为 |
|------|------|---------|
| `ANTHROPIC_API_KEY` | Claude（主创作模型）| `LLMAdapter` → mock fixture |
| `DEEPSEEK_API_KEY` | DeepSeek（副 / TTS）| 同上 |
| `OPENAI_API_KEY` | Embedding / 备用 | 走 hash 占位 embedding |
| `LANGSMITH_API_KEY` | 评估（可选）| 跳过上报 |
| `SMTP_*` / `RESEND_API_KEY` | 邮件 | console + 落 `outbox` 表 |
| `TENCENT_SMS_*` | 短信 | console |
| `WECHAT_PAY_*` | 微信支付 | mock 验签 + 假支付成功通知 |
| `S3_*` / `OSS_*` | 备份 | 跳过 |

> **新加 Key 流程**：在 `docker/env/.env.example` 增补占位 + 在 `docker/env/README.md` 写明申请方式 → adapter 增加 fallback 实现 → 提 PR。

## 四、数据库初始化

1. `docker/postgres/init/00-extensions.sql` 启用 `uuid-ossp`、`pgcrypto`、`vector`、`pg_trgm`。
2. `docker/postgres/init/01-schema.sql` 创建 `zhiyu_dev` schema。
3. 容器健康通过后，`api-app` 启动入口执行 `pnpm db:migrate`。
4. 失败立即退出，让 compose 标记不健康。

## 五、健康检查

| 服务 | 路径/命令 | 周期 |
|------|----------|------|
| api-app | `GET /api/v1/health` | 10s |
| api-admin | `GET /admin/v1/health` | 10s |
| web-* | `GET /` 200 | 30s |
| postgres | `pg_isready` | 10s |
| redis | `redis-cli ping` | 10s |

`/health` 响应：

```json
{ "code": 0, "data": { "uptime_s": 1234, "db": "up", "redis": "up", "adapters": { "llm": "mock" } } }
```

## 六、日志与可观测（dev 默认）

- 容器 stdout JSON → `docker logs`；不挂载到宿主，体积由 Docker logging driver 控制（`max-size=20m, max-file=5`）。
- 错误聚合（可选）：本地 Loki + Grafana 容器，默认关闭，由 `compose.yaml` 的 `profiles: [observability]` 提供。

## 七、备份与恢复

- dev 环境**默认不开启外部备份**（用户偏好）。
- 提供 `scripts/db/dump.sh` / `restore.sh` 供本地手动使用。

## 八、CI（容器内自动化）

- 流水线（如有）只依赖 Docker：`docker compose up -d` → `docker compose run --rm <svc> pnpm <task>` → `docker compose down -v`。
- 不集成 GitHub Actions / 第三方 SaaS（用户偏好）。
