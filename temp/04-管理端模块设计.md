# 04 · 管理端模块设计（web-admin）

> ⚠️ **2025-11 变更**：系统内生成工作台作废（题目/KP 由运营线下生成后导入）；审核仅保留学员举报处理；考试中心管理改为主题→阶段→章→节层级，阶段考和章测均展示题目卡片式试抽。
> 详见 [function/02-course/ai/CHANGELOG-2025-11.md](../function/02-course/ai/CHANGELOG-2025-11.md)。本文档历史段落若与变更冲突，以 CHANGELOG 为准。

> 这是给"内容运营 + 审核员 + 老板"用的后台。
> 设计目标：**让一个 6 人小团队能把 ~11 200 个 KP / ~5 万道题在 3–4 个月内出完并发布**（数量对齐 00/01）。

---

## 4.1 角色与权限

| 角色 | 看 | 能做 |
|---|---|---|
| `super` | 全部 | 全部，含发布、改用户、改账号、系统配置 |
| `content_admin` | 自己 `tracks_scope` 内 | 课程目录 CRUD、组卷、发布预览（不能终发布） |
| `reviewer` | 自己 `tracks_scope` 内的内容与学员举报 | 点检 KP / Question、处理学员举报、手动调整内容 |
| `readonly` | 全部数据 | 只看不动（数据/财务） |

行级权限通过 `tracks_scope` 数组控制：列表页 SQL 自带 `WHERE track_code = ANY($scope)`。

## 4.2 顶部菜单（一级）

```
仪表盘 | 课程内容 | 题库 | 举报处理 | 考试中心 | 媒资库 | 学员与运营 | 系统
```

---

## 4.3 模块 1：仪表盘

一屏看清楚"还差多少内容 / 还差多少点检 / 哪些举报最急"。

| 卡片 | 指标 |
|---|---|
| 今日新增 KP / Q | 草稿 / 待审 / 通过 |
| 内容缺口 | 按主题分阶段，未达标节数 |
| 导入质量 | 昨日 / 本月导入批次数、失败行、重复命中 |
| 审核 SLA | 平均待审天数、超 3 天数量 |
| 学员活跃 | DAU / WAU / MAU、付费转化（来自 user_*） |
| 题目质量 | 错题率最高 Top20、争议题（多人反馈） |

---

## 4.4 模块 2：课程内容（树形结构）

左侧是"主题 → 阶段 → 章 → 节"四级树，右侧是节详情。

### 4.4.1 树视图操作
- 拖拽排序（更新 `sort_order`）
- 右键：新增子节点 / 复制 / 移动到 / 改发布状态
- 节点上显示徽章：`草稿 12` / `待审 3` / `已发布`

### 4.4.2 节详情页（最常用页面，一屏要塞下）

```
─────────────────────────────────────────────────────────────
[节信息]  ec-2-3-1  优惠券与折扣        [编辑] [预览] [发布]
学习目标 | 预计时长 12 分钟 | 状态：草稿
─────────────────────────────────────────────────────────────
[KP 列表]  ▢全选            [+ 添加现有 KP] [导入 KP]
┌─排序─类型──标题──拼音──难度──题数──状态──操作─────────┐
│ 1   word  价格  jià gé   2     5     ✅审核通过 [编辑][换][删]│
│ 2   word  优惠券 yōu...   2     4     🟡待审   [审核]         │
│ 3   语法  把字句         3     6     📝草稿   [编辑]          │
│ ...                                                          │
└──────────────────────────────────────────────────────────────┘

[节末小测]  blueprint：6 题（按题型权重从本节题库抽取）
            [试抽 6 题] [重抽]
─────────────────────────────────────────────────────────────
```

### 4.4.3 KP 编辑抽屉（点 KP 任意行打开）

- 顶部：KP code / 类型 / 版本 / 来源批次
- 中部 Tab：
  - **基础**：标题/拼音/难度/标签
  - **内容**：按 KP 类型动态表单（word/grammar/dialog 各异）
  - **翻译**：5 种语言并排（zh/en/vi/th/id）
  - **媒资**：音频试听 + 手动替换、图片上传/替换
  - **题目**：本 KP 关联的所有 Question 列表（4–6 条）
- 右上角：保存 / 保存并发布 / 归档（带备注）

---

## 4.5 模块 3：题库（横向视角）

> 课程内容是"按章节看"，题库是"按题看"——审题专用。

筛选器：主题 / 题型 / 难度 / 状态 / 关联 KP 类型 / 来源批次。

每行：q_code / 题干预览 / 答案 / KP / 错题率 / 状态 / 操作（预览、编辑、归档、删除）。

**审题双开布局**：左半屏列表，右半屏题目预览（与用户端 1:1 渲染）+ 解析编辑框。

批量操作：批量通过 / 批量驳回 / 批量打标签 / 批量重出。

---

## 4.6 模块 4：批量导入与质量点检

题目与 KP 数据由运营在线下生成、整理后导入系统；后台只负责结构化导入、校验、点检和发布。

### 4.6.1 批量导入
- 支持 KP / Question 的 CSV、JSON 导入。
- 导入前先预览：展示总行数、错误行、重复命中、缺失字段。
- 导入后写来源批次、payload hash、操作者、导入时间，便于追溯。

### 4.6.2 质量点检
- 内容运营在 P-A-4 / P-A-5 中按主题、题型、状态筛选检查。
- 学员举报较多的题目在 P-A-6 聚合置顶。
- 点检后可保存、发布、归档或删除，不提供系统内 AI 生成任务。

## 4.7 模块 5：考试中心

| 子页 | 干嘛 |
|---|---|
| 主题层 | 先选电商 / 工厂 / HSK / 日常 / 共享主题 |
| 阶段层 | 展示阶段考配置、可抽题数、最近试抽结果 |
| 章层 | 展示章测配置、可抽题数、最近试抽结果 |
| 节层 | 只读展示节末小测 6 题配置和可抽题统计 |
| HSK 模考（HSK 主题专属） | 按官方等级题型比例配置，仅 HSK 主题展示 |

---

## 4.8 模块 6：媒资库

- 音频：按声线分桶（女声标准 / 男声标准 / 慢速版），支持试听、替换、软删
- 图片：场景图分类（电商场景、工厂车间、街景、菜市场…），支持上传、替换、软删
- 去重：上传时按 `hash` 命中已有就直接复用
- 关联：每个媒资能反查"被哪些 KP / Q 引用"，删除前必拦截

---

## 4.9 模块 7：学员与运营

| 子页 | 干嘛 |
|---|---|
| 学员列表 | 搜索、看学习画像、封禁、改订阅 |
| 学习数据 | 完课率、每节平均时长、题正确率 Top/Bottom |
| 反馈中心 | 学员对题目的"举报错题"，落 `course_content_reports`，由 P-A-6 处理 |

## 4.10 模块 8：系统

- 管理员账号 / 角色 / 范围
- 操作审计日志（admin_audit_log）
- 应用端国际化字典（UI 文案 zh/en/vi/th/id，给前端拉）；管理后台 UI 保持中文
- CDN 配置 / TTS 服务商 key / AI 模型 key（加密存）

## 4.11 关键工作面板：「内容缺口看板」

> 是上面"仪表盘"的展开版，单列出来因为它是运营的指挥中心。

```
主题：电商
┌──Stage──Chapter──Lesson────需要KP数──已生成──已审──已发布──缺口──[操作]
│ 1     1        ec-1-1-1   10        10     10    10       0   ✅
│ 1     1        ec-1-1-2   10        10      6     0       4   [批量审]
│ 1     2        ec-1-2-1    9         0      0     0       9   [导入 KP]
│ ...
└──
```

点"导入 KP"直接打开批量导入弹窗，并自动带入当前主题 / 阶段 / 章 / 节上下文。

---

## 4.12 发布流程（每个内容必经）

```
draft → reviewing → approved → (chapter/stage published=true) → CDN 同步
                ↘ rejected → 回到 draft（带审核备注）
                ↘ archived → 旧版本归档，必要时通过新导入批次修正
```

发布是"**节级**"开关：
- `lessons.is_published = true` 才会出现在用户端学习地图。
- 发布前自动校验：节内 KP 全部 `approved` ？每个 KP 至少 N 道题？内容 5 语言齐 ？音频齐 ？
