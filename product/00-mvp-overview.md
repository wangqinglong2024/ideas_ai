## 一、项目概述

### 1.1 一句话定位

> **PlayLingo = 了解中国的窗口 + 学中文的课堂 + 练中文的游乐场**

东南亚第一款融合中国文化探索、系统中文课程和趣味游戏的三合一中文学习 Web 应用。

### 1.2 核心数据

| 指标 | 数值 |
|------|------|
| 首发市场 | 越南 🇻🇳 |
| 定价 | $2.99/月（Duolingo 的 1/4） |
| MVP 开发周期 | 4 周（1 人全栈） |
| 内容体量 | 40 篇文化文章 + 20 课语音课程 + 20 关游戏 + 3 款迷你游戏 |
| 技术栈 | React + Phaser 3 + FastAPI + Supabase + Paddle MoR |
| 盈亏平衡 | ~100 付费用户（预计第 3 月达成） |
| 12 个月目标 | DAU 45K / MRR $14.4K / 年化 $170K+ |

### 1.3 竞争优势

| 文化内容深度 | ★★★★★ | ★☆☆☆☆ | ★★☆☆☆ | ★★☆☆☆ |
| 系统课程 | ★★★★☆ | ★★★☆☆ | ★★★★☆ | ★★★★★ |

## 二、三大核心模块

### 2.1 模块架构

```
┌─────────────────────────────────────────────────────────────┐
│                     PlayLingo 三合一架构                       │
├──────────────┬──────────────┬────────────────────────────────┤
│  🏛️ 发现中国   │  📚 系统课程   │  🎮 趣味游戏                    │
│              │              │                                │
│  8大文化板块   │  4级110课    │  4区域60关                       │
│  中越双语文章  │  HSK 1-3对标  │  3款迷你游戏                    │
│  互动测验     │  9种练习题型   │  4场Boss战                      │
│  SEO引流     │  证书认证     │  8段过场动画                     │
│              │              │  PvP对战(v1.1)                  │
├──────────────┴──────────────┴────────────────────────────────┤
│  跨模块基础设施：词汇SRS · 会员支付 · 推广返佣 · 成就系统 · 管理后台   │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 三模块飞轮

```
文化内容激发兴趣 → 用户想学中文 → 课程提供系统知识
        ↑                                    ↓
成就感驱动更深入的文化探索 ← 用户获得成就感 ← 游戏巩固应用能力
```

三模块互相导流：
- 文化阅读遇到不懂的词汇 → 引导去课程学习
- 课程中学到的语法 → 在游戏中实战运用
- 游戏中接触的文化背景 → 回到文化模块深入了解

### 2.3 模块详情

#### 🏛️ 发现中国 — 用文化点燃学习兴趣

| 维度 | 说明 | PRD |
|------|------|-----|

| MVP 范围 | 8板块 × 5篇 = **40 篇** | |

## 四、技术架构

### 4.1 系统总览

```
用户浏览器(PWA)                        管理后台(SPA)
       │                                    │
       ▼                                    ▼
   Cloudflare CDN ──→ Nginx 反代 ──→ /api/*  /api/admin/*
                                        │
                                        ▼
                                   FastAPI (Python, async)
                                        │
            ┌──────────┬────────────────┼──────────┐
            ▼          ▼                ▼          ▼
      Supabase     Redis (Upstash)  Cloudflare R2  外部服务
      (PG+Auth+RLS)  (缓存/排行)   (对象存储)    (Paddle/Azure TTS/Dify)
```

### 4.2 前端技术栈

| 技术 | 用途 |
|------|------|
| React 18 + TypeScript | 应用框架 |
| Vite | 构建工具 |
| Phaser 3 | 游戏引擎（关卡/迷你游戏/Boss战） |
| Tailwind CSS v4 | 样式系统 |
| Zustand | 状态管理 |
| React Query | 服务端状态 |
| i18next | 国际化（动态加载） |
| Workbox | PWA 离线缓存 |
| Ant Design Pro | 管理后台 UI |

### 4.3 后端架构

| 技术 | 用途 |
|------|------|
| FastAPI | 全异步 Python API |
| Supabase | PostgreSQL + Auth + RLS |
| Redis (Upstash) | 缓存 + 排行榜 |
| Cloudflare R2 | 对象存储（图片/音频） |
| Paddle | 支付网关（MoR 模式） |
| Azure TTS | 语音合成 |
| Dify | AI Workflow（内容生成/AI 增值） |

### 4.4 内容动态加载与反爬虫

| 内容类型 | 加载方式 | SEO |
|---------|---------|-----|
| 界面文字 | API 动态加载 → localStorage 缓存 | ❌ |
| 文化文章（App 内） | API 动态加载（需 Auth） | ❌ |
| 文化文章（SEO 公开页） | SSR 服务端渲染 | ✅ |
| 课程/游戏内容 | API 动态加载（需 Auth） | ❌ |
| 管理后台 | 纯 SPA + API 动态加载 | ❌ |

**关键原则**：所有内容通过管理后台配置后动态显示，发布无需更新代码。HTML 源码不含业务文本（SEO 公开页除外）。

### 4.5 部署方案

| 组件 | 方案 |
|------|------|
| 服务器 | 新加坡（腾讯云 / AWS），越南延迟 25-40ms |
| 容器化 | Docker Compose |
| CDN | Cloudflare（静态资源 max-age: 30d） |
| SSL | Cloudflare 全站 HTTPS |
| 数据库备份 | Supabase 自动每日备份 |
| 监控 | 基础 APM + 错误日志 + 自定义数据看板 |

| 佣金管理 | — | 推广员列表 + 提现审核 |
| i18n 文本 | 全站文本 CRUD + 动态加载 API | 缺失翻译检测 |
| 系统配置 | — | 功能开关 + 定价配置 |

### 5.2 权限体系

| 角色 | 说明 |
|------|------|
| 超级管理员 | 全部权限 + 创建/删除管理员（系统初始化时创建） |
| 普通管理员 | 内容管理 + 用户查看 + 数据查看（不可管理管理员和系统配置） |

认证方式：用户名 + 密码（独立于用户端 Supabase Auth），JWT 2h/7d。

### 5.3 资源上传规格

所有管理后台上传资源严格匹配应用端显示要求：

| 用途 | 尺寸 | 格式 | 上限 |
|------|------|------|------|
| 文章封面 | 1200×675px | WebP | 200KB |
| 文章/课程配图 | 800×auto | WebP | 150KB |
| Level 封面 | 800×600px | WebP | 150KB |
| 词汇/课文音频 | — | MP3 16kHz 单声道 | 100KB-2MB |

上传流程：原始文件 → 服务端校验 → 自动转 WebP/MP3 → 上传 R2 → 返回 CDN URL。
