## 一、产品定位

PlayLingo 管理后台是面向运营团队的内部管理系统。

> **管理后台 = 内容工厂 + 运营驾驶舱 + 系统控制台**

### 1.1 核心原则

| 原则 | 说明 |
|------|------|
| 配置即发布 | 所有内容在后台配置后即时生效，无需更新代码、无需重新部署 |
| 上传即映射 | 后台上传资源的格式/尺寸严格匹配应用端显示要求，所见即所得 |
| 动态文本 | 管理端和应用端页面文字均通过 API 动态加载，HTML 不含业务文本 |
| 简单权限 | 超级管理员 + 普通管理员两级，不搞复杂角色 |
| 审计可追溯 | 所有增删改操作记录操作人、时间、内容变更 |

### 1.2 技术选型

| 维度 | 方案 |
|------|------|
| 前端框架 | React + TypeScript + Ant Design Pro |
| 状态管理 | Zustand + React Query |
| 图表库 | ECharts 5 |
| 部署 | 与用户端同域 `/admin` 路径，Nginx 反代 |
| 后端 | 复用 FastAPI，`/api/admin/*` 前缀路由 |
| 认证 | JWT（独立于用户端 Supabase Auth） |

## 五、非功能性要求

| 维度 | 要求 |
|------|------|
| 页面加载 | ≤ 2s |
| 列表查询 | ≤ 1s |
| 并发 | 10 管理员同时在线 |
| 浏览器 | Chrome 90+ / Edge 90+（不做移动端适配） |
| 访问控制 | IP 白名单（可选） + HTTPS Only |
| 数据备份 | Supabase 自动每日备份 + 手动导出 |
| 操作审计 | 全部 CUD 操作记录至 audit_logs |
