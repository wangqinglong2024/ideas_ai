## 一、模块概述

### 1.1 目标

管理员认证系统为管理后台提供独立于用户端的登录与权限体系，确保只有授权人员可操作后台功能，所有操作可追溯。

### 1.2 核心指标

| 指标 | 目标 |
|------|------|
| 登录响应 | ≤ 500ms |
| JWT 刷新成功率 | ≥ 99.9% |
| 审计日志覆盖率 | 100%（所有 CUD 操作） |

```
└─────────────────────────────────────────────┘
```

**交互规则**：
- 居中登录卡片，毛玻璃背景
- 用户名：3-50 字符，字母数字下划线
- 密码：≥12 字符，含大小写 + 数字 + 特殊字符
- 密码可切换明文/密文（👁️ 图标）
- 回车键触发登录
- 登录失败显示剩余尝试次数
- 连续 5 次失败 → 锁定 30 分钟，显示倒计时

**登录成功后**：
- 跳转至 `/admin/dashboard`
- 颁发 Access Token（2h）+ Refresh Token（7d，HttpOnly Cookie）

## 四、安全措施

| 措施 | 规则 |
|------|------|
| 密码存储 | bcrypt 哈希，salt rounds = 12 |
| 登录锁定 | 连续 5 次失败 → 锁定 30 分钟 |
| JWT 过期 | Access Token 2h / Refresh Token 7d |
| HTTPS | 管理端强制 HTTPS，Cookie Secure + SameSite=Strict |
| IP 白名单 | 可选，超管在系统配置中设置 |
| 操作日志 | 所有 CUD 操作记录 admin_id + action + target + timestamp + IP |
| 会话管理 | 关闭浏览器后需重新登录（Session Cookie） |

## 五、数据模型

```sql
CREATE TABLE admin_users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  username VARCHAR(50) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  role VARCHAR(20) NOT NULL DEFAULT 'admin', -- 'super_admin' | 'admin'
  is_active BOOLEAN DEFAULT true,
  last_login_at TIMESTAMPTZ,
  failed_login_count INT DEFAULT 0,
  locked_until TIMESTAMPTZ,
  created_by UUID REFERENCES admin_users(id),
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);

CREATE TABLE admin_audit_logs (
  id BIGSERIAL PRIMARY KEY,
  admin_id UUID NOT NULL REFERENCES admin_users(id),
  action VARCHAR(50) NOT NULL,
  target_type VARCHAR(50) NOT NULL,
  target_id VARCHAR(100),
  details JSONB,
  ip_address INET,
  created_at TIMESTAMPTZ DEFAULT now()
);
```

## 七、用户故事

| # | 角色 | 故事 | 验收标准 |
|---|------|------|---------|
| 1 | 超管 | 初次部署后用 seed 凭据登录 | 可成功登录，被引导修改初始密码 |
| 2 | 超管 | 创建一个普通管理员账号 | 新管理员可用分配的凭据登录 |
| 3 | 超管 | 查看谁在什么时间做了什么操作 | 审计日志可按人/类型/时间筛选 |
| 4 | 管理员 | 修改自己的密码 | 旧密码验证通过后可设置新密码 |
| 5 | 管理员 | 连续输错密码 5 次 | 账号锁定 30 分钟，显示倒计时 |
| 6 | 管理员 | Token 过期后继续操作 | 自动用 Refresh Token 续签，无感刷新 |
