# 用户封禁/解封

> 关联文档：[用户列表页](01-user-list.md) | [用户详情页](02-user-detail.md) | [管理后台概述 §四](../00-admin-overview.md)
> 角色权限：超级管理员、用户运营
> 情绪目标：**严谨、公正、可追溯** — 每一次封禁/解封都有完整记录，操作不可轻率

---

## 一、封禁流程

### 1.1 入口

| 入口位置 | 操作方式 |
|---------|---------|
| **用户列表页** | 操作列"封禁"文字按钮（仅"正常"状态用户显示） |
| **用户详情页** | 基本信息区右侧"封禁用户"危险按钮（仅"正常"状态用户显示） |

点击上述按钮后，打开**封禁操作弹窗**。

### 1.2 封禁操作弹窗

| 属性 | 说明 |
|------|------|
| **弹窗类型** | 模态对话框（Modal），Level 4 阴影 |
| **弹窗宽度** | 480px |
| **标题** | H3 级别（20px），"封禁用户" |
| **副标题** | Body S 级别，灰色，显示"即将封禁用户：{昵称}（ID: {用户ID}）" |

#### 表单字段

| # | 字段 | 类型 | 必填 | 说明 |
|---|------|------|------|------|
| 1 | **封禁原因** | 下拉单选 | ✅ | 选项：违规言论 / 作弊行为 / 异常操作 / 其他 |
| 2 | **补充说明** | 多行文本输入框 | 选择"其他"时必填，其他选项可选 | 最大 200 字；Placeholder "请补充封禁原因详情…" |
| 3 | **封禁期限** | 下拉单选 | ✅ | 选项：1 天 / 7 天 / 30 天 / 永久 |

#### 封禁期限说明

| 期限选项 | 解封时间计算 | 说明 |
|---------|------------|------|
| 1 天 | 当前时间 + 24 小时 | — |
| 7 天 | 当前时间 + 168 小时 | — |
| 30 天 | 当前时间 + 720 小时 | — |
| 永久 | 无自动解封时间 | 仅可由管理员手动解封 |

#### 弹窗按钮

| 按钮 | 样式 | 位置 | 点击行为 |
|------|------|------|---------|
| **取消** | 次要按钮（`.glass-button-secondary`），圆角 `rounded-full` | 弹窗底部靠左 | 关闭弹窗，不执行任何操作 |
| **确认封禁** | 危险按钮（红色背景 `#ef4444`），圆角 `rounded-full` | 弹窗底部靠右 | 表单校验通过 → 打开二次确认弹窗 |

#### 表单校验

| 校验规则 | 错误提示 |
|---------|---------|
| 封禁原因未选择 | 下拉框红色边框 + "请选择封禁原因" |
| 选择"其他"但补充说明为空 | 输入框红色边框 + "请填写封禁原因详情" |
| 封禁期限未选择 | 下拉框红色边框 + "请选择封禁期限" |

### 1.3 二次确认弹窗

| 属性 | 说明 |
|------|------|
| **弹窗类型** | 确认对话框，叠加在封禁操作弹窗上方 |
| **弹窗宽度** | 400px |
| **图标** | Lucide `AlertTriangle`，48px，Amber 色 `#d97706`，居中 |
| **标题** | H3 级别，居中，"确认封禁？" |
| **内容** | Body 级别，居中，"你即将封禁用户 **{昵称}**，期限为 **{期限}**。封禁后该用户将无法使用任何功能。" |
| **永久封禁额外警告** | 当期限为"永久"时，内容下方追加红色文字："⚠️ 永久封禁仅可由管理员手动解除" |

#### 按钮

| 按钮 | 样式 | 点击行为 |
|------|------|---------|
| **取消** | 次要按钮 | 关闭二次确认弹窗，返回封禁操作弹窗 |
| **确认** | 危险按钮 | 执行封禁 → 按钮变 Loading 态 → 成功后关闭所有弹窗 + Toast"用户已封禁" + 刷新用户状态 |

### 1.4 封禁执行后

| 步骤 | 说明 |
|------|------|
| 1 | 用户账号状态变更为"封禁中" |
| 2 | 记录封禁日志（封禁时间、原因、期限、操作管理员） |
| 3 | 用户列表页和用户详情页状态实时更新 |
| 4 | "封禁用户"按钮变为"解封用户"按钮 |
| 5 | 写入系统操作日志（管理员 ID + 操作类型"封禁" + 目标用户 ID + 时间） |

## 二、封禁效果（用户端表现）

### 2.1 登录后封禁通知

| 属性 | 说明 |
|------|------|
| **触发条件** | 被封禁用户在封禁期间登录应用端 |
| **展示方式** | 全屏模态弹窗，不可关闭，不可绕过 |
| **弹窗内容** | — |

| 字段 | 说明 |
|------|------|
| **标题** | "账号已被封禁" |
| **封禁原因** | 显示管理员选择的原因分类（不显示补充说明详情，保护运营人员隐私） |
| **解封日期** | 限期封禁显示："预计解封时间：YYYY-MM-DD HH:mm"；永久封禁显示："该账号已被永久封禁" |
| **联系方式** | "如有疑问，请联系客服：support@zhiyu.app" |
| **操作** | 仅有"退出登录"按钮 |

### 2.2 封禁期间功能限制

| 功能 | 是否可用 | 说明 |
|------|---------|------|
| 登录 | ✅ 可登录 | 登录后弹出封禁通知 |
| 浏览发现中国 | ❌ 不可用 | 封禁弹窗遮挡，无法操作 |
| 课程学习 | ❌ 不可用 | 封禁弹窗遮挡，无法操作 |
| 游戏 | ❌ 不可用 | 封禁弹窗遮挡，无法操作 |
| 个人中心 | ❌ 不可用 | 封禁弹窗遮挡，无法操作 |
| 购买课程 | ❌ 不可用 | 封禁弹窗遮挡，无法操作 |

### 2.3 已购课程有效期规则

| 规则 | 说明 |
|------|------|
| **课程有效期不因封禁暂停** | 用户在 2026-01-01 购买 L4（有效期至 2029-01-01），2026-06-01 被封禁 30 天，解封后有效期仍为 2029-01-01，不延长 |
| **设计理由** | 封禁是对违规行为的惩罚，暂停有效期相当于减轻惩罚力度，不符合产品意图 |

## 三、解封流程

### 3.1 自动解封

| 属性 | 说明 |
|------|------|
| **触发条件** | 封禁期限到期（1天/7天/30天） |
| **执行方式** | 系统定时任务自动检测，到期后自动将用户状态改为"正常" |
| **记录** | 封禁历史记录中"解封时间"更新为实际解封时间，"解封方式"标记为"自动到期" |
| **用户通知** | 解封后用户下次登录时不再弹出封禁通知，正常使用所有功能 |

### 3.2 手动提前解封

#### 入口

| 入口位置 | 操作方式 |
|---------|---------|
| **用户列表页** | 操作列"解封"文字按钮（仅"封禁中"状态用户显示） |
| **用户详情页** | 基本信息区右侧"解封用户"成功按钮（仅"封禁中"状态用户显示） |

#### 解封确认弹窗

| 属性 | 说明 |
|------|------|
| **弹窗宽度** | 400px |
| **图标** | Lucide `ShieldCheck`，48px，成功色 `#22c55e`，居中 |
| **标题** | H3 级别，居中，"确认解封？" |
| **内容** | Body 级别，"即将解封用户 **{昵称}**。当前封禁原因：{原因}，原封禁期限：{期限}。解封后用户将立即恢复所有功能。" |

#### 按钮

| 按钮 | 样式 | 点击行为 |
|------|------|---------|
| **取消** | 次要按钮 | 关闭弹窗 |
| **确认解封** | 成功按钮（绿色背景 `#22c55e`） | 执行解封 → 按钮 Loading 态 → 成功后关闭弹窗 + Toast"用户已解封" + 刷新用户状态 |

#### 解封执行后

| 步骤 | 说明 |
|------|------|
| 1 | 用户账号状态变更为"正常" |
| 2 | 封禁历史记录更新：解封时间、解封方式"手动解封"、解封操作人 |
| 3 | "解封用户"按钮恢复为"封禁用户"按钮 |
| 4 | 写入系统操作日志（管理员 ID + 操作类型"解封" + 目标用户 ID + 时间） |

## 四、封禁历史记录

### 4.1 展示位置

- **用户详情页 Tab 6"封禁历史"**：展示该用户的完整封禁/解封历史记录（详见 [02-user-detail.md §九](02-user-detail.md)）

### 4.2 审计要求

每条封禁/解封记录必须包含以下字段，不可修改、不可删除：

| 字段 | 说明 |
|------|------|
| 封禁时间 | 精确到秒（UTC） |
| 封禁原因分类 | 违规言论 / 作弊行为 / 异常操作 / 其他 |
| 封禁原因补充 | 手动输入的详情（如有） |
| 封禁期限 | 1天 / 7天 / 30天 / 永久 |
| 封禁操作管理员 | 管理员姓名 + ID |
| 解封时间 | 精确到秒（UTC），未解封显示空 |
| 解封方式 | 自动到期 / 手动解封 |
| 解封操作管理员 | 手动解封时记录管理员姓名 + ID |

## 五、状态矩阵

| 状态 | 设计要求 |
|------|---------|
| **空状态 (Empty)** | 不适用（封禁/解封为操作弹窗，无独立页面空状态） |
| **加载中 (Loading)** | 弹窗内"确认封禁"/"确认解封"按钮变为 Loading 态（旋转图标 + 按钮 Disabled），防止重复提交 |
| **首次加载 (First Load)** | 不适用 |
| **成功 (Success)** | 封禁成功：Toast 绿色"用户已封禁"，2s 自动消失；解封成功：Toast 绿色"用户已解封"，2s 自动消失 |
| **错误 (Error)** | 封禁/解封失败：Toast 红色"操作失败，请重试"，需手动关闭；弹窗不自动关闭，用户可重试 |
| **部分加载 (Partial)** | 不适用 |
| **离线 (Offline)** | 点击封禁/解封按钮时检测网络状态：离线则 Toast 红色"网络连接已断开，请恢复网络后重试"，不打开弹窗 |
