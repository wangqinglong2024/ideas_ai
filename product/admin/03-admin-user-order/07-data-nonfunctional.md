## 一、数据流向说明

### 1.1 用户管理数据流

```
注册来源（邮箱/Google/Apple）
       ↓
  Supabase Auth（用户认证）
       ↓
  用户表 users（基本信息）
       ↓                        ↓                       ↓
  管理后台（用户列表/详情）   应用端（个人中心）    学习/游戏记录表
       ↓
  管理员操作（封禁/解封/重置密码）
       ↓
  操作日志表 admin_logs
```

### 1.2 订单数据流

```
用户端支付
       ↓
  Paddle Checkout 托管页面
       ↓
  Paddle 处理支付
       ↓
  Paddle Webhook → 后端接收
       ↓
  订单表 orders（写入订单记录）
       ↓                            ↓
  用户课程权限表 user_courses     推荐返利逻辑
  （解锁课程访问权）             （30天冷却→发放知语币）
       ↓
  管理后台（订单列表/详情）
```

### 1.3 退款数据流（核心链路）

```
管理员发起退款（管理后台）
       ↓
  ┌────────────────────────────────────────────┐
  │             事务性操作（全部成功或全部回滚）      │
  │                                              │
  │  ① 向 Paddle 发起退款请求                      │
  │     └→ 订单状态 → "退款处理中"                  │
  │                                              │
  │  ② 课程访问权回收                              │
  │     └→ user_courses 表标记"已回收"              │
  │     └→ 用户端立即不可访问，学习进度保留          │
  │                                              │
  │  ③ 推荐返利扣回（如有推荐关系）                  │
  │     ├→ 推荐人知语币余额扣除（可至负数）          │
  │     └→ 被推荐人自得返利扣除（可至负数）          │
  │                                              │
  │  ④ 写入退款日志 refund_logs                    │
  │  ⑤ 写入系统操作日志 admin_logs                  │
  └────────────────────────────────────────────┘
       ↓
  Paddle Webhook 通知退款结果
       ├→ 退款成功：订单状态 → "已退款"
       └→ 退款失败：订单状态 → "退款失败" → 回滚课程权限和返利
       ↓
  用户端推送通知（退款用户 + 推荐人）
```

### 1.4 知语币数据流

```
知语币来源（7 种触发方式）
  ├── 推荐返利（30天冷却后自动发放）
  ├── 每日签到（用户端签到触发）
  ├── 游戏连胜（5 连胜触发）
  ├── 新手引导（完成入学测试触发）
  ├── 节日活动（运营后台配置触发）
  ├── 管理员手动调整（管理后台操作触发）
  └── 退款扣回（退款事务中自动执行，负向）
       ↓
  知语币流水表 coin_transactions
  （每笔变动一条记录：用户ID+事由+金额+变动后余额+操作人+时间）
       ↓
  用户余额字段 users.coin_balance（实时更新）
       ↓
  管理后台展示                     用户端展示
  ├── 用户详情页知语币 Tab          ├── 个人中心余额
  ├── 知语币总览看板               └── 收支明细
  └── 负余额监控列表
```

## 二、非功能需求

### 2.1 性能要求

| 页面/操作 | 指标 | 目标值 |
|----------|------|--------|
| 用户列表页首屏加载 | 首屏渲染时间 | < 2s |
| 用户列表搜索/筛选 | 结果响应时间 | < 1s |
| 用户详情页加载 | 基本信息区渲染 | < 1s |
| 用户详情页 Tab 切换 | Tab 内容加载 | < 1.5s |
| 订单列表页首屏加载 | 首屏渲染时间 | < 2s |
| 订单详情页加载 | 全页渲染 | < 1.5s |
| 退款操作执行 | 从确认到反馈 | < 3s（不含 Paddle 异步处理） |
| 知语币调整执行 | 从确认到反馈 | < 1s |
| 用户数据导出（10,000 条） | 文件生成时间 | < 15s |
| 用户数据导出（50,000 条） | 文件生成时间 | < 30s |

### 2.2 并发要求

| 场景 | 预期并发量 |
|------|----------|
| 管理后台同时在线管理员 | 10-20 人 |
| 用户列表同时查询 | 10 并发 |
| 退款同时操作 | 5 并发（需防止对同一订单重复退款） |
| 知语币调整同时操作 | 5 并发（需防止对同一用户并发调整导致余额错误） |

### 2.3 退款事务一致性

| 规则 | 说明 |
|------|------|
| **原子性** | 退款链路中的课程回收+返利扣回+日志记录必须在同一事务中完成，任一步骤失败则全部回滚 |
| **幂等性** | 同一订单不可重复发起退款；退款按钮在发起后立即变为 Loading + Disabled |
| **防重复提交** | 前端：确认按钮点击后立即 Disabled；后端：订单号作为幂等键，重复请求返回"该订单已在退款处理中" |
| **最终一致性** | Paddle 退款结果通过 Webhook 异步通知，系统需处理 Webhook 延迟（最长 72 小时） |
| **补偿机制** | 如 Paddle 退款失败，系统自动回滚课程权限恢复和返利恢复，并通知管理员 |

### 2.4 数据导出合规（GDPR）

| 规则 | 说明 |
|------|------|
| **最小化原则** | 导出字段仅限运营必需（详见 [01-user-list.md §六](01-user-list.md)） |
| **禁止导出** | 密码（含哈希）、精确地理位置、IP 地址、设备标识符 |
| **导出审计** | 每次导出操作记录到系统日志（操作人+导出时间+筛选条件+导出行数） |
| **数据保护** | 导出文件不缓存在服务器，生成后直接流式传输至客户端浏览器下载 |

### 2.5 操作审计

所有管理员"写操作"必须记录至系统日志表 `admin_logs`：

| 操作类型 | 记录字段 |
|---------|---------|
| **封禁用户** | 管理员 ID + 目标用户 ID + 封禁原因 + 期限 + 时间 |
| **解封用户** | 管理员 ID + 目标用户 ID + 时间 |
| **发起退款** | 管理员 ID + 订单号 + 退款金额 + 退款原因 + 时间 |
| **知语币调整** | 管理员 ID + 目标用户 ID + 调整类型 + 金额 + 原因 + 调整前余额 + 调整后余额 + 时间 |
| **重置密码** | 管理员 ID + 目标用户 ID + 时间（不记录密码内容） |
| **数据导出** | 管理员 ID + 导出类型 + 筛选条件 + 行数 + 时间 |

日志记录**不可修改、不可删除**，保留期限 ≥ 2 年。

### 2.6 安全要求

| 规则 | 说明 |
|------|------|
| **权限校验** | 每个 API 请求校验管理员角色权限，越权请求返回 403 |
| **敏感操作二次确认** | 封禁、退款、知语币调整、重置密码必须二次确认弹窗 |
| **并发锁** | 退款操作和知语币调整需要对目标记录加行级锁，防止并发冲突 |
| **输入校验** | 所有管理员输入（搜索词、原因文本、金额）在后端做参数校验和防注入处理 |
| **会话安全** | 管理员会话超时 2 小时自动退出；敏感操作（退款、封禁）前校验会话有效性 |

## 三、验收标准汇总

### 3.1 用户列表页

| # | 验收项 | 验收条件 |
|---|--------|---------|
| 1 | 搜索功能 | 输入昵称/邮箱/ID → 500ms 内返回匹配结果 → 命中文字高亮 |
| 2 | 筛选功能 | 6 个筛选条件任意组合 → 列表实时刷新 → "重置筛选"一键清除所有条件 |
| 3 | 排序功能 | 点击注册日期/最近活跃/累计消费列头 → 三态循环切换 → 数据正确排序 |
| 4 | 分页 | 每页 20 条 → 翻页加载正常 → 总记录数准确 |
| 5 | 导出功能 | 筛选后点击导出 → 30s 内生成 CSV → 自动下载 → 字段符合 GDPR 要求 |
| 6 | 7 种状态 | 空/加载/首次加载/成功/错误/部分加载/离线 7 种状态表现正确 |

### 3.2 用户详情页

| # | 验收项 | 验收条件 |
|---|--------|---------|
| 1 | 基本信息区 | 10 个字段完整展示 → 用户 ID 可复制 → 状态标签正确 |
| 2 | 学习数据 Tab | 已购课程列表正确 → 学习进度百分比准确 → 评测记录按时间倒序 |
| 3 | 游戏数据 Tab | 段位信息正确 → 12 款游戏胜率数据完整 → 段位趋势图可交互 |

| 5 | 知语币 Tab | 余额正确（负数红色）→ 收支明细按时间倒序 → 管理员调整记录独立展示 |
| 6 | 推荐关系 Tab | 推荐人/被推荐人信息完整 → 昵称可跳转 → 累计返利数据准确 |

### 3.3 用户封禁/解封

| # | 验收项 | 验收条件 |
|---|--------|---------|
| 1 | 封禁流程 | 选择原因+期限 → 二次确认 → 执行成功 → 用户状态变"封禁中" |
| 2 | 封禁效果 | 用户端登录后弹出全屏封禁通知 → 无法使用任何功能 → 仅可退出登录 |
| 3 | 课程有效期不暂停 | 封禁期间课程有效期继续倒计时 → 解封后有效期不延长 |
| 4 | 自动解封 | 封禁期限到期 → 系统自动解封 → 用户状态恢复"正常" |
| 5 | 手动解封 | 管理员点击解封 → 确认后立即生效 → 用户恢复所有功能 |
| 6 | 封禁历史 | 每次封禁/解封完整记录 → 不可修改不可删除 |
| 7 | 审计日志 | 封禁和解封操作写入系统日志 → 包含操作人+时间+原因 |

### 3.5 退款管理

| # | 验收项 | 验收条件 |
|---|--------|---------|
| 1 | 退款弹窗 | 订单摘要正确 → 退款原因可选 → 金额可修改 → 影响预览准确 |
| 2 | 二次确认 | 显示退款金额+影响清单 → 确认后执行 |
| 3 | 课程回收 | 退款后用户立即失去课程访问权 → 学习进度保留 → 用户端显示"已回收" |
| 4 | 返利扣回 | 推荐人和被推荐人返利均扣回 → 可扣至负数 → 负数红色显示 |
| 5 | 负余额处理 | 推荐人余额不足时扣至负数 → 不阻断退款 → 用户端显示提示文案 |
| 6 | 退款失败回滚 | Paddle 退款失败 → 课程权限恢复 → 返利恢复 → 管理员收到通知 |
| 7 | 防重复退款 | 同一订单不可重复发起退款 → 发起后按钮立即 Disabled |
| 8 | 退款列表 | 退款记录完整 → 可按状态/日期/操作人筛选 |
| 9 | 审计日志 | 退款操作写入系统日志 → 包含操作人+金额+原因+时间 |

### 3.6 知语币管理

| # | 验收项 | 验收条件 |
|---|--------|---------|
| 1 | 总览看板 | 发行量/流通量/消耗量/负余额用户数 4 项数据准确 |
| 2 | 趋势图表 | 7/30/90 天切换正常 → Hover 数据准确 → 每 5 分钟自动刷新 |
| 3 | 手动调整 | 选择增加/扣除+金额+原因 → 二次确认 → 余额即时变更 |
| 4 | 调整审计 | 每次调整记录操作人+金额+原因+时间+调整前后余额 → 不可修改不可删除 |
| 5 | 上限校验 | 增加后超过 100,000 上限 → 阻止并提示 |
| 6 | 负余额允许 | 扣除导致负余额 → 显示警告但不阻止（退款扣回场景） |
| 7 | 负余额监控 | 负余额用户自动统计 → 列表按余额升序 → 可跳转用户详情 |
| 8 | 大额预警 | 单日变动超 5,000 币 → 预警列表展示 |
