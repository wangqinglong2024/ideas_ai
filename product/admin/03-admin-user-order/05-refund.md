# 退款管理

> 关联文档：[订单管理](04-order-management.md) | [知语币管理](06-coin-management.md) | [商业模式 §五](../../../plan/06-business-model.md) | [财务合规](../../../plan/07-financial-compliance.md)
> 路由路径：退款列表 `/admin/orders/refunds`
> 角色权限：超级管理员、用户运营（内容运营和游戏运营**无权操作退款**）
> 情绪目标：**严谨、可信、滴水不漏** — 退款是最复杂的业务流程，必须确保课程回收+返利扣回同步执行

---

| 入口位置 | 操作方式 | 条件 |

| **订单详情页** | 退款信息区"发起退款"危险按钮 | 仅"已完成"状态订单可操作 |
| **退款列表页** | 本页面仅查看退款记录，不发起退款 | — |

## 二、退款操作弹窗

### 2.1 弹窗基础

| 属性 | 说明 |
|------|------|
| **弹窗类型** | 模态对话框（Modal），Level 4 阴影 |
| **弹窗宽度** | 560px |
| **标题** | H3 级别（20px），"发起退款" |

### 2.2 订单信息摘要（只读区域）

弹窗顶部以卡片形式展示关联订单摘要，灰色背景 `rgba(255,255,255,0.03)`，圆角 `rounded-xl`，内边距 16px：

| 字段 | 展示方式 |
|------|---------|
| 订单号 | Body S 级别，灰色标签"订单号" + 值 |
| 用户 | 头像（24px）+ 昵称 + 邮箱 |
| 课程 | "Level {X}: {课程名称}" |
| 支付金额 | "$X.XX" |
| 支付日期 | YYYY-MM-DD |
| 推荐关系 | 有推荐人："推荐人：{昵称}，返利 XX 币"；无推荐人："无推荐关系" |

### 2.3 退款表单字段

| # | 字段 | 类型 | 必填 | 说明 |
|---|------|------|------|------|
| 1 | **退款原因** | 下拉单选 | ✅ | 选项：用户申请 / 课程质量 / 系统错误 / 其他 |
| 2 | **原因补充** | 多行文本输入框 | 选择"其他"时必填，其他选项可选 | 最大 500 字；Placeholder "请补充退款原因详情…" |
| 3 | **退款金额** | 数字输入框 | ✅ | 默认值：订单全额（如 $6.00）；支持手动修改（部分退款）；最小 $0.01，最大不超过订单原始金额；格式保留两位小数 |

### 2.4 退款方式说明（只读提示）

弹窗表单下方，蓝色信息提示框（`.info-banner`），Lucide `Info` 16px + 文字：

> 退款将通过 Paddle 原路退回至用户支付账户。Paddle 处理退款通常需要 5-10 个工作日。

### 2.5 退款影响预览（只读区域）

表单下方，Amber 色警告框（`.warning-banner`），Lucide `AlertTriangle` 16px + 标题"退款影响"：

| 影响项 | 说明 |
|--------|------|
| **课程访问权回收** | "用户将立即失去 Level {X} 的课程访问权。学习进度数据保留，但不可继续学习。" |
| **推荐返利扣回** | 有推荐人时："推荐人 {昵称} 的返利 XX 知语币将被扣回。"；被推荐人自得返利："用户自得的 XX 知语币将被扣回。"；无推荐人时不显示此项 |
| **负余额风险** | 如推荐人当前余额 < 扣回金额："⚠️ 推荐人当前余额 XX 币，扣回后余额将变为 -XX 币（允许负余额）。" |

### 2.6 弹窗按钮

| 按钮 | 样式 | 位置 | 点击行为 |
|------|------|------|---------|
| **取消** | 次要按钮（`.glass-button-secondary`） | 弹窗底部靠左 | 关闭弹窗 |
| **确认退款** | 危险按钮（红色背景 `#ef4444`） | 弹窗底部靠右 | 表单校验通过 → 打开二次确认弹窗 |

### 2.7 表单校验

| 校验规则 | 错误提示 |
|---------|---------|
| 退款原因未选择 | "请选择退款原因" |
| 选择"其他"但补充为空 | "请填写退款原因详情" |
| 退款金额为空或 ≤ 0 | "请输入有效的退款金额" |
| 退款金额 > 订单原始金额 | "退款金额不能超过订单金额 $X.XX" |

## 三、退款二次确认弹窗

| 属性 | 说明 |
|------|------|
| **弹窗宽度** | 440px |
| **图标** | Lucide `AlertTriangle`，48px，错误色 `#ef4444`，居中 |
| **标题** | H3 级别，居中，"确认退款？" |
| **内容** | Body 级别，居中文字 |

### 确认弹窗内容

```
你即将对订单 {订单号} 执行退款操作：

退款金额：${X.XX}
退款方式：原路退回（Paddle）

退款后将同时执行：
✓ 回收用户 Level {X} 课程访问权
✓ 扣回推荐人 {昵称} 返利 XX 知语币
✓ 扣回用户自得返利 XX 知语币

此操作不可撤销，是否确认？
```

如无推荐关系，则不展示返利扣回行。

### 按钮

| 按钮 | 样式 | 点击行为 |
|------|------|---------|
| **取消** | 次要按钮 | 关闭二次确认弹窗，返回退款操作弹窗 |
| **确认退款** | 危险按钮，文字"确认退款" | 执行退款流程 → 按钮 Loading 态 → 详见 §四 |

## 四、退款后自动操作（完整链路）

点击"确认退款"后，系统按以下顺序**事务性**执行（全部成功或全部回滚）：

### 4.1 步骤 1：向 Paddle 发起退款请求

| 属性 | 说明 |
|------|------|
| **操作** | 调用 Paddle Refund API，传入 Paddle 交易号和退款金额 |
| **订单状态** | 变更为"退款处理中" |
| **等待** | Paddle 异步处理，通过 Webhook 通知退款结果 |

### 4.2 步骤 2：课程访问权回收

| 属性 | 说明 |
|------|------|
| **触发时机** | 退款请求发起**立即执行**（不等待 Paddle 确认） |
| **操作** | 将用户对该 Level 的访问权限设置为"已回收" |
| **用户端表现** | 用户立即无法访问该 Level 课程内容；学习进度数据保留（不删除），后续如重新购买可恢复进度 |
| **课程列表显示** | 用户端"我的课程"中该 Level 显示"已回收"标签，点击弹出提示"该课程已因退款回收。如需继续学习，请重新购买。" |

### 4.3 步骤 3：推荐返利扣回

| 属性 | 说明 |
|------|------|
| **触发时机** | 与课程回收同步执行 |
| **条件** | 该订单存在推荐关系 |

#### 扣回逻辑

| 场景 | 操作 |
|------|------|
| **推荐人返利** | 从推荐人知语币余额中扣除该订单对应的返利金额（订单金额 × 20% ÷ $0.01 = 知语币数） |
| **被推荐人自得返利** | 从被推荐人（即退款用户）知语币余额中扣除同等金额 |
| **冷却期内未发放** | 如返利尚在 30 天冷却期内（尚未发放），则取消待发放的返利记录，无需扣回 |
| **推荐人余额不足** | 允许扣至负数（负余额），不阻断退款流程 |

#### 负余额处理

| 规则 | 说明 |
|------|------|
| **允许负余额** | 这是产品设计明确允许的场景（因退款扣回导致） |
| **负余额标识** | 管理后台和用户端均以红色显示负数余额 |
| **用户端提示** | 推荐人收到通知："因您推荐的用户发起了课程退款，对应的推荐奖励已扣回，您的知语币余额为 XX 币。后续获得新奖励时将自动抵充。" |
| **自动抵充** | 推荐人后续获得新的知语币（签到/推荐/活动）时，优先抵充负余额 |
| **功能不受限** | 负余额不阻止用户正常使用平台功能（不封禁、不限制），仅限制知语币消费（余额为负时无法兑换课程/皮肤） |

### 4.4 步骤 4：记录退款日志

| 字段 | 说明 |
|------|------|
| 退款单号 | 系统生成唯一 ID |
| 关联订单号 | 原订单号 |
| 退款金额 | USD |
| 退款原因 | 分类 + 补充说明 |
| 操作管理员 | 管理员姓名 + ID |
| 操作时间 | 精确到秒（UTC） |
| 课程回收状态 | 已回收 / 回收中 |
| 返利扣回状态 | 已扣回 / 无需扣回 |
| 返利扣回详情 | 推荐人 ID + 扣回金额；被推荐人 ID + 扣回金额 |

### 4.5 步骤 5：用户端通知

| 通知对象 | 通知方式 | 通知内容 |
|---------|---------|---------|
| **退款用户** | App 推送 + 站内信 | "您的 Level {X} 课程订单已退款 ${X.XX}，课程访问权已回收。退款将在 5-10 个工作日内退回原支付账户。" |
| **推荐人**（如有） | App 推送 + 站内信 | "因您推荐的用户发起了课程退款，对应的推荐奖励已扣回，您的知语币余额为 XX 币。后续获得新奖励时将自动抵充。" |

### 4.6 退款失败处理

| 场景 | 处理 |
|------|------|
| **Paddle 退款失败** | Paddle Webhook 通知退款失败 → 订单状态变更为"退款失败" → 课程访问权恢复 → 已扣回的返利恢复 → 管理后台 Toast 红色通知 |
| **课程回收失败** | 整个退款事务回滚，退款请求取消，Toast 错误"退款失败：课程回收异常，请联系技术支持" |
| **返利扣回失败** | 整个退款事务回滚，Toast 错误"退款失败：返利扣回异常，请联系技术支持" |

### 5.2 筛选条件

| 筛选项 | 组件 | 选项 |

### 5.3 退款列表表格

| # | 列名 | 宽度 | 内容说明 | 排序 |

### 5.4 表格布局与分页

与订单列表页一致：毛玻璃面板，行高 56px，每页 20 条。

### 5.5 状态矩阵

| 状态 | 设计要求 |
|------|---------|
| **空状态 (Empty)** | 表格区域居中 Lucide `RotateCcw` 48px 灰色 + "暂无退款记录" |
| **加载中 (Loading)** | 表格 10 行骨架屏 |
| **首次加载 (First Load)** | 筛选栏+表格全骨架屏 |
| **成功 (Success)** | 正常展示 |
| **错误 (Error)** | "数据加载失败" + "重试"按钮 |
| **部分加载 (Partial)** | 翻页降透明度+骨架 |
| **离线 (Offline)** | 顶部 Banner 黄色提示 |
