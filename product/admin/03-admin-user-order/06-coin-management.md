# 知语币管理

> 关联文档：[用户详情页 §七](02-user-detail.md) | [退款管理](05-refund.md) | [商业模式 §五](../../../plan/06-business-model.md) | [产品总纲 §五](../../00-product-overview.md)
> 路由路径：知语币总览 `/admin/coins` · 负余额监控 `/admin/coins/alerts`
> 角色权限：超级管理员、用户运营
> 情绪目标：**透明、审慎、可追溯** — 每一笔知语币变动都有审计记录，虚拟经济健康可见

## 二、管理员手动调整

### 2.1 入口

| 入口位置 | 操作方式 |
|---------|---------|
| **用户详情页** | 基本信息区右侧"调整知语币"按钮 |
| **用户详情页知语币 Tab** | Tab 内可增设快捷"调整"按钮（复用同一弹窗） |

### 2.2 调整操作弹窗

| 属性 | 说明 |
|------|------|
| **弹窗类型** | 模态对话框（Modal），Level 4 阴影 |
| **弹窗宽度** | 480px |
| **标题** | H3 级别（20px），"调整知语币" |
| **副标题** | Body S 级别，灰色，"操作用户：{昵称}（ID: {用户ID}）· 当前余额：{XX} 币" |

#### 当前余额强调

- 正数或零：Amber 色 `#d97706`
- **负数：红色 `#ef4444`**，如"当前余额：-230 币"

#### 表单字段

| # | 字段 | 类型 | 必填 | 说明 |
|---|------|------|------|------|
| 1 | **调整类型** | 分段按钮（Segmented Control） | ✅ | 选项："增加"（绿色高亮）/ "扣除"（红色高亮）；默认不选中 |
| 2 | **调整金额** | 数字输入框 | ✅ | 整数，最小 1，最大 100,000；Placeholder "请输入调整的知语币数量" |
| 3 | **调整原因** | 多行文本输入框 | ✅ | 最小 10 字，最大 500 字；Placeholder "请详细说明调整原因（必填，至少 10 字）…" |

#### 实时预览

表单下方显示调整结果预览卡片（灰色背景 `rounded-xl`，内边距 16px）：

| 字段 | 说明 |
|------|------|
| 当前余额 | "XX 币" |
| 调整量 | 增加显示"+XX 币"绿色；扣除显示"-XX 币"红色 |
| **调整后余额** | "XX 币"，**加粗**；如结果为负数则红色显示 |

#### 上限校验

| 校验规则 | 错误提示 |
|---------|---------|
| 调整类型未选择 | "请选择调整类型" |
| 金额为空或 ≤ 0 | "请输入有效的知语币数量" |
| 增加后余额超过 100,000 上限 | "调整后余额将超过上限（100,000 币），请减少金额" |
| 原因为空或 < 10 字 | "请填写调整原因（至少 10 字）" |
| 扣除导致负余额 | **不阻止**，但在预览区域显示 Amber 色警告："调整后余额将为负数" |

### 2.3 弹窗按钮

| 按钮 | 样式 | 点击行为 |
|------|------|---------|
| **取消** | 次要按钮 | 关闭弹窗 |
| **确认调整** | 主要按钮（Rose 色背景 `#e11d48`） | 表单校验通过 → 二次确认弹窗 |

### 2.4 二次确认弹窗

| 属性 | 说明 |
|------|------|
| **弹窗宽度** | 400px |
| **图标** | Lucide `Coins`，48px，Amber 色，居中 |
| **标题** | "确认调整？" |
| **内容** | "你即将对用户 **{昵称}** 的知语币余额进行调整：{增加/扣除} **{XX}** 币。调整后余额：**{XX}** 币。" |

| 按钮 | 样式 | 点击行为 |
|------|------|---------|
| **取消** | 次要按钮 | 关闭确认弹窗 |
| **确认** | 主要按钮 | 执行调整 → Loading 态 → 成功后关闭所有弹窗 + Toast"知语币已调整" + 刷新用户知语币数据 |

### 2.5 调整执行后

| 步骤 | 说明 |
|------|------|
| 1 | 用户知语币余额立即变更 |
| 2 | 写入知语币流水表：事由"管理员调整"，操作人为当前管理员姓名+ID |
| 3 | 写入系统操作日志（操作类型"知语币调整" + 管理员 ID + 用户 ID + 金额 + 原因 + 时间） |
| 4 | 用户详情页知语币 Tab 数据刷新 |

### 2.6 审计要求

每条管理员手动调整记录**不可修改、不可删除**，必须包含：

| 字段 | 说明 |
|------|------|
| 调整时间 | 精确到秒（UTC） |
| 操作管理员 | 姓名 + ID |
| 目标用户 | 昵称 + ID |
| 调整类型 | 增加 / 扣除 |
| 调整金额 | 整数 |
| 调整原因 | 全文 |
| 调整前余额 | 整数 |
| 调整后余额 | 整数 |

## 三、知语币异常监控

### 3.1 负余额用户列表

| 属性 | 说明 |
|------|------|
| **路由** | `/admin/coins/alerts` |
| **页面标题** | "负余额用户" |
| **数据来源** | 自动统计当前余额 < 0 的所有用户 |
| **刷新频率** | 页面加载时请求，支持手动刷新（右上角 Lucide `RefreshCw` 按钮） |

#### 表格列定义

| # | 列名 | 宽度 | 内容说明 |
|---|------|------|---------|
| 1 | 用户信息 | 200px | 头像（32px）+ 昵称（Sky 色可点击跳转用户详情页） |
| 2 | 用户ID | 120px | 等宽字体 |
| 3 | 当前余额 | 100px | 负数，红色加粗，等宽字体 |

| 5 | 负余额起始日期 | 120px | 余额首次变为负数的日期 |

### 3.2 单日大额变动预警（P1）

| 属性 | 说明 |
|------|------|
| **位置** | 负余额用户列表下方，间距 24px |
| **标题** | H3 级别，"大额变动预警" |
| **触发规则** | 单个用户单日知语币变动（发放或消耗）超过 **5,000 币**（≈ $50） |

#### 预警列表表格

| # | 列名 | 宽度 | 内容说明 |
|---|------|------|---------|
| 1 | 用户信息 | 200px | 头像 + 昵称（可点击） |
| 2 | 变动日期 | 120px | 格式 YYYY-MM-DD |
| 3 | 变动总量 | 100px | 当日变动绝对值之和，Amber 色 |
| 4 | 变动明细 | 240px | 主要变动事由摘要（如"推荐返利 +3,600 · 皮肤购买 -1,500"） |
| 5 | 操作 | 120px | "查看详情"跳转用户知语币 Tab |

**阈值配置**（P2）：未来支持管理员在系统设置中修改预警阈值，当前硬编码 5,000 币。

## 四、状态矩阵

### 知语币总览页

| 状态 | 设计要求 |
|------|---------|
| **空状态 (Empty)** | 汇总卡片显示全 0 值；趋势图表显示空坐标轴 + 居中"暂无数据" |
| **加载中 (Loading)** | 4 张卡片骨架（标题行+数值行）+ 图表区域矩形骨架 |
| **首次加载 (First Load)** | 全页骨架：卡片骨架+图表骨架+分布图骨架 |
| **成功 (Success)** | 正常展示 |
| **错误 (Error)** | 图表区域"数据加载失败" + "重试"按钮；卡片显示"--" |
| **部分加载 (Partial)** | 卡片先加载，图表独立加载 |
| **离线 (Offline)** | 顶部 Banner 黄色提示 |

### 知语币调整弹窗

| 状态 | 设计要求 |
|------|---------|
| **加载中 (Loading)** | "确认调整"按钮 Loading 态 + Disabled |
| **成功 (Success)** | Toast 绿色"知语币已调整"，2s 自动消失 |
| **错误 (Error)** | Toast 红色"调整失败，请重试"，需手动关闭；弹窗不关闭 |
| **离线 (Offline)** | 点击调整按钮时 Toast 红色"网络连接已断开" |
