# 管理后台 — 订阅 & 佣金管理

> **模块定位**: 订阅状态查看/操作、佣金概览、推广员管理、提现审批
> **版本**: v3.0 | **日期**: 2026-04-16
> **关联**: [管理后台总览](00-index.md) | [会员支付](../09-membership-payment.md) | [推广佣金](../10-referral-commission.md)

---

## 一、模块概述

### 1.1 目标

提供订阅和佣金的全流程管理能力：查看订阅状态、处理异常、审批提现请求、监控佣金数据。

### 1.2 核心指标

| 指标 | 说明 |
|------|------|
| 活跃订阅数 | 当前 status=active 的订阅总数 |
| MRR | 月度经常性收入（换算到月维度） |
| 佣金总支出 | 累计已发放佣金 |
| 待审提现 | 需要人工审核的提现申请数量 |

---

## 二、信息架构

```
/admin/commerce
  ├── /subscriptions        → 订阅管理
  ├── /subscriptions/:id    → 订阅详情
  ├── /commissions          → 佣金概览
  ├── /promoters            → 推广员列表
  ├── /promoters/:id        → 推广员详情
  └── /withdrawals          → 提现审批
```

**反欺诈规则**：

| 规则 | 阈值 | 动作 |
|------|------|------|
| 下线付费率 < 10% | 警告 | 需人工审核 |
| 单日推荐注册 > 20 | 拦截 | 自动标记 + 人工审核 |
| 同 IP 注册 > 5 | 拦截 | 关联账号全部标记 |
| 佣金/推荐比 > 均值 3σ | 警告 | 需人工审核 |

## 四、API 端点

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | `/api/admin/subscriptions` | 订阅列表 | 全部 |
| GET | `/api/admin/subscriptions/:id` | 订阅详情 | 全部 |
| PUT | `/api/admin/subscriptions/:id/extend` | 延长订阅 | 超管 |
| PUT | `/api/admin/subscriptions/:id/cancel` | 取消订阅 | 超管 |
| GET | `/api/admin/commissions/overview` | 佣金概览 | 全部 |
| GET | `/api/admin/promoters` | 推广员列表 | 全部 |
| GET | `/api/admin/promoters/:id` | 推广员详情 | 全部 |
| GET | `/api/admin/withdrawals` | 提现列表 | 全部 |
| PUT | `/api/admin/withdrawals/:id/approve` | 通过提现 | 超管 |
| PUT | `/api/admin/withdrawals/:id/reject` | 拒绝提现 | 超管 |

## 五、用户故事

| # | 角色 | 故事 | 验收标准 |
|---|------|------|---------|
| 1 | 管理员 | 查看某用户订阅详情 | 显示套餐/金额/状态/支付记录 |
| 2 | 超管 | 为用户延长 30 天订阅 | 到期时间更新 + 操作记录 |
| 3 | 管理员 | 查看本月佣金概览 | 5 指标卡 + 趋势图 + 等级分布 |
| 4 | 超管 | 审批推广员的 $50 提现 | 反欺诈检查 → 填意见 → 通过/拒绝 |
| 5 | 超管 | 发现并封禁一个刷量推广员 | 推广员列表标记 + 冻结佣金 + 封禁账号 |
