## 一、收藏操作

### 1.1 收藏入口

发现中国模块内有以下收藏入口，行为规则统一：

| 入口位置 | 图标 | 说明 |
|---------|------|------|
| 文章列表页 — 文章卡片右下角 | Lucide `Heart`（20px） | 可在列表中快速收藏/取消 |
| 文章详情页 — 元信息区 | Lucide `Heart`（20px）+ 文字标签 | 阅读过程中收藏/取消 |
| 个人中心 — 我的收藏列表 | Lucide `Heart`（20px），已收藏状态 | 可取消收藏 |

### 1.2 未登录用户收藏

未登录用户可以正常浏览前 3 类目（01 中国历史、02 中国美食、03 名胜风光）的文章，但收藏操作需要登录。

| 步骤 | 行为 |
|------|------|
| 1 | 未登录用户点击任意收藏按钮（空心心形） |
| 2 | 触发全局登录弹窗（Bottom Sheet，从底部上滑，带背景模糊遮罩） |
| 3 | 登录弹窗内文案：标题「登录后即可收藏」；副标题「登录后还可解锁全部 12 类目内容」 |
| 4a | 用户完成登录 → 弹窗关闭 → **自动执行收藏操作** → 心形变实心 Rose 色 + 弹跳动画（Spring 缓动，300ms）→ Toast「已收藏」（绿色，2s 自动消失） |
| 4b | 用户关闭登录弹窗（点击关闭按钮或点击遮罩区域）→ 弹窗关闭 → 不执行收藏 → 心形保持空心 |

**登录弹窗文案多语言**：

| 元素 | 汉语 | 英语 | 越南语 |
|------|------|------|--------|
| 标题 | 登录后即可收藏 | Sign in to save | Đăng nhập để lưu |
| 副标题 | 登录后还可解锁全部 12 类目内容 | Sign in to unlock all 12 categories | Đăng nhập để mở khóa tất cả 12 chủ đề |

### 1.3 已登录用户 — 添加收藏

| 步骤 | 行为 |
|------|------|
| 1 | 用户点击空心心形图标 |
| 2 | **乐观 UI**：心形立即变为实心 Rose 色（`#e11d48`） |
| 3 | 弹跳动画：心形执行 `scale(1) → scale(1.3) → scale(1)` 变化，Spring 缓动曲线（`cubic-bezier(0.34, 1.56, 0.64, 1)`），总时长 300ms |
| 4 | 同时后台发送收藏请求 |
| 5a | 请求成功 → Toast「已收藏」（绿色，顶部居中，2s 自动消失） |
| 5b | 请求失败 → 心形回退为空心状态（100ms 淡出过渡）→ Toast「收藏失败，请重试」（红色 Toast，需手动关闭） |

### 1.4 已登录用户 — 取消收藏

| 步骤 | 行为 |
|------|------|
| 1 | 用户点击实心心形图标 |
| 2 | **乐观 UI**：心形立即变为空心状态（继承次要文字色） |
| 3 | 缩小动画：心形执行 `scale(1) → scale(0.8) → scale(1)` 变化，100ms |
| 4 | 同时后台发送取消收藏请求 |
| 5a | 请求成功 → Toast「已取消收藏」（绿色，顶部居中，2s 自动消失） |
| 5b | 请求失败 → 心形回退为实心 Rose 色（100ms 淡出过渡）→ Toast「操作失败，请重试」（红色 Toast，需手动关闭） |

### 1.5 收藏状态同步

当用户在文章列表页收藏/取消某篇文章后：
- 该操作结果**实时同步**到文章详情页的收藏按钮状态
- 反之亦然：详情页操作后，返回列表页时该文章卡片的收藏图标同步更新
- 同步方式为前端本地状态管理，无需重新请求列表数据

### 1.6 Toast 文案汇总

| 场景 | 汉语 | 英语 | 越南语 |
|------|------|------|--------|
| 收藏成功 | 已收藏 | Saved | Đã lưu |
| 取消收藏成功 | 已取消收藏 | Unsaved | Đã bỏ lưu |
| 收藏失败 | 收藏失败，请重试 | Save failed, please retry | Lưu thất bại, vui lòng thử lại |
| 取消收藏失败 | 操作失败，请重试 | Action failed, please retry | Thao tác thất bại, vui lòng thử lại |

## 二、收藏数据同步

### 2.1 数据同步至个人中心

用户收藏文章后，该文章自动出现在「个人中心 → 我的收藏」列表中。

| 规则 | 说明 |
|------|------|
| 同步时机 | 收藏请求成功后即时同步（下次打开我的收藏列表时可见） |
| 排序 | 按收藏时间倒序（最近收藏的排在最前） |
| 数据一致性 | 收藏列表展示的文章信息（标题、封面、浏览量）为最新数据，不是收藏时的快照 |

### 2.2 收藏数量限制

| 属性 | 说明 |
|------|------|
| 最大收藏数量 | 不限制（MVP 阶段） |
| 极端情况 | 若用户收藏超过 500 篇，我的收藏列表采用分页加载（每页 20 篇） |

### 2.3 文章被下架

| 场景 | 行为 |
|------|------|
| 用户已收藏的文章被运营后台下架 | 该文章从我的收藏列表中**自动移除**；用户无感知（不推送通知） |
| 用户已收藏的文章内容被更新 | 收藏列表中的文章自动展示最新版本；用户无感知 |

## 三、取消收藏

### 3.1 文章详情页取消收藏

与 1.4 节一致：点击实心心形 → 乐观 UI 切换 → 发送请求 → 成功/失败处理。

### 3.2 文章列表页取消收藏

与文章详情页行为一致，操作后列表中该文章卡片的心形图标同步更新。

### 3.3 我的收藏列表页取消收藏

「个人中心 → 我的收藏」页面中的取消收藏操作：

| 步骤 | 行为 |
|------|------|
| 1 | 用户点击收藏列表中某篇文章的实心心形图标 |
| 2 | 弹出确认对话框（模态对话框，毛玻璃背景） |
| 3 | 对话框内容：标题「确认取消收藏？」；说明「取消后可重新收藏」 |
| 4 | 对话框按钮：「取消」（Ghost 按钮，左侧）+「确认取消」（Rose 色药丸按钮，右侧） |
| 5a | 用户点击「确认取消」→ 对话框关闭 → 该文章卡片执行向左滑出动画（300ms ease-out）→ 列表自动收缩填充空间 → Toast「已取消收藏」（绿色，2s） |
| 5b | 用户点击「取消」或点击对话框外区域 → 对话框关闭 → 不执行操作 |
| 5c | 取消请求失败 → 该文章卡片滑回原位 → Toast「操作失败，请重试」（红色，需手动关闭） |

**确认对话框多语言**：

| 元素 | 汉语 | 英语 | 越南语 |
|------|------|------|--------|
| 标题 | 确认取消收藏？ | Remove from saved? | Bỏ lưu bài viết này? |
| 说明 | 取消后可重新收藏 | You can save it again later | Bạn có thể lưu lại sau |
| 取消按钮 | 取消 | Cancel | Hủy |
| 确认按钮 | 确认取消 | Remove | Bỏ lưu |

> 注意：仅在「我的收藏」列表页取消收藏时才弹出确认对话框（因为取消后文章从列表中消失，属于破坏性操作）。在文章列表页和详情页取消收藏不需要确认（用户可以轻松重新收藏）。

### 3.4 我的收藏列表页设计

此页面归属「个人中心」模块，但在此简要说明其与发现中国的关联设计：

| 元素 | 说明 |
|------|------|
| 页面标题 | 「我的收藏」（跟随 UI 语言：英语="My Saves"；越南语="Bài đã lưu"） |
| 列表样式 | 与文章列表页的文章卡片样式一致（缩略图+标题+元信息+心形图标） |
| 排序 | 按收藏时间倒序，不可切换排序 |
| 点击行为 | 点击文章卡片 → Push 转场进入文章详情页 |
| 空状态 | 品牌插画 + 文案「还没有收藏任何文章」+ CTA「去发现中国看看 →」（Rose 色药丸按钮，点击切换至发现中国 Tab） |
| 空状态多语言 | 英语="No saved articles yet" + "Explore Discover China →"；越南语="Chưa lưu bài viết nào" + "Khám phá Trung Quốc →" |
