## 一、数据流向说明

### 1.1 金句数据流向

```
运营人员在管理后台「每日金句管理」模块操作：
  ① 创建金句内容（中文+拼音+出处+英文翻译+越南语翻译+解读文字+分享背景图）
  ② 设置发布日期（支持提前 30 天排期）
  ③ 标记是否为节日金句（关联节日名称）
  ④ 确认发布
       ↓
  写入 Supabase 数据库（金句表）
       ↓
  用户端每日 0:00 UTC 拉取当日金句
  （优先级：节日金句 > 排期金句 > 最近已发布金句兜底）
       ↓
  用户端渲染金句卡片（按用户多语言配置）
```

### 1.2 文章数据流向

```
内容策划选题排期
       ↓
  Dify 工作流生成多语言初稿
  （输入：选题关键词+类目+难度 → 输出：中文正文+拼音+英文翻译+越南语翻译+词汇表+测验题）
       ↓
  写入 Supabase 数据库（文章表，状态=草稿）
       ↓
  运营人员在管理后台「内容管理 → 发现中国文章」模块操作：
  ① 查看 Dify 生成的草稿文章
  ② 编辑调整内容（润色、纠错、补充封面图/音频）
  ③ 配置文章元数据（类目归属、标签、排序权重）
  ④ 确认发布（状态=已发布）或退回修改（状态=草稿）
       ↓
  已发布文章在用户端可见
  运营人员可随时上架/下架
       ↓
  用户端拉取文章列表/详情
  （仅返回状态=已发布的文章，按排序规则呈现）
```

### 1.3 收藏数据存储

```
用户点击收藏按钮
       ↓
  前端乐观 UI 更新（心形变实心）
       ↓
  发送收藏请求至 Supabase
  （写入收藏关系表：用户 ID + 文章 ID + 收藏时间）
       ↓
  成功：前端状态确认
  失败：前端回退 UI + 错误提示
       ↓
  「个人中心 → 我的收藏」查询收藏关系表
  关联文章表获取最新文章信息（标题、封面、浏览量等）
```

### 1.4 浏览量统计规则

| 规则 | 说明 |
|------|------|
| 统计时机 | 用户进入文章详情页时计数 +1 |
| 去重规则 | 同一用户对同一篇文章，**24 小时内仅计 1 次**浏览 |
| 未登录用户 | 基于设备指纹（浏览器 fingerprint）去重 |
| 登录用户 | 基于用户 ID 去重 |
| 显示格式 | ≥ 1000 显示为 1.2k 格式；≥ 10000 显示为 1.2w 格式 |
| 实时性 | 浏览量为准实时统计（允许 5 分钟延迟），不影响用户体验 |

## 二、非功能需求

### 2.1 性能要求

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 类目首页首屏加载 | ≤ 2 秒 | 包含金句卡片 + 12 类目卡片渲染完成 |
| 文章列表首页加载 | ≤ 1.5 秒 | 封面图 + 首批 10 篇文章卡片渲染完成 |
| 文章详情页加载 | ≤ 2 秒 | 标题+元信息+正文首屏内容渲染完成 |
| 金句分享图片生成 | ≤ 3 秒 | 从点击分享到图片预览面板弹出 |
| 文章分享卡片生成 | ≤ 3 秒 | 同上 |
| 收藏操作响应 | ≤ 300ms | 乐观 UI 即时响应，后台请求异步完成 |
| 音频播放启动 | ≤ 1 秒 | 点击播放到声音出来 |

### 2.2 图片加载策略

| 策略 | 说明 |
|------|------|
| 类目卡片插图 | **预加载**：类目首页加载时即请求 12 张类目插图（优化 LCP） |
| 文章列表缩略图 | **懒加载**：卡片进入可视区域前 200px 时开始加载；未加载完成时显示灰色占位（保持卡片布局稳定） |
| 文章封面大图 | **优先加载**：进入文章列表页时优先加载封面大图 |
| 文章正文图片 | **懒加载**：图片进入可视区域前 300px 时开始加载；显示灰色占位（保持原图宽高比） |
| 图片格式 | 优先使用 WebP 格式（减小文件体积）；浏览器不支持时降级 JPEG/PNG |
| 图片 CDN | 所有图片通过 CDN 分发，支持按尺寸裁剪（缩略图请求小尺寸，详情页请求大尺寸） |

### 2.3 音频加载策略

| 策略 | 说明 |
|------|------|
| 预加载 | 进入文章详情页后，若文章有音频，**不自动预加载**音频文件（节省流量） |
| 按需加载 | 用户点击播放按钮时开始加载音频 |
| 缓冲 | 音频加载采用流式播放（边下载边播放），无需等待整个文件下载完成 |
| 音频格式 | MP3（兼容性最广）或 AAC |
| 音频文件大小 | 建议单个音频 ≤ 5MB（约 3-5 分钟） |

### 2.4 分页与缓存

| 策略 | 说明 |
|------|------|
| 文章列表分页大小 | 每页 10 篇 |
| 分页加载机制 | 无限滚动（触底自动加载下一页），非传统翻页 |
| 列表数据缓存 | 用户返回列表页时使用内存缓存，避免重新请求（当次会话有效） |
| 文章详情缓存 | 已浏览的文章详情缓存至本地存储（缓存 50 篇，LRU 淘汰） |
| 金句缓存 | 当日金句缓存至本地存储，24 小时后过期 |
| 类目数据缓存 | 12 类目基础信息缓存至本地存储，1 小时刷新一次 |

### 2.5 离线体验

| 场景 | 行为 |
|------|------|
| 已缓存内容 + 无网络 | 展示缓存内容（标记离线状态），收藏/分享操作提示需联网 |
| 无缓存 + 无网络 | 显示离线空状态页（插画 + 文案 + 「连接网络后自动加载」） |
| 网络恢复 | 顶部离线 Banner 自动消失 + 自动刷新当前页面数据 |

### 2.6 可用性

| 指标 | 目标 |
|------|------|
| 服务可用性 | 99.9% SLA（全年宕机 < 8.7 小时） |
| 错误处理 | 所有网络请求失败均有用户友好的错误提示 + 重试按钮 |
| 无障碍 | WCAG 2.1 AA 合规（色彩对比度、触控尺寸、屏幕阅读器支持） |

### 2.7 安全

| 措施 | 说明 |
|------|------|
| 内容安全 | 所有文章内容经管理后台审核后才发布；正文中不含用户输入内容（无 XSS 风险） |
| 接口安全 | 收藏/浏览量等写入操作需要用户认证（Bearer Token） |
| 频率限制 | 收藏/取消收藏操作限制每用户每分钟 30 次（防刷） |
| 图片安全 | 分享图片在服务端生成，不暴露原始资源路径 |
