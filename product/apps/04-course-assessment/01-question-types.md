## 一、单选题

### 1.1 题干展示

```
┌─────────────────────────────────────────────┐
│  第 3/5 题                                    │
│─────────────────────────────────────────────│
│                                               │
│  [题干区域]                                   │
│  选择正确的词语填入横线处：                       │
│  Xuǎnzé zhèngquè de cíyǔ tián rù héngxiàn chù │ ← 拼音行（如开启）
│  Chọn từ đúng điền vào chỗ trống              │ ← 解释语言行（如开启）
│                                               │
│  昨天我___了一本很有趣的书。                      │
│                                               │
│  [附件区域]（如有）                              │
│  🖼️ 图片（点击可放大）                          │
│  🔊 音频（点击播放，详见听力题型）                  │
│                                               │
└─────────────────────────────────────────────┘
```

**题干规则**：
- 题干文本始终展示中文，拼音行和解释语言行按用户配置渲染
- 题干支持附带图片（点击可放大查看）或音频（详见听力选择题规范）
- 题干文本区域可滚动，当题干较长时不遮挡选项区域
- 题干内的关键词可加粗显示

### 1.2 选项布局

```
┌─────────────────────────────────────────────┐
│                                               │
│  ○  A. 看        Kàn                          │
│     ─────────────────────────────────         │
│  ○  B. 读        Dú                           │
│     ─────────────────────────────────         │
│  ○  C. 写        Xiě                          │
│     ─────────────────────────────────         │
│  ○  D. 听        Tīng                         │
│                                               │
└─────────────────────────────────────────────┘
```

**选项交互规则**：
- 选项以 A/B/C/D 纵向排列，每个选项为独立的可点击卡片
- 选项前方为圆形单选按钮（○），选中后变为实心圆（●）+ Rose 色填充
- 选中态视觉：选项卡片边框变为 Rose 色（`border: 2px solid #e11d48`），背景变为 Rose 10% 透明度（`rgba(225, 29, 72, 0.1)`）
- 点击已选中的选项可取消选中（变回未选中状态）
- 点击另一个选项自动取消前一个选中项
- 选项卡片最小点击区域：高度 ≥ 48px，宽度占满内容区
- 选项文本支持多语言渲染（中文 + 拼音，按用户配置）

### 1.3 提交方式（按考核层级不同）

| 考核层级 | 提交方式 | 说明 |
|---------|---------|------|
| 课时小测验 | 选中即提交 | 点击选项后自动判定，无需额外确认按钮 |
| 单元阶段测评 | 手动点击"下一题"前进 | 选中选项后需点击"下一题"按钮前进，可回看修改 |
| 级别综合考核 | 手动点击"下一题"前进 | 同上，且支持模块间切换 |

### 1.4 正确反馈（即时反馈模式——课时小测验适用）

```
┌─────────────────────────────────────────────┐
│                                               │
│  ● ✅ A. 看        Kàn                        │ ← 绿色高亮背景
│     ─────────────────────────────────         │    (#22c55e, 10% 透明度)
│  ○  B. 读        Dú                           │    + 绿色边框
│     ─────────────────────────────────         │
│  ○  C. 写        Xiě                          │
│     ─────────────────────────────────         │
│  ○  D. 听        Tīng                         │
│                                               │
│  ┌───────────────────────────────────────┐   │
│  │ ✅ 太棒了！                              │   │ ← 鼓励文案随机
│  └───────────────────────────────────────┘   │
│                                               │
│            [ 下一题 → ]                        │
└─────────────────────────────────────────────┘
```

**正确反馈规则**：
- 正确选项卡片背景变为成功色 10% 透明度（`rgba(34, 197, 94, 0.1)`），边框变为成功色（`#22c55e`）
- 选项前方显示 ✅ 图标，替代原有的圆形按钮
- 底部弹出鼓励文案卡片（毛玻璃面板），文案从以下列表中随机选取：
  - "太棒了！"
  - "答对啦！"
  - "继续加油！"
  - "真厉害！"
  - "完美！"
  - "学得真好！"
- 鼓励文案以 UI 语言展示（非中文 UI 时展示对应翻译）
- 其他选项变为 disabled 灰色态，不可再点击
- 1 秒后底部出现"下一题 →"按钮

### 1.5 错误反馈（即时反馈模式——课时小测验适用）

```
┌─────────────────────────────────────────────┐
│                                               │
│  ○  A. 看        Kàn                          │ ← 正确答案绿色高亮
│     ─────────────────────────────────         │
│  ● ❌ B. 读       Dú                          │ ← 用户选择红色高亮
│     ─────────────────────────────────         │
│  ○  C. 写        Xiě                          │
│     ─────────────────────────────────         │
│  ○  D. 听        Tīng                         │
│                                               │
│  ┌───────────────────────────────────────┐   │
│  │ ❌ 正确答案是 A                          │   │
│  │                                         │   │
│  │ 解析："看"表示用眼睛观看，常与"书"、     │   │
│  │ "电影"搭配。                              │   │
│  │ "读"侧重于出声朗读。                      │   │
│  │                                         │   │
│  │ 📋 已加入复习队列                         │   │
│  └───────────────────────────────────────┘   │
│                                               │
│            [ 下一题 → ]                        │
└─────────────────────────────────────────────┘
```

**错误反馈规则**：
- 用户选中的错误选项：背景变为错误色 10% 透明度（`rgba(239, 68, 68, 0.1)`），边框变为错误色（`#ef4444`），前方显示 ❌
- 正确选项自动高亮：背景变为成功色 10% 透明度，边框变为成功色，前方显示 ✅
- 底部弹出解析卡片（毛玻璃面板），包含：
  - "正确答案是 X" 标题行
  - 解析文字（说明为什么是正确答案，以及用户答案为什么不对）
  - "📋 已加入复习队列" 提示文字（表示该知识点已加入 SRS）
- 解析文字按用户语言配置渲染（中文 + 解释语言）
- 其他未选中选项变为 disabled 灰色态
- 1 秒后底部出现"下一题 →"按钮

## 二、多选题

### 2.1 与单选题的区别

```
┌─────────────────────────────────────────────┐
│  第 5/15 题                    ⚠️ 可多选      │ ← 多选提示标签
│─────────────────────────────────────────────│
│                                               │
│  以下哪些是中国传统节日？（可多选）              │
│                                               │
│  ☐  A. 春节                                   │ ← 方形复选框
│     ─────────────────────────────────         │
│  ☐  B. 情人节                                 │
│     ─────────────────────────────────         │
│  ☐  C. 中秋节                                 │
│     ─────────────────────────────────         │
│  ☐  D. 端午节                                 │
│                                               │
│            [ 确认提交 ]                        │
└─────────────────────────────────────────────┘
```

**多选题交互规则**：
- 题干右上角显示 Amber 色标签"⚠️ 可多选"
- 选项前方为方形复选框（☐），替代单选的圆形按钮
- 选中态：方形框内显示 ✓（☑），边框变为 Rose 色，背景变为 Rose 10% 透明度
- 可同时选中多个选项，点击已选中的选项可取消选中
- 必须点击"确认提交"按钮才能提交答案（不自动提交）

### 2.2 判定规则

| 判定方式 | 规则 | 适用场景 |
|---------|------|---------|
| 全对才算对 | 选中的选项必须与正确答案完全一致（不多不少） | 所有考核层级 |

**反馈展示**：
- 正确：所有正确选项绿色高亮 + ✅
- 错误：用户多选的错误项红色高亮 + ❌，漏选的正确项绿色高亮 + ✅
- 解析说明每个正确选项为什么对，每个错误选项为什么不对

## 四、拼音标注题

### 4.1 题面展示

```
┌─────────────────────────────────────────────┐
│  第 1/5 题                                    │
│─────────────────────────────────────────────│
│                                               │
│  请为以下汉字标注正确的拼音（含声调）：          │
│                                               │
│  ┌─────────────────────────────────────┐     │
│  │                                       │     │
│  │            图  书  馆                  │     │ ← 大字展示汉字
│  │                                       │     │
│  │    ┌──────┐ ┌──────┐ ┌──────┐       │     │
│  │    │      │ │      │ │      │       │     │ ← 拼音输入框
│  │    └──────┘ └──────┘ └──────┘       │     │
│  │                                       │     │
│  └─────────────────────────────────────┘     │
│                                               │
│  💡 输入提示：使用数字表示声调                   │
│     例如：tú = tu2，shū = shu1                 │
│                                               │
│            [ 确认 ]                            │
└─────────────────────────────────────────────┘
```

### 4.2 拼音输入方式

**输入规则**：
- 每个汉字对应一个独立输入框，输入框宽度自适应（最小 64px）
- 用户使用键盘输入拼音字母 + 数字声调（例：`tu2` 表示 `tú`，`shu1` 表示 `shū`，`guan3` 表示 `guǎn`）
- 声调数字映射：1 = 一声（ˉ），2 = 二声（ˊ），3 = 三声（ˇ），4 = 四声（ˋ），5 或不标 = 轻声
- 输入框内实时将"字母+数字"转换为带声调的拼音显示（例：用户输入 `tu2`，输入框内显示 `tú`）
- 多个输入框间支持 Tab 键切换到下一个输入框
- 输入框获得焦点时显示 Rose 色弥散光晕

**输入提示**：
- 首次展示拼音标注题时，在输入框下方显示灰色提示文字："使用数字表示声调，例如：tu2 = tú"
- 提示文字在用户首次输入后自动淡出
- 输入框上方展示待标注的汉字，字号为 Body L 以上（≥ 18px），确保汉字清晰可辨

### 4.3 校验规则

| 校验项 | 规则 | 说明 |
|--------|------|------|
| 拼音字母 | 必须完全正确 | `tu` 与 `du` 不同，判为错误 |
| 声调标注 | 必须正确 | `tú`(二声) 与 `tù`(四声) 不同，判为错误 |
| 大小写 | 不区分 | `Tu2` 与 `tu2` 均可 |
| 空格/特殊字符 | 自动忽略 | 用户误输入空格不影响判定 |

**反馈展示（即时反馈模式）**：
- 正确：输入框边框变为成功色，汉字上方显示绿色拼音 + ✅
- 错误：输入框边框变为错误色，显示用户输入（红色删除线）+ 正确拼音（绿色）+ 解析

## 五、排序组句题

### 5.1 打乱词语展示

```
┌─────────────────────────────────────────────┐
│  第 4/5 题                                    │
│─────────────────────────────────────────────│
│                                               │
│  请将以下词语拖拽排列成正确的句子：              │
│                                               │
│  [待选区域 — 打乱的词语卡片]                    │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐       │
│  │ 在    │ │ 看书  │ │ 图书馆│ │ 我   │       │
│  └──────┘ └──────┘ └──────┘ └──────┘       │
│                                               │
│  [答题区域 — 拖入/点击后排列]                    │
│  ┌──────────────────────────────────────┐    │
│  │  ___  ___  ___  ___  。               │    │ ← 空位占位符
│  └──────────────────────────────────────┘    │
│                                               │
│  💡 拖拽词语到正确位置，或点击词语按顺序排列       │
│                                               │
│        [ 重置 ]        [ 确认 ]               │
└─────────────────────────────────────────────┘
```

### 5.2 拖拽排序交互

**交互方式一：拖拽（主要）**
- 词语以独立卡片形式展示，卡片样式为毛玻璃卡片（`.glass-card`），圆角 `--radius-sm`（8px）
- 用户长按卡片（200ms）触发拖拽态：卡片略微放大（`scale(1.05)`）+ 阴影增强（Level 3）+ 半透明跟随手指
- 拖拽到答题区域的空位上松手 → 卡片吸附到该位置，伴随弹性动画（Spring 缓动）
- 答题区域的已有卡片可拖拽调整位置（拖出后原位空出，放入新位后其他卡片顺移）
- 从答题区域拖回待选区域 → 卡片回到待选区

**交互方式二：点击选择顺序（辅助）**
- 用户依次点击待选区域的词语卡片
- 每次点击后，该卡片自动按点击顺序填入答题区域的下一个空位
- 卡片从待选区域移到答题区域时有平滑飞入动画（300ms，Decelerate 缓动）
- 点击答题区域中已排列的卡片 → 该卡片回到待选区域（后续卡片自动前移）

**重置按钮**：
- 点击"重置"→ 所有卡片回到待选区域初始打乱位置
- 重置动画：所有卡片同时飞回（200ms）

### 5.3 排好后的展示效果

```
┌──────────────────────────────────────┐
│  我    在    图书馆    看书    。       │ ← 所有空位已填满
└──────────────────────────────────────┘
```

- 所有空位填满后，"确认"按钮变为可点击的 Rose 色主按钮
- 空位未填满时，"确认"按钮为 disabled 灰色态

### 5.4 提交校验

**正确**：
- 整个答题区域背景变为成功色 10% 透明度
- 各卡片下方出现绿色下划线
- 显示完整正确句子（含拼音和解释语言）

**错误**：
- 错误位置的卡片红色高亮，正确位置的卡片绿色高亮
- 下方显示正确排列的完整句子
- 解析说明句子的语法结构和词序规则

## 六、填空题

### 6.1 句子中的空位展示

```
┌─────────────────────────────────────────────┐
│  第 7/15 题                                   │
│─────────────────────────────────────────────│
│                                               │
│  请在横线处填入合适的词语：                       │
│                                               │
│  昨天下午，我和朋友 ______① 公园里               │
│  ______② 了很长时间。                           │
│                                               │
│  ┌───────────────────────────┐               │
│  │ ①  [          ]           │               │ ← 输入框 1（当前焦点）
│  │ ②  [          ]           │               │ ← 输入框 2
│  └───────────────────────────┘               │
│                                               │
│            [ 确认 ]                            │
└─────────────────────────────────────────────┘
```

### 6.2 输入框样式和位置

**空位展示**：
- 句子中的空位以下划线 + 圆圈数字编号（①②③…）标识
- 下划线长度固定为 6 个中文字符宽度（约 96px），不暗示答案长度
- 下划线颜色为中性灰（`#a3a3a3`）

**输入框**：
- 句子下方的独立输入区域，每个空位对应一个输入框
- 输入框前方标注对应编号（①②③…）
- 输入框样式：毛玻璃输入框（`.glass-input`），圆角 `--radius-md`（12px）
- 获得焦点时：Rose 色弥散光晕（`box-shadow: 0 0 0 4px rgba(225, 29, 72, 0.15)`）
- 输入框内显示灰色占位文字："请输入答案"

### 6.3 多个空位时的 Tab 切换

- 按 Tab 键或点击手机键盘上的"下一个" → 焦点跳到下一个输入框
- Shift+Tab 或点击"上一个" → 焦点跳到上一个输入框
- 最后一个输入框按 Tab → 焦点跳到"确认"按钮
- 输入框旁边显示小型上下箭头按钮（移动端），用于在多个空位间切换

### 6.4 校验规则

| 校验项 | 规则 |
|--------|------|
| 完全匹配 | 答案必须与标准答案完全一致 |
| 同义词接受 | 系统预设的同义词列表中的词语也判为正确（如"在"和"正在"） |
| 标点忽略 | 用户输入中的标点符号自动忽略 |
| 前后空格 | 自动 trim，不影响判定 |

### 7.2 文章与题目的分屏/联动展示

**移动端（默认）**：
- 文章区域和题目区域上下分布，以分隔线隔开
- 文章区域占屏幕上半部分（约 40%），可独立滚动

- 用户可在各子题间自由切换，已作答的题目保留答案

### 8.2 图片展示

- 图片以卡片形式展示，宽度占满内容区，高度自适应（最大高度 300px）
- 图片支持点击放大查看（全屏模态查看，双指缩放）
- 如果是纯文字情境描述（无图片），则以引用块样式展示情境文字

### 8.3 文本输入区

**输入区样式**：
- 多行文本输入框，高度最小 160px，可随内容自动扩展（最大高度 320px）
- 毛玻璃输入框样式（`.glass-input`），圆角 `--radius-lg`（16px）
- 获得焦点时显示 Rose 色弥散光晕

**字数限制与统计**：
- 输入区右下角实时显示字数统计："当前字数 / 最大字数"（如 "32 / 200 字"）
- 字数不足最低要求时：字数统计文字为 Amber 色，提交按钮 disabled
- 字数达到最低要求后：字数统计文字变为中性灰色，提交按钮变为可点击
- 字数接近上限（≥90%）时：字数统计文字变为 Amber 色警告
- 字数达到上限：无法继续输入，字数统计文字变为错误色

### 8.4 AI 批改说明

**提交后流程**：
1. 点击"提交" → 提交按钮变为 Loading 态（"AI 评分中..."） + disabled
2. 页面展示 AI 评分等待动画（详见 `04-level-exam.md` AI 批改等待页）
3. AI 批改完成后（预计 10-30 秒），自动跳转到结果或继续下一题

**AI 批改评分维度**（对用户不直接展示维度，仅展示总分和评语）：
- 内容完整性：是否描述了图片/情境中的关键元素
- 语言准确性：语法、用词是否正确
- 表达流畅性：句子是否通顺自然
- 字数要求：是否满足最低字数

## 九、通用交互规范

### 9.1 答题区域通用样式

| 元素 | 样式 |
|------|------|
| 题目卡片 | 毛玻璃面板（`.glass-card`），圆角 `--radius-2xl`（24px），内边距 24px |
| 选项卡片 | 毛玻璃面板，圆角 `--radius-lg`（16px），内边距 16px，最小高度 48px |
| 输入框 | `.glass-input`，圆角 `--radius-md`（12px），高度 48px |
| 按钮（主） | Rose 色，药丸形（`--radius-full`），高度 48px，文字白色 |
| 按钮（次） | 毛玻璃，药丸形，高度 40px，文字跟随主题色 |

### 9.2 答题操作防误设计

| 场景 | 处理方式 |
|------|---------|
| 意外退出（杀进程/切后台） | 单元测评和综合考核：已作答内容自动保存，重新进入后恢复到上次的答题位置；课时小测验：不保存，需重新开始 |
| 网络中断 | 答题过程不依赖实时网络（题目已预加载）；提交时如果断网，显示"网络连接失败"Toast + "重试提交"按钮；恢复网络后自动重试提交 |
| 重复点击提交 | 首次点击后按钮立即变为 Loading + disabled 态，防止重复提交 |
| 误触返回 | 单元测评和综合考核中，点击系统返回键/浏览器后退 → 弹出确认弹窗："确定退出考核？您的答题进度已保存，可稍后继续。" → "继续答题" / "保存并退出" |
