## 一、触发条件

### 1.1 解锁规则

- 当前单元的**全部课时**学习完毕（包括每课时的小测验）后，单元测评入口解锁
- 解锁时系统推送一条应用内通知："单元 X 全部课时已完成，快来测试一下吧！"（如推送通知开启，同步推送至系统通知栏）
- 未完成全部课时时，测评入口显示为锁定态（灰色 + 🔒 图标），点击后提示"请先完成全部课时学习"

### 1.2 前置条件检查

| 检查项 | 条件 | 未满足时行为 |
|--------|------|------------|
| 课时完成度 | 本单元所有课时标记为"已完成" | 测评入口锁定 + 点击提示"请先完成全部课时学习" |
| 登录状态 | 用户已登录 | 跳转登录弹窗（课程模块整体需登录） |
| 课程权限 | L1-L3 免费 / L4+ 已购买 | 触发付费墙 |

### 4.2 顶部信息栏

| 元素 | 位置 | 说明 |
|------|------|------|
| 题目进度 | 左上 | "第 X/N 题"，Body S（14px） |
| 计时器 | 右上 | ⏱️ 图标 + MM:SS 格式，从 00:00 开始正计时 |
| 进度条 | 标题下方全宽 | 已答题部分 Rose 色，当前题 Rose 色脉冲，未答部分灰色 |

**计时器说明**：
- 单元测评**不强制限时**，计时器仅供参考（帮助用户了解自己的答题速度）
- 计时器以中性灰色展示，不使用红色等警告色
- 如果用户中途退出恢复进度，计时器从上次暂停处继续计时

### 4.3 非即时反馈模式

- 单元测评采用**统一批改模式**：答完所有题目后一次性判分
- 选中选项后**不显示对错**，仅将选中状态标记在选项上（Rose 色选中态）
- 选中后可随时更改答案（点击其他选项切换选中）
- 已作答的题目在进度条上显示为实心圆点，未作答显示为空心圆点

### 4.4 "上一题"/"下一题"按钮

| 按钮 | 位置 | 行为 |
|------|------|------|
| ← 上一题 | 底部左侧 | 返回上一道题，保留已选答案；第 1 题时隐藏此按钮 |
| 下一题 → | 底部右侧 | 进入下一道题；未作答时也可跳过进入下一题 |
| 提交测评 | 底部右侧（最后一题时替代"下一题"） | 提交全部答案进行判分 |

- 两个按钮之间居中对齐，间距 16px
- "下一题"为 Rose 色主按钮，"上一题"为毛玻璃次要按钮
- 切换题目时带向左/向右滑动动画（300ms，Standard 缓动）

### 4.5 题目导航（快速跳转）

- 点击顶部进度条上的任意圆点 → 直接跳转到对应题目
- 进度条上的圆点状态：
  - ● 已作答（Rose 色实心）
  - ○ 未作答（灰色空心）
  - ◉ 当前题（Rose 色脉冲动画）

### 4.6 提交前确认弹窗

当用户点击"提交测评"时，弹出确认弹窗：

## 五、题型范围

单元测评使用以下 5 种题型：

| 题型 | 使用频率 | 分值 | 说明 |
|------|---------|------|------|
| **单选题** | 高（40-50%） | 每题 5-8 分 | 词义、语法、文化知识选择 |
| **听力选择题** | 中（15-20%） | 每题 8-10 分 | 听音频选择正确答案（最多播放 3 次） |
| **拼音标注题** | 中（10-15%） | 每题 5-8 分 | 为 2-4 个汉字标注拼音 |
| **排序组句题** | 中（10-15%） | 每题 8-10 分 | 4-6 个词语排列成正确句子 |
| **填空题** | 中（10-15%） | 每题 5-8 分 | 在句子空位填入合适词语 |

**分值分配**：
- 总分 100 分，各题分值根据题型难度和数量分配
- 具体每题分值在提交后的结果页中展示

## 七、重考规则

| 规则项 | 说明 |
|--------|------|
| 等待间隔 | **无等待间隔**，可立即重考 |
| 题目变化 | 重考时题目从题库中随机重新抽取，与上次不完全相同 |
| 重考次数 | 不限次数 |
| 成绩记录 | 记录所有测评成绩，以**最高分**为最终成绩 |
| 通过后重考 | 已通过的单元测评可重新测评（用于刷分），不影响已解锁的下一单元 |
| 复习提示 | 未通过时结果页展示薄弱知识点和推荐复习课时 |

## 八、通过后行为

### 8.1 解锁下一单元

- 通过单元测评（≥70 分）后，下一单元自动解锁
- 返回 Level 详情页（单元列表）时，可看到：
  - 当前单元测评卡片状态变为 ✅ 已通过 + 分数
  - 下一单元的课时从 🔒 变为可点击状态
  - 解锁动画：下一单元卡片从灰色渐变为正常色 + 🔒 图标消失动画（300ms）

### 8.2 进度更新

- Level 详情页的进度百分比更新（反映已通过的单元数/总单元数）
- Level 地图中当前 Level 的进度百分比同步更新

### 8.3 全部单元通过后

- 当该 Level 的所有单元测评全部通过 → 级别综合考核入口解锁
- 级别综合考核入口在 Level 详情页底部从 🔒 变为可点击的"开始考核"按钮
- 推送通知："Level X 全部单元测评已通过，快来挑战综合考核吧！"

## 九、状态矩阵

| 状态 | 描述 | 设计要求 |
|------|------|---------|
| **空状态 (Empty)** | 不适用 | 测评始终有 10-15 题，不存在空状态 |
| **加载中 (Loading)** | 测评题目加载中 | 开始页点击"开始测评"后显示全屏骨架屏（题干 + 4 选项占位），1-2 秒内加载完毕 |
| **首次加载 (First Load)** | 首次进入测评 | 展示测评开始页（说明 → 开始按钮）→ 加载题目 → 展示第一题 |
| **成功 (Success)** | 提交成功 + 通过 | 提交按钮 Loading 态（1-2 秒）→ 跳转通过结果页（向上滑入动画）→ 🎉 文案 + 分数 |
| **错误 (Error)** | 提交成功 + 未通过 | 提交按钮 Loading 态 → 跳转未通过结果页 → 😔 文案 + 分数 + 薄弱知识点 |
| **部分加载 (Partial)** | 不适用 | 题目一次性加载完毕 |
| **离线 (Offline)** | 网络断开时 | 如果题目已加载：可离线继续答题，提交时缓存结果等待联网上传；如果题目未加载：显示"网络连接失败，请检查网络后重试"错误页 + 重试按钮 |
| **意外退出** | 杀进程/切后台 | 每答一题自动保存进度到本地 + 服务端（如有网络）；重新进入后展示"继续测评"选项；保存进度有效期 24 小时，过期后清除 |
| **提交网络失败** | 提交时断网 | "提交失败，请检查网络连接"Toast + "重试提交"按钮；已作答内容不丢失 |
