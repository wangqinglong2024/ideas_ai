## 一、数据流向说明

### 1.1 考核题目数据流

```
管理后台（内容运营）
  │
  │  ① 课程内容 + 练习题库上架/发布
  ↓
Supabase 数据库
  │
  │  ② 考核系统根据考核层级规则，
  │     从题库中抽取随机题目组合
  ↓
应用端（考核界面）
  │
  │  ③ 用户答题
  ↓
应用端本地缓存（进度保存）
  │
  │  ④ 提交答卷（联网时实时提交，断网时缓存后重试）
  ↓
Supabase 数据库（答卷存储）
  │
  │  ⑤ 自动评分（选择/填空/排序等客观题）
  │     + AI 批改（书写/简答等主观题，调用 Dify 工作流）
  ↓
评分结果写回 Supabase
  │
  │  ⑥ 应用端拉取结果，展示结果页
  ↓
应用端（结果页 + 证书生成）
```

### 1.2 各层级考核的数据流差异

| 数据流环节 | 课时小测验 | 单元测评 | 级别综合考核 |
|-----------|----------|---------|------------|
| 题目加载时机 | 课时学习内容加载时一并预加载 | 点击"开始测评"后加载 | 点击"开始考核"后加载 |
| 本地进度保存 | 不保存 | 每答一题保存到本地 + 服务端 | 每答一题保存到本地 + 服务端 |
| 提交方式 | 每题即时提交结果 | 全部做完后统一提交 | 全部做完后统一提交 |
| 评分方式 | 客户端即时判定 + 服务端记录 | 服务端统一评分 | 服务端统一评分 + AI 批改（书写模块） |
| AI 批改 | 无 | 无 | L7+ 书写模块需要 |
| 结果存储 | 最近一次结果覆盖 | 所有历次记录保留 | 所有历次记录保留 |

### 1.3 SRS 复习队列数据流

```
课时小测验 / 单元测评 / 综合考核
  │
  │  答错的知识点
  ↓
SRS 复习队列（Supabase）
  │
  │  艾宾浩斯间隔计算
  │  1天 → 2天 → 4天 → 7天 → 15天 → 30天
  ↓
每日推送复习提醒
  │
  ↓
复习答题结果
  │
  │  答对 → 延长间隔
  │  答错 → 重置间隔
  ↓
更新 SRS 队列
```

### 1.4 证书数据流

```
综合考核通过（≥85 且单项 ≥60）
  │
  ↓
服务端自动生成证书记录（Supabase）
  │  · 用户 ID
  │  · Level 信息
  │  · HSK/CEFR 等级
  │  · 考核分数
  │  · 通过日期
  │  · 证书编号（唯一）
  ↓
应用端拉取证书数据
  │
  ↓
证书展示页渲染证书卡片
  │
  │  用户操作：保存图片 / 分享
  ↓
客户端渲染证书为高清 PNG 图片
  │
  ↓
系统保存/分享接口
```

### 1.5 考核进度保存数据流

```
用户每答一题
  │
  ↓
写入本地存储（IndexedDB / localStorage）
  │  · 当前考核 ID
  │  · 已答题目列表 + 用户答案
  │  · 当前题目位置
  │  · 计时器时间
  │
  ├── 联网时：同步写入 Supabase
  │
  └── 断网时：缓存到本地，联网后自动同步
  
用户重新进入考核
  │
  ↓
检查服务端是否有保存进度
  │
  ├── 有且未过期（<24h）→ 提示"继续考核"
  │
  └── 无或已过期 → 从头开始
```

## 二、非功能需求

### 2.1 性能要求

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 考核题目加载时间 | < 2 秒 | 从点击"开始"到第一题展示完毕 |
| 题目切换响应 | < 100ms | 点击"下一题"到新题目展示 |
| 答案提交响应 | < 500ms | 课时小测验即时反馈的判定和显示 |
| 全卷提交响应 | < 3 秒 | 单元测评和综合考核提交后到结果页展示（不含 AI 批改） |
| AI 批改响应 | 10-30 秒 | 书写模块 AI 评分（调用 Dify 工作流） |
| AI 批改超时上限 | 60 秒 | 超过 60 秒提供"先查看其他成绩"选项 |
| 证书图片生成 | < 3 秒 | 从点击"保存图片"到图片生成完毕 |
| 进度保存延迟 | < 200ms | 每答一题写入本地存储的时间 |

### 2.2 并发与容量

| 指标 | 目标值 |
|------|--------|
| 同时在线考核用户 | 支持 500 人同时考核（MVP 阶段） |
| 题库规模 | 每 Level 至少 300 道题（确保随机不重复） |
| 单用户考核记录 | 不限历史记录数量 |
| 证书存储 | 单用户最多 12 张级别证书 + 4 张阶段证书 |

### 2.3 考核数据防作弊

| 防作弊措施 | 说明 |
|-----------|------|
| 答题时间异常检测 | 单题答题时间 < 2 秒（排除单选即时提交的课时小测验）标记为可疑 |
| 全卷用时异常检测 | 15 题单元测评 < 2 分钟完成标记为可疑 |
| 异常处理方式 | 仅标记记录（写入 `is_suspicious` 字段），不阻止提交或影响成绩；后台可查询可疑记录 |
| 题目随机 | 每次考核题目从题库随机抽取，降低答案传播风险 |
| 选项随机 | 选择题的选项顺序随机排列（同一题不同用户看到的 ABCD 顺序可能不同） |
| 进度加密 | 本地存储的答题进度数据进行签名校验，防止篡改 |

**不做的防作弊措施**（MVP 阶段不需要）：
- 不做摄像头监控
- 不做屏幕切换检测
- 不做 IP 异常检测
- 不做设备指纹校验

### 2.4 可用性

| 指标 | 目标值 |
|------|--------|
| 服务可用性 | 99.9% SLA |
| 考核数据一致性 | 提交的答卷不允许丢失（本地缓存 + 服务端双写） |
| 离线支持 | 题目预加载后支持离线答题，联网后自动同步结果 |
| 跨设备一致性 | 同一账号在不同设备上看到相同的考核记录、证书、倒计时 |

### 2.5 无障碍

| 规则 | 要求 |
|------|------|
| 色彩对比度 | 正文 ≥ 4.5:1，大标题 ≥ 3:1（正确/错误反馈不仅依赖颜色，同时使用 ✅/❌ 图标） |
| 触控尺寸 | 选项卡片高度 ≥ 48px，按钮 ≥ 44×44pt |
| 屏幕阅读器 | 选项使用 `role="radio"` / `role="checkbox"`；音频播放按钮有 `aria-label="播放音频"` |
| 键盘导航 | Tab 切换选项 → Enter 选中 → Tab 到下一题按钮 → Enter 前进 |
| 动效敏感 | 支持 `prefers-reduced-motion`，可关闭庆祝粒子动画和进度条动画 |
| 听力题辅助 | 听力题音频提供文字转录选项（无障碍模式下，在音频区域下方展示音频文本） |

## 三、验收标准汇总

### 3.1 课时小测验验收标准

| # | 验收项 | 验收标准 |
|---|--------|---------|
| 1 | 自动弹出 | 课时全部内容学完后 ≤ 1 秒内弹出小测验界面 |
| 2 | 题目数量 | 每次测验 3-5 题，覆盖本课时核心知识点 |
| 3 | 即时反馈 | 选中选项后 ≤ 100ms 显示正确/错误反馈 |
| 4 | 正确反馈 | 显示绿色高亮 + ✅ + 随机鼓励文案（6 条中随机选 1 条） |
| 5 | 错误反馈 | 显示红色高亮 + ❌ + 正确答案绿色高亮 + 解析文字 + "已加入复习队列"提示 |
| 6 | SRS 加入 | 答错题对应知识点在 SRS 队列中可查到，复习间隔为 1 天 |
| 7 | 总结页 | 展示环形正确率图 + 各题正误列表 + "继续学习"按钮 |
| 8 | 题目随机 | 复习再做时题目与首次不同（从题库随机抽取） |
| 9 | 离线答题 | 断网状态下可正常答题，联网后结果自动上传 |

### 3.2 单元测评验收标准

| # | 验收项 | 验收标准 |
|---|--------|---------|
| 1 | 解锁条件 | 单元全部课时完成后测评入口从🔒变为可点击，点击锁定态提示"请先完成全部课时学习" |
| 2 | 测评开始页 | 展示题目数量、通过标准、题型范围、进度保存提示 |
| 3 | 题目数量 | 10-15 题，覆盖本单元全部课时知识点 |
| 4 | 非即时反馈 | 选中选项后仅标记选中态（Rose 色），不显示对错 |
| 5 | 前后翻页 | "上一题"/"下一题"按钮可正常切换，已答题目保留选中状态 |
| 6 | 提交确认 | 点击"提交测评"弹出确认弹窗；有未作答题目时显示未作答数量警告 |
| 7 | 通过结果 | ≥70 分：🎉 文案 + 绿色分数 + 答题详情 + "继续学习下一单元"按钮 |
| 8 | 未通过结果 | <70 分：😔 文案 + 红色分数 + 薄弱知识点列表 + "去复习"/"重新测评"按钮 |
| 9 | 解锁下一单元 | 通过后返回 Level 详情页，下一单元显示为已解锁 |
| 10 | 进度保存 | 中途退出后重新进入可恢复进度，保存进度 24 小时有效 |
| 11 | 即时重考 | 未通过后可立即重考，题目随机重新抽取 |

### 3.3 级别综合考核验收标准

| # | 验收项 | 验收标准 |
|---|--------|---------|
| 1 | 解锁条件 | 全部单元测评通过后综合考核入口解锁 |
| 2 | 考核说明页 | 展示考核模块列表（含各模块题目数）、通过标准（85 分 + 单项 60）、预计用时 |
| 3 | 模块切换 | 4 个模块 Tab 可自由切换，切换不丢失已答内容 |
| 4 | 模块进度 | 每个 Tab 显示完成状态微标（已答/总题数或 ✅） |
| 5 | 提交确认 | 弹窗显示各模块作答情况汇总；有未作答题目时警告 |
| 6 | AI 批改等待 | 书写模块提交后展示等待页（AI 图标动画 + 模拟进度条 + 趣味小知识） |
| 7 | AI 批改超时 | >60 秒提供"先查看其他成绩"选项 |
| 8 | 通过结果 | 庆祝粒子动画 + 总分 + 雷达图 + 各模块分数 + 证书预览 + "查看证书"/"继续学习下一级"按钮 |
| 9 | 未通过结果 | 总分/单项未达标详细说明 + 薄弱模块红色高亮 + 薄弱知识点 + 24h 倒计时 |
| 10 | 24h 重考间隔 | 未通过后考核入口显示倒计时（精确到分钟），倒计时结束后自动变为"重新考核"按钮 |
| 11 | 进度保存 | 中途退出后可恢复进度，24 小时有效 |
| 12 | 长时间无操作 | >30 分钟无操作弹出提示，计时器暂停 |

### 3.4 电子证书验收标准

| # | 验收项 | 验收标准 |
|---|--------|---------|
| 1 | 自动生成 | 综合考核通过后自动生成证书，无需用户操作 |
| 2 | 证书内容 | 包含：Logo + 标题 + 用户昵称 + Level 名称 + HSK/CEFR + 分数 + 日期 + 编号 + Slogan |
| 3 | 证书展示 | 证书详情页完整展示证书卡片，支持双指缩放和全屏查看 |
| 4 | 保存图片 | 点击"保存图片"生成 1080×1440 PNG 图片，≤3 秒完成 |
| 5 | 分享功能 | 点击"分享证书"调用系统分享面板，分享图片含品牌水印 |
| 6 | 个人中心列表 | "我的证书"列表按 Level 从高到低排列，每项显示缩略图 + Level + 分数 + 日期 |
| 7 | 空状态 | 无证书时展示品牌插画 + 引导文案 + "去学习"按钮 |
| 8 | 跨设备一致 | 换设备登录同一账号可看到所有已获得的证书 |

### 3.5 通用验收标准

| # | 验收项 | 验收标准 |
|---|--------|---------|
| 1 | 多语言 | 所有界面文案（按钮标签、提示、反馈）支持汉语/英语/越南语切换 |
| 2 | 深色/浅色模式 | 所有页面在深色和浅色模式下均正确显示，颜色对比度达标 |
| 3 | 骨架屏 | 所有加载态使用 Skeleton 骨架屏，禁止纯 Spinner |
| 4 | 操作反馈 | 所有按钮点击后 ≤ 100ms 有视觉响应（缩放/变色/Loading） |
| 5 | 网络异常 | 断网时显示友好提示 + 重试按钮，不出现空白页或技术错误信息 |
| 6 | 防重复提交 | 提交按钮点击后立即 disabled + Loading，防止重复提交 |
| 7 | 误触保护 | 考核中点击返回键弹出确认弹窗，不直接退出 |
| 8 | 进度保存可靠性 | 进度数据本地 + 服务端双写，单点故障不丢失数据 |
