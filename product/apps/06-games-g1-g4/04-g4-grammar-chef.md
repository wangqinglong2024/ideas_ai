## 二、游戏子模式

### 2.1 子模式总览

| 子模式 | 人数 | 影响段位 | 结束条件 | 特殊规则 |
|--------|------|---------|---------|---------|
| **单人·经营** | 1 人 | 否 | 连续 3 位顾客离开（耐心耗尽） | 无尽顾客，速度递增 |
| **单人·限时** | 1 人 | 否 | 90 秒倒计时归零 | 90 秒内完成最多订单 |
| **1v1 PK** | 2 人 | 是 | 90 秒倒计时归零 | 各自厨房，同时接单，比完成数量 |
| **多人协作** | 2 人 | 是 | 90 秒倒计时归零 | 合作厨房：一人选词，一人排序 |

### 2.2 单人·经营模式

- 顾客源源不断排队等待
- 每位顾客有独立耐心计时条（15-25 秒，视句子难度）
- 耐心耗尽 → 顾客生气离开 → 累计离开人数 +1
- 累计 3 位顾客离开 → Game Over
- 连续正确完成订单获得 Combo 加速（顾客等待时间延长、分数加成）
- 挑战目标：服务最多顾客

### 2.3 单人·限时模式

- 90 秒内尽可能多完成订单
- 顾客不会因为超时离开（仅限时间制约）
- 每完成一单 +10 秒奖励时间（最多延长至 120 秒总计）
- 挑战目标：总订单数和总分

### 2.4 1v1 PK 模式

- 双方各自独立厨房，接收**不同的随机订单**
- 90 秒内比谁完成的订单更多
- 完成数量相同 → 比总分（正确句子的词汇难度越高分越高）
- 段位影响：胜 +1 星，负 -1 星，平局不变

### 2.5 多人协作模式

- 2 人共用一个厨房
- **分工机制**：
  - 玩家 A（选词手）：只能从传送带上抓取食材放到操作台
  - 玩家 B（排序手）：只能在操作台上排列食材顺序并放入锅中提交
- 双方均可看到顾客订单和当前操作台状态
- 需要配合默契：A 快速选对词，B 快速排对序
- 90 秒倒计时结束，按双方共同完成的订单数计分
- 胜负判定：与其他协作组比较（匹配两组，比谁做得多）

### 3.3 顾客等待区（180px）

**顾客队列**：
- 最多同时显示 **3 位**排队顾客（从左到右排列）
- 最左侧为当前服务的顾客（最大、最清晰），后面两位依次缩小（90%、80%）

**顾客角色**：
- 中国风卡通人物头像（80×80px），有多种样式随机分配
- 每位顾客头像上方有**名字**（如 "小明"、"李阿姨"、"王老板"），Body S 14px
- VIP 顾客有金色边框 + ⭐ 标记

**耐心计时条**：
- 位于顾客头像下方，宽度 120px，高度 6px
- 初始满格绿色 → 逐渐缩短并变色：绿 → 黄 → 橙 → 红
- 颜色变化阈值：100%-60% 绿色 → 60%-30% 黄色 → 30%-10% 橙色 → <10% 红色闪烁
- 耐心计时秒数显示在计时条右侧（Caption 12px）

**订单显示**（当前顾客）：
- 当前顾客下方显示「目标句子的意思」（即 "菜名"）
- 格式：引号包裹的目标语义描述（如 `"虽然天气冷，但我们还是去了公园"`）
- Body（16px），白色，最多两行，超长截断
- 如果目标句子涉及关联词 → 关联词部分高亮为 Amber 色

### 3.4 厨房操作区（500px）

整个操作区分为三列：

#### 3.4.1 传送带（左侧，宽 360px）

**传送带视觉**：
- 竖向传送带，词语食材方块从上方持续滚入
- 传送带背景：金属质感深灰色（`#2a2a2a`），两侧有传动齿轮装饰
- 可上下滚动浏览所有可用食材

**词语食材方块**：
- 尺寸：根据词语长度自适应（最小 80×56px，最大 160×56px）
- 圆角 12px
- 背景色按词性分类：
  - 名词：Sky 蓝淡底（`rgba(2,132,199,0.2)`）
  - 动词：绿色淡底（`rgba(34,197,94,0.2)`）
  - 形容词/副词：Amber 淡底（`rgba(217,119,6,0.2)`）
  - 关联词：Rose 红淡底（`rgba(225,29,72,0.2)`）+ 金色边框（特殊食材标识）
  - 标点符号：灰色淡底
- 方块中心显示词语（Body S 14px，白色，字重 600）
- 方块上方小标签显示词性（Caption 11px，半透明白色）

**传送带内容**：
- 包含当前订单所需的全部正确词语 + 3-5 个干扰词语
- 干扰词语在语义上接近正确答案（如目标 "因为" 的干扰项是 "所以"、"但是"）
- 食材排列顺序随机（不按正确语序）

#### 3.4.2 操作台（中间，宽 760px）

**操作台视觉**：
- 木质纹理台面（中国风砧板质感），浅棕色（`#8B7355`）
- 周围有微弱的暖光效果

**词语排列区**：
- 水平排列的词语槽位，最多 **12 个槽位**（适应最长句子）
- 每个槽位为虚线框（56×56px 最小），间距 4px
- 空槽位：虚线边框（白色，透明度 30%）
- 已放置词语：与传送带食材方块样式一致，但尺寸略大（高度 64px）

**操作方式**：
- 从传送带拖动词语到操作台的任意槽位
- 操作台上的词语可**左右拖动调整顺序**
- 双击操作台上的词语 → 退回传送带

**语序辅助**：
- 操作台上已放置 2 个以上词语时，如果当前顺序可组成部分正确序列 → 词语间连线变绿
- 如果当前顺序有明显错误 → 词语间连线变灰虚线（不直接提示哪里错，仅暗示需调整）

#### 3.4.3 锅区（右侧，宽 280px）

**锅视觉**：
- 中国风大铁锅造型（直径 200px），有火焰动画在锅底
- 锅中有微弱的热气/蒸汽上升动画（白色半透明，持续循环）
- 锅上方文字提示 `将排好的词语放入锅中`（Caption 12px）

**提交操作**：
- 将操作台上排列好的词语整体拖入锅中 **或** 点击底部「提交」按钮
- 提交后触发烹饪动画（见 §4.3）

### 3.5 底部操作按钮区（80px）

## 四、核心交互

### 4.1 第一步：接单 — 查看顾客订单

**触发**：新顾客到达（或上一位顾客处理完毕后，下一位自动前移）

**视觉反馈**：
1. 新顾客从屏幕右侧走入等待区（步行动画，300ms）
2. 顾客到达后，头像上方弹出对话气泡，显示订单（目标句子语义）
3. 对话气泡持续 2 秒后收起，订单信息转移到顾客下方固定显示区
4. 同时，传送带开始滚入与本订单相关的词语食材

**音效**：门铃 "叮铃" 音（顾客到达）

### 4.2 第二步：选词 — 从传送带抓取食材

**用户动作**：在传送带上**点击或拖动**词语食材方块

**点击模式**：
1. 点击传送带上的词语 → 词语自动飞入操作台的第一个空槽位（200ms 弧线动画）
2. 词语飞行时有轻微旋转 + 缩放弹跳

**拖动模式**：
1. 长按传送带上的词语（150ms 触发拖动）→ 词语跟随手指
2. 拖到操作台区域松手 → 词语落入最近的空槽位（弹跳落地动画）
3. 拖到操作台之外松手 → 词语回弹到传送带原位

**选择干扰词时的反馈**：
- 干扰词和正确词在传送带上无视觉区别（玩家需要自行判断）
- 将干扰词放入操作台本身不会触发错误提示（错误在提交时判定）

**音效**：
- 抓取：轻微 "嗒" 拾取音
- 放置：短促 "咚" 落地音

### 4.3 第三步：排序 — 在操作台上调整词语顺序

**用户动作**：在操作台上**拖动**词语方块调整前后位置

**操作细节**：
1. 长按操作台上的词语（150ms）→ 词语浮起（上移 8px + 阴影加深）
2. 左右拖动 → 其他词语自动左右挤让（200ms 平滑位移）
3. 松手 → 词语落入当前位置（弹跳 100ms）

**双击退回**：
- 双击操作台上的词语 → 词语飞回传送带（弧线动画 200ms）

**语序辅助提示**：
- 操作台上有 ≥ 2 个词语时，词语之间显示连接线
- 部分正确的连接：连接线为绿色实线
- 不确定/可能有误的连接：连接线为灰色虚线
- 此提示仅供参考，不保证 100% 准确（避免过度降低难度）

### 4.4 第四步：提交 — 放入锅中烹饪

**用户动作**：点击「放入锅中」按钮 或 将操作台词语整体拖入锅区

#### 4.4.1 提交正确

**判定**：操作台上的词语序列与目标句子的正确语序完全一致

**视觉反馈**（按时间顺序）：
1. **烹饪动画**（0ms）：操作台上的词语依次飞入锅中（每个间隔 80ms），锅中火焰变大
2. **蒸汽升腾**（500ms）：锅中升起大量白色蒸汽 + 金色光芒
3. **菜品呈现**（800ms）：锅中弹出一盘精美中国菜的卡通图标（随机：饺子/面条/炒菜/汤品等）
4. **上菜动画**（1000ms）：菜品飞向当前顾客
5. **顾客满意**（1200ms）：顾客头像变为笑脸 😊 + 头上冒出 ❤️ + 竖起大拇指
6. **得分飘字**（1200ms）：顾客上方飘出 `+50`（Amber 色，H2 24px）
7. **顾客离开**（1500ms）：顾客满意地走向左侧离开（步行动画）
8. **Combo 更新**（1200ms）：连击计数 +1

**音效**：锅中 "嗞啦" 烹饪音 → 叮铃完成音 → 顾客满意 "谢谢" 语音

**分数变化**：

| 情况 | 基础分 | Combo 加成 | 特殊加成 |
|------|-------|-----------|---------|
| 普通顾客正确 | 50 | Combo ×1.5（3连）/ ×2（5连）/ ×3（10连） | — |
| VIP 顾客正确 | 50 | 同上 | ×2（VIP 双倍） |
| 含关联词且正确使用 | 50 | 同上 | +20 关联词加分 |
| 快速完成（<8秒） | 50 | 同上 | +10 速度加分 |

#### 4.4.2 提交错误

**判定**：词语序列与正确语序不一致（包括：缺词、多词、语序错误）

**视觉反馈**：
1. **锅冒黑烟**（0ms）：词语飞入锅中后，锅中冒出黑色烟雾（持续 800ms）
2. **失败音效 + 锅震动**（200ms）：锅左右震动 3 次（±8px，200ms）
3. **顾客不满**（500ms）：顾客表情变为不满 😤 + 头上冒出 💢
4. **错误提示**（500ms）：操作台上方显示正确语序（绿色文字，Body 16px），持续 2 秒供学习
5. **词语退回**（1500ms）：操作台上的词语退回传送带
6. **Combo 中断**（0ms）：Combo 归零 + `连击中断!` 提示

**音效**：沉闷 "噗嗤" 失败音 + 锅碰撞音

**分数变化**：扣 10 分（不低于 0 分）

**注意**：错误提交**不直接导致顾客离开**，顾客耐心条继续倒计时，玩家可重新选词排序再次提交

#### 4.4.3 顾客超时离开

**触发**：顾客耐心条耗尽

**视觉反馈**：
1. 耐心条归零时最后一格红色闪烁 3 次
2. 顾客表情变为愤怒 😡 + 头上冒出 💢💢
3. 顾客转身走向右侧（愤怒步行动画，比正常步行快）
4. 顾客离开时留下 `-20` 飘字（Rose 红色）
5. 失败计数 +1（对应的 ✗ 图标变红 + 放大脉冲）
6. 操作台自动清空，传送带更新为下一位顾客的食材

**音效**：不耐烦的 "哼" 音 + 脚步声

**分数变化**：扣 20 分（不低于 0 分）

### 4.5 关联词特殊食材

关联词（"因为""所以""虽然""但是""不但""而且""如果""就"等）在游戏中作为**特殊食材**：

**视觉区别**：
- Rose 红淡底 + 金色 2px 边框 + 方块右上角 ⭐ 小标记
- 传送带中关联词食材有微弱金色发光脉冲

**使用正确时的额外奖励**：
- 每正确使用一个关联词 → 额外 +20 分
- 正确使用一对关联词（如 "因为...所以..."）→ 额外 +50 分（成对使用奖励）
- 成对关联词得分飘字为金色特效（区别于普通得分的 Amber 色）

**相关题型示例**：
- 目标句意：`虽然天气很冷，但是我们还是去了公园。`
- 传送带食材：`虽然` `天气` `很冷` `但是` `我们` `还是` `去了` `公园` `。` + 干扰词 `因为` `所以` `可是`
- 正确排序：`虽然` → `天气` → `很冷` → `，` → `但是` → `我们` → `还是` → `去了` → `公园` → `。`

## 五、难度递增

### 5.1 经营模式难度曲线

| 累计完成订单数 | 句子长度 | 顾客耐心 | 干扰词数量 | 句型复杂度 |
|-------------|---------|---------|----------|-----------|
| 0-5 单 | 4-6 个词 | 25 秒 | 2 个 | 简单句（主谓宾） |
| 6-10 单 | 5-7 个词 | 22 秒 | 3 个 | 含形容词/副词修饰 |
| 11-20 单 | 6-8 个词 | 20 秒 | 3-4 个 | 含单关联词（因为/所以） |
| 21-30 单 | 7-10 个词 | 18 秒 | 4 个 | 含成对关联词 |
| 31-40 单 | 8-12 个词 | 16 秒 | 5 个 | 复句 + 句式转换（反问/感叹） |
| 41 单+ | 10-14 个词 | 15 秒 | 5-6 个 | 多重复句 + 混合关联词 |

### 5.2 限时模式难度

- 90 秒内不递增（保持中等难度：6-8 个词、20 秒耐心、3 个干扰词）
- 重点在于速度和准确率的平衡

### 5.3 PK 模式难度

- 固定中等难度（同限时模式）
- 双方接收相同难度但不同内容的订单（确保公平）

### 5.4 VIP 顾客

- 每 5 位普通顾客后出现 1 位 VIP 顾客
- VIP 顾客特征：金色边框头像 + ⭐ 标记 + 出场时金色光芒
- VIP 订单难度 = 当前难度 +1 级
- VIP 正确完成 → 双倍分数
- VIP 超时离开 → 双倍扣分（-40 分）

## 六、生命值系统

G4 使用「顾客离开数」替代传统生命值：

| 规则 | 说明 |
|------|------|
| **失败上限** | 3 位顾客离开 → Game Over（经营模式） |
| **计数图标** | 顶部 HUD 3 个 ✗ 图标，初始灰色 |
| **触发条件** | 顾客耐心耗尽离开 或 跳过本单 |
| **不触发条件** | 提交错误但顾客耐心未耗尽（可重新提交） |

**失败图标动画**（顾客离开时）：
1. 对应 ✗ 图标从灰色变红色
2. 放大至 150% 后回弹至 100%（200ms）
3. 短暂震动（100ms）

**最后机会提醒**：已有 2 位顾客离开时（仅剩 1 次容错）：
- 第三个 ✗ 图标持续缓慢闪烁（红色，每秒 1 次）
- 屏幕边缘持续微弱红色光晕

**限时/PK 模式**：无失败上限，顾客离开仅扣分

## 七、PK 模式特殊规则

### 7.1 独立厨房

- 双方各自接收不同的随机订单（由服务端分别生成）
- 订单难度相同（服务端控制难度平衡）
- 各自独立完成，互不影响

### 7.2 对方状态反馈

| 事件 | 己方屏幕反馈 |
|------|------------|
| **对方完成一单** | 对手区闪烁绿色 `完成 +1` + 对手完成数跳动 |
| **对方连续完成（Combo ≥ 3）** | 己方屏幕顶部短暂显示 `对手连击 ×3！` 提示条（800ms） |
| **对方失败一单** | 对手区短暂红色闪光 |
| **对方订单数超越己方** | 己方订单计数变 Rose 红闪烁 + `加油！` 提示 |

### 7.3 PK 结束条件

1. **90 秒倒计时归零** → 完成订单数多者胜
2. **订单数相同** → 总分高者胜
3. **仍相同** → 平局

### 7.4 协作模式评分

- 双方共同的完成订单数 × 100 = 协作分
- 与另一匹配组比较协作分
- 高分组 +1 星，低分组不扣星

## 九、状态矩阵

### 9.1 游戏进行中状态

| 状态 | 描述 | 设计要求 |
|------|------|---------|
| **等待接单** | 顾客刚到达，传送带正在更新食材 | 传送带滚动动画，操作台清空 |
| **选词中** | 玩家正在从传送带选取食材 | 传送带可交互，操作台可放入 |
| **排序中** | 玩家正在操作台排列词语顺序 | 操作台可拖动调整 |
| **提交中** | 词语飞入锅中 + 烹饪动画 | 禁止操作，播放烹饪动画 |
| **正确反馈** | 上菜 + 顾客满意 | 得分特效 + Combo 更新 |
| **错误反馈** | 黑烟 + 正确答案展示 | 显示正确语序 2 秒 |
| **顾客离开** | 耐心耗尽，顾客走了 | 失败计数 +1 + 扣分 |
| **Combo 激活** | 连续正确 ≥ 3 单 | Combo 特效 + 火焰增大 |
| **暂停** | 玩家暂停 | 参见 `08-hud-landscape.md` |
| **Game Over** | 3 位顾客离开 | 厨房灯光熄灭动画 → Game Over → 结算 |

### 9.2 PK 模式状态

| 状态 | 描述 | 设计要求 |
|------|------|---------|
| **双方对战中** | 各自独立厨房操作 | 己方完整画面 + 对手缩略进度 |
| **对手领先** | 对方完成订单数 > 己方 | 己方订单计数变红 + 提示文字 |
| **时间即将结束** | 最后 10 秒 | 倒计时变红闪烁 + 紧迫音效 |
| **对手断线** | 对手网络断开 | 等待 15 秒重连，超时判己方获胜 |

### 9.3 协作模式状态

| 状态 | 描述 | 设计要求 |
|------|------|---------|
| **等待队友** | 匹配/等待队友连接 | 参见 `05-game-common/03-matching.md` |
| **协作进行中** | A 选词 + B 排序 | 各自操作区高亮，非操作区半透明 |
| **队友选词中** | A 正在抓取食材 | B 端操作台实时显示 A 放入的词语 |
| **队友排序中** | B 正在排列词语 | A 端可看到 B 的排列操作（实时同步） |
| **队友断线** | 队友网络断开 | 等待 15 秒重连；超时则 A/B 均可操作全部功能（降级为单人） |

### 9.4 空状态 / 异常状态

| 状态 | 描述 | 设计要求 |
|------|------|---------|
