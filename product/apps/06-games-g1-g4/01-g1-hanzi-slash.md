# G1 汉字切切切（L1 · 切水果式）

> 关联文档：[游戏通用系统](../05-game-common/00-index.md) | [本轮索引](00-index.md)
> 通用 HUD 参见 `05-game-common/08-hud-landscape.md`，以下仅描述 G1 独特画面和交互。

---

## 一、游戏概述

| 属性 | 说明 |
|------|------|
| **一句话描述** | 汉字从屏幕飞出，滑动手指切对正确汉字，切错扣命——像切水果一样上瘾的汉字识别游戏 |
| **灵感来源** | Fruit Ninja（切水果） |
| **题库范围** | L1 课程 300 个基础汉字（M1 汉字与书写 + M2 拼音与语音），含拼音、释义、配图 |
| **单局时长** | 1-3 分钟（经典模式视生命值消耗而定；限时模式固定 60 秒） |
| **情绪目标** | 爽快、紧张、成就感 |

## 二、游戏子模式

### 2.1 子模式总览

| 子模式 | 人数 | 影响段位 | 结束条件 | 特殊规则 |
|--------|------|---------|---------|---------|
| **单人·经典** | 1 人 | 否 | 3 条命耗尽 | 无尽模式，速度逐渐加快 |
| **单人·限时** | 1 人 | 否 | 60 秒倒计时归零 | 切错不扣命，仅扣分 |
| **1v1 PK** | 2 人 | 是 | 60 秒倒计时归零 或 一方 Game Over | 双方同题序列，抢答机制 |
| **多人混战** | 2-4 人 | 是 | 90 秒倒计时归零 | 积分最高者胜 |

### 2.2 单人·经典模式

- 汉字卡片无限飞出，速度逐渐加快
- 切对得分，切错扣命（3 条命）
- 漏切（卡片落出屏幕）不扣分不扣命
- 3 条命耗尽 → Game Over → 进入结算页
- 挑战目标：个人最高分

### 2.3 单人·限时模式

- 60 秒倒计时，尽可能多地切对汉字
- 切对得分，切错**仅扣 5 分**，不扣命
- 漏切不扣分
- 倒计时归零 → 进入结算页
- 挑战目标：60 秒内最高分

### 2.4 1v1 PK 模式

- 双方看到**完全相同的出题序列**（同一时间出现相同汉字卡片）
- **抢答机制**：同一张卡片，谁先切对谁得分；另一方该卡片自动消失
- 切错仍扣自己的命（3 条命）
- 结束条件：60 秒倒计时归零 或 一方 Game Over
- 胜负判定：分数高者胜；分数相同则切对数多者胜；仍相同则平局
- 段位影响：胜 +1 星，负 -1 星，平局不变

### 2.5 多人混战模式

- 2-4 人同时游戏，所有人看到**相同的出题序列**
- 抢答机制同 PK 模式
- 每人 3 条命，命耗尽则提前淘汰（可继续观战）
- 90 秒倒计时归零 → 按积分排名
- 排名规则：第 1 名 +1 星，其他不扣星（多人模式宽容策略）

## 四、核心交互

### 4.1 汉字卡片飞出

**触发**：游戏开始后，服务端按节奏下发题目，卡片从屏幕底部飞出

**卡片视觉**：
- 尺寸：120×120px 正方形，圆角 16px
- 背景：毛玻璃白色半透明（`rgba(255,255,255,0.15)` + `backdrop-filter: blur(8px)`）
- 内容：居中显示一个汉字（H1 32px，白色，字重 700）
- 声调色带：卡片顶部 4px 高色带，颜色对应该汉字声调（一声蓝/二声绿/三声橙/四声红）

**飞行轨迹**：
- 从屏幕底部外（y > 1080）以抛物线轨迹飞入游戏区
- 水平起始位置随机（x 范围：200px - 1720px）
- 初始速度向上（vy: -600 ~ -900 px/s），受重力影响（g: 400 px/s²）减速后下落
- 水平速度随机（vx: -100 ~ 100 px/s），产生弧线
- 卡片飞行时缓慢旋转（角速度 30°-60°/s）
- 卡片完全落出屏幕底部视为"漏切"

**同时出现的卡片数量**：
- 初始：每波 2 张卡片
- 每 30 秒增加 1 张，最多同时 5 张
- 卡片间飞出间隔：500ms → 逐渐缩短至 200ms

### 4.2 切割操作 — 切对

**用户动作**：滑动手指划过**正确的汉字**卡片（即底部提示对应的汉字）

**判定规则**：滑动轨迹与卡片碰撞框（卡片尺寸 + 8px 容差）相交 → 判定为切割

**视觉反馈**（按时间顺序）：
1. **卡片裂开**（0ms）：卡片沿切割轨迹方向裂成两半，两半向两侧飞散并旋转，持续 400ms 后淡出
2. **得分飘字**（50ms）：在卡片原位置向上飘出 `+10` 字样（Amber 色，H3 20px），上移 60px 后淡出，持续 600ms
3. **粒子爆炸**（0ms）：卡片中心迸发 12-16 个彩色粒子（使用该汉字声调对应颜色），向四周扩散，持续 500ms
4. **Combo 更新**（100ms）：Combo 计数器 +1，数字弹跳放大 120% 后回弹

**音效反馈**：清脆的"唰"切割音（类似切水果音效），Combo ≥ 3 时叠加升调和弦

**分数变化**：
| Combo 状态 | 基础分 | 倍率 | 实际得分 |
|-----------|-------|------|---------|
| 无 Combo（首切） | 10 | ×1 | 10 |
| Combo 2 | 10 | ×1 | 10 |
| Combo 3-4 | 10 | ×1.5 | 15 |
| Combo 5-9 | 10 | ×2 | 20 |
| Combo 10-19 | 10 | ×3 | 30 |
| Combo 20+ | 10 | ×5 | 50 |

### 4.3 切割操作 — 切错

**用户动作**：滑动手指划过**非当前提示对应的汉字**卡片

**视觉反馈**：
1. **屏幕边缘红闪**（0ms）：屏幕四周边缘出现 Rose 红色径向渐变闪光，持续 300ms 淡出
2. **卡片震动**（0ms）：被切错的卡片原地左右震动 3 次（位移 ±6px），持续 200ms
3. **生命值减少**（100ms）：对应心形图标播放碎裂动画（心形裂开 → 碎片飞散 → 变灰），持续 400ms
4. **Combo 断裂**（0ms）：Combo 计数器显示 `Combo Break!`（Rose 红色），持续 500ms 后淡出，计数归零
5. **错误标注**（200ms）：卡片上方短暂显示正确答案（该卡片的拼音 + 释义），持续 1000ms

**音效反馈**：沉闷的"嗡"错误音 + 玻璃碎裂音（心碎）

**分数变化**：不扣分，但 Combo 归零

### 4.4 漏切

**触发**：卡片飞出屏幕底部且未被切割

**视觉反馈**：卡片落出屏幕时底部边缘出现短暂的白色涟漪波纹（持续 200ms）

**音效反馈**：无（不惩罚漏切）

**分数变化**：不扣分、不扣命

### 4.5 Combo 特效详情

| Combo 等级 | 触发条件 | 视觉特效 | 音效 |
|-----------|---------|---------|------|
| **3 连** | 连续切对 3 次 | 屏幕中央闪现 `GOOD!` 文字（Sky 蓝色，H1 32px），渐变放大后淡出 | 短促升调音效 |
| **5 连** | 连续切对 5 次 | 屏幕中央闪现 `GREAT!!` 文字（Amber 金色，Display 48px），带金色光芒粒子环绕 | 中等升调和弦 |
| **10 连** | 连续切对 10 次 | 全屏金色光芒爆发 + `AMAZING!!!` 文字（Rose 红+Amber 金渐变，Display 48px），带彩色粒子喷泉 | 华丽升调管弦乐短片段（1.5 秒） |
| **20 连** | 连续切对 20 次 | 全屏彩虹光环 + `LEGENDARY!!!!` 文字（彩虹渐变），背景短暂变为星空特效 | 史诗级管弦乐片段（2 秒） |

### 4.6 Game Over 流程

**触发**：3 条命全部耗尽（经典模式）或 倒计时归零（限时/PK/多人模式）

**视觉流程**：
1. 画面慢动作（0.3x 速度）持续 1 秒
2. 所有飞行中的卡片向下坠落消失
3. 屏幕中央浮现 `Game Over`（Display 48px，白色）或 `Time's Up!`（限时模式）
4. 持续 1.5 秒后自动过渡到游戏结算页（参见 `05-game-common/04-settlement.md`）

**音效**：低沉的鼓声 + 余韵回响

## 五、难度递增

### 5.1 经典模式难度曲线

| 时间段 | 卡片飞出速度 | 同时在屏卡片数 | 卡片间隔 | 干扰特征 |
|-------|------------|--------------|---------|---------|
| 0-30 秒 | 慢（vy: -600px/s） | 2 张 | 500ms | 无干扰，题目汉字与干扰汉字外形差异大 |
| 30-60 秒 | 中（vy: -700px/s） | 3 张 | 400ms | 开始出现同偏旁字（如 清/请/情） |
| 60-90 秒 | 快（vy: -800px/s） | 3-4 张 | 300ms | 形近字增多（如 大/太/犬） |
| 90-120 秒 | 很快（vy: -850px/s） | 4 张 | 250ms | 同声调干扰增多 |
| 120 秒+ | 极快（vy: -900px/s） | 4-5 张 | 200ms | 形近字 + 同声调 + 同偏旁综合干扰 |

### 5.2 限时模式难度

- 60 秒内速度固定为中等难度（不递增）
- 同时在屏卡片数固定 3 张
- 重点在于准确率和速度的平衡

### 5.3 PK/多人模式难度

- 速度固定为中等偏快
- 同时在屏 3-4 张卡片
- 难度不递增（确保公平性）
- 出题序列由服务端在匹配成功时生成，对所有参与者一致

## 六、生命值系统

### 6.1 生命值规则

| 规则 | 说明 |
|------|------|
| **初始生命** | 3 条命 |
| **扣命条件** | 切错汉字 -1 命 |
| **不扣命** | 漏切不扣命 |
| **不可恢复** | 经典模式中生命值不可恢复 |
| **Game Over** | 生命值归零时触发 Game Over |

### 6.2 生命值图标表现

**存活态**：
- 红色实心心形图标（28×28px）
- 每次切对时微微跳动一下（缩放 105%，持续 100ms）

**失去动画**（触发：切错扣命时）：
1. 心形从中间裂开（0ms，裂缝从中心向两侧扩展）
2. 碎片向两侧飞散（100ms，4-6 个心形碎片）
3. 碎片淡出，位置变为灰色空心心形轮廓（400ms）

**最后一命闪烁**：仅剩 1 条命时，该心形图标持续缓慢闪烁（每秒 1 次），提醒玩家注意

### 6.3 限时模式特殊处理

- 限时模式无生命值图标
- 切错仅扣 5 分，无生命值概念
- HUD 中生命值位置替换为「切对数/总切数」准确率显示

## 七、PK 模式特殊规则

### 7.1 出题同步

- 匹配成功后，服务端生成一条固定长度的出题序列（60 秒，约 80-100 题）
- 双方客户端接收相同序列，按相同时间节点飞出卡片
- 时间同步精度 < 50ms（基于服务端时间戳）

### 7.2 抢答机制

| 事件 | 处理 |
|------|------|
| **A 先切对** | A 得分 + 正确特效；B 端该卡片播放「已被对手切走」动画（卡片变灰缩小消失） + 短暂提示"对方得分 +10" |
| **B 先切对** | 同上，角色互换 |
| **同时切对**（时间差 < 50ms） | 双方均得分（不扣命、不扣分），视为双方都答对 |
| **A 切错** | 仅 A 扣命，不影响 B |
| **双方都漏切** | 卡片正常落出，无惩罚 |

### 7.3 对方得分时的反馈

- 对手每得分一次，己方屏幕右上角对手分数数字**跳动放大**（120%，持续 200ms）
- 对手连续 3 次抢先得分时，己方屏幕中央短暂闪现 `追上去！`（Sky 蓝色，Body 16px，持续 800ms）

### 7.4 PK 结束条件

1. **60 秒倒计时归零** → 分数高者胜
2. **一方 Game Over**（3 命耗尽）→ 另一方直接获胜（不等倒计时）
3. **双方同时 Game Over** → 分数高者胜
4. **分数相同** → 切对数多者胜 → 仍相同则平局

## 八、上瘾机制详情

### 8.1 Combo 特效

详见 §四 4.5（Combo 特效详情表格）。核心设计思路：
- Combo 越高，特效越华丽，分数倍率越高
- Combo 中断有明确的「Combo Break」反馈，激发重新连击欲望
- 高 Combo 时背景音乐节奏感增强（BPM 提升 10%）

### 8.2 个人最高分记录

- 经典模式和限时模式各自独立记录个人最高分
- 每次游戏结束时，若本局分数 > 历史最高分 → 结算页显示 `NEW RECORD!`（金色大字 + 金色粒子喷射）
- 个人最高分在游戏大厅对应卡片上展示

### 8.3 每日挑战模式

每日 00:00 UTC 刷新一个特殊限时模式（60 秒），规则有特殊变化：

| 星期 | 挑战主题 | 特殊规则 |
|------|---------|---------|
| 周一 | **形近字地狱** | 全部出形近字组合（大/太/犬、清/请/情） |
| 周二 | **同声调挑战** | 全部出同声调汉字，必须凭字形识别 |
| 周三 | **极速模式** | 卡片速度为普通模式的 1.5 倍 |
| 周四 | **反向切割** | 提示显示汉字，需要切对应的拼音卡片 |
| 周五 | **单手模式** | 同时在屏仅 1 张卡片，但速度极快 |
| 周六 | **混合大乱斗** | 随机混合以上规则 |
| 周日 | **轻松日** | 速度慢，Combo 加分翻倍，适合刷分 |

- 每日挑战有独立排行榜（当日全服排名）
- 参与每日挑战获得知语币奖励（参与 +5，前 10% +20，第 1 名 +50）

### 8.4 段位星数

- 参见通用段位系统（`05-game-common/05-rank-system.md`）
- G1 的 PK 和多人对战胜负影响统一段位
- 段位晋级时的庆祝动画叠加在游戏结算页之上

## 九、状态矩阵

### 9.1 游戏进行中状态

| 状态 | 描述 | 设计要求 |
|------|------|---------|
| **正常游戏** | 卡片飞出，玩家正在切割 | 所有 HUD 正常显示，提示条实时更新 |
| **Combo 激活** | Combo ≥ 3 | Combo 计数器高亮 + 对应等级特效 + 背景音乐节奏增强 |
| **最后一命** | 仅剩 1 条命 | 最后一颗心闪烁 + 屏幕边缘持续微红光晕 |
| **暂停** | 玩家点击暂停 | 参见 `08-hud-landscape.md` 暂停菜单 |
| **Game Over** | 3 命耗尽 | 慢动作 → Game Over 文字 → 过渡结算页 |
| **Time's Up** | 倒计时归零 | 钟声音效 → Time's Up 文字 → 过渡结算页 |

### 9.2 PK 模式状态

| 状态 | 描述 | 设计要求 |
|------|------|---------|
| **双方对战中** | 正常 PK 进行 | 双方 HUD 对称显示，抢答实时反馈 |
| **对手已出局** | 对方 3 命耗尽 | 己方屏幕显示 `对手已出局！` + 自动获胜 → 结算 |
| **己方已出局** | 自己 3 命耗尽 | Game Over 动画 → 可观战对手剩余时间 → 结算 |
| **对手断线** | 对手网络断开 | 等待 15 秒重连，超时则判己方获胜 |
| **己方断线** | 自己网络断开 | 参见 `08-hud-landscape.md` 断线处理 |

### 9.3 空状态 / 异常状态

| 状态 | 描述 | 设计要求 |
