# G6 汉字华容道（L6 · 滑块解谜）

> 关联文档：[游戏通用系统](../05-game-common/00-index.md) | [本轮索引](00-index.md)
> 通用 HUD 参见 `05-game-common/08-hud-landscape.md`，以下仅描述 G6 独特画面和交互。

## 一、游戏概述

| 属性 | 说明 |
|------|------|
| **一句话描述** | 滑动打乱的汉字方块，拼出正确的成语、句子或古诗句——步数越少越厉害 |
| **灵感来源** | 华容道 / 数字滑块谜题（15-puzzle） |
| **题库范围** | L6 课程的成语 150 个 + 缩句扩句 + 古诗句 + 综合语序练习，每题标注难度（简单/中等/困难） |
| **单局时长** | 1-3 分钟/题，可连续解题；极速模式固定 3 分钟 |
| **情绪目标** | 烧脑、满足感、挑战欲 |

## 二、游戏子模式

### 2.1 子模式总览

| 子模式 | 人数 | 影响段位 | 结束条件 | 特殊规则 |
|--------|------|---------|---------|---------|
| **单人·无尽** | 1 人 | 否 | 3 次超时（单题 120 秒）结束 | 连续解题，难度递增 |
| **单人·极速** | 1 人 | 否 | 3 分钟倒计时归零 | 3 分钟内解最多题 |
| **1v1 PK** | 2 人 | 是 | 双方完成同一题或倒计时归零 | 同题竞速 |
| **多人竞速** | 2-4 人 | 是 | 所有人完成或倒计时归零 | 同题竞速，排名计分 |

### 2.2 单人·无尽模式

- 系统随机出题，玩家解完一题自动出下一题
- 每题 120 秒时限，超时视为失败（扣一次机会）
- 3 次超时 → Game Over → 进入结算页
- 难度随题数递增：前 5 题 4×4 简单 → 第 6-10 题 4×4 中等 → 第 11 题起 5×5
- 记录连续解题数和总得分

### 2.3 单人·极速模式

- 3 分钟倒计时，连续解题
- 所有题目均为 4×4（保证速度节奏）
- 解完一题立刻出下一题，无超时单题限制
- 3 分钟结束时统计总解题数和总得分

### 2.4 1v1 PK 模式

- 双方看到**完全相同的题目**（相同的打乱状态）
- 谁先完成谁获胜（单题制，Bo3 三局两胜）
- 每题时限 90 秒，超时判双方本题平局
- 胜 +1 星，负 -1 星

### 2.5 多人竞速模式

- 2-4 人同题竞速
- 每题时限 90 秒
- 按完成顺序排名：第 1 名完成的分数最高
- Bo3 三局两胜，按总排名积分决定最终名次
- 第 1 名 +1 星，其余不扣星

## 四、核心交互

### 4.1 题目生成与展示

**触发**：进入游戏 / 上一题完成后

**题目生成规则**：
- 服务端从题库中抽取目标文本（成语/句子/古诗句）
- 将目标文本的每个字分配到棋盘格中（按阅读顺序：从左到右、从上到下）
- 随机空出 1 格作为空位
- 通过服务端执行的随机滑动打乱方块位置（确保有解且可计算最优步数）
- 将打乱后的棋盘状态和最优步数一并下发给客户端

**展示流程**（按时间顺序）：
1. **目标展示**（0ms）：左侧目标区先展示目标文字（淡入 300ms）
2. **棋盘入场**（300ms）：中间棋盘区方块依次从上到下淡入（每块间隔 30ms）
3. **打乱动画**（800ms）：方块从正确位置快速打乱到初始状态（自动播放打乱过程，持续 1000ms，速度 3x）
4. **操作提示**（1800ms）：短暂显示「滑动方块拼出正确答案」引导文字（Body 16px，白色 60%），持续 1.5 秒淡出
5. **倒计时启动**（2000ms）：底部倒计时开始运行，棋盘进入可操作状态

### 4.2 滑块操作 — 拖拽滑动

**操作方式**：玩家通过**拖拽**移动方块（非点击方向键）

**拖拽规则**：
- 仅与空位**水平或垂直相邻**的方块可被拖拽
- 拖拽方向仅限**朝向空位**的方向（水平或垂直）
- 不可对角线拖拽
- 不与空位相邻的方块拖拽无响应

**拖拽交互细节**：

| 阶段 | 交互表现 |
|------|---------|
| **按下** | 手指/鼠标按下方块 → 方块轻微抬起（阴影从 Level 1 升至 Level 2，缩放 102%），持续 100ms |
| **拖动** | 方块跟随手指/鼠标移动，仅在合法方向上滑动（限制在空位方向的单轴移动）；超出一格距离时不继续移动 |
| **释放（有效）** | 拖动距离超过方块尺寸 50% → 方块自动吸附到空位位置（弹性动画 200ms，easeOutBack），原位置变为新空位 |
| **释放（无效）** | 拖动距离不足 50% → 方块弹回原位（弹性动画 150ms） |

**快速滑动（Flick）**：
- 在方块上快速划过（速度 > 300px/s）→ 直接将方块滑到空位，无需拖满 50%
- 适合熟练玩家快速操作

**连续滑动**：
- 在同一行/列上连续快速划过多个方块时，沿划过方向的所有方块依次向空位方向滑动（类似推多米诺骨牌）
- 例如：空位在最左边，快速从右向左划过同行的 3 个方块 → 3 个方块依次向左移动一格

**音效**：方块滑动时发出轻柔的木块滑动音（"咔哒"），吸附到位时发出轻微的"咔"落位音

**步数计数**：每次有效滑动（方块成功移到空位）计为 1 步

### 4.3 滑块操作 — 桌面端键盘支持

- 桌面端额外支持方向键操作
- 按 ↑/↓/←/→ 键 → 将空位**对应方向**的方块滑入空位
- 例如：按 ← 键 → 空位右侧的方块向左滑入空位
- 键盘操作同样触发滑动音效和动画

### 4.4 提示功能

**触发**：点击右侧信息区的「提示」按钮

**规则**：每局限 3 次提示，用完后按钮变灰不可点击

**视觉反馈**：
1. 下一步应移动的方块**边框高亮**（Sky 蓝色发光边框 3px，脉冲动画持续 2 秒）
2. 该方块应滑向的空位方向显示微弱箭头引导（Sky 蓝色半透明箭头）
3. 提示剩余次数 -1
4. 2 秒后高亮和箭头自动淡出

**音效**：轻柔的提示音（"叮咚"）

### 4.5 解题成功

**判定规则**：所有方块都在正确位置（每个方块的汉字对应目标文本中的正确位置）

**视觉反馈**（按时间顺序）：
1. **方块齐亮**（0ms）：所有方块同时发出金色光芒（Amber 色边框发光），持续 300ms
2. **文字连线**（300ms）：方块间出现从左到右、从上到下的阅读顺序连线（Amber 色流光线条），汉字依次亮起
3. **成语/句子朗读**（500ms）：自动播放目标文字的朗读音频
4. **得分飘字**（600ms）：棋盘上方飘出得分（如 `+500`，Amber 色），上移 60px 后淡出
5. **最优解对比**（800ms）：右侧信息区显示最优步数对比：
   - 你的步数 ≤ 最优步数 → 显示 `完美！` 绿色 + 星形粒子
   - 你的步数 ≤ 最优步数 × 1.5 → 显示 `优秀！` Sky 蓝色
   - 其他 → 显示 `完成！` 白色
6. **下一题过渡**（2000ms）：棋盘方块向四周飞散淡出 → 新题目入场

**分数规则**：

| 评价 | 条件 | 基础分 | 步数奖励 | 速度奖励（<30s 完成） |
|------|------|-------|---------|-------------------|
| 完美 | 步数 ≤ 最优解 | 500 | +200 | +100 |
| 优秀 | 步数 ≤ 最优解 × 1.5 | 500 | +100 | +50 |
| 完成 | 步数 > 最优解 × 1.5 | 500 | 0 | +30 |

**使用提示的分数影响**：每使用 1 次提示扣 50 分（最低不低于基础分 500）

### 4.6 解题失败（超时）

**触发**：单题倒计时归零（无尽模式 120 秒 / PK 模式 90 秒）

**视觉反馈**：
1. **倒计时归零**（0ms）：进度条变 Rose 红 + 闪烁 3 次
2. **蜂鸣音**（0ms）：短促蜂鸣
3. **棋盘暗淡**（200ms）：棋盘整体降低至 40% 透明度
4. **正确答案展示**（400ms）：方块自动滑动到正确位置（自动演示最优解过程，速度 2x），伴随朗读音频
5. **失败处理**（2000ms）：
   - 无尽模式：扣一次机会（显示剩余机会 ×2 / ×1），进入下一题
   - 极速模式：不扣机会，直接进入下一题
   - PK 模式：本题判平（双方均未完成时）或对方获胜（对方已完成时）

## 五、PK 模式特殊规则

### 5.1 同题竞速机制

| 规则 | 说明 |
|------|------|
| **同题** | 双方看到完全相同的棋盘初始状态（相同的打乱方式） |
| **独立操作** | 各自在自己的棋盘上操作，互不影响 |
| **实时进度** | 对手的完成进度百分比实时同步显示（不显示具体棋盘状态，仅百分比） |
| **先完成获胜** | 一方先拼好正确答案 → 该题该方获胜 |
| **对手完成提示** | 对手完成时，己方屏幕顶部闪现 `对手已完成！`（Rose 红色，持续 1 秒） |

### 5.2 Bo3 三局两胜制

- 每局一道题，先完成者该局获胜
- 双方均超时则该局平局
- 先赢 2 局者获胜
- 三局结束后自动跳转结算页

### 5.3 局间结算

局间展示结算卡片（同 G5 回合间结算样式）：
- 本局结果（胜/负/平）
- 双方步数对比
- 双方用时对比
- 总比分
- 3 秒后进入下一局

## 六、上瘾机制详情

### 6.1 最少步数挑战

核心驱动力：追求最优解。

| 元素 | 说明 |
|------|------|
| **理论最优解** | 每题完成后展示理论最少步数（服务端预计算），与玩家实际步数对比 |
| **完美标记** | 步数 = 最优解 → 题目标记为「完美」，在历史记录中展示金色 ⭐ |
| **全完美挑战** | 连续 N 题全部完美完成 → 触发特殊成就 |
| **步数排行榜** | 每日一题（特殊超难题），全服排名按最少步数排列 |

### 6.2 解题速度排行榜

- 每日一题有全服速度排行榜
- 展示前 100 名的用时和步数
- 激发竞速欲望

### 6.3 每日一题

- 每日 00:00 UTC 发布一道特殊超难题（5×5 棋盘 + 复杂语序）
- 全服参与者共同挑战
- 按步数排名 → 步数相同按速度排名
- 前 3 名获得知语币奖励（第 1 名 +50，第 2 名 +30，第 3 名 +20）

### 6.4 难度递增的满足感

| 阶段 | 棋盘 | 题目类型 | 平均最优步数 |
|------|------|---------|------------|
| 入门（题 1-5） | 4×4 | 四字成语 | 6-10 步 |
| 进阶（题 6-10） | 4×4 | 四字成语（形近字干扰） | 10-15 步 |
| 挑战（题 11-15） | 5×5 | 五字/七字古诗句 | 15-25 步 |
| 大师（题 16+） | 5×5 | 长句+语序复杂 | 25-40 步 |

## 七、状态矩阵

### 7.1 游戏进行中状态

| 状态 | 描述 | 设计要求 |
|------|------|---------|
| **正常解题** | 方块可拖拽，玩家正在操作 | 棋盘可交互，所有信息面板正常显示 |
| **方块拖拽中** | 玩家正在拖拽某个方块 | 被拖拽方块抬起 + 阴影增强，跟随手指移动 |
| **方块吸附** | 方块滑入空位 | 弹性动画 + 落位音效 + 步数 +1 |
| **使用提示** | 玩家点击提示按钮 | 目标方块高亮 2 秒 + 方向箭头 |
| **解题成功** | 所有方块就位 | 金色齐亮 + 连线 + 朗读 + 得分 + 最优解对比 |
| **即将超时** | 倒计时 <15 秒 | 进度条变 Rose 红闪烁 + 催促音效（每 5 秒一次） |
| **超时失败** | 单题倒计时归零 | 棋盘暗淡 + 自动演示正确答案 |
| **暂停** | 玩家点击暂停（仅单人） | 参见 `08-hud-landscape.md` |
| **Game Over** | 无尽模式 3 次超时 | Game Over 文字 → 过渡结算页 |
| **Time's Up** | 极速模式 3 分钟归零 | 钟声 → Time's Up → 过渡结算页 |

### 7.2 PK 模式状态

| 状态 | 描述 | 设计要求 |
|------|------|---------|
| **双方竞速中** | 各自操作棋盘 | 对手进度条实时更新，己方棋盘正常交互 |
| **对手已完成** | 对手先拼好 | 顶部闪现 `对手已完成！` + 己方可继续操作至超时 |
| **己方已完成** | 己方先拼好 | 庆祝动画 + 等待对手完成/超时 |
| **局间结算** | 单局结束 | 结算卡片 + 3 秒后下一局 |
| **最终结算** | Bo3 结束 | 总结果 → 过渡结算页 |
| **对手断线** | 对手网络断开 | 等待 15 秒重连，超时判己方获胜 |

### 7.3 空状态 / 异常状态

| 状态 | 描述 | 设计要求 |
|------|------|---------|
| **题目加载中** | 等待服务端生成题目 | 棋盘区显示 Skeleton + 方块占位 + "生成中..." |
| **题目加载失败** | 网络/服务端异常 | 居中错误提示 + "重试" + "返回大厅" |
| **PK 匹配中** | 等待对手 | 参见 `05-game-common/03-matching.md` |
