## 一、游戏概述

| 属性 | 说明 |
|------|------|
| **一句话描述** | 阅读一篇短文，像侦探一样找出隐藏的 5 个谜题并解答——速度越快、正确越多，评级越高 |
| **灵感来源** | 推理解谜类游戏 |
| **题库范围** | L8 课程阅读材料（现代文 2000-3000 字）+ AI 生成扩展短文（300-800 字），每篇文章附带 5 个谜题（修辞手法识别、作者意图判断、逻辑漏洞发现、情感变化分析、文体特征辨别） |
| **单局时长** | 3-5 分钟（视阅读速度和推理难度） |
| **情绪目标** | 沉浸、烧脑、成就感、推理快感 |

## 二、游戏子模式

### 2.1 子模式总览

| 子模式 | 人数 | 影响段位 | 结束条件 | 特殊规则 |
|--------|------|---------|---------|---------|
| **单人·破案** | 1 人 | 否 | 5 个谜题全部作答完毕 或 时间耗尽（5 分钟） | 独立解谜，追求完美评级 |
| **1v1 PK** | 2 人 | 是 | 双方均完成 或 时间耗尽（5 分钟） | 同文章，比谁先找完且正确率高 |
| **多人竞速** | 2-4 人 | 是 | 所有人完成 或 时间耗尽（5 分钟） | 同文章，积分制排名 |
| **侦探排位** | 2 人 | 是 | 同 1v1 PK | 段位赛，赢升输降，匹配同段位对手 |

### 2.2 单人·破案模式

- 系统随机从题库中抽取一篇短文（300-800 字）+ 5 个预设谜题
- 玩家阅读文章后，逐个发现并解答谜题
- 5 分钟时限，全部答完或时间耗尽 → 进入结算
- 追求最高评级（SSS）和最快速度

### 2.3 1v1 PK 模式

- 双方阅读**完全相同的文章**和**完全相同的 5 个谜题**
- 各自独立阅读和答题
- 综合正确数 + 速度判定胜负
- 胜 +1 星，负 -1 星

### 2.4 多人竞速模式

- 2-4 人同文同题
- 综合正确数 + 速度积分排名
- 第 1 名 +1 星，其余不扣星

### 2.5 侦探排位模式

- 与 1v1 PK 规则相同
- 按段位匹配对手，段位赛专用模式
- 连续胜利可触发段位晋级赛

### 3.3 左侧·文章阅读区（1100px）

**整体面板**：
- 毛玻璃背景面板（`rgba(255,255,255,0.05)` + `backdrop-filter: blur(12px)`），圆角 16px
- 内 padding：32px（左右）、24px（上下）
- 纵向滚动（文章可能超出一屏高度）

**文章标题**：
- H2 字号（24px），白色，字重 700
- 标题下方分隔线（1px，白色 10% 透明度）

**文章正文**：
- Body L 字号（18px），白色 80%，行高 1.8（宽松阅读）
- 段落间距 24px
- 重点词汇可点击（下划线虚线 + 白色 60%），点击后弹出浮层释义（Body S 14px，毛玻璃浮层）

**线索区域标记**（文章中隐藏的谜题线索位置）：

| 状态 | 视觉标记 | 说明 |
|------|---------|------|
| **未发现** | 无任何标记，与正文完全相同 | 玩家需要通过阅读自行发现 |
| **可发现（提示后）** | 文字下方出现微弱的 Amber 色波浪下划线（2px，40% 透明度） | 使用提示后出现，引导玩家注意此处 |
| **已发现** | 文字高亮背景（`rgba(217,119,6,0.15)`，Amber 色半透明高亮） + 右侧出现放大镜图标 🔍（16px） | 玩家点击该区域后标记为已发现 |
| **已解答** | 文字高亮背景变为绿色（`rgba(34,197,94,0.15)`）+ 右侧勾号 ✅ | 谜题已被正确解答 |
| **解答错误** | 文字高亮背景变为 Rose 红色（`rgba(225,29,72,0.15)`）+ 右侧叉号 ❌ | 谜题解答错误 |

**线索发现交互**：
- 玩家在阅读过程中，点击/长按自己认为有谜题的段落或句子
- 如果点击的位置**正好是**某个谜题线索区域 → 触发「发现线索」
- 如果点击的位置**不是**线索区域 → 无响应（不惩罚错误点击，鼓励探索）
- 每个线索区域的点击判定范围为该句/该段落（非精确到单字，给予一定容差）

- 同时在提示内容区显示文字提示（如 `注意第三段中比喻手法的使用`）
- 使用提示会影响最终评级（每用 1 次降低评级上限 1 级）

## 四、核心交互

### 4.1 案件加载

**触发**：进入游戏

**视觉流程**：
1. **案件卷宗入场**（0ms）：屏幕中央出现一个「卷宗文件夹」图形（侦探办公室风格，250×300px），从下方滑入
2. **卷宗打开**（500ms）：文件夹打开动画（翻开封面），露出文章标题
3. **文字展开**（800ms）：文章内容从标题开始逐渐展开显示（非打字机效果，而是从上到下淡入展开），持续 1000ms
4. **谜题面板入场**（1200ms）：右侧谜题面板从右侧滑入，5 个谜题全部显示为 🔒 未发现
5. **操作提示**（1800ms）：底部短暂显示引导文字 `仔细阅读文章，点击可疑之处发现线索`（Body 16px，白色 60%），持续 2 秒淡出
6. **倒计时启动**（2000ms）：5 分钟倒计时开始

### 4.2 阅读文章

**基本阅读**：
- 玩家在左侧文章区纵向滚动阅读
- 支持手指/鼠标滚动，滚动流畅
- 重点词汇（下划线虚线标记）可点击查看释义浮层：
  - 点击 → 浮层从词汇下方弹出（毛玻璃面板，160×80px）
  - 内容：汉字 + 拼音 + 简短释义
  - 点击浮层外任意区域关闭
  - 查看释义不消耗时间也不影响评级

### 4.3 发现线索

**用户动作**：点击/长按文章中自认为有谜题的段落或句子

**点击判定**：
- 每个谜题对应文章中一个「线索区域」（可能是一个句子、一个词语、或一个段落）
- 点击判定范围为线索区域所在的完整句子（给予容差）
- 点击非线索区域 → 无任何响应（不惩罚，鼓励探索）
- 点击线索区域 → 触发「发现线索」

**发现线索视觉反馈**（按时间顺序）：
1. **高亮动画**（0ms）：线索区域文字背景从透明变为 Amber 色半透明高亮（`rgba(217,119,6,0.15)`），300ms 渐变
2. **放大镜图标**（100ms）：线索区域右侧弹出放大镜 🔍 图标（16px），轻微弹跳入场
3. **发现提示音**（0ms）：清脆的「发现」音效（类似探测器蜂鸣声）
4. **右侧谜题更新**（200ms）：对应谜题从 🔒 变为 🔍，序号圆形背景变 Amber，文字变白 80%
5. **谜题弹窗出现**（400ms）：从发现位置弹出谜题作答弹窗（详见 §4.4）

### 4.4 解答谜题

**谜题弹窗**：发现线索后自动弹出谜题作答弹窗

**弹窗视觉**：
- 尺寸：600×400px，居中偏右（不遮挡左侧文章阅读区的大部分内容）
- 背景：毛玻璃面板（`rgba(20,20,30,0.85)` + `backdrop-filter: blur(16px)`），圆角 20px
- 顶部：谜题类型标签（如 `修辞手法识别`，Caption 12px，Amber 背景标签）
- 关闭按钮：右上角 × 图标（可暂时关闭弹窗，稍后从右侧谜题列表点击 🔍 重新打开）

**弹窗内容结构**：

```
┌─────────────────────────────────────────────────┐
│  [修辞手法识别]                              ✕   │
│                                                  │
│  线索原文：                                       │
│  「那些泛黄的书页仿佛在低声诉说着往事」           │
│                                                  │
│  问题：这句话使用了什么修辞手法？                  │
│                                                  │
│  ○ A. 比喻                                       │
│  ● B. 拟人  ← 选中态                             │
│  ○ C. 夸张                                       │
│  ○ D. 排比                                       │
│                                                  │
│           [提交答案]                              │
└─────────────────────────────────────────────────┘
```

**谜题类型与问题格式**：

| 谜题类型 | 问题形式 | 选项数量 |
|---------|---------|---------|
| **修辞手法识别** | 「这句话使用了什么修辞手法？」 | 4 选 1 |
| **作者意图判断** | 「作者在第 X 段中主要想表达什么？」 | 4 选 1 |
| **逻辑漏洞发现** | 「文中哪处存在逻辑不一致？」 | 4 选 1（选项为文中句子） |
| **情感变化分析** | 「从第 X 段到第 Y 段，作者的情感如何变化？」 | 4 选 1（选项为情感变化描述） |
| **文体特征辨别** | 「本文属于什么文体？它最显著的文体特征是？」 | 4 选 1 |

**选项交互**：
- 点击选项 → 选项前圆点变为实心填充（Sky 蓝色）+ 选项行背景微亮
- 可更换选择（点击其他选项切换）
- 选中任一选项后「提交答案」按钮从灰色变为 Rose 红可点击态

### 4.5 谜题解答 — 正确

**判定**：提交的答案与服务端预设正确答案一致

**视觉反馈**：
1. **选项变绿**（0ms）：正确选项背景变为绿色半透明（`rgba(34,197,94,0.2)`），前方显示 ✅
2. **弹窗标题变绿**（100ms）：弹窗顶部谜题类型标签变绿色
3. **解析展示**（300ms）：弹窗下方展开解析区域（淡入 300ms）：
   - 「正确！」绿色 H3 20px
   - 解析文字（Body S 14px，白色 60%，2-3 行解释为什么这是正确答案）
4. **得分飘字**（400ms）：弹窗上方飘出 `+200`（Amber 色）
5. **文章中标记更新**（500ms）：线索区域高亮从 Amber 变绿色
6. **右侧谜题更新**（500ms）：对应谜题从 🔍 变 ✅，序号背景变绿
7. **弹窗自动关闭**（2000ms）：弹窗向上缩小淡出

**音效**：清脆的正确确认音 + 轻快的短旋律

**分数规则**：

| 正确回答 | 基础分 | 速度奖励（发现后 <10s 答对） | 无提示奖励 |
|---------|-------|-------------------------|----------|
| 每题 | 200 | +50 | +30（未使用过提示） |

### 4.6 谜题解答 — 错误

**判定**：提交的答案与正确答案不一致

**视觉反馈**：
1. **所选选项变红**（0ms）：错误选项背景变为 Rose 红半透明，前方显示 ❌
2. **正确选项标记**（200ms）：正确选项背景变绿色 + ✅（让玩家知道正确答案）
3. **弹窗震动**（0ms）：弹窗整体左右震动 2 次
4. **解析展示**（300ms）：弹窗下方展开解析区域：
   - 「可惜，答错了」Rose 红色 Body 16px
   - 正确答案：`正确答案是 B. 拟人`
   - 解析文字（2-3 行解释）
5. **文章中标记更新**（500ms）：线索区域高亮变 Rose 红色
6. **右侧谜题更新**（500ms）：对应谜题从 🔍 变 ❌
7. **弹窗自动关闭**（3000ms）：弹窗淡出（比正确时多停留 1 秒，给玩家看解析的时间）

**音效**：沉闷的错误音

**分数影响**：答错不扣分，但该题得 0 分（影响总评级）

### 4.7 全部谜题完成

**触发**：5 个谜题全部作答完毕（无论对错）

**视觉流程**：
1. **画面暗淡**（0ms）：全屏覆盖半透明暗层
2. **案件结案动画**（300ms）：屏幕中央出现「案件报告」卷宗（400×500px），从下方滑入
3. **评级揭晓**（800ms）：卷宗上方大号显示评级字母（如 `S`，Display 64px，评级颜色），伴随光芒特效
4. **详细数据**（1200ms）：卷宗上逐行展示：
   - 正确数：`4 / 5`
   - 用时：`3:21`
   - 使用提示：`1 次`
   - 总得分：`850`
5. **停留 3 秒**后自动过渡到游戏结算页

## 五、案件评级系统

### 5.1 评级规则

综合正确数、速度和提示使用次数评定：

| 评级 | 图标 | 条件 | 评级颜色 |
|------|------|------|---------|
| **SSS** | 🏆 | 5/5 正确 + 用时 <2 分钟 + 未使用提示 | 彩虹渐变 |
| **SS** | ⭐ | 5/5 正确 + 用时 <3 分钟 | Amber 金色 |
| **S** | 💎 | 5/5 正确 或 4/5 正确 + 用时 <2 分钟 | Sky 蓝色 |
| **A** | 🔷 | 4/5 正确 或 3/5 正确 + 用时 <2.5 分钟 | 绿色 |
| **B** | 🔹 | 3/5 正确 或以下 | 白色 |

### 5.2 评级视觉特效

| 评级 | 揭晓特效 |
|------|---------|
| **B** | 评级字母淡入，无额外特效 |
| **A** | 评级字母弹跳入场 + 绿色光芒 |
| **S** | 评级字母旋转入场 + Sky 蓝色粒子爆发 |
| **SS** | 评级字母放大入场 + Amber 金色光芒 + 星形粒子喷射 |
| **SSS** | 全屏彩虹光芒爆发 + 评级字母彩虹渐变 + 大量金色粒子 + 史诗级音效（2 秒） |

### 5.3 提示对评级的影响

| 使用提示次数 | 评级上限 |
|------------|---------|
| 0 次 | SSS（无限制） |
| 1 次 | SS（最高 SS） |
| 2 次 | S（最高 S） |
| 3 次 | A（最高 A） |

## 七、上瘾机制详情

### 7.1 案件评级追求

- 核心驱动力：追求 SSS 评级
- 每篇文章的最高评级记录保存
- 已获得 S 评级的文章可重新挑战追求 SS/SSS
- 评级分布统计（B 几篇 / A 几篇 / S 几篇 / SS 几篇 / SSS 几篇）

### 7.2 破案手记收集

- 每次完成案件后，该篇文章的**解析笔记**自动收入「破案手记」
- 破案手记按文体分类（记叙文 / 说明文 / 议论文 / 散文）
- 每类文体集齐一定数量的解析笔记可获得特殊成就
- 玩家可随时回顾已收集的破案手记，复习修辞手法和分析方法

### 7.3 每日特案

- 每日 00:00 UTC 发布一篇「特别案件」（更长/更难的文章）
- 全服参与，按评级+速度排名
- 前 3 名获得知语币奖励（第 1 名 +50，第 2 名 +30，第 3 名 +20）
- 「特别案件」标记金色边框，区别于普通案件

### 7.4 侦探排位的竞争感

- 排位赛匹配同段位对手，胜负直接影响段位
- 段位晋级有庆祝动画，降级有提醒
- 赛季结算奖励激发长期活跃

## 八、状态矩阵

### 8.1 游戏进行中状态

| 状态 | 描述 | 设计要求 |
|------|------|---------|
| **阅读中** | 玩家滚动阅读文章 | 文章区可滚动，谜题面板全部 🔒，底部倒计时运行 |
| **发现线索** | 玩家点击线索区域 | 高亮动画 + 放大镜图标 + 发现音效 + 弹出谜题弹窗 |
| **解答谜题** | 谜题弹窗打开，等待玩家选择答案 | 弹窗覆盖部分画面，左侧文章仍可参考（弹窗不完全遮挡） |
| **解答正确** | 玩家答对 | 选项变绿 + 解析展示 + 得分 + 弹窗自动关闭 |
| **解答错误** | 玩家答错 | 选项变红 + 正确答案标记 + 解析展示 + 弹窗自动关闭 |
| **使用提示** | 点击获取提示按钮 | 文章中目标区域出现 Amber 波浪下划线 + 文字提示 |
| **即将超时** | 倒计时 <60 秒 | 进度条变 Rose 红闪烁 + 催促音效（每 15 秒一次） |
| **全部完成** | 5 个谜题全部作答 | 案件结案动画 + 评级揭晓 |
| **时间耗尽** | 5 分钟归零 | 未答完的谜题自动标记为未解答（得 0 分）→ 结案动画 |
| **暂停** | 仅单人模式可暂停 | 参见 `08-hud-landscape.md` |

### 8.2 PK 模式状态

| 状态 | 描述 | 设计要求 |
|------|------|---------|
| **双方阅读中** | 各自独立阅读和答题 | 对手进度实时同步显示 |
| **对手进度领先** | 对手解答谜题数 > 己方 | 对手进度数字 Rose 红色闪烁（轻微压力） |
| **对手已完成** | 对手 5 题全部作答 | 顶部闪现 `对手已完成破案！` + 己方可继续直到超时 |
| **己方已完成** | 己方先完成 | 等待对手完成或超时 → 结算 |
| **双方均完成** | 双方都作答完毕 | 直接进入对比结算 |
| **对手断线** | 对手网络断开 | 等待 15 秒重连，超时判己方获胜 |

### 8.3 PK 胜负判定

| 优先级 | 判定依据 | 说明 |
|--------|---------|------|
| 1 | 正确数 | 正确数多者胜 |
| 2 | 用时 | 正确数相同时，用时少者胜 |
| 3 | 提示使用 | 仍相同时，使用提示少者胜 |
| 4 | 平局 | 以上全部相同则平局 |

### 8.4 空状态 / 异常状态

| 状态 | 描述 | 设计要求 |
|------|------|---------|
| **案件加载中** | 等待服务端下发文章和谜题 | 左侧文章区 Skeleton + 右侧谜题面板占位 + "案件调配中..." |
| **案件加载失败** | 网络/服务端异常 | 居中错误提示 + "重试" + "返回大厅" |
| **PK 匹配中** | 等待对手 | 参见 `05-game-common/03-matching.md` |
