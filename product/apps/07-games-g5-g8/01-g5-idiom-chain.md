## 一、游戏概述

| 属性 | 说明 |
|------|------|
| **一句话描述** | 回合制成语接龙，用最后一个字接出新成语——接不上就输，用生僻成语额外加分 |
| **灵感来源** | 文字类棋牌对战 |
| **题库范围** | L5 课程 100+ 成语 + 扩展成语库（含释义、出处、例句），每个成语标注常见度（常见 / 进阶 / 生僻） |
| **单局时长** | 2-5 分钟（无尽模式视玩家水平而定；限时模式固定 60 秒；PK 三回合制约 3-5 分钟） |
| **情绪目标** | 紧张、博弈、成就感、知识积累 |

## 二、游戏子模式

### 2.1 子模式总览

| 子模式 | 人数 | 影响段位 | 结束条件 | 特殊规则 |
|--------|------|---------|---------|---------|
| **单人·无尽** | 1 人 | 否 | 玩家接不上（倒计时归零且无法从提示中选择） | 与 AI 无限接龙，记录最长链 |
| **单人·限时** | 1 人 | 否 | 60 秒倒计时归零 | 60 秒内接出最多成语 |
| **1v1 PK** | 2 人 | 是 | 三回合制，每回合一方接不上则判负 | 真人对战，回合交替 |
| **多人车轮战** | 3-6 人 | 是 | 淘汰到最后一人 | 轮流接龙，接不上即淘汰 |

### 2.2 单人·无尽模式

- 系统给出一个起始成语（随机从题库中抽取常见成语）
- 玩家需要在 10 秒内用**尾字同字**或**尾字同音字**接出新成语
- 接出后系统 AI 在 1-3 秒内自动接龙（AI 难度自适应）
- 如此交替进行，直到玩家接不上
- 记录最长连续接龙数（个人最高记录）
- AI 回合时玩家可预判准备

### 2.3 单人·限时模式

- 60 秒倒计时，系统给出起始成语
- 玩家单独接龙（无 AI 对手），每接出一个成语立刻给出下一个尾字
- 接龙速度越快分越高（每个成语基础 100 分 + 速度奖励）
- 60 秒结束时统计总得分和接龙数

### 2.4 1v1 PK 模式

- 三回合制，每回合独立起始成语
- 双方交替接龙，每人每次 10 秒倒计时
- 一方接不上（倒计时归零且未选择/输入有效成语）→ 本回合判负
- 三回合胜者（先赢 2 回合）获胜
- 胜 +1 星，负 -1 星

### 2.5 多人车轮战模式

- 3-6 人参加，系统给出起始成语
- 按匹配顺序轮流接龙，每人 10 秒倒计时
- 接不上即淘汰出局（可继续观战）
- 最后一人存活获胜
- 排名规则：第 1 名 +1 星，第 2 名不变，其余 -1 星

## 四、核心交互

### 4.1 系统给出起始成语

**触发**：游戏开始 / 新回合开始（PK 模式）

**视觉流程**（按时间顺序）：
1. **成语卡牌入场**（0ms）：第一张成语卡牌从画面中央放大入场（从 50% 缩放到 100%），持续 500ms
2. **成语朗读**（200ms）：自动播放该成语的朗读音频（如「一马当先」）
3. **释义浮层**（500ms）：卡牌下方弹出释义浮层（背景 `rgba(0,0,0,0.7)` + blur 12px，Body S 14px，白色），展示成语释义 + 出处，持续 2 秒后淡出
4. **尾字高亮**（1000ms）：卡牌中的尾字变为 Rose 红色 + 放大 120%，持续 300ms 后恢复
5. **操作区激活**（1500ms）：底部操作区从灰色变为可操作状态，输入框获取焦点，10 秒倒计时启动
6. **提示面板加载**（1500ms）：右侧联想提示面板加载以尾字/同音字开头的可用成语列表

### 4.2 玩家输入接龙 — 键盘输入方式

**用户动作**：在输入框中打字输入成语

**输入过程**：
1. 输入第 1 个字 → 联想面板实时过滤，显示所有以该字开头的四字成语
2. 输入第 2 个字 → 联想面板进一步缩小范围
3. 输入第 3 个字 → 联想面板通常仅剩 1-2 个候选
4. 输入第 4 个字 → 如果匹配到有效成语，「确认」按钮变为高亮 Rose 红（可点击状态）
5. 点击「确认」或按回车键 → 提交

**键盘辅助**：
- 移动端弹出中文输入法键盘，支持拼音输入
- 桌面端支持直接打字 + 回车提交
- 输入过程中联想面板高亮匹配文字（已输入的部分加粗）

### 4.3 玩家输入接龙 — 联想提示选择方式

**用户动作**：直接点击右侧联想提示面板中的成语

**交互流程**：
1. 玩家浏览联想提示列表
2. 点击目标成语 → 成语自动填入输入框（无需手动输入）
3. 自动触发提交（点击即提交，不需二次确认）
4. 如果成语有效 → 进入「接龙成功」流程
5. 选择生僻成语（带 ⭐ 标签） → 额外加分

### 4.4 接龙成功

**判定规则**：
- 提交的成语必须以当前尾字**同字**或**同音字**开头
- 提交的成语必须存在于成语库中（服务端校验）
- 提交的成语在本局中未被使用过

**视觉反馈**（按时间顺序）：
1. **输入框绿色闪光**（0ms）：输入框边框变为绿色 `#22c55e`，持续 300ms
2. **新卡牌入场**（100ms）：新成语卡牌从底部操作区飞向链条末端，滑入到位 + 轻微弹跳，持续 400ms
3. **箭头连接**（300ms）：新卡牌与前一张之间自动绘制箭头连接线 + 接龙关键字高亮
4. **得分飘字**（200ms）：在新卡牌上方飘出得分（如 `+100`，Amber 色，H3 20px），上移 60px 后淡出
5. **链长度更新**（300ms）：顶部链长度计数器 +1，数字弹跳放大 120% 后回弹
6. **释义浮层**（500ms）：新成语的释义浮层自动展示 1.5 秒（帮助玩家学习）
7. **操作区重置**（600ms）：输入框清空，尾字更新为新成语的尾字，联想面板刷新，新一轮 10 秒倒计时启动

**生僻成语额外反馈**：
- 飘字变为 `+150 博学!`（Amber 金色 + 星形粒子特效）
- 卡牌顶部出现 ⭐ 标记
- 音效叠加清脆的铃铛音

**音效反馈**：清脆的"叮"确认音 + 成语朗读音频

**分数规则**：

| 成语类型 | 基础分 | 速度奖励（<3s 提交） | 合计 |
|---------|-------|-------------------|------|
| 常见成语 | 100 | +20 | 100-120 |
| 进阶成语 | 120 | +30 | 120-150 |
| 生僻成语 | 150 | +50 | 150-200 |

### 4.5 接龙失败 — 提交无效成语

**触发**：提交的成语不满足接龙规则（非同字/同音开头、不存在、已使用过）

**视觉反馈**：
1. **输入框震动**（0ms）：输入框左右震动 3 次（位移 ±6px），边框变 Rose 红色，持续 300ms
2. **错误提示**（50ms）：输入框下方显示具体错误原因，Rose 红色 Body S 14px：
   - 「成语不存在，请重新输入」
   - 「必须以"X"字或同音字开头」
   - 「该成语已被使用」
3. **倒计时继续**（不暂停）：提交失败不暂停倒计时，玩家需快速修改

**音效反馈**：短促的"嗡"错误音

### 4.6 接龙失败 — 倒计时归零

**触发**：10 秒倒计时归零且玩家未提交有效成语

**视觉反馈**（按时间顺序）：
1. **倒计时归零**（0ms）：进度条填充色变为 Rose 红 + 闪烁 3 次
2. **蜂鸣音**（0ms）：短促的蜂鸣提示音
3. **链条断裂**（200ms）：链条末端的 `?` 占位卡牌变为碎裂动画（裂开 + 碎片飞散），Rose 红色
4. **判负/扣分**（500ms）：
   - 无尽模式：显示 `接不上了！` → Game Over → 进入结算页
   - 限时模式：扣 50 分 + 系统自动给出新起始成语 → 继续游戏
   - PK 模式：本回合判负 → 回合结算动画
   - 车轮战：玩家淘汰 → 淘汰动画 + 可观战
5. **正确答案展示**（800ms）：短暂展示「其实可以接：XXX」（灰色浮层，持续 2 秒），帮助玩家学习

## 五、PK 模式特殊规则

### 5.1 三回合制流程

```
第 1 回合：系统给出起始成语 A → 双方交替接龙 → 一方接不上 → 判负
         ↓
第 2 回合：系统给出起始成语 B → 双方交替接龙 → 一方接不上 → 判负
         ↓
（若前两回合各胜一局）
第 3 回合：系统给出起始成语 C → 双方交替接龙 → 一方接不上 → 判负
         ↓
三回合结果汇总 → 先赢 2 回合者获胜
```

### 5.2 回合交替机制

| 事件 | 处理 |
|------|------|
| **己方回合** | 操作区可用，输入框获取焦点，10 秒倒计时启动 |
| **对方回合** | 操作区灰色半透明遮罩（opacity 40%）+ 中央显示 `等待对方` 文字（白色 Body 16px，缓慢呼吸闪烁） |
| **对方接龙成功** | 新卡牌加入链条 + 对方得分显示 + 轮到己方（操作区解除遮罩） |
| **对方接龙失败** | 链条断裂动画 + 「对手接不上了！」提示 + 本回合己方获胜 |
| **己方接龙成功** | 新卡牌加入 + 得分 + 轮到对方（操作区加遮罩） |
| **己方接龙失败** | 链条断裂 + 本回合己方判负 |

### 5.3 回合间结算

每回合结束后，画面中央展示回合结算卡片（毛玻璃面板 400×200px）：

| 显示内容 | 视觉规范 |
|---------|---------|
| 回合结果 | 「第 X 回合」H2 24px + 「你赢了！」（绿色）或「你输了」（Rose 红） |
| 本回合链长 | 「链长度 ×8」Amber 色 Body 16px |
| 比分 | 「1 : 0」H1 32px，己方绿色/对方 Rose 红 |
| 倒计时 | 「下一回合 3 秒后开始」Caption 12px |

### 5.4 PK 最终结算

三回合结束后 → 自动跳转游戏结算页（`05-game-common/04-settlement.md`），额外展示：
- 三回合比分（如 2:1）
- 本局最长单回合链长度
- 本局使用的生僻成语列表
- 段位星数变化

### 5.5 多人车轮战特殊规则

| 规则 | 说明 |
|------|------|
| **轮流顺序** | 按匹配进入顺序排列，固定轮转 |
| **当前操作者** | 头像高亮 + 操作区仅该玩家可用 |
| **淘汰** | 接不上即淘汰，头像变灰 + 「已淘汰」标记 |
| **观战** | 被淘汰后可继续观战，但不可操作 |
| **跳过已淘汰** | 轮到已淘汰玩家时自动跳过 |
| **最终排名** | 按淘汰顺序反序排名（最后存活 = 第 1 名） |

## 六、上瘾机制详情

### 6.1 生僻成语额外加分

核心差异化设计：鼓励玩家学习和使用生僻成语。

| 成语等级 | 判定标准 | 基础分 | 额外反馈 |
|---------|---------|-------|---------|
| **常见** | 题库中标记为常见（如一马当先、守株待兔） | 100 | 无额外 |
| **进阶** | 题库中标记为进阶（如殚精竭虑、韦编三绝） | 120 | Sky 蓝色 `进阶！` 飘字 |
| **生僻** | 题库中标记为生僻（如济人利物、赤绳系足） | 150 | Amber 金色 `博学！` 飘字 + ⭐ + 铃铛音 |

### 6.2 成语释义自动展示

- 每个成语接龙成功后，释义浮层自动展示 1.5 秒
- 浮层包含：成语、拼音、释义、出处（如有）
- 玩家在游戏中「不知不觉学到了成语」的体验
- 结算页展示「本局学到的成语」列表

### 6.3 每日「成语之星」排名

- 每日 00:00 UTC 重置排名
- 统计当日所有模式的总接龙数
- 前 3 名展示在游戏大厅 G5 卡片上
- 第 1 名获得「今日成语之星」头衔（当日有效）

### 6.4 段位与专属称号

- 通用段位系统（青铜→王者）通过 PK/车轮战胜负影响
- 此外有**成语接龙专属称号**（与通用段位并行展示）：

| 称号 | 获取条件 |
|------|---------|
| 成语新手 | 累计接龙 50 次 |
| 成语达人 | 累计接龙 200 次 |
| 成语高手 | 累计接龙 500 次 + 使用过 20 个生僻成语 |
| 成语大师 | 累计接龙 1000 次 + 使用过 50 个生僻成语 |
| 成语宗师 | 单局无尽模式链长 ≥ 30 + 累计接龙 2000 次 |

## 七、状态矩阵

### 7.1 游戏进行中状态

| 状态 | 描述 | 设计要求 |
|------|------|---------|
| **正常游戏（己方回合）** | 操作区可用，等待玩家输入 | 输入框获取焦点，联想面板显示候选，倒计时运行 |
| **正常游戏（对方回合）** | 等待对方操作 | 操作区灰色遮罩 + 「等待对方」呼吸闪烁，链条末端 `?` 闪烁 |
| **输入中** | 玩家正在输入成语 | 联想面板实时过滤，输入框显示输入光标 |
| **倒计时紧迫** | 倒计时 <3 秒 | 进度条变 Rose 红闪烁 + 心跳音效 + 秒数放大 |
| **接龙成功** | 有效成语已提交 | 新卡牌入场动画 + 得分 + 释义浮层 |
| **接龙失败（无效）** | 提交了无效成语 | 输入框震动 + 错误提示 + 倒计时继续 |
| **接龙失败（超时）** | 倒计时归零 | 链条断裂 + 蜂鸣音 + 判负/扣分 |
| **暂停** | 玩家点击暂停（仅单人） | 参见 `08-hud-landscape.md` 暂停菜单 |
| **Game Over** | 无尽模式接不上 | 链条断裂 → Game Over 文字 → 过渡结算页 |
| **Time's Up** | 限时模式 60 秒归零 | 钟声 → Time's Up 文字 → 过渡结算页 |

### 7.2 PK 模式状态

| 状态 | 描述 | 设计要求 |
|------|------|---------|
| **双方对战中** | 回合交替进行 | 当前操作方高亮，另一方灰色遮罩 |
| **回合结算** | 一方接不上，本回合结束 | 回合结算卡片弹出 + 3 秒后进入下一回合 |
| **最终结算** | 三回合结束 | 汇总比分 → 过渡到游戏结算页 |
| **对手断线** | 对手网络断开 | 等待 15 秒重连，超时判己方获胜 |
| **己方断线** | 自己网络断开 | 参见 `08-hud-landscape.md` 断线处理 |

### 7.3 车轮战模式状态

| 状态 | 描述 | 设计要求 |
|------|------|---------|
| **轮到己方** | 操作区可用 | 己方头像高亮 + 操作区激活 |
| **轮到他人** | 等待其他玩家 | 操作区遮罩 + 当前操作者头像高亮 |
| **己方淘汰** | 己方接不上 | 淘汰动画 + 进入观战模式（仅观看链条和其他玩家操作） |
| **他人淘汰** | 其他玩家接不上 | 该玩家头像变灰 + 「已淘汰」标记 |
| **仅剩一人** | 最后存活者 | 自动获胜 → 胜利动画 → 结算页 |

### 7.4 空状态 / 异常状态

| 状态 | 描述 | 设计要求 |
|------|------|---------|
| **题库加载中** | 进入游戏，等待服务端下发起始成语 | 游戏区显示 Skeleton 加载动画 + "准备中..." 文字 |
| **题库加载失败** | 网络错误或服务端异常 | 居中错误提示 + "重试" 按钮 + "返回大厅" 按钮 |
| **联想提示为空** | 当前尾字无可接成语 | 联想面板显示空状态图标 + 「暂无可接成语，试试输入同音字开头的？」 |
| **PK 匹配中** | 等待对手 | 参见 `05-game-common/03-matching.md` |
