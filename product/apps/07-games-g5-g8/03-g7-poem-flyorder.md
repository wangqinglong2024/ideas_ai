## 一、游戏概述

| 属性 | 说明 |
|------|------|
| **一句话描述** | 系统给出关键字，双方轮流说出含该字的古诗句——说不出就输，冷门诗句额外加分 |
| **灵感来源** | 中国诗词大会·飞花令 |
| **题库范围** | L7 课程累计 76 首古诗词（含全文、作者、朝代、注释、赏析），关键字覆盖高频诗词用字（如「月」「花」「春」「风」「山」「水」「云」「雨」等） |
| **单局时长** | 2-5 分钟（取决于双方储备量） |
| **情绪目标** | 文雅、紧张、对弈感、文化自豪 |

## 二、游戏子模式

### 2.1 子模式总览

| 子模式 | 人数 | 影响段位 | 结束条件 | 特殊规则 |
|--------|------|---------|---------|---------|
| **单人·练习** | 1 人 | 否 | 玩家答不出（倒计时归零） | vs AI，AI 难度可调（初级/中级/高级） |
| **1v1 对决** | 2 人 | 是 | 一方答不出 | 真人飞花令，严格规则 |
| **多人淘汰** | 4-8 人 | 是 | 淘汰到最后一人 | 轮流作答，答不出即淘汰 |
| **主题挑战** | 不限 | 否 | 个人最长连续作答数 | 每日一个特殊关键字，全服参与排名 |

### 2.2 单人·练习模式

- 系统随机给出关键字，玩家与 AI 轮流对弈
- 每轮 15 秒倒计时
- AI 难度可选：
  - **初级**：AI 仅用题库内常见诗句，响应慢（5-8 秒）
  - **中级**：AI 使用题库内全部诗句，响应中等（3-5 秒）
  - **高级**：AI 使用扩展诗库，响应快（1-3 秒），偶尔使用冷门诗句
- 玩家答不出 → Game Over → 结算页展示本局对弈诗句列表

### 2.3 1v1 对决模式

- 系统随机给出关键字
- 双方轮流作答，每轮 15 秒
- 严格规则：诗句必须包含关键字 + 不可重复 + 必须是真实诗句（服务端校验）
- 一方答不出 → 另一方获胜
- 胜 +1 星，负 -1 星

### 2.4 多人淘汰模式

- 4-8 人参加，系统给出关键字
- 按匹配顺序轮流作答，每人 15 秒
- 答不出即淘汰，可继续观战
- 最后存活者获胜
- 第 1 名 +1 星，第 2 名不变，其余 -1 星

### 2.5 主题挑战模式

- 每日 00:00 UTC 发布一个特殊关键字（可能是较难的字，如「愁」「梦」「酒」）
- 单人 vs AI（高级），记录玩家连续作答数
- 全服按连续作答数排名
- 前 3 名获得知语币奖励

## 四、核心交互

### 4.1 关键字公布

**触发**：游戏开始 / 新一轮关键字更换（主题挑战模式）

**视觉流程**：
1. **画面暗淡**（0ms）：全屏覆盖半透明暗色层（`rgba(0,0,0,0.6)`）
2. **关键字入场**（200ms）：关键字从画面中央以放大动画入场（从 200% 缩小到 100%），伴随毛笔书写风格的笔触动效（笔画依次绘出）
3. **朗读**（500ms）：播放关键字的朗读音频（如 "月"）
4. **装饰展开**（700ms）：关键字周围展开古风装饰（如水墨云纹、祥云）
5. **暗层淡出**（1500ms）：暗色层淡出，关键字缩小飞入顶部 HUD 位置
6. **游戏开始**（2000ms）：操作区激活，先手方（随机决定）的倒计时启动

### 4.2 玩家作答 — 键盘输入方式

**用户动作**：在输入框中打字输入完整诗句

**输入流程**：
1. 获取焦点 → 弹出中文键盘（移动端）
2. 输入诗句文字（如 `举头望明月`）
3. 输入过程中右侧提示列表实时过滤匹配
4. 若输入内容包含关键字 → 确认按钮变为可点击态
5. 点击「确认」或按回车 → 提交到服务端校验

**输入辅助**：
- 输入拼音时，候选词联想（依赖系统输入法）
- 输入框支持复制粘贴（但粘贴内容同样需通过服务端校验）

### 4.3 玩家作答 — 提示列表选择方式

**用户动作**：直接点击右侧提示列表中的诗句

**交互流程**：
1. 浏览提示列表（可滚动）
2. 点击目标诗句 → 自动填入输入框 + 自动提交
3. 服务端校验 → 进入「作答成功」或「作答失败」流程

### 4.4 诗句有效性判定（服务端三重校验）

提交后服务端依次校验，任一项失败即判为无效：

| 校验项 | 规则 | 失败提示 |
|--------|------|---------|
| **包含关键字** | 提交的诗句中必须包含本局关键字 | 「诗句中未包含"X"字」 |
| **不可重复** | 提交的诗句不可与本局中任一方已出的诗句相同 | 「这句诗已被使用过」 |
| **真实诗句** | 提交的内容必须是题库中存在的真实诗句 | 「未找到这句诗，请确认」 |

**校验响应时间**：< 200ms（服务端预索引）

### 4.5 作答成功

**判定**：诗句通过三重校验

**视觉反馈**（按时间顺序）：
1. **输入框绿色闪光**（0ms）：输入框边框变绿 `#22c55e`，持续 300ms
2. **诗句卡片入场**（100ms）：中间展示区展示诗句卡片（详见 §3.5 卡片入场动画）
3. **背景画渐显**（300ms）：诗句意境背景画从透明淡入（持续 500ms）
4. **朗诵音频**（500ms）：自动播放诗句朗诵音频（清晰有感情的朗读）
5. **得分飘字**（600ms）：卡片上方飘出得分（如 `+100`，Amber 色），持续 600ms 淡出
6. **已出列表更新**（300ms）：该诗句加入己方已出诗句列表
7. **回合切换**（3000ms）：诗句卡片淡出 → 轮到对方 → 操作区加遮罩 → 对方倒计时启动

**冷门诗句额外反馈**：
- 飘字变为 `+150 诗词达人!`（Amber 金色 + 星形粒子）
- 卡片边框出现 Amber 金色光晕
- 音效叠加古筝弹拨音

**分数规则**：

| 诗句类型 | 基础分 | 速度奖励（<5s 提交） | 合计 |
|---------|-------|-------------------|------|
| 常见诗句 | 100 | +20 | 100-120 |
| 进阶诗句 | 120 | +30 | 120-150 |
| 冷门诗句 | 150 | +50 | 150-200 |

### 4.6 作答失败 — 提交无效诗句

**视觉反馈**：
1. **输入框震动**（0ms）：左右震动 3 次，边框变 Rose 红，持续 300ms
2. **错误提示**（50ms）：输入框下方显示具体失败原因（Rose 红色 Body S 14px）
3. **倒计时继续**：提交失败不暂停倒计时
4. **输入框自动清空**（500ms）：清空输入框，玩家可重新输入

**音效**：短促"嗡"错误音

### 4.7 作答失败 — 倒计时归零

**触发**：15 秒倒计时归零且玩家未提交有效诗句

**视觉反馈**：
1. **倒计时归零**（0ms）：进度条 Rose 红闪烁 3 次
2. **蜂鸣音**（0ms）：短促蜂鸣
3. **己方区域暗淡**（200ms）：己方区域降至 40% 透明度
4. **判负宣告**（400ms）：画面中央显示毛玻璃面板（500×200px）：
   - 1v1：`你未能接上，对手获胜！`（Rose 红色 H2 24px）
   - 多人淘汰：`你已淘汰！` → 进入观战模式
   - 单人练习：`词穷了！本局结束` → Game Over → 结算页
5. **正确提示**（800ms）：短暂展示 `其实可以答：XXX`（灰色浮层，展示 2 个可用诗句），持续 3 秒

## 五、专属称号系统

### 5.1 称号层级

古诗飞花令拥有独立于通用段位的**专属称号系统**，与通用段位并行展示：

| 称号 | 图标 | 获取条件 | 称号颜色 |
|------|------|---------|---------|
| **诗童** | 📒 | 初始称号 | 灰色 |
| **秀才** | 📗 | 累计作答 50 句有效诗句 | 白色 |
| **举人** | 📘 | 累计作答 150 句 + 使用过 10 句冷门诗句 | Sky 蓝色 |
| **进士** | 📕 | 累计作答 300 句 + 单局连续作答 ≥ 10 句 | Amber 金色 |
| **状元** | 🏆 | 累计作答 600 句 + 使用过 30 句冷门诗句 + 1v1 胜率 ≥ 60% | Rose 红色 |
| **诗仙** | ✨ | 累计作答 1000 句 + 全部关键字至少对战过一次 + 单局最长连续 ≥ 20 句 | 彩虹渐变 |

### 5.2 称号展示

- 称号标签展示在头像下方（玩家昵称旁）
- 对局中双方称号均可见，形成「秀才 vs 进士」的对弈氛围
- 称号升级时弹出庆祝动画（金色光芒 + 新称号入场 + 升级音效）
- 称号在个人中心、游戏大厅、排行榜中均展示

### 5.3 诗词储备量统计

- 个人中心展示「诗词储备量」：你已掌握 X 首古诗中含 Y 个关键字的 Z 句诗
- 以可视化图表呈现：
  - 横轴：关键字（月、花、春、风...）
  - 纵轴：该关键字下你成功作答过的诗句数量
  - 颜色深浅表示掌握程度

## 六、PK 模式特殊规则

### 6.1 回合交替机制

| 事件 | 处理 |
|------|------|
| **己方回合** | 操作区可用，输入框获取焦点，15 秒倒计时启动 |
| **对方回合** | 操作区灰色遮罩 + `对方作答中...`，可看到对方 15 秒倒计时 |
| **己方作答成功** | 诗句展示 + 得分 + 轮到对方 |
| **对方作答成功** | 对方诗句在中间展示区展示（含背景画和朗诵）+ 轮到己方 |
| **己方作答失败（超时）** | 判负 → 对方获胜 |
| **对方作答失败（超时）** | 判对方负 → 己方获胜 |

### 6.2 先手决定

- 匹配成功后随机决定先手（显示 3 秒倒计时 + 「你先/对方先」提示）
- 先手方可能略有优势（有更多可选诗句），因此先手方首轮倒计时为 12 秒（比后手少 3 秒）以平衡

### 6.3 多人淘汰特殊规则

| 规则 | 说明 |
|------|------|
| **轮流顺序** | 按匹配进入顺序固定轮转 |
| **当前作答者** | 头像高亮 + 中间区域边框发光标识 |
| **淘汰** | 答不出即淘汰，头像变灰 + 「已淘汰」 |
| **观战** | 淘汰后可继续观看，操作区不可用 |
| **最终排名** | 按淘汰顺序反序排名 |

### 6.4 对手作答时的反馈

- 对手成功作答的诗句同样在中间展示区完整展示（含背景画 + 朗诵音频）
- 这是共同欣赏诗句的过程，不仅仅是竞争
- 对手使用冷门诗句时，己方屏幕显示 `对方好厉害！`（Sky 蓝色，800ms 淡出）

## 七、上瘾机制详情

### 7.1 诗句自动配背景画和朗诵音频

**美感设计亮点**：每一句成功提交的诗句都不仅仅是文字，还伴随：

| 元素 | 说明 |
|------|------|
| **背景画** | 从预设的水墨画/国画素材库中，根据诗句意境标签（如「思乡」「战争」「爱情」「自然」）匹配一幅背景画，20% 透明度叠加 |
| **朗诵音频** | 预录的标准普通话朗诵音频，有感情地朗读诗句，自动播放 |
| **沉浸感** | 两者结合营造「诗词大会」般的文化沉浸体验 |

### 7.2 冷门诗句加分

- 冷门诗句（题库中标记，出现频率较低的诗句）额外加分 +50
- 触发「诗词达人」飘字 + 特殊音效
- 鼓励玩家扩展诗词储备，而非只用最常见的几句

### 7.3 诗词储备量可视化

- 结算页展示「诗词储备量 +X」（本局新增使用的诗句数）
- 个人中心展示总储备量图表
- 形成「收集」心理——「我要把所有关键字的诗句都答过一遍」

### 7.4 每日主题挑战

- 每日一个特殊关键字，可能是较难的字
- 全服排名激发竞争欲
- 参与即获知语币 +5，前 10% +20，第 1 名 +50

## 八、状态矩阵

### 8.1 游戏进行中状态

| 状态 | 描述 | 设计要求 |
|------|------|---------|
| **己方回合·输入中** | 玩家正在输入诗句 | 输入框获取焦点，提示列表实时过滤，倒计时运行 |
| **己方回合·等待提交** | 输入了内容但未提交 | 确认按钮高亮（含关键字时）或灰色（不含时） |
| **对方回合** | 等待对方作答 | 操作区遮罩 + 对方倒计时可见 |
| **诗句展示中** | 一方成功作答，展示诗句 | 中间区域展示卡片 + 背景画 + 朗诵音频，持续 3 秒 |
| **倒计时紧迫** | 倒计时 <5 秒 | 进度条 Rose 红闪烁 + 心跳音效 |
| **作答失败（无效）** | 提交的诗句未通过校验 | 输入框震动 + 错误提示 + 倒计时继续 |
| **作答失败（超时）** | 15 秒归零 | 蜂鸣 + 判负/淘汰 |
| **暂停** | 仅单人练习模式可暂停 | 参见 `08-hud-landscape.md` |
| **Game Over** | 单人练习词穷 | Game Over → 结算页 |

### 8.2 PK / 多人模式状态

| 状态 | 描述 | 设计要求 |
|------|------|---------|
| **双方对弈中** | 回合交替进行 | 当前作答方高亮，另一方遮罩 |
| **对手已淘汰/判负** | 对手答不出 | 己方获胜庆祝 → 结算页 |
| **己方已淘汰** | 己方答不出 | 判负 → 观战模式（多人淘汰）或结算页（1v1） |
| **对手断线** | 对手网络断开 | 等待 15 秒重连，超时判己方获胜 |

### 8.3 空状态 / 异常状态

| 状态 | 描述 | 设计要求 |
|------|------|---------|
| **关键字加载中** | 等待服务端生成关键字和题库 | 中间区域 Skeleton + "准备中..." |
| **加载失败** | 网络/服务端异常 | 居中错误提示 + "重试" + "返回大厅" |
| **提示列表为空** | 所有含关键字的诗句都已被使用 | 提示面板显示 `所有可选诗句已用完` + 玩家只能通过输入框尝试 |
| **PK 匹配中** | 等待对手 | 参见 `05-game-common/03-matching.md` |
