## 一、匹配触发

### 1.1 触发入口

| 入口 | 触发操作 | 匹配类型 |
|------|---------|---------|
| 模式选择页 → 点击"1v1 PK"按钮 | 直接进入匹配 | 1v1 匹配 |
| 模式选择页 → 点击"多人对战"按钮 | 直接进入匹配 | 多人匹配 |
| 游戏结算页 → 点击"再来一局"按钮 | 直接进入匹配（保持上一局相同模式） | 1v1 或多人匹配 |

### 1.2 匹配前校验

进入匹配页面前，系统执行以下校验：

| 校验项 | 失败时行为 |
|--------|----------|
| 网络连接 | 弹出 Toast："请连接网络后再匹配"，停留在模式选择页 |
| 用户登录态 | 弹出登录墙（理论上已在大厅拦截，此处为兜底） |
| 用户是否在其他游戏中 | 弹出对话框："你正在进行另一场游戏，确认退出当前游戏？" → 确认退出当前游戏并进入匹配 / 取消 |

### 2.2 页面状态：匹配中

#### 我方区域（左侧）

| 元素 | 说明 | 字体/尺寸 |
|------|------|----------|
| **头像** | 用户头像，圆形 | 64×64px，圆形（`border-radius: 50%`） |
| **昵称** | 用户昵称，单行截断 | Body（16px），字重 600，白色 |
| **段位名称** | 当前段位名称 | Body S（14px），Amber 色 |
| **星数图标** | 当前小段已获星数 | 12×12px 星星图标排列 |

#### 中间区域

| 元素 | 说明 |
|------|------|
| **"VS" 文字** | 匹配中显示为半透明状态 |
| **"寻找对手..." 文字** | VS 下方，脉冲动画（透明度 0.5↔1.0 循环），时长 1.5s |
| **匹配动画** | 以 VS 文字为中心的旋转扫描雷达效果：半透明 Rose 色扇形区域绕中心顺时针旋转，转速 3 秒/圈 |
| **匹配计时** | VS 下方显示已等待时间，格式"00:05"，每秒更新 | 

#### 对手区域（右侧）

| 元素 | 说明 |
|------|------|
| **"?" 占位头像** | 灰色圆形，中央白色问号 | 
| **"???" 昵称占位** | 三个问号，灰色文字 |
| **"???" 段位占位** | 三个问号，灰色文字 |

#### 底部按钮

| 按钮 | 标签 | 样式 | 行为 |
|------|------|------|------|
| **取消匹配** | "取消匹配" | 灰色边框按钮（非实心），药丸形 | 取消匹配队列，返回模式选择页 |

### 2.3 视觉规范

- 页面背景：深色（暗色模式下 `#0e0e0e`，亮色模式下 `#171717`），增强对战氛围
- 我方和对手区域左右对称布局，各占页面 35% 宽度
- 中间区域占 30% 宽度
- 整体垂直居中
- 匹配动画的雷达效果使用 Rose 色 20% 不透明度的扇形

## 三、匹配成功展示

### 3.1 匹配成功过渡

匹配到对手后，执行以下序列动画：

#### 步骤 1：对手信息揭晓（时长 800ms）

```
对手"?"头像 → 缩小为 0 → 弹性放大为对手真实头像（Spring 缓动）
"???" 昵称 → 淡出 → 对手真实昵称淡入
"???" 段位 → 淡出 → 对手真实段位淡入
```

#### 步骤 2：VS 强调（时长 500ms）

```
"VS" 文字放大到 150% → 回弹至 100%（Spring 缓动）
颜色从半透明变为 Rose 色全亮
匹配雷达动画停止
"寻找对手..." 文字消失
```

#### 步骤 3：倒计时开始（时长 3000ms）

```
VS 文字位置替换为倒计时数字：
  "3" → 放大淡出（500ms）
  "2" → 放大淡出（500ms）
  "1" → 放大淡出（500ms）
  "开始！" → 放大后快速缩小消失（500ms）

倒计时数字样式：
  · Display 字号（48px），字重 800
  · 颜色：3=白色，2=Amber 色，1=Rose 色，开始=Rose 色
  · 每个数字出现时伴随轻微震动反馈（如支持 Haptic）
```

### 3.2 倒计时规则

- 倒计时期间（3 秒）**不可取消匹配**，"取消匹配"按钮隐藏
- 倒计时结束后自动触发横屏切换，进入游戏画面
- 倒计时期间若任一方断线，参见§七网络断线处理

### 3.3 对手信息展示

| 元素 | 说明 |
|------|------|
| **对手头像** | 对手用户头像，圆形 64×64px |
| **对手昵称** | 对手昵称，单行截断，最长 12 个字符 |
| **对手段位** | 对手当前段位名称 + 星数图标 |

## 四、匹配规则

### 4.1 段位匹配

| 规则 | 说明 |
|------|------|
| **匹配范围** | 优先匹配同段位对手（同大段 ± 1 小段） |
| **范围扩大** | 等待 10 秒后扩大至同大段 ± 2 小段 |
| **再次扩大** | 等待 20 秒后扩大至相邻大段 |
| **最终兜底** | 等待 30 秒后触发超时处理（§五） |

### 4.2 匹配队列

- 匹配请求发送至对战服务器
- 服务端按段位计算匹配分数，优先选择分数差值最小的对手
- 同一用户不可与最近 3 局的对手连续匹配（避免重复对战体验差）

## 五、匹配超时处理

### 5.1 超时触发

- 1v1 PK 匹配等待超过 **30 秒**后触发超时

### 5.2 超时弹窗

```
┌─────────────────────────────────────┐
│                                     │
│          😔                         │
│     未找到合适对手                    │
│                                     │
│  ┌─────────────────────────────┐    │
│  │     🤖 进入 AI 对战          │    │ ← 主按钮（Rose 色实心）
│  │     AI 对战不影响段位         │    │
│  └─────────────────────────────┘    │
│                                     │
│  ┌─────────────────────────────┐    │
│  │     ⏳ 继续等待              │    │ ← 次按钮（灰色边框）
│  └─────────────────────────────┘    │
│                                     │
│  ┌─────────────────────────────┐    │
│  │     ← 返回                  │    │ ← 文字按钮（灰色文字）
│  └─────────────────────────────┘    │
│                                     │
└─────────────────────────────────────┘
```

### 5.3 按钮行为

| 按钮 | 标签 | 副文字 | 行为 |
|------|------|-------|------|
| **进入 AI 对战** | "进入 AI 对战" | "AI 对战不影响段位" | 关闭弹窗 → 直接进入游戏（对手为 AI）→ 结算页标注为 AI 对战 |
| **继续等待** | "继续等待" | — | 关闭弹窗 → 继续匹配 → 再等 30 秒后再次触发超时弹窗 |
| **返回** | "返回" | — | 取消匹配 → 返回模式选择页 |

### 5.4 AI 对战说明

- AI 对战结算页明确标注"AI 对战"标识
- AI 对战**不影响段位**（不增星、不减星）
- AI 对战记录**不计入**胜场/总场统计
- AI 难度默认与用户当前段位匹配（青铜→简单 AI，白银→简单 AI，黄金→中等 AI，铂金+→困难 AI）

### 6.2 元素说明

| 元素 | 说明 |
|------|------|
| **标题** | "多人对战 · {游戏名称}" |
| **人数进度** | "已匹配 X / Y 人"，X 为当前已进入人数，Y 为该游戏要求的人数 |
| **玩家头像网格** | 按总人数排列头像位（如 4 人则 4 个位），已匹配的显示头像+昵称，未匹配的显示"?"占位 |
| **匹配动画** | 脉冲光圈效果：从中心向外扩散的圆环，Rose 色，间隔 2 秒 |
| **取消匹配按钮** | 同 1v1 匹配 |

### 6.3 人数达标后行为

- 所有位置匹配满后（如 4/4 人），自动进入 3 秒倒计时
- 倒计时动画和规则同§三（匹配成功展示）
- 倒计时期间不可取消

### 6.4 多人匹配超时

| 规则 | 说明 |
|------|------|
| **超时时间** | 等待 **30 秒**后触发超时 |
| **超时弹窗** | 与 1v1 超时弹窗相同布局 |
| **进入 AI 补位** | 未匹配满的位置由 AI 补充，AI 不影响段位（仅对 AI 方无效，真人间胜负仍计段位） |
| **继续等待** | 继续匹配，再等 30 秒后再次触发 |
| **返回** | 取消匹配返回模式选择页 |

### 6.5 中途玩家退出匹配

- 如果已匹配但未开始游戏（倒计时前），某玩家退出：
  - 其位置恢复为"?"占位
  - 继续等待新玩家匹配
  - 不触发超时重置

## 七、网络断线处理

### 7.1 匹配中断线

| 场景 | 行为 |
|------|------|
| 匹配等待过程中网络断开 | 顶部展示离线 Banner："网络已断开"；自动从匹配队列移除；网络恢复后 Banner 消失，需用户手动重新点击匹配 |
| 匹配成功倒计时期间断线 | 对手方进入游戏，断线方 5 秒内未重连则判负（-1 星）；断线方看到："网络异常，对局已结束" |

### 7.2 游戏进行中断线

| 场景 | 行为 |
|------|------|
| **短暂断线（≤ 10 秒）** | 游戏画面冻结，展示全屏半透明遮罩 + "网络重连中..."文字 + 旋转加载指示器；自动尝试 WebSocket 重连；重连成功后恢复游戏，双方状态同步 |
| **长时间断线（> 10 秒）** | 停止重连尝试；对于 PK/多人模式：判断断线方负（-1 星）；对方获胜（+1 星）；断线方展示："网络断开超时，对局已结束，判定为失败" + "返回大厅"按钮 |
| **单人模式断线** | 游戏暂停；展示"网络已断开"；网络恢复后可继续；如果游戏数据全在本地则不受影响 |

### 7.3 断线重连成功

- 重连成功后，移除"重连中"遮罩
- 游戏状态从服务端同步恢复
- 底部 Toast 提示："网络已恢复"（绿色，2 秒自动消失）

## 八、状态矩阵

| 状态 | 触发条件 | UI 表现 |
|------|---------|---------|
| **空状态** | 不适用 | — |
| **加载中** | 正在连接对战服务器并加入匹配队列 | "连接中..."文字替代匹配动画，持续 1-2 秒 |
| **首次加载** | 同"加载中" | — |
| **成功状态（匹配中）** | 已加入匹配队列，等待对手 | 正常展示匹配页面，雷达/脉冲动画播放 |
| **成功状态（匹配成功）** | 找到对手 | 播放匹配成功动画序列（§三） |
| **错误状态** | 连接对战服务器失败 | 页面居中展示："服务器连接失败" + 重试按钮 + 返回按钮；重试按钮点击后重新连接 |
| **部分加载** | 多人匹配中，部分位置已匹配 | 已匹配位置显示玩家信息，未匹配位置显示"?"占位 |
| **离线状态** | 设备无网络 | 顶部离线 Banner；匹配自动取消；展示"请连接网络后重试" + 返回按钮 |
