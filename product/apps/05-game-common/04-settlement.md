### 1.2 大标题区域

| 元素 | 说明 | 视觉规范 |
|------|------|---------|
| **胜利图标** | 奖杯图标 | 48×48px，Amber 色 |
| **"胜利！"文字** | 多语言（"胜利！" / "Victory!" / "Chiến thắng!"） | Display（48px），字重 800，Amber 色 |
| **入场动画** | 标题从 0 缩放至 100% + 弹性回弹，同时金色粒子从两侧向中心汇聚 | Spring 缓动，时长 600ms |
| **背景特效** | 金色光芒从标题向外辐射扩散，渐变衰减 | 持续 2 秒后衰减为静态微光 |

### 1.3 得分展示区域

| 元素 | 说明 | 视觉规范 |
|------|------|---------|
| **"得分"标签** | "得分" | Body S（14px），灰色 |
| **分数数字** | 本局获得的分数 | H1（32px），字重 700，白色 |
| **滚动动画** | 数字从 0 滚动到实际分数 | 时长 1200ms，Decelerate 缓动，数字逐帧递增 |

### 1.4 段位变化展示区域

#### 普通加星（未晋级）

```
黄金 II ⭐⭐⭐ → 黄金 II ⭐⭐⭐⭐
                              ↑ 新增的星闪烁高亮
+1 ⭐
```

| 元素 | 说明 |
|------|------|
| **变化前段位** | 左侧展示对局前的段位名称 + 星数 |
| **箭头** | "→" 分隔符 |
| **变化后段位** | 右侧展示对局后的段位名称 + 星数 |
| **新增星高亮** | 新增的那颗星使用 Amber 色闪烁动画（透明度 0.5↔1.0 循环 3 次） |
| **"+1 ⭐" 标签** | 段位变化行下方，Rose 色文字 |

#### 晋级展示（升入下一小段或大段）

当本局胜利导致晋级时，触发晋级动画：

```
动画序列（总时长 2500ms）：
  1. 旧段位图标居中放大至 120%（300ms）
  2. 旧段位图标碎裂/消散效果（500ms）
  3. 新段位图标从碎片中心凝聚成型 + 放大至 150% + 回弹至 100%（800ms，Spring 缓动）
  4. 金色光环从新段位图标向外扩散（400ms）
  5. 粒子特效：金色/Rose 色粒子从图标四周喷射（500ms）
  6. 新段位名称文字从下方滑入 + 淡入（300ms）
  7. 伴随音效：升段庆祝音（明亮、上升的音阶）
```

| 元素 | 说明 |
|------|------|
| **旧段位图标** | 晋级前的段位专属图标 |
| **新段位图标** | 晋级后的段位专属图标，尺寸 96×96px |
| **新段位名称** | "晋级至 {段位名}！"，如"晋级至 黄金 I！" |
| **晋级标签** | Rose 色标签"晋级！"，脉冲动画 |

> 晋级动画在得分滚动完成后自动播放，用户不可跳过（2.5 秒内自然结束）。

| 按钮 | 标签 | 样式 | 行为 |

## 二、失败结算页（PK / 多人对战模式）

### 2.1 页面布局

布局结构与胜利结算页相同，差异在视觉风格和内容。

### 2.2 大标题区域

| 元素 | 说明 | 视觉规范 |
|------|------|---------|
| **失败图标** | 伤心表情图标 | 48×48px |
| **"惜败！"文字** | 多语言（"惜败！" / "Defeat!" / "Thất bại!"） | Display（48px），字重 800，灰白色 |
| **入场动画** | 标题从上方轻微下落 + 轻缓淡入，无粒子特效 | Decelerate 缓动，时长 400ms |
| **背景特效** | 无特效，保持暗淡背景 | — |

### 2.3 段位变化展示区域

#### 普通掉星（未掉段）

```
黄金 II ⭐⭐⭐⭐ → 黄金 II ⭐⭐⭐
-1 ⭐
```

| 元素 | 说明 |
|------|------|
| **变化前段位** | 对局前的段位名称 + 星数 |
| **变化后段位** | 对局后的段位名称 + 星数 |
| **失去的星** | 变化前显示的最后一颗星使用灰色 + 缩小消失动画（300ms） |
| **"-1 ⭐" 标签** | 灰色文字 |

#### 掉段展示（降至上一小段或大段）

当本局失败导致掉段时，触发掉段动画：

```
动画序列（总时长 1500ms）：
  1. 当前段位图标震动（左右晃动 3 次，200ms）
  2. 段位图标缩小至 80% + 透明度降至 50%（300ms）
  3. 新（更低）段位图标从底部滑入 + 淡入至 100%（500ms，Decelerate 缓动）
  4. 段位名称文字更新，灰色文字
  5. 伴随音效：下降音（低沉、下行的音阶）
```

| 元素 | 说明 |
|------|------|
| **旧段位图标** | 掉段前的段位图标 |
| **新段位图标** | 掉段后的段位图标，尺寸 64×64px（比晋级动画小） |
| **新段位名称** | "{段位名}"，无额外标签 |

> 掉段动画比晋级动画更简短、更低调，避免加剧用户挫败感。

### 2.4 知识点回顾区域

- 与胜利结算页相同的知识点列表
- 答错项同样可点击查看详解

### 2.5 推荐复习引导

- **触发条件**：本局答错率 ≥ 50%（答错题数 / 总题数）
- **展示位置**：知识点回顾区域下方
- **内容**：

```
┌─────────────────────────────────────┐
│  💡 答错较多，建议复习对应课程        │
│                                     │
│  [去学习 Level X 课程]               │ ← 文字链接按钮，Sky 色
└─────────────────────────────────────┘
```

| 元素 | 说明 |
|------|------|
| **引导文案** | "答错较多，建议复习对应课程" | 
| **课程链接** | "去学习 Level X 课程"，X 为该游戏对应的 Level |
| **点击行为** | 跳转至对应 Level 的课程详情页（退出游戏横屏，恢复竖屏） |

- 答错率 < 50% 时，此引导区域不显示

### 2.6 底部按钮区域

- 与胜利结算页相同："再来一局" + "返回大厅"

## 三、单人模式结算页

### 3.1 与 PK 模式的差异

| 维度 | PK/多人模式 | 单人模式 |
|------|-----------|---------|
| 大标题 | "胜利！"或"惜败！" | "游戏结束" |
| 段位变化 | 展示段位增减 | **不展示段位变化**（单人不影响段位） |
| 个人最高分 | 不展示 | 展示个人最高分对比 |
| 晋级/掉段动画 | 有 | 无 |
| 知识点回顾 | 有 | 有 |
| 推荐复习 | 答错率高时展示 | 答错率高时展示 |
| 再来一局 | 重新匹配 | 直接开始同模式新局 |

```
│           最高：920 分 🏅                    │
```

- 知识点回顾区域下方、底部按钮区域上方
- 文字链接样式："📤 分享战绩"，Sky 色，居中

```
点击"分享战绩"
```

## 五、状态矩阵

| 状态 | 触发条件 | UI 表现 |
|------|---------|---------|
| **空状态** | 不适用 | — |
| **加载中** | 正在从服务端获取结算数据 | 页面居中显示知语 Logo 呼吸动画，持续 1-2 秒 |
| **首次加载** | 同"加载中" | — |
| **成功状态** | 结算数据加载完成 | 按序播放动画：标题入场 → 分数滚动 → 段位变化 → 知识点列表淡入 → 按钮淡入 |
| **错误状态** | 结算数据请求失败 | "结算数据加载失败" + 重试按钮；重试后如仍失败，展示"返回大厅"按钮兜底 |
| **部分加载** | 不适用（单次请求） | — |
| **离线状态** | 设备无网络 | 如果游戏数据已在本地（单人模式），正常展示结算；如果需要服务端数据（PK 模式），展示"网络异常，请连接网络查看结算" + "返回大厅"按钮 |

## 六、数据流向

| 数据 | 来源 | 去向 | 说明 |
|------|------|------|------|
| 本局得分 | 对战服务器结算 / 本地游戏引擎计算 | 得分展示区域 | PK 模式由服务端计算；单人模式由本地计算 |
| 段位变化 | 对战服务器计算 | 段位变化展示 | 服务端返回 `before_rank`、`after_rank`、`star_change` |
| 知识点列表 | 对战服务器 / 本地游戏引擎 | 知识点回顾 | 每题的知识点名称 + 答对/答错状态 |
| 知识点详解 | Supabase `knowledge_points` 表 | 答错详解面板 | 按需加载（点击时请求） |
| 个人最高分 | Supabase `game_records` 表 | 单人模式最高分对比 | 游戏结束时与历史记录对比 |
| 战绩卡片 | 前端本地渲染（Canvas） | 分享/保存 | 纯前端生成图片 |
