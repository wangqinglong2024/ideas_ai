## 三、分类 Tab

### 3.1 Tab 定义

| Tab | 标签 | 筛选规则 |
|-----|------|---------|
| **全部** | "全部" | 展示所有可售皮肤 |
| **角色皮肤** | "角色皮肤" | 类型为角色外观的皮肤 |
| **背景** | "背景" | 类型为游戏场景背景的皮肤 |
| **特效** | "特效" | 类型为游戏特效（如切割特效、泡泡样式）的皮肤 |
| **音效** | "音效" | 类型为游戏音效包的皮肤 |

### 3.2 Tab 视觉规范

- Tab 栏横向可滚动（如屏幕宽度不够）
- 选中 Tab：Rose 色背景 + 白色文字，药丸形
- 未选中 Tab：透明背景 + 灰色文字
- Tab 间距 8px，padding 12px 16px
- Tab 文字：Body S（14px），字重 500

### 3.3 游戏筛选下拉

| 元素 | 说明 |
|------|------|
| **默认值** | "全部游戏" |
| **选项** | "全部游戏" + 12 款游戏名称（含游戏图标 20×20px） |
| **选中后** | 仅展示适用于该游戏的皮肤 |
| **与分类 Tab 组合** | 分类 Tab 和游戏筛选可同时生效（AND 关系） |

## 四、皮肤卡片设计

### 4.1 卡片布局

```
┌──────────────────────┐
│                      │
│   [预览图/动画]      │  ← 占卡片 65% 高度
│                      │
│  赛季限定             │  ← 条件展示：赛季限定标签
├──────────────────────┤
│  火焰刀光            │  ← 皮肤名称
│  汉字切切切           │  ← 适用游戏标签
│  99 💰               │  ← 价格
└──────────────────────┘
```

### 4.2 卡片元素定义

| 元素 | 说明 | 字体/尺寸 |
|------|------|----------|
| **预览图/动画** | 皮肤的静态预览图或短循环动画（GIF/WebP 动图） | 宽度 100%，高度占卡片 65%，`object-fit: cover` |
| **皮肤名称** | 多语言展示 | Body S（14px），字重 600 |
| **适用游戏标签** | 该皮肤适用的游戏名称 | Caption（12px），灰色文字 |
| **价格** | 知语币价格 | Body S（14px），字重 600，Amber 色 |
| **已拥有标识** | 用户已购买时，价格替换为"已拥有" | Body S（14px），绿色文字 |
| **赛季限定标签** | 赛季限定皮肤在预览图右上角展示 | Caption（12px），Amber 色背景 + 白色文字，圆角 8px |
| **付费用户折扣** | 付费用户可享折扣时，在价格旁展示 | 原价删除线 + 折扣价，Rose 色 |

### 4.3 卡片网格规范

- 2 列网格布局
- 卡片间距 16px，左右 padding 16px
- 卡片背景：毛玻璃面板（`.glass-card`）
- 卡片圆角：24px
- 点击态：`scale(0.97)`，时长 100ms

### 4.4 点击行为

- 点击皮肤卡片 → 弹出皮肤详情弹窗（§五）

### 5.3 购买按钮状态

| 用户状态 | 按钮文字 | 按钮样式 | 点击行为 |
|---------|---------|---------|---------|
| **未拥有 + 余额充足** | "购买" | Rose 色实心按钮，药丸形 | 弹出二次确认弹窗（§五.4） |
| **未拥有 + 余额不足** | "购买" | Rose 色实心按钮（正常样式） | 弹出余额不足提示（§五.5） |

| **已拥有 + 已装备** | "卸下" | 灰色边框按钮，药丸形 | 直接卸下，Toast 提示"已卸下" |

#### 确认购买后流程

```
点击"确认购买"
  → 按钮变为 Loading 态（旋转 + 不可点击）
  → 服务端扣除知语币 + 写入用户皮肤记录
  → 成功：
     · 关闭确认弹窗
     · 详情弹窗中价格区域变为"已拥有"
     · 按钮变为"装备"
     · 顶部知语币余额更新
     · Toast 提示："购买成功！"（绿色，2 秒）
  → 失败：
     · Toast 提示："购买失败，请重试"（红色，需手动关闭）
     · 按钮恢复可点击状态
```

## 六、装备管理

### 6.1 装备规则

| 规则 | 说明 |
|------|------|
| **装备粒度** | 每款游戏可独立装备不同皮肤 |
| **同类互斥** | 每款游戏的每个皮肤类型（角色/背景/特效/音效）同一时间只能装备一个 |
| **装备生效** | 装备后，进入对应游戏时自动使用该皮肤 |
| **卸下** | 卸下后恢复默认外观 |
| **跨游戏** | 如果皮肤适用多款游戏，在一款游戏中装备不影响其他游戏 |

### 6.2 装备操作

- **装备**：在皮肤详情弹窗中点击"装备"按钮 → Toast"已装备"（绿色）→ 按钮变为"卸下"
- **卸下**：在皮肤详情弹窗中点击"卸下"按钮 → Toast"已卸下"（蓝色）→ 按钮变为"装备"
- **游戏大厅联动**：装备/卸下后，游戏大厅中对应游戏卡片的皮肤缩略图实时更新

## 七、状态矩阵

| 状态 | 触发条件 | UI 表现 |
|------|---------|---------|
| **空状态** | 当前分类/游戏筛选下无可用皮肤 | 卡片区域展示插画 + "暂无此类皮肤" |
| **加载中** | 首次进入商城，皮肤列表加载中 | 6 张骨架屏卡片（2 列 × 3 行） |
| **首次加载** | 同"加载中" | — |
| **成功状态** | 皮肤列表加载完成 | 正常展示皮肤卡片网格 |
| **错误状态** | 皮肤列表请求失败 | "加载失败" + 重试按钮 |
| **部分加载** | 列表触底加载更多 | 列表底部加载指示器 |
| **离线状态** | 设备无网络 | 顶部离线 Banner；展示缓存的皮肤数据（如有）；点击购买/装备时提示"请连接网络" |

## 八、数据流向

| 数据 | 来源 | 去向 | 说明 |
|------|------|------|------|
| 皮肤列表 | Supabase `skins` 表 | 商城卡片网格 | 含名称、描述、价格、类型、适用游戏、预览图 URL |
| 用户已购皮肤 | Supabase `user_skins` 表 | 已拥有标识 + 装备/卸下按钮 | 筛选 `user_id` 对应的记录 |
| 用户知语币余额 | Supabase `users` 表 `coin_balance` 字段 | 顶部余额展示 + 购买校验 | 实时查询 |
| 购买交易 | 前端 → API → Supabase `coin_transactions` + `user_skins` | 扣除知语币 + 写入皮肤拥有记录 | 服务端事务性操作 |
| 装备状态 | Supabase `user_skins` 表 `equipped` 字段 | 装备/卸下状态 | 更新对应记录 |
