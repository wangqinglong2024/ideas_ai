## 一、页面布局

### 1.1 整体结构

```
┌─────────────────────────────────────┐
│  ← 返回                             │ ← 顶部导航栏
├─────────────────────────────────────┤
│                                     │
│  [游戏封面大图区]                     │
│  游戏封面 + 名称 + Level + 描述       │
│                                     │
├─────────────────────────────────────┤
│                                     │
│  [模式按钮列表]                       │
│  单人练习 / 1v1 PK / 多人对战         │
│                                     │
├─────────────────────────────────────┤
│                                     │
│  [历史战绩摘要]                       │
│  胜场 / 总场 / 胜率                   │
│                                     │
├─────────────────────────────────────┤
│  [底部 Tab 导航栏]（全局）              │
└─────────────────────────────────────┘
```

### 1.2 布局规则

- 页面为竖屏状态（尚未进入游戏，不触发横屏）
- 内容区可纵向滚动
- 顶部导航栏含返回按钮，标题为空（封面区已有游戏名）


### 3.2 按钮详细定义

| 按钮 | 图标 | 主标签 | 副标题 | 点击行为 |
|------|------|-------|--------|---------|
| **单人练习** | Lucide `Target` | "单人练习" | "不影响段位" | 展开子模式选择面板（§四） |
| **1v1 PK** | Lucide `Swords` | "1v1 PK" | "实时匹配，影响段位" | 进入匹配页面（`03-matching.md`） |
| **多人对战** | Lucide `Users` | "多人对战" | "X 人匹配，影响段位"（X 根据游戏类型确定） | 进入多人匹配页面（`03-matching.md`） |

### 3.3 按钮视觉规范

- 按钮为全宽卡片式按钮，高度 72px，左右 padding 16px
- 左侧图标 24×24px，图标与文字间距 16px
- 主标签：**Body L**（18px），字重 600
- 副标题：**Body S**（14px），字重 400，灰色文字
- 按钮背景：毛玻璃面板（`.glass-card`）
- 按钮间距：12px
- 点击态：`scale(0.97)` + 背景亮度微增，时长 100ms

## 四、单人模式的子模式选择

### 4.1 触发方式

点击"单人练习"按钮后，按钮下方展开子模式选择面板（手风琴展开动画，时长 200ms）。

### 4.2 通用子模式框架

每款游戏的单人模式提供 2-3 种子模式，以下为通用定义框架：

```
┌─────────────────────────────────────┐
│  🎯 单人练习                    ▼    │ ← 已展开状态，▼ 变为 ▲
├─────────────────────────────────────┤
│    ○ 经典模式                        │
│      无尽挑战，直到失败               │
│    ○ 限时模式                        │
│      XX 秒内获取最高分               │
│    ○ 无尽模式                        │
│      连续关卡，难度递增               │
│                                     │
│    [开始游戏]                        │ ← 确认按钮
└─────────────────────────────────────┘
```

### 4.3 子模式选择规则

- 子模式以单选列表展示（Radio Button 形态）
- 默认选中第一个子模式
- 每个子模式包含：模式名称 + 一句话说明
- 子模式名称使用 **Body**（16px），字重 500
- 子模式说明使用 **Body S**（14px），灰色文字
- 选中态：左侧圆点变为 Rose 色实心圆，文字变为 Rose 色
- 底部"开始游戏"确认按钮：Rose 色实心按钮（`.glass-button`），药丸形

### 4.4 各游戏子模式对照

| 游戏 | 子模式 1 | 子模式 2 | 子模式 3 |
|------|---------|---------|---------|
| G1 汉字切切切 | 经典（无尽挑战个人最高分） | 限时（60 秒切最多汉字） | — |
| G2 拼音泡泡龙 | 生存（泡泡下降坚持最久） | 清屏（逐屏消除无限循环） | — |
| G3 词语消消乐 | 无尽（无限消除最高分） | 限步（30 步内最高分） | — |
| G4 语法大厨 | 经营（无尽顾客挑战服务数） | 限时（90 秒最多订单） | — |
| G5 成语接龙大战 | 无尽（与 AI 无限接龙） | 限时（60 秒接最多成语） | — |
| G6 汉字华容道 | 无尽（连续解题 3 超时结束） | 极速（3 分钟解最多题） | — |
| G7 古诗飞花令 | 练习（vs AI，AI 难度可调） | 主题挑战（每日特殊关键字） | — |
| G8 阅读侦探社 | 破案（独立解谜追求评分） | — | — |
| G9 HSK 大冒险 | 冒险（无尽地牢角色养成） | — | — |
| G10 辩论擂台 | 练习（vs AI 辩手难度可调） | — | — |
| G11 诗词大会 | 挑战（无尽模式 3 错出局） | 限时（2 分钟答对最多） | — |
| G12 文豪争霸 | 无尽（连续闯轮越来越难） | — | — |

### 4.5 AI 难度选择

- 适用游戏：G7 古诗飞花令、G10 辩论擂台（单人练习模式）
- 在子模式选择面板底部、"开始游戏"按钮上方，展示难度选择：
  - "AI 难度"标签 + 三档横向选择按钮：**简单** / **中等** / **困难**
  - 默认选中"中等"
  - 选中态：Rose 色背景 + 白色文字
  - 未选中态：透明背景 + 灰色文字

### 4.6 点击"开始游戏"

- 点击后按钮变为 Loading 态（转圈 + 不可点击），时长最多 3 秒
- 触发横屏切换（详见 `08-hud-landscape.md`）
- 进入对应游戏画面

## 五、历史战绩摘要

### 5.1 布局结构

```
┌─────────────────────────────────────┐
│  📊 本游戏战绩                       │
│                                     │
│  胜场        总场         胜率       │
│   12          20          60%       │
└─────────────────────────────────────┘
```

### 5.2 元素定义

| 元素 | 说明 | 字体/尺寸 |
|------|------|----------|
| **区块标题** | "本游戏战绩"，左侧带 📊 图标 | Body（16px），字重 600 |
| **胜场数字** | 该游戏累计胜场（含单人+PK+多人） | H2（24px），字重 700，Rose 色 |
| **总场数字** | 该游戏累计总场次 | H2（24px），字重 700 |
| **胜率百分比** | 胜场 / 总场 × 100%，保留整数 | H2（24px），字重 700，Amber 色 |
| **数据标签** | "胜场"/"总场"/"胜率" | Caption（12px），灰色 |

### 5.3 特殊状态

- 未玩过该游戏时：三个数字均显示"—"，数据标签正常显示
- 三列数字水平等间距排列，居中对齐

## 六、返回按钮

### 6.1 位置

- 顶部导航栏左侧，图标按钮
- 图标：Lucide `ArrowLeft`，尺寸 24px

### 6.2 行为

- 点击返回游戏大厅页
- 返回动画：横向滑出（向右滑出），时长 300ms

## 七、状态矩阵

| 状态 | 触发条件 | UI 表现 |
|------|---------|---------|
| **空状态** | 不适用（页面数据基于游戏 ID 确定加载） | — |
| **加载中** | 正在获取该游戏的战绩数据和模式配置 | 封面区正常展示（封面图和基础信息为本地传参）；战绩区域显示骨架屏横条 |
| **首次加载** | 用户首次进入该游戏的模式选择页 | 同"加载中"，无特殊差异 |
| **成功状态** | 所有数据加载完成 | 正常展示全部内容 |
| **错误状态** | 战绩数据加载失败 | 战绩区域展示"数据加载失败"文案 + 重试图标；模式按钮正常可用（不依赖战绩数据） |
| **部分加载** | 不适用（数据为单次请求） | — |
| **离线状态** | 设备无网络连接 | 战绩区域显示"—"；点击"1v1 PK"或"多人对战"时弹出 Toast："请连接网络后再匹配"；"单人练习"正常可用（如果游戏资源已缓存） |
