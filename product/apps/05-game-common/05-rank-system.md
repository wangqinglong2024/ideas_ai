## 二、胜负规则

### 2.1 星数增减

| 对战结果 | 星数变化 | 适用模式 |
|---------|---------|---------|
| **PK 胜利** | +1 ⭐ | 1v1 PK、多人对战（最终排名第 1） |
| **PK 失败** | -1 ⭐ | 1v1 PK、多人对战（最终排名非第 1） |
| **单人模式** | 无变化 | 单人练习所有子模式 |
| **AI 对战** | 无变化 | 匹配超时后进入的 AI 对战 |
| **对手逃跑** | +1 ⭐ | 对手在 PK 中退出游戏 |
| **自己逃跑** | -1 ⭐ | 自己在 PK 中退出游戏 |

### 2.2 多人对战特殊规则

| 排名 | 星数变化 |
|------|---------|
| 第 1 名 | +1 ⭐ |
| 第 2 名 | 0（不变） |
| 第 3 名及以后 | -1 ⭐ |

> 多人对战中，仅冠军获得加星，亚军保平，其余扣星。

### 2.3 最低段位保护

- 段位最低不跌破 **青铜 III · 1 星**
- 当用户处于青铜 III 且仅剩 1 星时，PK 失败不再扣星，段位保持青铜 III 1 星

## 三、晋级规则

### 3.1 小段晋级

| 条件 | 行为 |
|------|------|
| 当前小段星数已满 + PK 胜利 | 自动升入下一小段，新小段星数为 1 |
| 示例 | 青铜 III 3/3 星 → 胜利 → 青铜 II 1/3 星 |

### 3.2 大段晋级

| 条件 | 行为 |
|------|------|
| 当前大段最高小段（I）星数已满 + PK 胜利 | 自动升入下一大段的最低小段，新小段星数为 1 |
| 示例 | 青铜 I 3/3 星 → 胜利 → 白银 III 1/3 星 |

### 3.3 晋级结算页表现

- 小段晋级：在结算页段位变化区域展示新段位，无特殊动画（仅新星闪烁高亮）
- 大段晋级：触发晋级动画（详见 `04-settlement.md` §一.1.4）

## 四、掉段规则

### 4.1 小段掉段

| 条件 | 行为 |
|------|------|
| 当前小段星数为 0 + PK 失败 | 降至上一小段满星 |
| 示例 | 白银 II 0/3 星 → 失败 → 白银 III 3/3 星（注意：不是 2 星，而是满星减 1 即 2 星） |

> 修正：掉段后回到上一小段，星数为该小段满星数 - 1。
> 示例：白银 II 1/3 星 → 失败（到 0 星）→ 再失败 → 白银 III 2/3 星（满星 3 - 1 = 2 星）

### 4.2 大段掉段

| 条件 | 行为 |
|------|------|
| 当前大段最低小段（III 或 IV 或 V）星数为 0 + PK 失败 | 降至上一大段最高小段（I），星数为该小段满星数 - 1 |
| 示例 | 白银 III 1/3 星 → 失败（到 0 星）→ 再失败 → 青铜 I 2/3 星 |

### 4.3 掉段结算页表现

- 小段掉段：段位变化区域展示新段位，无特殊动画
- 大段掉段：触发掉段动画（详见 `04-settlement.md` §二.2.3）

## 五、保护机制

### 5.1 新手保护

| 规则 | 说明 |
|------|------|
| **适用范围** | 新注册用户的前 10 局 PK 对战 |
| **保护效果** | PK 失败时不扣星（只加不减） |
| **目的** | 让新手快速体验到升段的成就感，降低初期挫败感 |
| **展示** | 结算页段位变化区域显示"🛡️ 新手保护中（剩余 X 局）" |

### 5.2 晋级保护

| 规则 | 说明 |
|------|------|
| **适用范围** | 大段晋级后的前 3 局 PK 对战 |
| **保护效果** | PK 失败时不掉段（可掉星但不掉回上一大段） |
| **目的** | 避免刚晋级就掉段的糟糕体验 |
| **展示** | 结算页段位变化区域显示"🛡️ 晋级保护中（剩余 X 局）" |

### 5.3 保护标识视觉

- 保护状态时，游戏大厅顶部段位展示栏的段位名称旁显示小型盾牌图标（Lucide `Shield`，16px，Sky 色）
- 保护剩余局数以 tooltip 形式展示（点击盾牌图标时）

## 六、晋级动画描述

### 6.1 大段晋级动画

> 详见 `04-settlement.md` §一.1.4，此处为完整视觉效果描述。

**动画序列**（总时长约 2500ms）：

| 步骤 | 时长 | 视觉效果 | 音效 |
|------|------|---------|------|
| 1. 旧段位放大 | 300ms | 旧段位图标从 32px 居中放大至 96px（150%） | — |
| 2. 旧段位碎裂 | 500ms | 图标裂纹扩散 → 碎片向四周飞散 → 碎片变为光点 | 清脆碎裂音 |
| 3. 新段位凝聚 | 800ms | 光点从四周汇聚至中心 → 凝聚成新段位图标 → 图标放大至 150% → Spring 回弹至 100% | 上升音阶 + 叮咚 |
| 4. 光环扩散 | 400ms | 金色光环从新图标中心向外扩散两圈，逐渐衰减 | — |
| 5. 粒子喷射 | 500ms | 金色 + Rose 色粒子从图标四周喷射 → 抛物线轨迹 → 自然衰减消失 | 庆祝烟花音 |
| 6. 段位名文字 | 300ms | "晋级至 {段位名}！"从下方 20px 滑入 + 淡入 | — |

**配色规则**：
- 光点/粒子颜色与新段位主色调一致（如晋级至黄金 → 金色粒子）
- 光环颜色为新段位主色调 + 白色混合

### 6.2 小段晋级动画

- 无独立动画
- 仅在结算页段位变化区域，新增的星使用 Amber 色闪烁动画（透明度 0.5↔1.0，循环 3 次，1.5 秒）

## 七、掉段动画描述

### 7.1 大段掉段动画

**动画序列**（总时长约 1500ms）：

| 步骤 | 时长 | 视觉效果 | 音效 |
|------|------|---------|------|
| 1. 段位震动 | 200ms | 当前段位图标左右晃动 3 次（±4px） | 低沉震动音 |
| 2. 段位衰减 | 300ms | 图标缩小至 80% + 透明度降至 50% + 灰度滤镜 | — |
| 3. 新段位出现 | 500ms | 新（更低）段位图标从底部 30px 滑入 + 淡入至 100% | 下行音阶 |
| 4. 段位名更新 | 300ms | 旧段位名淡出 → 新段位名淡入 | — |

**设计原则**：
- 掉段动画刻意**比晋级动画短**（1.5s vs 2.5s）
- 无粒子、无光环、无庆祝效果
- 配色偏冷灰，不使用明亮色
- 目的：快速传达信息，不加剧用户负面情绪

### 7.2 小段掉段动画

- 无独立动画
- 结算页段位变化区域，失去的星以灰色 + 缩小消失动画展示（300ms）

## 八、赛季机制

### 8.1 赛季时长

| 属性 | 说明 |
|------|------|
| **赛季周期** | 3 个自然月为一赛季 |
| **赛季命名** | "第 X 赛季"（S1、S2、S3...），按自然季度划分 |
| **赛季起止** | 每年 1 月 1 日、4 月 1 日、7 月 1 日、10 月 1 日 0:00 UTC+7 开始 |
| **首赛季** | 产品上线日所在季度 |

### 8.2 赛季结束时段位软重置规则

赛季结束时，所有用户段位按以下规则重置：

| 赛季结束段位 | 重置后段位 | 说明 |
|------------|----------|------|
| 青铜 III ~ 青铜 I | 青铜 III 1 星 | 青铜段位重置为起点 |
| 白银 III ~ 白银 I | 青铜 I 1 星 | 下降约 2 个大段的最高小段 |
| 黄金 IV ~ 黄金 I | 白银 III 1 星 | 下降至白银起点 |
| 铂金 IV ~ 铂金 I | 白银 I 1 星 | 下降至白银最高 |
| 钻石 V ~ 钻石 I | 黄金 IV 1 星 | 下降至黄金起点 |
| 星耀 V ~ 星耀 I | 黄金 I 1 星 | 下降至黄金最高 |
| 王者 | 铂金 IV 1 星 | 下降至铂金起点 |

> 软重置原则：段位越高下降越多，但不会直接降至青铜，保持一定的段位优势。

### 8.3 赛季奖励

赛季结束时，根据用户**赛季最终段位**发放奖励：

| 赛季最终段位 | 奖励内容 |
|------------|---------|
| **王者 Top 10** | 赛季限定头像框 + "S{X} 传奇王者"专属称号 + 500 知语币 |
| **王者** | 赛季限定段位边框 + 300 知语币 |
| **星耀** | 赛季纪念徽章 + 200 知语币 |
| **钻石** | 赛季参与徽章 + 100 知语币 |
| **铂金** | 50 知语币 |
| **黄金** | 30 知语币 |
| **白银** | 10 知语币 |
| **青铜** | — |

- 奖励在赛季结束后下一赛季开始时自动发放
- 用户登录后弹出赛季结算弹窗，展示赛季最终段位 + 获得的奖励列表
- 赛季限定头像框和称号在个人中心展示

### 8.4 赛季倒计时展示

| 展示位置 | 展示条件 | 内容 |
|---------|---------|------|
| 游戏大厅页顶部段位栏 | 赛季剩余 ≤ 7 天 | 段位名称下方展示小字："赛季倒计时 X 天" |
| 游戏大厅页 | 赛季剩余 ≤ 3 天 | 顶部段位栏增加 Amber 色脉冲边框 |
| 推送通知 | 赛季剩余 3 天 / 1 天 | "赛季即将结束，冲刺最后段位！" |

## 九、王者段位特殊规则

### 9.1 进入王者

- 星耀 I 满星后再 PK 胜利 → 自动晋级王者
- 晋级动画为所有段位中最华丽的版本（额外增加金色烟花 + 屏幕边缘金色光晕）

### 9.2 积分规则

| 行为 | 积分变化 |
|------|---------|
| PK 胜利 | +10 分 |
| PK 失败 | -10 分 |
| 初始积分 | 0 分 |
| 积分下限 | 无下限（可为负数） |
| 掉出条件 | 积分 < 0 时，降至星耀 I 满星 |

### 9.3 王者排名

- 王者段位内按积分从高到低排名
- 积分相同时，按达到该积分的时间先后排序（先到者排名靠前）
- 排名实时更新

### 9.4 王者展示

- 游戏大厅顶部段位栏：展示"王者"+ 当前积分 + 当前排名
- 格式示例：👑 王者 · 120 分 · #42
- 排名 Top 10 的用户在排行榜中展示金色边框高亮

## 十、12 款游戏共享段位说明

### 10.1 共享规则

| 规则 | 说明 |
|------|------|
| **统一段位** | 12 款游戏使用同一个段位系统，无独立段位 |
| **任意游戏影响** | 在任何一款游戏中的 PK/多人对战胜负都直接影响统一段位 |
| **跨游戏累积** | 在 G1 中赢了 +1 星，在 G5 中输了 -1 星，段位一致计算 |
| **战绩独立** | 每款游戏的胜场/总场/胜率独立统计，仅段位共享 |

### 10.2 设计意图

- 降低用户理解成本：只需关注一个段位，不用在多个游戏间切换
- 鼓励尝试多款游戏：玩任何游戏都对段位有贡献
- 减少前端开发复杂度：段位系统单一数据源

### 10.3 用户说明

- 游戏大厅页展示的段位为统一段位
- 排行榜中的"全服总榜"按统一段位排序
- 各游戏排行榜按该游戏的胜场数排序（非段位）

### 10.4 新用户起始段位

| 属性 | 值 |
|------|-----|
| 初始段位 | 青铜 III |
| 初始星数 | 1 ⭐ |
| 初始累计星数 | 1 |
