## 一、暂停按钮

### 1.1 位置

- 游戏画面右上角，距顶部 16px，距右侧 16px
- 图标：Lucide `Pause`，尺寸 28px，白色，半透明背景圆形（`rgba(0, 0, 0, 0.4)`，直径 44px）
- 按钮可点击区域：44×44px（符合触控最小尺寸标准）

### 1.2 点击行为

```
点击暂停按钮
  → 游戏逻辑暂停（计时器停止、动画冻结、对手操作队列暂存）
  → 画面冻结在当前帧
  → 弹出暂停菜单（§二）
```

### 1.3 模式差异

| 模式 | 暂停行为 |
|------|---------|
| **单人模式** | 游戏完全暂停，无时间限制 |
| **1v1 PK** | 暂停仅暂停本方画面，对手继续操作；暂停时长最多 15 秒，超时自动恢复（显示倒计时） |
| **多人对战** | 与 1v1 PK 相同，暂停仅影响自己 |

### 1.4 PK 模式暂停限制

- 每局 PK 最多暂停 **2 次**
- 每次暂停最多 **15 秒**
- 暂停按钮旁显示剩余暂停次数（如"⏸ ×2"）
- 暂停次数用完后，暂停按钮变灰不可点击
- 暂停期间，暂停菜单上方显示倒计时："自动恢复 XX 秒"

## 二、暂停菜单

### 2.1 弹窗布局

```
┌─────────────────────────────────────┐
│                                     │
│            ⏸️ 暂停                   │ ← 标题
│                                     │
│  ┌─────────────────────────────┐    │
│  │     ▶️ 继续游戏              │    │ ← 主按钮
│  └─────────────────────────────┘    │
│                                     │
│  🔊 声音      ──────●──            │ ← 音效开关
│  🎵 音乐      ──────●──            │ ← 背景音乐开关
│                                     │
│  ┌─────────────────────────────┐    │
│  │     🚪 退出游戏              │    │ ← 退出按钮
│  └─────────────────────────────┘    │
│                                     │
└─────────────────────────────────────┘
```

### 2.2 弹窗视觉规范

- 弹窗居中展示，宽度 320px
- 背景：毛玻璃面板（`.glass-card`）+ 背景遮罩（`rgba(0, 0, 0, 0.6)`）
- 弹窗圆角：24px
- 内部 padding：24px

### 2.3 菜单元素

| 元素 | 说明 | 交互 |
|------|------|------|
| **"暂停"标题** | H2（24px），字重 700，居中 | — |
| **继续游戏按钮** | Rose 色实心按钮，药丸形，全宽 | 关闭暂停菜单，恢复游戏 |
| **声音开关** | 左侧 🔊 图标 + "声音"文字，右侧 Toggle 开关 | 切换游戏音效开关（默认开启） |
| **音乐开关** | 左侧 🎵 图标 + "音乐"文字，右侧 Toggle 开关 | 切换背景音乐开关（默认开启） |
| **退出游戏按钮** | 灰色边框按钮，药丸形，全宽 | 触发退出确认（§三） |

### 2.4 声音/音乐控制

- Toggle 开关使用标准滑块样式
- 开启态：Rose 色滑块
- 关闭态：灰色滑块
- 设置持久化至本地存储，下次进入游戏保持上次设置
- 切换即时生效（无需确认）

## 三、退出游戏确认

### 3.1 触发

- 在暂停菜单中点击"退出游戏"按钮

### 3.2 确认弹窗（单人模式）

```
┌─────────────────────────────────────┐
│                                     │
│          确认退出？                   │
│                                     │
│  退出后本局进度将丢失。               │
│                                     │
│  ┌────────────┐ ┌────────────┐      │
│  │   取 消    │ │  确认退出   │      │
│  └────────────┘ └────────────┘      │
│                                     │
└─────────────────────────────────────┘
```

### 3.3 确认弹窗（PK / 多人模式）

```
┌─────────────────────────────────────┐
│                                     │
│          确认退出？                   │
│                                     │
│  ⚠️ PK 模式退出将判定为失败，         │
│  段位扣除 1 星，并记录为逃跑。        │
│                                     │
│  ┌────────────┐ ┌────────────┐      │
│  │   取 消    │ │  确认退出   │      │
│  └────────────┘ └────────────┘      │
│                                     │
└─────────────────────────────────────┘
```

### 3.4 按钮行为

| 按钮 | 行为 |
|------|------|
| **取消** | 关闭确认弹窗，返回暂停菜单 |
| **确认退出**（单人模式） | 结束游戏 → 不展示结算页 → 横屏切换恢复竖屏 → 返回模式选择页 |
| **确认退出**（PK/多人模式） | 判定为失败 → 段位 -1 星 → 记录逃跑 → 横屏切换恢复竖屏 → 返回游戏大厅页 |

## 四、PK 模式退出惩罚

### 4.1 惩罚规则

| 惩罚项 | 说明 |
|--------|------|
| **判定为失败** | 等同于 PK 失败 |
| **段位扣星** | -1 ⭐（与正常失败一致） |
| **逃跑次数记录** | 服务端记录该用户逃跑次数 |
| **对手获胜** | 对手判定为胜利，+1 ⭐ |

### 4.2 逃跑惩罚机制

| 逃跑累计次数（当前赛季） | 额外惩罚 |
|------------------------|---------|
| 1-3 次 | 无额外惩罚 |
| 4-6 次 | 匹配等待时间增加 30 秒（惩罚性延长） |
| 7+ 次 | 匹配等待时间增加 60 秒 + 段位扣 2 星（双倍扣星） |

### 4.3 逃跑提示

- 确认退出弹窗中展示当前赛季已逃跑次数："本赛季已逃跑 X 次"
- 如已达到惩罚阈值，额外展示红色警告："再次逃跑将受到更严厉的惩罚"

## 五、游戏中网络断线处理

> 与 `03-matching.md` §七 网络断线处理 配合使用，此处补充 HUD 侧的展示规范。

### 5.1 短暂断线（≤ 10 秒）

| 元素 | 说明 |
|------|------|
| **遮罩** | 全屏半透明黑色遮罩（`rgba(0, 0, 0, 0.7)`），覆盖游戏画面 |
| **文字** | "网络重连中..." 居中显示，Body L（18px），白色 |
| **加载指示器** | 文字下方旋转圆环动画，白色 |
| **倒计时** | "剩余 XX 秒" 显示在加载指示器下方，每秒递减 |
| **HUD** | 暂停按钮不可点击（灰色） |

### 5.2 长时间断线（> 10 秒）

| 元素 | 说明 |
|------|------|
| **遮罩** | 保持全屏遮罩 |
| **文字** | "网络断开超时，对局已结束" |
| **PK 模式** | 追加文字："判定为失败" |
| **按钮** | "返回大厅"按钮（Rose 色实心，居中） |
| **段位变化** | 已在服务端处理（-1 星），返回大厅后段位自动更新 |

### 5.3 单人模式断线

| 元素 | 说明 |
|------|------|
| **遮罩** | 全屏半透明黑色遮罩 |
| **文字** | "网络已断开" |
| **按钮** | "继续离线游戏"（如游戏支持）/ "返回大厅" |
| **说明** | 如游戏数据全在本地处理（如 G1 汉字切切切单人模式），网络断线不影响游戏 |

## 六、进入游戏时自动切换横屏

### 6.1 触发时机

以下场景触发自动横屏切换：

| 场景 | 触发点 |
|------|--------|
| 单人模式点击"开始游戏" | 按钮 Loading 完成后 |
| 1v1 PK 匹配成功倒计时结束 | "开始！"文字消失后 |
| 多人对战人数达标倒计时结束 | "开始！"文字消失后 |

### 6.2 横屏切换流程

```
竖屏页面（模式选择 / 匹配页面）
  → 触发横屏切换
  → 全屏渐黑过渡（300ms）
  → 屏幕方向旋转至 landscape
  → 游戏画面加载
  → 全屏渐亮过渡（300ms）
  → 游戏画面可见，开始游戏
```

### 6.3 移动端横屏实现

| 步骤 | 说明 |
|------|------|
| 1. 请求全屏 | 调用 `document.documentElement.requestFullscreen()` |
| 2. 锁定方向 | 调用 `screen.orientation.lock('landscape')` |
| 3. 游戏容器 | Phaser 游戏画布占满全屏，ScaleManager 自适应 |
| 4. 系统栏隐藏 | 全屏模式下自动隐藏浏览器地址栏和系统状态栏 |

### 6.4 不支持方向锁定的兜底

- 部分浏览器/设备不支持 `screen.orientation.lock()`
- 兜底方案：使用 CSS `transform: rotate(90deg)` 旋转页面容器
- 同时设置 `width: 100vh; height: 100vw` 适配旋转后的尺寸
- 此时系统状态栏仍可见，但游戏画面已横向展示

## 七、横屏锁定提示

### 7.1 触发条件

- 用户**首次**进入任何游戏时（按用户维度，仅首次触发一次）
- 已展示过一次的用户不再重复展示

### 7.2 提示内容

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│         📱↔️🖥️                                      │
│                                                     │
│     请将设备横置                                     │
│     游戏以横屏模式运行，获得最佳体验                    │
│                                                     │
│              [知道了]                                │
│                                                     │
└─────────────────────────────────────────────────────┘
```

| 元素 | 说明 |
|------|------|
| **图标** | 手机旋转动画图标 |
| **标题** | "请将设备横置" |
| **说明** | "游戏以横屏模式运行，获得最佳体验" |
| **按钮** | "知道了"，Rose 色实心按钮 |
| **位置** | 全屏半透明遮罩 + 居中弹窗 |
| **自动关闭** | 点击"知道了"关闭，或设备已横置时自动关闭 |

### 7.3 持久化

- 展示一次后，在本地存储中记录 `game_landscape_tip_shown = true`
- 后续进入游戏不再展示此提示
- 清除缓存后会重新展示

## 八、退出游戏后自动恢复竖屏

### 8.1 触发时机

以下场景触发恢复竖屏：

| 场景 | 触发点 |
|------|--------|
| 游戏结算页点击"返回大厅" | 页面跳转前 |
| 游戏中退出确认后"确认退出" | 确认后立即触发 |
| 网络断线判负后"返回大厅" | 按钮点击后 |
| 游戏结算页点击"再来一局" | **不恢复竖屏**（继续横屏进入匹配/新局） |

### 8.2 恢复竖屏流程

```
横屏游戏画面
  → 触发竖屏恢复
  → 全屏渐黑过渡（300ms）
  → 解除方向锁定（screen.orientation.unlock()）
  → 退出全屏模式（document.exitFullscreen()）
  → 屏幕方向旋转至 portrait
  → 目标页面加载（大厅/模式选择）
  → 全屏渐亮过渡（300ms）
  → 竖屏页面可见
```

### 8.3 结算页的特殊处理

- 游戏结算页**仍然保持横屏**展示（结算页是游戏流程的一部分）
- 只有在用户点击"返回大厅"时才恢复竖屏
- 点击"再来一局"时保持横屏，直接进入下一局匹配或开始

## 九、Web 端横屏实现说明

### 9.1 Web 桌面端

| 维度 | 说明 |
|------|------|
| **屏幕方向** | 桌面端浏览器天然为横屏，无需方向旋转 |
| **全屏** | 进入游戏时调用全屏 API（`requestFullscreen()`）获得沉浸体验 |
| **画面比例** | 保持 16:9 比例，居中展示，两侧留黑 |
| **退出全屏** | 用户按 Esc 或点击退出时自动退出全屏 |

### 9.2 移动端 H5 浏览器

| 维度 | 说明 |
|------|------|
| **首选方案** | `screen.orientation.lock('landscape')` 强制横屏 |
| **兜底方案** | CSS `transform: rotate(90deg)` + 尺寸调换 |
| **全屏 API** | `requestFullscreen()` 隐藏浏览器 UI |
| **viewport meta** | 游戏画面使用独立 viewport 配置，禁止缩放 |

### 9.3 PWA 模式

| 维度 | 说明 |
|------|------|
| **独立窗口** | PWA standalone 模式下无浏览器地址栏 |
| **方向锁定** | manifest.json 中游戏页面不设全局方向，由 JS 动态控制 |
| **状态栏** | iOS PWA 可通过 `apple-mobile-web-app-status-bar-style` 控制 |

## 十、游戏中 HUD 完整布局

### 10.1 横屏 HUD 布局

```
┌──────────────────────────────────────────────────────────┐
│  [段位] 黄金II ⭐⭐⭐    [游戏名]        ⏸ [暂停] 0:00│ ← 顶部 HUD
│                                                          │
│                                                          │
│                                                          │
│                    [游戏主画面区域]                         │
│                                                          │
│                                                          │
│                                                          │
│  [HP/分数等游戏特定UI]              [对手信息/排名等]       │ ← 底部 HUD
└──────────────────────────────────────────────────────────┘
```

### 10.2 顶部 HUD 元素

| 元素 | 位置 | 说明 |
|------|------|------|
| **段位信息** | 左上角 | 段位图标（20px）+ 段位名 + 星数，Body S（14px），半透明背景 |
| **游戏名称** | 顶部居中 | 当前游戏名称，Body S（14px），白色 50% 不透明度 |
| **暂停按钮** | 右上角 | 暂停图标 + PK 模式剩余暂停次数 |
| **计时器** | 暂停按钮左侧 | 游戏已进行时间 "M:SS"，Body S（14px） |

### 10.3 底部 HUD 元素

- 底部 HUD 的具体内容由各游戏自行定义（如 HP、分数、技能等）
- 本文档仅定义通用的顶部 HUD 和暂停/退出系统

### 10.4 HUD 透明度

- 所有 HUD 元素背景使用半透明黑色（`rgba(0, 0, 0, 0.3)`）
- 不遮挡游戏核心画面
- 游戏进行中 HUD 可在 3 秒无操作后自动降低透明度（40% → 20%），触摸屏幕任意位置恢复

## 十二、数据流向说明

| 数据 | 来源 | 去向 | 说明 |
|------|------|------|------|
| 暂停/继续指令 | 用户点击 HUD 暂停按钮 | 游戏引擎暂停/恢复 | 本地处理 |
| 声音/音乐设置 | 用户切换开关 | 本地存储 + 游戏引擎 | 持久化至 localStorage |
| 退出指令 | 用户确认退出 | 对战服务器（PK 模式判负）+ 本地路由跳转 | PK 模式需通知服务端 |
| 屏幕方向 | Screen Orientation API / CSS transform | 浏览器渲染引擎 | 横竖屏切换 |
| 断线检测 | WebSocket 连接状态监听 | HUD 断线遮罩 + 重连逻辑 | 心跳检测间隔 3 秒 |
| 逃跑记录 | 服务端记录 `user_escape_records` 表 | 匹配惩罚逻辑 | 按赛季累计 |

## 十三、非功能需求

| 维度 | 指标 | 说明 |
|------|------|------|
| **匹配响应时间** | 1v1 匹配 P50 ≤ 10 秒，P90 ≤ 30 秒 | 30 秒为超时上限 |
| **多人匹配响应** | P50 ≤ 15 秒，P90 ≤ 30 秒 | 30 秒为超时上限 |
| **游戏帧率** | 稳定 60fps，最低不低于 30fps | Phaser 3 渲染性能 |
| **WebSocket 延迟** | P50 ≤ 50ms，P90 ≤ 150ms | 对战实时通信 |
| **断线重连** | 重连尝试间隔 1 秒，最多尝试 10 次（10 秒） | 指数退避策略 |
| **横屏切换时间** | 切换完成（含过渡动画）≤ 800ms | 包含渐黑+旋转+渐亮 |
| **结算页加载** | ≤ 2 秒内展示完整结算内容 | 含动画开始播放 |
| **皮肤加载** | 皮肤预览图加载 ≤ 1 秒 | CDN 加速 |
| **游戏资源加载** | 首次加载 ≤ 5 秒，后续加载 ≤ 2 秒（有缓存） | Service Worker 缓存策略 |

## 十四、验收标准汇总

| # | 验收项 | 验收条件 |
|---|--------|---------|
| 1 | 横屏切换 | 进入游戏时自动切换为横屏；退出游戏时自动恢复竖屏；首次横屏有操作引导 |
| 2 | 暂停功能 | 单人模式暂停完全冻结游戏；PK 模式暂停有次数和时长限制 |
| 3 | 声音控制 | 声音/音乐开关即时生效且持久化 |
| 4 | 退出确认 | 单人模式退出弹窗无惩罚提示；PK 模式退出弹窗明确提示判负和扣星 |
| 5 | PK 退出惩罚 | 退出 PK 判负 -1 星，记录逃跑次数，累计逃跑有额外惩罚 |
| 6 | 短暂断线 | 10 秒内展示重连遮罩，重连成功后恢复游戏 |
| 7 | 长时间断线 | 超过 10 秒判负，展示失败提示和返回按钮 |
| 8 | 匹配动画 | 1v1 匹配展示雷达扫描动画；多人匹配展示人数进度 |
