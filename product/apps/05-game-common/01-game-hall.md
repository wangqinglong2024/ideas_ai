## 一、未登录用户访问行为

### 1.1 触发场景

用户在未登录状态下点击底部导航栏"游戏模式"Tab。

### 1.2 行为流程

```
点击"游戏模式"Tab
  → 弹出登录墙（半屏弹窗，非全屏覆盖）
  → 用户选择登录/注册方式
  → 登录成功后自动进入游戏大厅页
  → 登录失败或用户关闭弹窗 → 停留在当前页面，不进入游戏模块
```

### 1.3 规则说明

- 未登录用户**无法**以任何方式进入游戏大厅页或其子页面
- 登录墙的交互规范遵循全局框架 `02-auth-system.md` 定义
- 登录成功后，底部 Tab 自动切换至"游戏模式"并展示游戏大厅页

## 二、页面布局

### 2.1 整体结构

```
┌─────────────────────────────────────┐
│  [顶部段位展示栏]                      │
│  段位图标 + 段位名称 + 星数 + 总星数     │
├─────────────────────────────────────┤
│                                     │
│  [游戏列表区域]                       │

## 三、顶部段位展示栏

### 3.1 布局结构

```
┌─────────────────────────────────────┐
│  🥇 黄金 II   ⭐⭐⭐☆           47 ⭐  │
│  [段位图标] [段位名] [星数排列] [总星数]  │
└─────────────────────────────────────┘
```

### 3.2 展示元素

| 元素 | 说明 | 示例 |
|------|------|------|
| **段位图标** | 当前大段对应的图标，尺寸 32×32px | 🥇（黄金段位图标） |
| **段位名称** | 大段名称 + 罗马数字小段，多语言展示 | "黄金 II" / "Gold II" / "Vàng II" |
| **当前星数排列** | 该小段已获星数以实心星展示，未获星数以空心星展示 | ⭐⭐⭐☆（3/4 星） |
| **总星数数字** | 累计总星数的数字展示 | "47 ⭐" |

### 3.3 视觉规范

- 段位图标使用对应段位的专属彩色图标（非 emoji 占位，需设计师提供）
- 段位名称使用 **Body** 字号（16px），字重 600
- 星数图标尺寸 16×16px，实心星使用 Amber 色（`#d97706`），空心星使用中性灰（`#a3a3a3`）
- 总星数数字使用 **Body S** 字号（14px），右对齐
- 整栏高度 56px，水平 padding 16px，背景使用毛玻璃面板（`.glass-card`）

### 3.4 交互行为

| 操作 | 行为 |
|------|------|
| 点击段位图标或段位名称区域 | 跳转至段位详情页面（`05-rank-system.md` 定义） |
| 点击总星数区域 | 跳转至排行榜页面（`06-leaderboard.md` 定义） |

### 3.5 新用户初始态

- 新注册用户初始段位为：**青铜 III · 1 星**
- 展示为：🥉 青铜 III ⭐☆☆ 1 ⭐

## 四、游戏列表

### 4.1 列表/网格切换

- 列表区域右上角展示切换按钮（图标按钮，无文字）
  - 网格图标：`grid` 图标（Lucide `LayoutGrid`）
  - 列表图标：`list` 图标（Lucide `List`）
- 默认视图：**网格视图**（2 列）
- 切换动画：淡入淡出过渡，时长 200ms
- 用户偏好持久化至本地存储（`localStorage`），下次进入保持上次选择

### 4.2 网格视图

- 2 列网格布局，卡片间距 16px，左右 padding 16px
- 每行 2 张游戏卡片，共 6 行展示 12 款游戏

### 4.3 列表视图

- 单列纵向排列，卡片高度 88px，卡片间距 12px，左右 padding 16px
- 封面缩略图在左侧（正方形 64×64px），信息在右侧

### 4.4 游戏卡片设计（网格视图）

```
┌──────────────────────┐
│                      │
│   [游戏封面插图]      │
│    占卡片 60% 高度    │
│                      │
│  推荐先学 L4          │ ← 角标（条件展示）
├──────────────────────┤
│  汉字切切切           │ ← 游戏名称
│  Level 1             │ ← 对应课程 Level
│  12 胜 / 20 场       │ ← 胜场/总场
│  [皮肤缩略图]         │ ← 当前装备皮肤
└──────────────────────┘
```

### 4.5 卡片元素详细定义

| 元素 | 位置 | 说明 | 字体/尺寸 |
|------|------|------|----------|
| **游戏封面插图** | 卡片上部 | 每款游戏的专属封面插图，圆角与卡片一致 | 图片高度占卡片 60% |
| **游戏名称** | 封面下方第 1 行 | 多语言显示，如"汉字切切切" / "Hanzi Slash" | H3（20px），字重 600 |
| **对应 Level 标签** | 封面下方第 2 行 | 标签样式，如 "Level 1"、"Level 5" | Caption（12px），标签背景使用 Sky 色 10% 不透明度 |
| **胜场/总场** | 第 3 行 | 格式："X 胜 / Y 场"，未玩过显示"—" | Body S（14px），灰色文字 |
| **当前装备皮肤缩略图** | 第 3 行右侧 | 已装备皮肤的 24×24px 缩略图，未装备皮肤时不显示 | 24×24px 圆角图片 |
| **推荐先学角标** | 封面右上角 | 用户未学对应 Level 课程时显示，半透明黑底白字 | Caption（12px），格式"推荐先学 LX" |

### 4.6 游戏解锁状态

- **所有 12 款游戏对登录用户全部解锁**，无锁定态
- 卡片无任何锁定遮罩或锁定图标
- 即使用户未学习对应 Level 课程，游戏仍可正常进入和游玩

### 4.7 未学对应课程的提示

- 当用户尚未开始学习该游戏对应 Level 的课程时：
  - 游戏卡片右上角显示半透明角标："推荐先学 LX"（X 为对应 Level 编号）
  - 角标样式：背景 `rgba(0, 0, 0, 0.6)`，文字白色，圆角 8px，padding 4px 8px
  - **角标不阻止用户点击进入游戏**，仅为引导建议
- 当用户已开始学习（课程进度 > 0%）或已完成对应课程时，角标不显示

### 4.8 卡片点击行为

- 点击任意游戏卡片 → 页面跳转至该游戏的**模式选择页**（`02-mode-select.md`）
- 跳转动画：横向滑入（从右侧滑入），时长 300ms，缓动 `cubic-bezier(0.2, 0, 0, 1)`

## 五、底部快捷入口栏

### 5.1 布局结构

```
┌─────────────────────────────────────┐
│  🛍️ 皮肤商城          🏆 排行榜      │
└─────────────────────────────────────┘

### 5.2 元素定义

| 按钮 | 图标 | 文字标签 | 点击行为 |
|------|------|---------|---------|

## 六、状态矩阵

| 状态 | 触发条件 | UI 表现 |
|------|---------|---------|
| **空状态** | 理论上不存在（12 款游戏为固定列表），仅在服务端游戏配置异常时出现 | 品牌插画 + "暂无可用游戏" + "刷新试试"按钮 |
| **加载中** | 页面首次加载，正在获取段位信息和游戏列表数据 | 顶部段位栏：骨架屏矩形占位；游戏列表：6 张骨架屏卡片（2 列 × 3 行） |
| **首次加载** | 用户第一次进入游戏大厅，数据尚未缓存 | 全屏骨架屏 + 知语 Logo 呼吸动画 |
| **成功状态** | 段位信息和游戏列表数据加载完成 | 正常展示所有内容，段位栏实时更新 |
| **错误状态** | 网络请求失败或服务端错误 | 列表区域展示错误插画 + "加载失败，点击重试"文案 + 重试按钮；顶部段位栏展示上次缓存数据或占位符"—" |
| **部分加载** | 列表已展示但部分卡片的战绩数据尚未返回 | 已返回的卡片正常展示，未返回的卡片中"胜场/总场"区域显示骨架屏横条 |
| **离线状态** | 设备无网络连接 | 顶部展示离线 Banner："当前无网络连接"；使用上次缓存的段位和游戏数据展示页面；点击任何需要网络的操作（进入游戏）时提示"请连接网络后再试" |

## 七、数据流向

| 数据 | 来源 | 去向 | 说明 |
|------|------|------|------|
| 用户段位信息 | Supabase `user_ranks` 表 | 顶部段位展示栏 | 登录后实时查询 |
| 游戏列表配置 | Supabase `games` 表 | 游戏列表区域 | 12 款游戏基础信息 |
| 各游戏战绩数据 | Supabase `game_records` 表 | 游戏卡片胜场/总场 | 按 `user_id` + `game_id` 聚合 |
| 用户课程进度 | Supabase `user_courses` 表 | 推荐先学角标判断 | 判断是否已开始对应 Level |
| 已装备皮肤 | Supabase `user_skins` 表 | 卡片皮肤缩略图 | 筛选 `equipped = true` 的皮肤 |
